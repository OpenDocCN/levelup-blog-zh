## 作为数据科学家需要掌握的 10 项软件工程技能

数据科学是一个不断发展的领域。为了跟上这种快节奏的环境，数据科学家需要能够快速学习新的东西，并不断更新他们的技能。

数据科学家首先是软件工程师。他们可能不会在日常工作中编写机器学习模型或自然语言处理算法，但他们作为数据科学家所做的工作需要软件工程和编程技能，以便能够对数据应用所有数据科学项目生命周期。除此之外，他们还应该能够理解用户的需求，并为这些需求开发解决方案，这对于在组织中工作的任何数据科学家来说都是至关重要的。

这篇博客文章列出了一些最重要的软件工程技能，如果数据科学家想在这个领域继续蓬勃发展，他们应该掌握这些技能。

# 1.编写高效干净的代码

![](img/45f8a3816851b8969c8a61c94e3756c7.png)

由[萨曼莎·加德斯](https://unsplash.com/@srosinger3997?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的照片

作为一名数据科学家，你应该把大部分时间花在从数据中获取可操作的见解上，而不是等待代码运行完毕。这就是为什么您需要能够编写高效的代码来帮助您减少运行时间并节省尽可能多的计算资源。您还应该能够找到代码瓶颈，并消除这些瓶颈和糟糕的设计模式。

你也将在团队中工作，你的代码将被其他人使用，这就是为什么你需要一个清晰的代码，其他人可以很容易地理解和遵循它。而且在你的大多数项目中，你将不得不回到你的旧代码来重用它，所以用一种干净的方式写它将会使它容易理解和重用。

# 4.单元测试

有效的单元测试过程减少了开发和维护时间，改进了文档，增加了用户的信任，并节省了生产系统的时间。因为单元测试是数据科学中的常见实践，所以它已经成为一项重要的技能。因为几乎每一个走向市场的数据科学项目都需要测试。公司几乎普遍使用它。因此，单元测试是业内必备技能，几乎每个公司都在使用。

Python 最流行的测试框架 pytest 可用于单元测试，因为您在项目中会经常使用 python。作为一名数据科学家，你应该能够为数据预处理、模型和可视化创建单元测试，解释测试结果并修复错误代码。您还应该了解一些高级概念，比如 TDD、测试组织、fixtures 和 mocking，这样您就可以正确地测试您的数据科学项目。

# 5.Bash 脚本

![](img/61226bcafc7078e775fed770a15c5d73.png)

加布里埃尔·海因策在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上的照片

在工作中使用命令行将是至关重要的，因为您将在生产环境中使用脚本。如果您使用命令行脚本，您将能够创建自动化常见任务的例程，以及更快地处理大量操作系统。

Bash 是一种快速、简洁且健壮的数据和文件操作语言。它可以用来在云中构建分析管道，特别是对于跨多个文件使用数据的 Linux 用户。在输入参数和输出结果方面，您必须掌握 Bash 脚本的实际经验。理解变量和数组等数据结构以及循环和条件等控制语句也很重要。最后，您应该能够编写 Bash 函数，并使用 cron 自动运行它们。

# 6.命令行自动化

![](img/7bbc521030f4c9f6805d2c4e1c475f36.png)

由[安德里亚·德·森蒂斯峰](https://unsplash.com/@santesson89?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄

在后台运行命令和脚本的能力可以节省时间和自动化繁琐的任务。有几种不同的工具和技术可以用来实现这一点，包括编程自动化、脚本和命令行自动化。使用命令行自动化有很多好处。首先，有可能自动化复杂的任务，否则可能需要大量的脚本或编程。第二，无需安装软件或创建定制脚本或模板就可以实现任务自动化。最后，这样做是可能的，而不必显式地记录需要运行的命令。使用命令行自动化的一个缺点是没有视觉反馈。这意味着不可能看到正在执行什么命令或者完成这些命令需要多长时间。这也意味着错误不容易被发现。
命令行自动化对于数据科学家来说非常有用，他们需要自动化数据清理、数据加载和数据集成等任务，并且希望对代码运行的环境有更多的控制。

# 7.使用 Git 进行版本控制

![](img/a7293ef20d108db4c35d03514278bc5e.png)

在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上 [Praveen Thirumurugan](https://unsplash.com/@praveentcom?utm_source=medium&utm_medium=referral) 拍摄的照片

在生产环境中工作要求数据科学家了解版本控制。因为你将与其他数据科学家、数据工程师和软件工程师合作。因此，能够分享和更新你的代码，并且以专业的方式跟进他们的更新是很重要的。此外，如果您正在处理个人项目，使用版本控制将允许您对代码进行更好的版本控制。此外，由于在数据科学领域，您可能会有许多尝试来为您的问题找到正确的解决方案，因此最好能够将所有尝试及其结果版本化。

您应该能够创建一个新的 Git repo，提交更改，并查看现有 repo 的提交历史。您应该能够使用标记和分支来组织您的提交，并且您将通过粉碎那些讨厌的合并冲突来掌握合并更改的艺术，以及如何编辑提交、恢复更改，甚至删除提交。

# 8.云工程

![](img/6ebecb3436b428c90644b2947d05f6a7.png)

图片由 [C Dustin](https://unsplash.com/@dianamia?utm_source=medium&utm_medium=referral) 在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄

公司可以通过互联网访问一系列计算服务。如今，世界上最大的数据中心，如亚马逊网络服务，可以以最低的成本让公司运行他们的应用程序。因为小型企业或新兴经济体国家可以利用这项技术进行雄心勃勃的复杂工作，否则如果他们必须在自己的服务器上托管这些工作，成本可能会非常高，所以云计算是数据科学应用程序的重要组成部分。由于云计算，数据分析、数据管理和模型部署都得到了简化。因为服务器的生产成本很高，特别是对于较小的组织，因为他们需要相当大的空间来容纳它们，不断的维护和备份以防出现任何问题，以及大量的规划和研究，公司可能会根据其数据需求获得比所需更多或更少的服务器。

以下是数据科学家使用的最著名的云计算平台:

## 1.亚马逊网络服务

亚马逊的云计算业务创建于 2006 年，目前是数据科学领域最受欢迎的云计算平台之一。[亚马逊网络服务](https://aws.amazon.com/)提供的数据分析产品包括亚马逊 QuickSight(商业分析服务)、亚马逊 RedShift(数据仓库)、AWS 数据管道、AWS 数据交换、亚马逊 Kinesis(实时数据分析)、亚马逊 EMR(大数据处理)、亚马逊 SageMaker(模型部署)。Amazon Web Services 提供的数据库解决方案包括 Amazon Aurora(关系数据库)和 Amazon DynamoDB(非关系数据库)。AWS 被网飞、NASA 等组织所利用。

## 2.谷歌云

[谷歌云](https://cloud.google.com/)平台是谷歌运营的数据仓库平台。这与谷歌用于支持其内部服务如谷歌搜索、YouTube、Gmail 等的基础设施相同。Google Cloud 提供各种数据分析服务，如 BigQuery(数据仓库)、Dataflow(实时分析)、Dataproc(运行 Apache Hadoop 和 Apache Spark 集群)、Looker(商业智能分析)、Google Data Studio(可视化仪表盘和数据报告)、Dataprep(数据准备)等。

## 3.微软 Azure

微软 Azure 于 2010 年作为云计算平台发布。微软 Azure 是流行的数据科学和数据分析云计算平台。Azure Synapse Analytics(数据分析)，Azure Stream Analytics(流分析)，Azure Data bricks(Apache Spark Analytics)，Azure Data Lake Storage(数据湖)，Data Factory(混合数据集成)，以及其他 Azure 产品可用于数据分析。也支持 Azure Cosmos DB (NoSQL 数据库)、Azure SQL 数据库(SQL 数据库)和其他数据库。

# 9.DevOps

开发运维是数据科学家的一项关键技能。DevOps 可以帮助数据科学家优化数据管道、模型训练、模型测试、模型预测和模型部署中涉及的软件组件的部署。

CI/CD 是一种在许多软件项目中使用的 DevOps 技术。如果没有使用 CI/CD 管道的必要基础知识，数据科学家将无法有效地与软件团队合作。除了学习和利用云基础架构中现有的 DevOps 功能之外，数据科学家还能够凭借对云计算的熟练掌握，更快地将其技能应用到生产环境中。

# 10.数据库和大数据

作为一名数据科学家，您应该能够查询和使用结构化和非结构化数据库技术。这些数据库可能包括像 PostgreSQL 这样的 SQL 数据库或像 MongoDB 这样的 NoSQL 数据库。由于这是任何软件工程项目的标准组成部分，您应该能够处理它，尤其是在数据收集阶段。如果您的项目包括处理大数据，您可能还需要处理 Spark 和 Hive 等大数据技术，这些技术支持 Hadoop 集群操作。
