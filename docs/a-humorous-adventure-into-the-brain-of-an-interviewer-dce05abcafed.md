## 面试官到底是怎么想的

![](img/438241464c10e7c4a75fc56f13254af2.png)

由[塞巴斯蒂安·赫尔曼](https://unsplash.com/@officestock?utm_source=unsplash&utm_medium=referral&utm_content=creditCopyText)在 [Unsplash](https://unsplash.com/s/photos/interview?utm_source=unsplash&utm_medium=referral&utm_content=creditCopyText) 上拍摄的照片

面试一个候选人非常有趣，因为你可以被动地表明你比他们优越。你有机会问最复杂的问题(当然是来自谷歌)，当他们没有回答时，你会傻笑着自鸣得意。这是一个证明你在研究生院花了一百年学习编译器优化只是为了在工作中运行 SQL 分组查询的机会。

但是老实说，面试对面试官来说也很难。你必须像面试者一样为他们做好准备。你必须花几个晚上谷歌最难的面试问题，但不仅仅是这样。你还必须首先尝试解决它们，因为，说实话，你从来没有解决过它们。

你可以问一些与工作相关的问题，比如，

> 如何调优这个特定的查询？或者
> 
> 如何着手修复“内存不足异常？”

但你没有。因为，如果他们那么容易回答呢？你必须问一些他们很难回答的问题。这就是目标…对吗？

我经历过面试的双方，我在这里向你倾吐大量关于面试官问的常见问题和他们真实想法的真相。

# 解释你在上一个项目中的角色

*面试官真实想法:*

这是我最擅长的部分。我希望他想说多久就说多久。这是他的时代。我可以看着他的简历说:

> “你在这里做了什么？”
> 
> “你为什么要这样做”

我不需要知道任何事情。我可以一直点头，好像我明白了。

另外，它消磨时间。这次采访不会在规定的时间内结束。

# **合并一组** `**k**` **链表** `**lists**`

*面试官真实想法:*

这项工作与链表无关。事实上，我认为公司里没有人使用过百万年以上的链表。但是我一直在采访中看到有人问他们，怎么能遗漏呢？

我只知道网上提到的解决方案。我希望他们不要尝试任何其他解决方案——我可能连一半都不明白。

我可能问了项目需要的普通 SQL 问题，但我不会问你任何关于实际工作要求的问题，因为这不会给我机会变得病态地迂腐。

# 这段代码的时间复杂度是多少？

*面试官真实想法:*

我自己不知道这个问题的解决方案，也没有正式证明“正确的”答案是正确的。我刚刚在网上看到有人把这个问题标为棘手。

因此，如果你很快就想出了答案，我会认为你要么是猜的，要么是作弊，在互联网上查找编程面试问题。但无论发生什么，我都不会接受你聪明到可以回答 2358 人标记为“难”的问题。

如果你说你不知道答案，让我带着吃屎的笑容解释正确答案(来自网络)，我会最开心。

# 你将如何解决团队内部的分歧？

*面试官真实想法:*

这是当今的新事物。在我那个时代，没有人在乎成为团队成员——每个人都在乎。

我希望他开始说话，让这个房间不要变成死气沉沉的。我真的不在乎，说什么都行。反正我也不听。我也没有任何跟进问题。现在，我只是担心这次采访不会太快结束。

# 你有什么问题要问我吗？

*面试官的真实想法*

终于结束了**。我希望他没有任何问题要问我。如果他问了一个复杂的问题，一些我不知道的问题，怎么办？为了避免这种尴尬，我特意没有为这一部分留出任何时间。当我找到 2 秒钟的停顿时，我会说“*谢谢你的时间*”然后离开。**

不要！他有一个问题。他想知道我在项目中做什么。我真不知道答案。嗯！我到底是做什么的？很奇怪，我没怎么想过。我知道我有很多喝咖啡的休息时间，我和同事们一起散步，也浪费了他们的时间。是的，是的，我参加了很多会议，如果那被认为是工作的话。伙计。成为团队领导是很棒的。

这不是一篇严肃的文章，也不打算以这种方式发表。这不是一篇讨论应该如何进行采访的文章。

这只是我试图给你的一天带来一点幽默。如果你有任何关于面试的有趣故事，请告诉我。
