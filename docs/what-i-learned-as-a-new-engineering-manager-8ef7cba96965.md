# 作为一名新的工程经理，我学到了什么

> 原文：<https://levelup.gitconnected.com/what-i-learned-as-a-new-engineering-manager-8ef7cba96965>

大约两年半前，我得到了从软件工程师到软件工程经理的机会。我过去申请过经理职位，但这是我第一次真正尝试成为一家新公司的经理，并了解作为一名工程经理真正意味着什么。

一路上，我学到了很多东西，做出了一些好的决定和一些坏的决定，这可能并不奇怪。在我研究了几年之后，这里有一些重要的经验，我想和任何一个刚刚进入这个职位的工程经理分享一下。

# 每周 1-1 次

这听起来可能是老生常谈，但是每周召开一次常务会议，每周至少留出 30 分钟来叙旧，建立关系，了解工作之外的斗争或胜利，而不必为此安排一次明确的会议。

当有人想倾诉时，通常最好尽快倾诉，把问题公开，而不是任其恶化。他们等待的时间越长，他们就有越多的机会决定情况是不可改变的，并将开始寻找不同的机会。任何经理的工作都有一部分是留住员工，员工不必要的离开会带来很多成本。

如果你的直接下属在分享他们团队的问题时开始慌张或情绪化，确保你没有反映他们的情绪。寻求理解他们为什么会经历那些情绪，并一定要感谢他们与你分享他们的挫折(显然，作为经理，你不能在不知道发生了什么的情况下做一些事情)。我犯过这样的错误，当有人对我不高兴时，我会变得不高兴，事后我总是后悔，我们的谈话也没有那么有影响力，因为我们都被情绪笼罩，无法解决问题。

每周 1-1 是一个非常有用的工具，你可能会想取消或跳过它们，但至少见面、签到、聊一会儿，然后提前结束是一个好得多的结果，你会感谢自己。

# 不是每个人都会留下来，这没关系

如果你在内部得到提升，或者即将加入一个现有的团队，那么这个团队的潜在成员可能会离开。这可能有很多原因，下面是我听到的一些。

*   我不想再次更换经理，重新开始建立关系的过程
*   我想成为一名经理，我将在另一家公司找到那个职位
*   我是你的同事，你现在做我的经理感觉太怪了

这些原因都不能反映你作为管理者的能力，但希望你能预见到这些，并提前做好计划。最好熟悉面试流程，了解你的公司在招聘后备职位方面的情况，以防这种意外发生在你身上。做好准备总比没有计划就被抓到好。

# 成为团队的倍增器

![](img/2c5aa365060acfb13a74cf922046a2ca.png)

作为一名经理和领导者，在可能的情况下加倍团队的努力对你来说更为重要。您可能会参加团队中其他人不会参加的会议，您有责任影响和倡导不同的流程和团队结构变化，以帮助整个团队。常见的团队结构变化包括在团队之间调动人员和/或雇佣更多人员。你越了解公司的发展方向，你就越能更好地倡导有利于你的团队和更广泛的业务目标的变革。

有时你需要亲自动手，这样你的团队才会相信你能完成工作，并在真正重要的时候成为一个帮手。这样做太多是很容易的，所以了解你自己和你的极限，并与团队沟通，这样他们就不会期望太多。

# 明确地重复你自己

如果你希望其他人记住一个重要的日期或改变他们的行为，那么你应该明确这些期望，并以不同的形式重复这些期望(例如，休闲、电子邮件、会议等)。).不同的团队成员会以不同的方式记住信息，多次提供这些信息会增加他们记住和理解信息的机会。如果你已经为表现不佳的人提供了明确的指导方针，那么随后发一封电子邮件，概述你所涵盖的内容，这将有助于双方记住，也可以作为一种工具，以防你需要制定绩效改进计划。

作为经理，现在你也有责任了解公司政策的变化(如假期、病假等)。)并能够与您的直接下属谈论他们。看看他们是否有问题，他们是否确切了解正在发生的变化以及这些变化将如何影响他们。

# 反馈需要更长的时间，但回报可能是巨大的

作为一名软件工程师，关于你是否做得很好的反馈可以每天测量和感受到。我能编码多少？我做了多少测试？我向生产部门发货了多少次？但是作为一名经理，需要更长的时间才能看到你的劳动成果，看到团队成就的提高和直接下属实现他们的目标。然而，作为一名经理，最大的成就之一就是把某人放在他们擅长的位置上，或者让他们更接近他们的职业目标。我试图更快地获得关于我工作表现的反馈的一个方法是简单地问“你有什么反馈给我吗?”？我可以开始、停止或应该继续做什么？”。这似乎很明显，但有时你会得到一些关于你已经擅长的事情或你应该立即停止的不良行为的非常好的信息。

# 与其做导师，不如做教练

作为一名经理，我希望自己开始做更多的事情，那就是更多地做教练，而不是导师。我曾经是一名导师，也有过许多优秀的导师，所以作为一名新经理，我觉得这很舒服，但我剥夺了其他人以自己的方式解决问题的权利，并让他们感受到那种成就感。作为一名教练，你需要建立一种关系，这样你就有了最低程度的信任和理解，你关心对方，没有恶意。

如果你在思考或解决问题，那么你不是在指导别人。你考虑的是如何解决问题，而不是要求被辅导者进行批判性思考，依靠他们所知道的和能够用来解决手头问题的东西。问一些试探性的问题，重新构建你对他们的理解，看看他们能想出什么。你会对其他人的能力感到惊讶和印象深刻，这将提高工作满意度和整体参与度。蔻驰其他人，以便在你的组织中有更多的领导人来接管你的位置，这样你就可以去更大更好的事情！
