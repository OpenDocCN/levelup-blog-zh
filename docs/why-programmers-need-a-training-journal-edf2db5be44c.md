# 为什么程序员需要培训日志

> 原文：<https://levelup.gitconnected.com/why-programmers-need-a-training-journal-edf2db5be44c>

![](img/575c683e9fbb383052ec60a8d85608f9.png)

编程的领域既广又深。不管你已经编程一个月，一年，十年，或者更久，我们所有人都有一些共同点:有很多信息要学习，我们需要不断地提高我们的技能。从这个意义上说，所有的程序员都在接受“培训”

因此，我作为一名长跑运动员所使用的许多训练习惯现在都应用在我目前的耐力活动中:编程。这是其中之一:使用培训日志。

# 培训杂志

我在寻找模式。

每天晚上睡觉前，我会走向我的办公桌，拿起一本拍纸簿和一支笔，写下一些关于我当天跑步锻炼的笔记。这个非正式的过程包括我跑的英里数，我的感觉，我吃的食物。

例如，“不要在长跑的前一天晚上吃一顿丰盛的海鲜大餐。”或者“我起跑太快了。”或者“我今天跑步的时候感觉很棒…昨晚吃了简单的晚餐(鸡肉和意大利面)。”

我用这些笔记作为反馈，为以后的跑步和比赛做一些调整。换句话说，我坚持写跑步训练日记来帮助我变得更好，达到我的目标。成功了。

所以我现在应用同样的想法来成为一名更好的程序员。每天晚上，我都会打开我的海军鼹鼠皮笔记本，写下日期，并记下一些关于我当天编程工作的笔记。

# 一天比一天好

自我提高是我开始写编程日记的主要原因。我想到什么就写什么，但我的评论倾向于涵盖以下主题:

*   **我犯过的错误**和学到的重要教训。
*   **我尝试的学习策略**，这就是我如何使用[抽查](https://amymhaddad.com/four-ways-to-learn-programming-topics)技术的。
*   **生产力**:我在[巅峰](https://amymhaddad.com/program-better-in-ten-mode.mdx)的工作方式。
*   **话题**现在在学的和以后想学的。

就像我跑步的日子一样，我在寻找随着时间推移而出现的模式。类似的评论开始再次出现，这表明是时候做出调整了。或者，同样重要的是，我知道该继续做什么。

最重要的是，我希望变得更好。花几分钟时间反思有助于我在这个过程中前进。

# 保持势头

我现在写日记的过程中有一件事是我跑步时没有做的:每天发现一些积极的事情。这是我日记的第二个主题。

学习编程并不容易。这个过程也不是很快。这需要很大的毅力:旅程更像一场马拉松，而不是短跑。从你的表现中找到积极的一面，这可以每天提醒你，你正在创造 progress⁠——即使是在那些你感到沮丧、觉得自己没做多少事情的日子里。

你的工作总是有积极的一面。为此，我问自己一个问题:我现在比早上更好吗？然后，我写几个字来回答一下。

所以问题还是没有解决。但是你找到了正确的数据结构。或者你想到了正确的算法，并把它写在纸上。这就是进步。

或者你一直在纠结的一个概念终于有了意义。那也是进步。

花几分钟时间反思有助于你回忆起这些小小的成功，这会激发你的动力和信心。

# 不确定的时刻

随着时间的推移，每天的小收获可以导致指数级的大进步，这正是我跑步时的成功秘诀。

在长跑训练中，没有一次特别的跑步让我保持良好的状态。相反，这是一个每天出现并努力工作的问题。收益很小。但是足够多的小收获让我得到了我想要的结果。

同样，成为一名优秀的程序员并不意味着写出一个伟大的程序或者解决一个棘手的问题。相反，它是关于每天变得更好一点。

一本编程培训杂志提醒我:每天的努力很重要。毕竟，成功只不过是日积月累的小胜利。

# 让它成为现实

理论上，写训练日志似乎是个好主意。但是实际上是什么样的呢？如何让它成为一种习惯？

我发现一个简单的方法来整合一些新的东西到你的日常生活中，那就是把它和你已经做的事情联系起来。例如，刷完牙后，写日记。这样，你就把一些已经成为你日常生活一部分的事情(刷牙)和一些新的事情(写在你的训练日志上)联系起来了。这样做可以让新事物更容易整合和记忆。

最终，如果你不写，你会觉得你错过了什么。一旦我花些时间思考一下，我就觉得这一天的工作又回到了原点。

记住，你不一定要写段落，尽管有时候你可能会写。通常，几个句子甚至几个要点就足够了。

但是当你开始看到结果的时候，另一个鼓励你定期写训练日志的动力就来了。很快模式就会出现，你会找到不同的做事方式——把事情做得更好。你会对你正在处理的问题或你试图修复的 bug 有想法。

当你通读几个月前的笔记时，你也会有很多材料可以回顾。这不仅有助于自我提升，也有助于了解你已经走了多远。
