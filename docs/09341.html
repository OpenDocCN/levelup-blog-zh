第10章—缺少时间、成本或质量

这是关于编码艺术的第10章，按照古代书籍《孙子兵法》的松散格式编写，是关于如何编写软件代码的良好意图和观点的系列文章的一部分。

我们可以区分软件开发的各种类型的约束，即:

  • 时间 —由于展览等外部事件，截止日期可能很难确定且不可移动。
  • 成本——由于与客户的固定价格协议，它限制了资源的数量，因此限制了开发软件的团队规模。
  • 质量 —由于所需或期望的过程、团队的技能和培训，以及这些质量改进和缺陷减少步骤的时间分配。
  • 功能 —通常是质量、数量以及因果意图或更正式的需求和规范的详细程度的子集。
  • 非功能性——包含客户期望满足道德法则(软件必须工作)的所有东西，但通常不明确指定包括负载、性能、安全性、兼容性、可伸缩性、模块化、可支持性、可升级性、可观察性。

可以自由遍历的编码称为“基于时间和材料的”。所有人都可以看到风险，并理解风险。

有很多约束的编码称为“固定价格”。风险在于这些约束会给编码者带来压力。这种类型的参与需要更多的计划和风险考虑，以减少以后压力的可能性。

在 时间 有限的情况下，应优先对以下项目进行编码:高重要性功能、新的复杂算法、执行时间关键功能。

在 成本 受限的情况下，评估加工的投资回报率和收益，什么是必要的，什么是最好的。

如果 质量 是优先考虑的事情，那么必须将精力放在代码的手工和自动化检查上。应尽早使用真正的工具“lint”和其他静态分析仪。编码中的质量应该是一件全面的事情，编码标准、库的方向、使用和避免的特性、单元测试以及“ Done ”术语的定义。高质量的代码应该运行良好，并且能被机器和人很好地阅读。高级经理应该经常提到质量这个词，并对值得注意的项目给予表扬。

为了鼓励高性能，应该考虑使用专业的战斗勇士——敏捷教练、首席工程师。然后通过午餐、书籍、杂志和订阅在线学习资源来鼓励部队。那些在认证和考试的战场上证明自己的人应该受到尊重，接受培训，并为他们的盔甲佩戴徽章。

那些成为scrum大师或产品负责人的人应该在他们的艺术方面得到良好的训练和实践。

编码是一项技能，那些更有经验的人应该教育和指导那些资历更浅的人。从实习生到初级到中级再到高级再到校长，开悟的路径要标得清清楚楚。

如果 功能 是优先考虑的，那么要小心不要忘记和放错非功能需求。当产品所有者关注特性时，编码者必须时刻注意每一个标准。开发人员不能做任何假设，否则以后会反咬他们一口。

  • 因为身处战争迷雾中的编码人员不会混淆功能性和函数式编程，也不会忘记面向对象的侧面。

如果 非功能性 物品对编码者来说是清楚的，那么这是一支你不应该离开的军队！因为这是一个罕见的发现。仅次于有时间重构和处理技术债务的军队，可能还有一个星期五啤酒冰箱。

  • 清楚框架的用途和你使用它们的理由，把它写下来以防你忘记。
  • 采用2层、3层模型或其他你选择的模型，并有一个精确的命名约定和文件夹结构。
  • 通过使用存储过程和带有调优索引的良好代码来优化数据库性能。不要仅仅依靠缓存的灵活性来帮助糟糕的编码。
  • 记住要给表添加索引，即使是小的索引，因为它们可能会意外地增长。
  • 如果数据应该保持安全，那么使用只读视图。
  • 不要忘记总是在晚上提防攻击，因为忘记它是一个基于零的指数。
  • 不要使用神奇的数字，因为它们隐藏着除了作者之外任何人都无法理解的黑魔法，因为作者经常会带着混乱离开去一支新的军队。
  • 结构化代码来分担工作和责任是一件好事，使用MVC。

除了项目之外，还有其他可能影响发布的条件:

  • 缺乏编码标准或对代码布局没有一致意见
  • 关于哪个是唯一的IDE和开发平台的争论
  • 配置分支策略的扩展讨论
  • 会议
  • 工具的责备
  • 缺少凭据和访问权限，包括本地管理权限
  • “互联网瘫痪了”

如果没有的编码标准或关于代码布局、函数命名、注释的协议，混乱就会接踵而至。一个人的速度会阻碍另一个人，个人的贡献与一个统一一致的团队的力量不匹配。

如果对 一个真正的IDE 无休止的争论，那么不仅时间和精力会被浪费，内讧也会变得很普遍。所有的路都通向同一条路，所以选择一条并坚持走下去。如果你不能选一个，那就选一个聪明的人来选，如果没有聪明的人，那就找一个硬币来扔。

当主干上的喋喋不休和版本控制 系统的 分支策略停止了项目工作或减慢了进度，那么就需要一个勇敢的人。这个人应该描绘一个愿景，每个人都应该跟随，句号！

会议 是可以忍受的事情，但需要努力像滚石一样转移它们，所以它们要瞄准正确，不要转移注意力。总是有一个议程，明确谁参加，有明确的角色和任务，提前准备，决定和同意行动，做会议记录。大多数情况下，尽量让人们站起来，因为会议很短。把一天中只有编码可以进行而没有会议可以预定的神圣时段屏蔽掉。另一方面，每日站立对于一个平稳运行的团队至关重要，所以请积极参加并及时更新。

与“拙匠常怨工具”结盟，是“别人的错”。有许多工具，努力训练并使用给定的工具。提出新的工具只是为了在战争中获得直接的实际利益和优势。如果将军们没有意识到新工具的存在或好处，那么就在营火周围大声歌唱这些。

【互联网瘫痪】

这是一个从山顶回响的呼喊，它误导，误导就像来自敌人的话语。只能有一个互联网，但有许多附加条件。买条备用线，柜子里有个备用路由器，在安静的时刻试试移动热点。最重要的是向IT团队要求更多的带宽，因为数据贪婪之神总是需要更多的带宽。

有经验的开发人员一旦行动起来，就不会不知所措，一旦项目开始，他就不会不知所措。

因此

知己知彼，百战不殆。

如果你知道问题从何而来，并且知道如何编码，那么你就可以做出成功的软件。
