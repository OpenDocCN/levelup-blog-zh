# 作为软件开发人员说“我不知道”

## 这只是需要时间——短暂而甜蜜

![](img/840edb67ba69c5f9369d2d452d9d0620.png)

蒂姆·高在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的照片

让我们面对现实吧，我们不是上帝，尽管我们有时会觉得我们已经连续编码了 16 个小时，只是构建了客户想要的最棒的按钮…当然，创建一个按钮实际上需要大约 16 秒，按照客户要求的方式设计它需要整整一分钟；但是你明白了。

我们每天都在做一些很酷的事情，在这个过程中会遇到障碍和问题。浏览器兼容性、库版本过时、竞争条件、状态管理中的漏洞等。我们将花费令人痛苦的数小时、数天或数周时间来尝试解决这些问题，并将告诉客户或管理层这是不可能的…用当前的应用程序是不可能的。你害怕告诉他们你不知道如何解决这个问题，并把它归咎于当前的技术，这样你就不会显得无能。

嘿！说“我不知道”没关系。什么都不知道其实也没什么。我已经说了很长时间了，我会像这样继续下去，“我不知道，但让我找到答案，然后给你答复。”

你只是需要时间来研究，找到一个解决方案，并估计需要多长时间来实现。

通常，你会发现无论你想做什么，都没有人做过。这很酷，我让客户知道这一点。我告诉人们，任何东西都可以建造；只是需要时间。

就是这样。没有疯狂的故事发生在我身上，我如何克服它。你会经历这些请求/时刻/问题，只是不知道如何解决它们。您将研究如何实现它。只是需要时间。
