编码是一个特定的术语，但它实际上包含了一些不同的东西。

您可以出于不同的原因进行编码，并且根据您编码的原因，成就会有所不同。

如果你不小心的话，你可以每天都写代码，并且仍然保持同样的技能水平。

# 编码意图

你在编码时的意图很重要，因为如果你过于倾向于一个方向，你将开始缺乏其他领域的技能水平。

基本上有 3 种类型的编码你需要保持熟练。

*   学习——你寻找新信息和学习技术或工具如何工作的能力
*   理解——理解比学习更深刻。这本质上是学习+时间+把握概念。
*   应用——应用是指您从概念上理解技术，并且能够将您的知识应用/添加到项目或代码库中。

## 学习编码

编码学习是关于你对一项技术的介绍。您可以跟随教程，并充分理解这些概念，以便就它们展开表层对话。

这就是你应该以知识广度为目标的地方。你应该试着找出新技术提供的所有特性，它的优点/缺点是什么，作为一个程序员，这项技术实际上为你做了什么？

编码学习是你倾听、收集输入、阅读和思考的时间。您可以并且应该开始测试代码样本，并开始修补事情是如何工作和实现的。在某些情况下，您可能无法在此阶段随意修改代码。

例如，如果您对自己尝试一些 GraphQL 查询感兴趣，您可以通过快速设置来实现，但是如果您试图开发一个联邦服务器，则需要花费更长的时间来设置，并且概念上的理解将是一项艰巨的任务。

## 编码理解

理解编码不仅仅是寻找如何做，而是寻找代码背后的*为什么*。教程是有帮助的，但是在这个阶段阅读可能会对你有更好的帮助，因为文档可能会提供更多关于技术细节的解释。

你的目标是逐渐减少对视频的依赖，跟随*别人写的*并理解*为什么*写代码。这里的主要重点是让自己做好准备，根据自己的知识开始编写代码，并能够理解您正在使用的技术需要哪些元素才能正常工作。

这个过程永远不会随着技术的出现而结束。如果你遇到了新事物，这是一个获得更好理解的好机会。

## 要构建的编码

最重要的是能够运用你的知识。是的，你可能已经在学习和理解阶段实现了一些代码，但是环境要求不同的编码推理。

现在是时候从一个空白文件开始，看看你有什么发现。试着让一切独立工作，测试你对这项技术应该做什么的理解，以及你需要写什么代码来让它工作。

卡住的时候，参考资料就可以了。但是我看过一次这个规则，我觉得应该遵守:

> 不要复制你不理解的代码——未知的

如果你想节省一些时间；这完全没问题，但是如果你正在复制和粘贴你不熟悉的代码，这是一个很好的学习机会。堆栈溢出是了解其他人如何解决问题的一个很好的资源，但它不是仅仅复制代码的地方。

遵循这个规则，随着时间的推移，你的每一种不安全感都会被理解。

## 结论

为了确保你的成长，你应该一直处于这些阶段中的一个以上。无论你是在学习一门新语言的基础知识的同时加深对一门你已经知道的技术的理解，还是用一门你刚刚掌握的新技术建立一个新项目，你至少应该永远处于两个阶段。

你可能有一个最喜欢的阶段，这没关系，但是你必须经历所有的阶段，以确保你作为一个开发者正在发展你的技能。

保重。
