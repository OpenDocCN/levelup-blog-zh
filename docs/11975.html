给科特林泉增添了霸气

Spring使得创建REST API变得极其容易，但是作为API的消费者，我们需要文档来了解什么是可用的以及如何使用它。这就是 Swagger 介入并解决问题的原因。

简而言之，Swagger以json模式的形式定义了API文档的结构，指定了可用的路径、各种路径所需的参数以及可以预期的结果类型。因此，人类和机器都可以有一个标准的方式来读取API。阅读对人来说很容易，命名约定对大多数人来说定义得很好，不需要参考模式文档就能理解。然而，写它可能是棘手的，因为你必须记住每一项的确切措辞。

对于Spring用户来说，有一个简单的方法。只需将 SpringDoc 依赖项添加到您的类路径中，就可以在运行项目时自动生成Swagger文档。

dependencies {
  ...
  implementation("org.springdoc:springdoc-openapi-data-rest")
  implementation("org.springdoc:springdoc-openapi-ui")
  implementation("org.springdoc:springdoc-openapi-kotlin")
}

在您bootRun应用程序后，swagger ui将在您的应用程序上可用，可通过路径- /swagger-ui/index.html访问，例如http://localhost:8080/swagger-ui/index.html 。

而实际json模式文件的链接在页面/v3/api-docs上，例如http://localhost:8080/v3/api-docs 。
