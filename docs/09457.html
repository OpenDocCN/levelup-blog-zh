准备数据工程面试是很棘手的。不一定非要这样。我将分享我在2021年年中接受15+ Jr .数据工程师采访的收获。

数据工程是2021年增长最快的工作之一。难的是，你不知道从哪里开始，你也无法在网上找到很多真正的面试问题和经验。

下面我将和你分享一般的面试过程，这样你就能知道会发生什么:

数据工程面试流程:

这是大多数公司遵循的一般流程:

  1. 筛选轮:在线测试(1–2道编码题:leet code Easy-Medium+2–4 SQL(Advanced))。
  2. 第二轮:Face2Face面试:DS Algo + SQL Advanced + Spark Basics。
  3. 第三轮:DS Algo + SQL Advanced +分布式系统设计或 Spark +项目讨论。
  4. 第四轮:最后一轮(项目讨论/设计模式知识+ Spark + Scala/Python + Hive问题)。
  5. 第五轮:人力资源讨论。

数据工程师vs软件开发人员面试:

作为一名数据工程师，你不必专注于难懂的Leetcode问题。此外，对问题进行编码更像是数据工程工作，而不是核心算法问题。

省省力气，只准备LC容易和中等。

进来可怕的高级SQL + Spark:
作为一名数据工程师，编写复杂的SQL查询一定是你的强项。这意味着不仅仅是INSERT、DELETE、WHERE语句，您还需要了解以下内容:

  1. 窗口功能
  2. 子查询
  3. 递归子查询
  4. 通用表表达式
  5. 使用连接回答问题。

从 HackerRank(做题全)，Leetcode(免费的就够好了)学习和练习高级SQL。我很快会发布SQL +面试经验。敬请关注或关注我的李/Medium。

学习Spark和Hadoop概念:

最佳资源:

  1. 数据悟性
  2. 技术与应用
  3. Simplilearn

Spark编码实践:这方面的内容不是很容易得到。只有少数网站为您提供Spark面试问题。未来几天我会整理一些问题。

数据工程师编码回合看起来像这样:

  1. DS & Algo问题 : LC Easy / medium:经常涉及读出数据和使用字典进行处理。
  2. SQL问题:2–3:肯定会被问到窗口函数和高级连接的工作知识。
  3. Spark编码:将上述SQL代码转化为spark/ word count类型的问题。必须知道:map，flatmap，reduce，reduceByKey，joining，collect_list/collect_set和wordcount类型问题。
  4. Spark概念:与通用Spark和优化相关的概念。
