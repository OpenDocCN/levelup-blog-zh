程序员最佳实践—更好的命名

原文：https://levelup.gitconnected.com/programmer-best-practices-better-naming-9785bfa4b754?source=collection_archive---------15-----------------------#2020-06-21

要成为一名职业程序员，我们必须做很多事情才能成为一名优秀的程序员。

在本文中，我们将探讨成为一名优秀程序员的最基本要求，即通过更好地命名事物来清理自己。

编写干净的代码

童子军总是被告知要把他们的营地打扫得比他们发现的还要干净。

我们可以将同样的原则应用于代码。我们离开时比发现时干净。

通过这种方式，我们将始终拥有一个清晰的、易于阅读和使用的代码。

命名是编写易于使用的代码的开始。

使用透露意图的名字

我们可以通过以揭示意图的方式命名来开始清理代码。名字应该有足够的描述性，我们可以通过阅读它的名字来理解它。

例如，不写:

let x;

我们写道:

let elapsedTimeInMinutes;

两个名字差别很大。

我们马上就知道elapsedTimeInMinutes是什么意思，但是我们不知道x是什么意思。

因此，我们应该用我们理解的方式命名事物。代码应该明确地告诉我们它在做什么，而不是隐含地做所有的事情。

我们不知道x是什么，所以我们不知道它有什么。

同样，如果我们有这样一个函数:

const getItems = () => {
  const list = [];
  for (const a of arr) {
    if (a % 2 === 0){
      list.push(arr)
     }
  }
  return list;
}

我们不知道list和arr有什么。

因此，我们只是不知道它试图做什么，即使间距和缩进是好的。

没有假消息

我们不应该用我们的名字留下错误的线索。例如，如果变量实际上是一个数字，我们应该只给变量加上前缀num。

此外，我们应该保持我们的拼写一致，这样我们就没有机会误导人们。

有意义的区别

我们应该在程序中给我们的东西命名，这样我们就可以区分它们。否则，我们会被名称相似但行为不同的事物所迷惑。

例如，如果我们有两个变量名叫做productInfo和productData，那么它们太相似了。如果它们不同，那么它们应该有更容易区分的名称。

使用易发音的名字

应该使用容易发音的名字，这样我们就可以记住它们并谈论它们。

如果我们不能说出这些名字，那么我们就很难讨论它们。

使用可搜索的名称

名字应该是可搜索的，以便我们以后可以找到他们。因此，我们应该像前面提到的那样使用描述性名称。

此外，我们应该避免幻值，并将它们赋给命名的常量。这样，我们可以很容易地找到它们，只需在一个地方进行更改。

成员前缀

我们不需要成员前缀，因为我们可以通过它们的位置来判断它们是类或对象的成员。

例如，不写:

const obj = {
  mName: 'joe',
  //...
};

我们可以写:

const obj = {
  name: 'joe',
  //...
};

心理映射

我们不应该在不必要的时候让大脑多工作。

因此，我们应该把一切都写清楚，这样我们就不必用大脑来映射它们了。

类名

类名应该是像Car或Dog这样的名词短语。

方法名称

方法名应该是动词或动词短语，如save或speak.

每个概念一个词

每个概念应该有一个词，而不是同义词。

例如，我们可以用fetch来代替fetch、retrieve或get。

语境

我们应该给我们的名字添加一些上下文，这样我们就可以理解为什么它们会以这样的方式命名。

例如，如果某个东西是地址的一部分，那么我们应该在它们前面加上表示它是地址的前缀。

因此，我们可以不写state，而是写addrState，这样我们就可以确定我们引用的是地址中的州，而不是其他州。

结论

我们可以通过正确命名来使代码更加清晰。

名字应该是描述性的，容易发音，并且一致。

此外，我们应该在名称中给出一些上下文，这样我们就不会被它们搞混了。

虚假信息也应该纠正，以避免混乱。
