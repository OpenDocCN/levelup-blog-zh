如何日常提高软件架构技能

原文：https://levelup.gitconnected.com/how-to-improve-software-architecture-skills-daily-6f362d4e6493?source=collection_archive---------0-----------------------#2021-06-07

你可以在任何编程任务中做3个实践练习。

作为一名软件开发人员，成长的途径有很多，所有这些都需要时间和努力。我建议实践练习可以成为你日常任务的一部分，不需要开始你自己的项目或参加训练营。

为你遇到的每个问题找到两个或更多的解决方案

提高解决问题和创造力

无论您当前的任务是调试、编写新代码还是重构，开发任务都是关于问题和解决方案的。问题的范围可以有所不同，从重用现有代码到选择数据库。

即使你已经选择了解决方案，也要花时间做这个思想实验。

列出权衡并选择一个解决方案

提高优先级和细节思考。

让自己接触更多的标准，了解为什么我们应该或不应该做事情的其他考虑因素。

查看解决方案，并对每个解决方案满足每个标准的程度进行排序。保持你的判断清晰；即使是“糟糕”的解决方案也应该在某些方面获得好的排名。

最后，对标准进行优先排序——试着理解哪些方面对你的公司最重要，并选择能为你的案例带来最大利益的解决方案。这是一个技术领导者的工作。

与非技术人员进行技术讨论

增进沟通和技术理解

不同的人理解你作品的能力不同。与他们交谈会迫使你缩小视野，将细节概括成高层次的概念或放大细节。我会积极地开始讨论或尝试向某人解释某个问题，以测试我自己对该问题的理解。

供讨论的详细示例

我写下了4种类型的人的例子，你的互动是什么，以及一个解释听起来如何。

高级工程师
是在相关领域比你更有经验的人，或者是擅长问正确问题的人。向他们展示需要你对主题和内幕有深刻的理解。在这种演讲之后，我经常发现自己会回去做更多的研究——这太棒了！

“我集成了WebRTC:我添加了MAOI API，集成了Janus库，并将WS转换为WSS”

团队成员/初级
是你需要指导的具有扎实技术技能的人。这需要你把细节归纳成清晰的指导方针和流程。

“我集成了WebRTC:添加了一个名为…的新API，集成了一个名为…的库，它的文档是…并将web套接字转换为安全web套接字，我这样做是为了……”

非开发人员的技术人员，如QA或项目经理
出席讨论，你需要排除障碍或获得他们的支持。如果你能在他们不知道你工作的每一个技术细节的情况下与他们交流，那将是最好的。

“WebRTC集成预计在3个sprints内完成，需要与后端团队协作。我想多花4天时间来增强安全性”

非开发人员和非技术人员，比如人力资源或你的朋友
是这样一种人，你应该能够在求职面试中分享你的工作或展示你的品质，而完全不用讨论项目的技术方面。

对HR:“我集成了WebRTC，这是一个在web上传输视频的协议。我的工作包括与其他团队合作，由我主导对话，但我们都同意共同解决问题。我还主动增强了该功能的安全性。”

最后

我建议你可以在你正在做的每个开发任务的范围内做3个练习，这将帮助你提高你的架构技能。
