WebAssembly成为继HTML、CSS和JavaScript之后的第四种网络语言

在相当平静的一年后，WebAssembly在12月初收到了一些重大消息——它被W3C联盟正式推荐为web语言。万维网联盟(W3C)是万维网的主要国际标准组织。

自2017年宣布WebAssembly以来，它已经获得了大量关注和快速采用。在前几年，我们看到1.0规范在所有主流浏览器中被创建和集成。

2019年WebAssembly的另一个消息是字节码联盟的成立，该联盟看起来“通过合作实施标准和提出新标准来打造WebAssembly在浏览器之外的未来”。

我们仍在等待WebAssembly真正站稳脚跟并获得大规模采用，随着每次更新，我们离目标越来越近。毫无疑问，W3C声明对于公司来说是一个巨大的进步，我们需要继续降低使用WebAssembly的门槛，使产品更容易用它来构建。

TypeScript的使用激增——许多开发人员坠入爱河

2019年是打字稿年。不仅TypeScript已经成为向JS代码添加数据类型的实际选择，许多开发人员也经常选择在个人项目和工作中使用它而不是普通的JavaScript。

在2019年初发布的 StackOverflow调查中，TypeScript与Python并列第二，是最受欢迎的语言，仅次于Rust。在2020年初发布的新调查中，看到TypeScript攀升得更高并不奇怪。

TypeScript已经占据了web开发世界——包括前端和后端。一些开发人员试图将TS视为一种时尚，并认为它会像Coffeescript一样，但TypeScript已被证明解决了JS开发人员的一个核心问题，并且似乎只会越来越多地被使用。

TypeScript通过集成所有主要的文本编辑器为web开发人员提供了更好的开发体验。JavaScript开发人员将TypeScript视为一种工具，这种工具可以减少错误，同时也更易于阅读代码，因为类型和对象接口提供了自我文档。

值得注意的是，随着TypeScript在2019年通过NPM下载量的React，它变得多么受欢迎。它的下载量也远远超过了Flow和Reason等竞争对手。

T ypeScript和React解决完全不同的问题，所以这并不意味着直接的比较。这只是TypeScript受欢迎程度的一个证明。

TypeScript v3.0 于2018年末发布，到2019年已经发布了3.7版，其中包括更新的ECMAScript功能，如可选的链接和nullish运算符，以及对类型检查功能的改进。

HTML和CSS更新

感谢克里斯·科伊尔在他的文章中的反馈。我包括了他提到的显著变化。

HTML和CSS是web的核心组件，尽管它们没有以JavaScript的极快速度发展，但它们在2019年仍有一些显著的改进，这将对我们如何构建应用程序和改善我们网站的UX产生巨大影响。

HTML最大的两个变化是原生惰性加载和非jank流体图像加载。大图片一直是web性能的一个问题，我们已经解决了这个问题，以更好地处理我们加载它们的方式。有了对惰性加载和长宽比识别的原生支持，我们可以获得无缝的图像，而不需要在JS中实现任何额外的功能。

对于本机惰性加载，只需在映像中包含loading="lazy":

<img src="celebration.jpg" loading="lazy" alt="..." />

对于非jank图像，我们可以使用位于旗帜后面的intrinsicsize，或者您可以使用width、height、aspect-ratio。

// intrinsicsize
<img src="image.jpg" intrinsicsize="800x600" />

或者

// width, height, aspect-ratio<!-- HTML -->
<img src="image.jpg" width="800" height="600" />// CSS
img, video {
  aspect-ratio: attr(width) / attr(height);
}

对于CSS来说，2019年已经有很多慢慢到来但实际落地的变化。两个最令人兴奋的更新是prefers-reduced-motion和可变字体

在HTML/CSS方面有一个坚实的基础并跟上最新的改进是非常重要的。即使你在React和Vue中构建应用程序，它最终也会在浏览器中被转换成纯HTML和CSS。

Svelte获得了关注，但在推出第3版时采用率落后

在已经拥挤不堪的前端框架世界中，Svelte找到了一种将自己融入对话的方式。然而，正如我们在文章开头看到的，这还没有转化为大量的实际采用。最能概括苗条的就是“简单却有力”。在苗条网站上注明的3点是:

  1. 少写代码
  2. 没有虚拟DOM
  3. 真正的反应

Svelte试图将其大部分工作转移到编译步骤，而不是在运行时在浏览器中完成。Svelte有一个基于组件的架构，可以编译成纯粹的HTML和普通的JavaScript，同时也承诺减少样板代码。它使用反应式编程，直接更新DOM，而不是区分虚拟DOM。

苗条提供了一些新的和令人兴奋的前端景观…提供更少。在2020年，观察Svelte如何成长和发展将是一件有趣的事情，希望我们能够获得一些大规模使用它的例子，看看它如何与React、Vue和Angular等更大的竞争对手进行比较。

静态站点继续被利用，开发人员采用JAMStack

随着像Gatsby这样的框架使用率的增加，Netlify这样的静态网站主机的快速增长，以及无数无头CMS公司的涌现，静态网站证明它们将成为web不可或缺的一部分。

静态网站将旧网站与新工具、库和更新结合起来，提供了无与伦比的体验。我们能够使用React这样的现代库来构建我们的站点，但是在构建时将它们编译成静态HTML页面。由于所有页面现在都是预先构建的，因此不需要服务器时间来根据请求将它们与数据混合在一起，这些页面可以立即提供，并利用在全球cdn中缓存的优势，使内容尽可能靠近您的用户。

静态站点使用的一个流行的编程模式是 JAMStack (JavaScript、API、标记)。这是一种混合的静态/SPA方法，其中页面被静态地提供，但是一旦在客户端上，它被更多地视为一种SPA，使用API和用户交互来发展UI的状态。

pwa看到了更多的增长和采用

静态网站是获得非常快速的产品的方法之一，但它们并不适合所有的应用程序——另一个很好的选择是PWAs(渐进式网络应用程序)。PWAs允许您在浏览器中缓存资源，以使页面立即响应，并提供离线支持。此外，它们允许后台工作人员提供本地功能，如推送通知。

一些人甚至声称PWAs甚至可能取代原生移动应用。无论事情最终如何，毫无疑问，PWAs将在很长一段时间内成为公司产品制造的重要组成部分。

GraphQL继续受到开发人员的喜爱，在科技公司中得到进一步采用

GraphQL有望解决传统的基于REST的应用程序带来的许多问题。开发人员很快爱上了GraphQL，科技公司也终于开始采用它。几年前，GitHub用GraphQL编写了它的最新API，许多其他组织也在做这种改变。

GraphQL应用程序是数据驱动的，而不是端点驱动的，允许客户端声明他们需要的确切数据，并从服务器接收相应的JSON响应。GraphQL APIs提供了一个模式来记录所有数据及其类型，使开发人员能够全面了解API。

由于GraphQL APIs提供了一个完全类型化的模式，它还可以与使用TypeScript的应用程序很好地集成。使用类似于 GraphQL代码生成器的工具，它可以读取我们的客户端代码中的查询，并将它们与模式进行匹配，以提供将流经我们整个应用程序的类型脚本类型。

GraphQL下载量在过去的一年中翻了一倍多，Apollo开始成为使用最多的框架。

如果我们看看JS调查的状态，我们会看到 GraphQL现在是最受欢迎的数据层工具，而 Redux已经大幅下降。GraphQL在开发人员采用方面进展顺利。

CSS-in-JS势头强劲

web开发的进展感觉像是已经走上了在JavaScript下统一一切的道路，这表现在采用CSS-in-JS，其中使用JavaScript字符串创建样式。

这允许我们通过import / export使用普通的JavaScript语法来共享样式和依赖关系。它还简化了动态样式，因为CSS-in-JS组件可以将props插入到样式字符串中。如前所述，脸书甚至可能将CSS-in -JS视为前端的未来，并将发布他们自己的库。

下面是经典CSS和CSS-in-JS的一个例子。要用普通CSS处理动态样式，你必须管理组件中的类名，并根据state / props更新它。您还需要为每个变体创建一个CSS类:

// Component JS file
const MyComp = ({ isActive }) => {
  const className = isActive ? 'active' : 'inactive';  return <div className={className}>HI</div>
}// CSS file
.active { color: green; }
.inactive { color: red; }

使用CSS-in-JS，您不再需要管理CSS类。您只需将props传递给一个样式化组件，它就会用声明性语法处理动态样式。代码更加简洁，我们对样式的关注有了更清晰的分离，并允许CSS基于props管理动态样式。它现在读起来就像普通的React和JavaScript代码一样:

const Header = styled.div`
  color: ${({ isActive }) => isActive ? 'green' : 'red'};
`;const MyComp = ({ isActive }} => (
  <Header isActive={isActive}>HI</Header>
)

CSS-in-JS的两个领先库是样式组件和情感，带有情感传递样式组件，供2019年下载。这两个库已经从其他CSS-in-JS选项中分离出来，看起来它们将继续快速增长。

对2020年的预测

  • 随着代码拆分和PWAs的进一步利用，性能仍然是web最重要的方面。
  • WebAssembly变得越来越常见，看到了真正的采用，并在产品中使用。
  • GraphQL在新创公司和新项目方面超过REST，而老牌公司则进一步向它转移。
  • TypeScript成为新创业公司和项目的默认选择。
  • 我们开始看到真正的应用程序建立在没有服务器的区块链上，使得网络更加开放。
  • CSS-in-JS可能会成为默认的样式方法，而不是普通的CSS。
  • “无代码”应用变得更加流行。随着人工智能的改进和应用程序的更多抽象层，构建应用程序变得越来越容易。2020年，我们可能会看到无需编写代码就能创建应用程序的显著转变。
  • Flutter可能会取代React Native，成为构建跨平台移动应用的首选方式。
  • Svelte将会看到更多使用该技术建造的真实项目。
  • Deno(Node的创建者搭建的TypeScript运行时)看实际使用。
  • AR/VR使用像 A-Frame 、 React VR 和 Google VR 这样的库，以及对浏览器中原生AR/VR工具的改进，取得了长足的进步。
  • 集装箱化的影响(即Docker，Kubernetes)在前端过程中变得更加普遍。
