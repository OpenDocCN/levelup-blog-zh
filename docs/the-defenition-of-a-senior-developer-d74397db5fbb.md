每个有软件开发人员的公司都有他们对高级开发人员的定义。对一些人来说，这是年龄，经验的长度，但对其他人来说，这是两者的结合。

许多初级/中级开发人员想知道你什么时候成为高级开发人员。但是你应该问的问题是，你是怎么认可一个资深开发者的？

在这篇文章中，我想告诉你我对此的看法。它对每个人或组织来说都不一样。但是你可以用它来定义你的定义。

# 为什么？

我写这个是因为我听到开发人员使用术语“高级”,而不理解它是什么。看完这篇文章，我希望每个人都明白什么是高级开发人员。

# 聊什么

1.  如何认可一个资深开发者？
2.  不可能什么都是学长
3.  学长应该解释原因
4.  高年级学生可能是首席开发人员

# 1.如何认可一个资深开发者？

要识别一个高级开发人员，我们应该知道如何才能发现他或她。我们必须知道我们在寻找什么样的技能和特征。

我的清单是这样的:

*   以身作则
*   强大的沟通能力
*   不同技术和环境的经验
*   5 年以上开发工作经验
*   缺乏傲慢
*   有益的
*   很强的指导、教学和解释技能
*   病人

这些是高年级学生应该具备的最重要的技能。(也许我忘了什么)。技术技能和沟通技能同样重要。

如果一个高级开发人员不能用他的专业知识来解释事情，他或她就不是这方面的专家。或者他或她不是高级开发人员。

有 10 年经验的开发人员就是高级开发人员。如果你不喜欢这个术语，就不要用它😉。

# 2.不可能什么都是学长

如果因为什么都不懂而怀疑自己是不是高级开发者？让我告诉你，那是不可能的！

a 不可能什么都是高级开发者！你可能会成为一些领域的高级开发人员，但不是全部。

我称自己为高级开发人员，我对 HTML、CSS 和 JavaScript 的大部分很有信心。我知道这些语言中有我没有接触过或者没有花时间接触过的领域。这总是取决于你参与的项目类型。

默认情况下，高级开发人员不是向导(有些是🧙‍♂️)。高级开发人员是指在其工作的几个领域拥有丰富经验的人。大多数时候这与多年的经验有关。

# 3.学长应该解释原因

一个高级开发人员应该能够解释为什么他或她以那种方式而不是另一种方式做事。这对于其他经验较少的开发人员来说很重要。

高级开发人员知道如何编写代码听起来很合理。说出并理解“为什么”很重要。一个高级开发人员可能是错误的。

> *一个开发团队和它最弱的开发人员一样好*

你不可能事事都对。最后，一个开发团队和它最弱的开发人员一样好。所以解释原因非常重要。

# 4.高年级学生可能是首席开发人员

不知何故，我注意到很多时候团队中的开发人员希望一个资深开发人员成为首席开发人员。但事实并非总是如此。

因为一些高级开发人员没有领导团队中其他开发人员的愿望。大多数高级开发人员喜欢分享他们的知识，但是领导团队是一个不同的角色。

作为一个开发团队的领导者，你不仅要写代码。你的大部分时间将会变得不同。例如在会议中，帮助其他开发人员和管理来自上层管理人员的期望。

如果你是不想当领导的资深开发者。不要为此感到内疚！有这些目标的开发者将填补这个空缺。

# 谢谢

我希望在读到这里之后，你会更清楚一个高级开发人员是做什么的。以及你对他、她或你自己的期望。

但是请记住，每个人都是不同的，所以不是每个高级开发人员都是一样的。
