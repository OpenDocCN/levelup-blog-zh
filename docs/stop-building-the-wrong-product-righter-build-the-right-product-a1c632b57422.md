*制造人们想用的产品很难。您没有获得客户期望的结果可能有很多原因。如果你做的东西不能解决他们的问题或满足他们的需求，那么你再怎么努力让他们使用你的产品也没用。*

> ***你的问题不是你没有打造合适的产品。你的问题是你在制造错误的产品。***

*这里有两个主要方面**,它们应该始终适用于让你的时间值得花在你的产品上:***

## ***你需要确认你正在解决的问题是一个真实的问题。***

***最重要的事情是**验证**你的产品。你需要确认你正在解决的问题是一个真实的问题。***

***比如我因为一个问题开始做[网页集锦](https://web-highlights.com/)。我在网上阅读了许多文章，但无法将它们组织起来，以便在需要时可以重新找到信息。我在寻找 chrome 扩展，使我能够在网站上突出显示文本，并从任何地方访问突出显示的内容。有一些，但是根据我的需要没有。这就是我打造自己产品的原因。***

> ***我喜欢它，因为它解决了我的问题。***

***这给了我信心，相信一定有人有同样的问题和需求。似乎有。***

重要的是要明白产品不是你建造的东西。产品是您的客户所面临问题的解决方案。

> 产品验证就是确定你是否有一个值得解决的问题。

但是，即使解决了自己的问题，也不意味着其他人也面临同样的问题。你需要确定你的问题有市场。市场验证是关于了解市场有多大，你能赚多少钱，以及如何接触你的客户。

人们倾向于认为他们总是不得不强制性地发明一些新的东西。这个想法被称为“先动优势”，由利伯曼和蒙哥马利在 1988 年的一篇论文中提出。但事实上，经常成为“先行者”对许多初创公司来说是一种劣势。

十年后，同样的作者发表了一篇名为**“先动者(劣势)优势”**(Lieberman and Montgomery 1998)的新论文，在这篇论文中，他们放弃了自己的许多主张。

**【先行者】**的问题在于，他们在一个尚未得到验证的市场中行动。此外，他们支付勘探费用，并允许第二推动者免费利用他们的发现。

以下是一些不是先行者的公司的例子:

*   **Google** :在 Google 之前，有雅虎、Infoseek 等搜索引擎。
*   脸书:社交网络上还有许多其他先行者，比如 Friendster 和 MySpace。
*   **Slack** :不是团队沟通的第一个产品
*   **星巴克**:在星巴克成立之前，有很多地方可以买到咖啡。

从这些发现中得出的结论是，你的产品不必解决一个全新的问题。如果已经有类似的产品，你可以肯定你的产品有一个现有的市场。这保证了你在一个已经被验证的市场上生产产品。

从先行者的辛勤工作中学习，通过做得更好让你的产品与众不同。**不要多此一举。做得更好。***** 

## ***你需要验证你的解决方案确实解决了问题。***

***一旦你知道你发现了一个值得解决的问题，确保你的产品解决了实际问题。***

***如果你解决了自己的问题，你是幸运的，因为它更容易确定你的产品是否满足你解决问题的需要。如果你解决了别人的问题，确保尽快收到顾客的反馈。***

*****顾客反馈**是你打造人们想要的产品的最有价值的工具。你应该问的问题是:我们走的方向对吗？我们的客户喜欢什么，讨厌什么，他们会为什么买单？***

***最后一段隐藏了一个症结:为了获得客户反馈，你必须已经有一个现有的 MVP。***

***老实说，当我在大学期间开始构建 Web 亮点时，我并不关心这篇文章的任何一个事实。我只是一个积极上进的年轻开发人员，想要开发一些很酷的东西来解决我的问题。但是我做了一件非常重要的事情，这件事让我付出了一些努力:**我只是在 Chrome 网上商店发布了我的产品。*****

我不记得当时我的产品是什么样的了。然而，我敢肯定它不会赢得任何奖项。我只是发表了它，然后等待。

> 如果你不为你的第一次发布感到尴尬，那你发布得太晚了。—雷德·霍夫曼

最终，一个拥有近 4000 名用户的产品和一个在 Github 上某处腐烂的副业项目之间只有一线之隔。谁知道有多少潜在的伟大产品正在那里腐烂？不要犯同样的错误，大胆尝试。

如果你不能做到这一点，就不值得在你的产品上花更多的时间。

如果你不想直接发布你的产品，把它展示给朋友、家人或同事。一旦你发布了你的产品，确保用户可以尽可能容易地提供客户反馈。

> “通过确保在产品的明显位置提供反馈电子邮件地址或电话号码，以及/或者使用 ZenDesk、Groove 和 Get Satisfaction 等工具来管理与用户的沟通，让他们感到轻松”。—你好，叶夫根尼·布里克曼的创业*** 

# ***最后的想法***

***关键是产品验证是整个过程中最重要的一步。不仅仅是打造一个产品。它是关于建造人们想要和需要的东西。为了验证您的产品能够解决问题，请确保尽早收到客户反馈。***
