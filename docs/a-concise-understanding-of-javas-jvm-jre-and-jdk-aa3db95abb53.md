对于任何开始 Java 开发之旅的人来说，你会在某个时候(很快)碰到缩写词 JVM、JRE 和 JDK。写这篇文章是为了给你一个非常清晰和简明的定义，以及它们所解决的问题。

> 有趣的事实:JAVA 最初的目的不是互联网！

# **Java 的解决方案**

有趣的是，像烤面包机这样的设备(是的，就像烤面包的烤面包机一样)在早期是 Java 创造者的优先选择。他们想创造一种独立于平台的语言，用于为消费类设备编写软件，比如电视机、遥控器甚至烤面包机。当时人们所知的 Java 或 Oak 在消费电子产品市场上做得不太好，但对该团队来说幸运的是，万维网的出现给了他们另一条追求的途径。

> 一次编写，随处运行

作为计算机、操作系统和各种设备的全球网络，互联网迫切需要独立于平台的程序。意识到可移植性的问题也存在于应用程序设计中，Java 的创造者们决定放弃他们最初的软件驱动的消费设备的概念，而专注于解决更大的问题，如可移植程序。鉴于其“编写一次，在任何地方运行”的原则，Java 被引入作为可移植性问题的最终解决方案，因为它允许程序员在一个系统上编写代码，并在任何其他系统上运行它——这一功能是通过 JVM 实现的。

# 虚拟机（Java Virtual Machine 的缩写）

Java 对可移植性问题的解决方案是让编译器输出字节码，而不是可执行代码。Java 字节码是由 Java 虚拟机或 JVM 执行的一系列指令。通过将 Java 代码翻译成字节码，它变得可移植，并且可以在许多环境中运行，因为所需要的只是在每个平台上实现一个 Java 运行时环境——也包括 JVM。一旦 JRE 出现在任何系统上，那么 Java 应用程序就可以在其上运行，因为所有 JRE 都理解 Java 字节码。

# Java 运行时环境

想象一下，你有一个特殊的烤箱，这个特殊的烤箱有烘焙你最喜欢的蛋糕所需的所有材料，说明，材料，蛋糕罐本身——一切。有趣的是，应用程序有这样一个神奇的烤箱，只是它被称为运行时环境。运行时环境包括执行任何给定应用程序所需的一切。运行时环境处理内存分配、CPU 管理、库、源代码等。如上所述，Java 运行时环境包括运行 Java 程序所需的一切，包括 JVM。在很多方面，Java 语言的美丽和独特之处在于 JRE，因为它可以在 web 和手机之外的系统中实现。

# JDK

Java 开发工具包是一个包，它使程序员能够创建可以由 JVM 和 JRE 执行的应用程序。所以在开始用 Java 开发之前，您需要首先下载 JDK，JDK 将与 JVM 和 JRE 捆绑在一起。截至本文撰写之时，JDK 的最新版本是 JDK 14，但 JDK 8 仍在广泛使用，JDK 11 很快紧随其后。
