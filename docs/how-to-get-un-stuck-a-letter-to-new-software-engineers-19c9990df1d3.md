# 如何摆脱困境:给新软件工程师的一封信

> 原文：<https://levelup.gitconnected.com/how-to-get-un-stuck-a-letter-to-new-software-engineers-19c9990df1d3>

作为一名新的软件工程师，摆脱困境的概念经常被忽视。我总是看到这个主题被总结为一个答案:使用谷歌！

![](img/5c0e9d76dde247ba05dcccee7b935c35.png)

弗拉达·卡尔波维奇的照片

当然，谷歌是有帮助的……事实上，这显然是大多数有经验的软件工程师呼吸的空气，但是让我们更深入地看看使用谷歌对崭露头角的软件工程师意味着什么。

*注意:最初这篇文章是作为帮助新程序员跨越编程障碍的指南开始的，但我认为它对技术领导者和更有经验的开发者来说也是很好的见解(和提醒)。随意与你的同事和上司分享，这样他们就能理解并记住你的感受。*

像“使用谷歌”这样笼统的陈述的第一个问题是，他们经常做出不现实的假设。谷歌声明的假设是一定水平的编程知识。需要明确的是，你向其寻求建议的人并不是想开玩笑。谷歌可能是他们陷入困境时寻找答案的方式，也可能是他们在很长一段时间里寻找答案的方式。主要问题是他们不*知道*这对你来说不像对他们来说那么简单。

首先，经验会带来一定程度的自信。有经验的开发人员在心理上已经很有优势，因为他们已经看到了无数的问题、错误和争论。简而言之，这不是他们的第一次竞技，所以**他们知道他们可以解决大多数遇到的问题。**

作为一个新的程序员，我们没有同样的信心。对于有经验的开发人员来说，这似乎是第二天性的单一动作，新开发人员在过程的每一步都可能有问题。

## 这个问题

让我们进入现实世界的细节。

场景是这样的:

假设我正在做我的第一个需要 CRUD 特性的网页。假设我正在建立一个拥有多个餐厅的网站，而这些餐厅的汉堡是他们被分配到的餐厅所独有的(一对多关系)。我碰壁了，因为我不知道如何遍历从数据库查询返回的字典列表。我决定听从每个人的建议，我导航到谷歌。

众所周知，你不能真的像和人说话一样在谷歌上输入一些问题。你必须使用关键词，所以首先，作为新开发人员，我们试图弄清楚如何向谷歌提出正确的问题，以获得想要的结果。我们的编程知识和词汇不像有经验的程序员那样丰富。

一旦我们输入一些东西，我们就会找到看起来和我们要找的东西很接近的东西。“如何遍历字典列表”似乎达到了目的。排名靠前的结果通常是一个网站…..*鼓声* StackOverflow。我们根据预览点击看起来最接近我们问题的链接。

现在，让我们谈谈我们在 StackOverflow 上的发现。

**答案(结果)**

StackOverflow 是命中或错过。有时你可以找到你正在寻找的答案，有时这些答案比你目前对代码的理解更高。**这是我对 StackOverflow 最大的问题；**它是由有经验的开发者写的*，通常*写给*有经验的开发者。想想吧。一个人有足够的自信去论坛上回答其他人的问题，可能对代码了解很多，虽然我喜欢 7 年零 3 个月前回答这个问题的人，但我有时希望他或她能把它降低到一个较低的水平。*

现在你在 StackOverflow 页面上，你意识到你在 Google 中输入的问题不是你想问的正确问题。你真的想问“如何遍历字典列表，然后遍历字典并返回每个键的值”。你把它输入谷歌，然后进入另一个 StackOverflow 页面。

你有最上面的答案，在它下面，有 2-10 个其他的答案，这些答案似乎在过去也对其他人有效，因为他们有更多的投票。你只是有时候不知道哪个答案是你的问题的正确答案。

如果这还不够，还有最大的恐惧**就是用你几乎不理解的代码片段毁掉完美的代码(除了你目前遇到的这个问题)。**

**通常情况下是这样的**:我看了一些我能看懂的文字，然后我把代码片段(最上面的答案)抄下来，贴上，手指交叉。

这似乎是软件工程领域的自然规律，对吗？

## 那么，你如何摆脱困境呢？

老实说，使用谷歌是我知道的摆脱困境的两种方法之一。

是的，我知道斗争。这种事在我身上发生过很多次，而且还在发生。虽然，一旦你弄明白并发现软件工程最伟大的事情，这一切都是值得的——我在这里写了一篇关于这个的文章。

另一种伟大的感觉是当你向另一个人寻求帮助并得到帮助时；这是我用来摆脱困境的另一种方法。你可以问一个人比谷歌更能理解你最关心的问题。这是我支持训练营的主要原因之一。带着团体意识学习对我帮助很大。能够提出 5 个问题并得到这些问题的答案是无价的。

如果你是编程新手并且有工作，问问你的主管。求助不是失败。如果你走的是自学之路，找一个社区加入进来。一旦你对代码有了更好的理解，有一天你甚至可以驾驭伟大的堆栈溢出。

## 结论

我自己还是个初学者。我只是在转述别人告诉我的信息。另外，我对我所说的事情有点经验。我一度不知道如何向谷歌提问。现在，我至少可以找到一个与我的问题相对接近的页面。这是一个没有清晰路线图的过程，但我恳求你为小小的胜利而欢欣鼓舞，并认可你的进步。

你必须打破东西；你必须阅读你可能不完全理解的东西。你得不穿救生衣跳进谷歌的海洋里试着游。目标是岸边，但在这样的水域里自学游泳很重要。一旦你这样做了，到达岸边将不再是你的目标，因为你会喜欢游泳。目标是找到所有可以游泳的新水域，尝试不同的泳姿，潜入水下。
