开发人员很容易犯错误，但如果我们事先知道这些错误，我们就可以防止它们。

在这篇文章中，我们将看看新手程序员犯的错误，我们都可以避免。

# 自我怀疑

怀疑自己是正常的，因为我们没有编程经验。这也适用于有经验的开发人员学习新东西的时候。

然而，我们可以通过不那么质疑自己并让人们引导我们向正确的方向前进来度过这个难关。

我们只是需要一些反馈。如果我们在做副业，就要一些。如果我们的日常工作是作为一名开发人员，那么很多人会指引我们正确的方向，因为他们可能会对我们写的东西进行工作。

所以我们不用担心，通过获得反馈来度过自我怀疑。

# 害怕

恐惧是阻止我们做任何事情的东西，包括提升自己。这是我们在编程中需要解决的问题。

编程没什么好担心的，因为大多数程序都是对人们来说不太重要的应用。

如果不是太关键，那就没什么好害怕的。否则，就请更多的人帮忙检查关键部位。

# 错误的代码格式

糟糕的格式对读者不利，包括我们自己，所以我们应该确保我们的代码格式良好。

制表符和空格应该被很好地格式化，以便我们都能阅读它们。

我们可以通过用 linter 或代码格式化程序自动格式化我们的代码来轻松做到这一点。

这很容易做到，让每个人的生活更轻松。

# 名称的大小写

名字的大小写应该遵循我们语言中的惯例。大多数编程语言，所以我们要确保案例是一致的。

这样，在引用事物时就不太可能出错。

例如，JavaScript 变量和函数应该总是 camelCase，构造函数和类应该是 PascalCase。

单词分隔符可能符合也可能不符合某种语言的约定。如果是，那么我们添加它们。否则，不要添加它们。

# 糟糕的名字

不好的名字会让我们的代码难以理解。所以我们应该确保它们是有意义的。

这样，当我们阅读别人的代码时，我们可以在同一页上。

所以好的名字是描述性的，就像`numApples`，它描述了一个变量包含什么。

此外，词序应该是一致的，所以如果我们从`num`开始，那么我们从`num`开始。

# 评论太多

太多的注释并不好，因为代码应该足够清晰，我们不需要太多的注释。

它们只是占用空间，做我们的代码应该做的事情。

此外，它们中的大多数都会过时，所以我们可以跳过重复代码应该做的事情的注释。

像`// add 5 to numApples`这样的注释不需要出现在我们的代码中，因为代码应该告诉我们这些。

# 没有使用富有表现力的速记

有许多新手可能还没有学会的富有表现力的速记。

因此，我们应该花时间去学习它们。例如，在 JavaScript 中，我们可以使用数组的`reduce`方法来做这件事，而不是编写循环来对数组中的所有数字求和。

例如，我们可以写:

```
const sum = [1, 2, 3].reduce((a, b) => a + b, 0);
```

而不是写:

```
let sum = 0;
for (const n of [1, 2, 3]) {
  sum += n;
}
```

正如我们所见，使用`reduce`比使用循环要短得多。

同样，我们不需要写像`condition === true`这样的表达式，相反，我们可以写`condition`。

在编程语言中有更多的语法糖，在学习它们之后会节省我们的时间。

# 令人困惑的语言、框架和 ide

语言、框架、IDE 对于新手来说很容易混淆。例如，新手可能会混淆 jQuery 和 Angular 这样的框架。

或者 jQuery 可能会与 JavaScript 混淆。

我们必须确保在使用它之前清楚它们是什么。

# 结论

我们可以避免的新手错误包括过多的注释、不一致的大小写和混乱的格式。

我们很容易避开它们。

此外，我们应该学习大多数语言的表达能力。通常有语法上的好处可以使我们的代码更短。

语言、IDE 和框架也可能会混淆，所以我们应该把它们弄清楚。

最后，恐惧和自我怀疑不应该阻止我们做事。
