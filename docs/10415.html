为软件公司制定创新战略的六个步骤

遵循以下六个步骤实现持续创新

你可能听说过这句话，‘每个公司都是软件公司’。这可能会给你一种错觉，认为创办一家软件公司很容易，但事实远非如此。据估计，90%的新创企业在前三年都会失败。如果你想在市场上占有一席之地，并作为一家软件公司保持领先，你需要不断创新。为了不断创新，你需要一个创新战略。

制定创新战略的步骤

软件公司在创新时经常犯的一个错误是，他们没有对行动有一个坚实的理解就直接跳到战术上。以产品为导向的创新战略旨在解决这一问题，方法是全面审视公司的经营空间，建立目标以确定所确定行动的有效性。

制定这样一个战略有六个步骤。让我们开始吧。

第一步:对你的现状进行审核

通常，一家科技公司已经有了促进创新的计划。这可以是零星的努力，比如偶尔的黑客马拉松，也可以是更有规律的，比如开发人员有20%的时间来修补和学习新技术，就像谷歌和其他知名硅谷公司一直在做的那样。

作为制定创新战略的技术领导者，您需要对当前的状况进行审核，以了解哪些方面做得好，哪些方面需要改进。

问自己三个问题:

  1. 公司应该开始做什么？(发现机会)
  2. 公司应该继续做什么？(利用当前优势)
  3. 公司应该停止做什么？(消除浪费时间的因素)

问题1:公司应该开始做什么？(识别机会)

例如，在审计期间，你可能意识到你的公司目前没有专门的时间来修补或学习。虽然修补和学习应该是日常工作的一部分，但留出专门的时间来表明公司完全支持这些活动是有帮助的。它还能起到提醒的作用，让员工更有创造力和创新精神。你可能会发现，这样的战略举措是你的公司应该开始做的事情。然后你可以定义细节，比如多久和多少，当你在最后一步——第六步:投资战略计划。

你可能会发现软件公司和初创公司的另一个例子是，他们的软件工程师缺乏产品知识和对客户的了解。他们拥有出色的技术技能，但在打造客户喜爱的创新产品方面，他们还有改进的空间。他们的创新观严格关注他们使用的技术。在这种情况下，作为一名工程领导者，你会希望加强你的工程师的客户同理心，使他们体现出制造创新产品所必需的以下特征:

1.他们从为什么开始。他们了解客户想要什么，并在进入编码或解决方案之前分析问题。

2.他们有平衡的观点。他们每次都在技术、时间/精力和客户价值之间取得平衡，并提供各种功能。

3.他们永远不会爱上自己的代码。他们不会过早优化；他们是务实的，如果他们意识到他们正在构建的不是客户想要或需要的，他们愿意从头开始。

4.他们知道衡量什么。要知道客户真正想要或需要什么，他们知道衡量什么，他们衡量什么是重要的。例如，参与度指标、绩效指标、业务指标，并且他们验证他们已经发布的内容。

问题二:公司应该让继续做什么？(利用当前优势)

许多软件公司已经有了一些围绕创新的计划，有些可能做得很好。例如，你的公司可能已经有了一年一度的黑客马拉松，有相当多的工程师参与，产生了公司需要的想法。因此，这是一个可以增加业务价值的计划。

你可能还想通过围绕它建立一个结构来进一步利用这种优势，例如，跟踪不成功的想法，并确保它们要么被重新审视，要么失败和学习在整个公司公开分享。

问题3:公司应该阻止做什么？(消除浪费时间的因素)

同样，你的公司可能正在努力开展一些活动来促进创新，但这些活动根本不起作用。识别这些活动的最佳方式是向整个工程部门发送一份匿名调查，请他们对您当前的所有活动和流程进行评级和反馈。没有什么比效率低下和不得不千方百计完成工作更让工程师们鄙视的了。通过匿名征求他们的反馈，他们会更加乐于助人。

第二步:看看别人在做什么

不断变化的技术领域的美妙之处在于:永远不缺乏学习。这种学习可能来自竞争对手，他们在构建和维护行业内外的创新解决方案方面已经证明了自己的实力。事实上，我强烈建议超越你自己的行业，这样你可以扩展你目前的能力和知识基础，并在你自己的行业中领先他人。例如，如果你在教育领域，看看其他人在电信领域做什么。同样，如果你在法律领域，看看健身和娱乐领域。看起来他们似乎是彼此对立的，但是这样做你会得到更多创造性和非传统的想法。

我们知道有很多知名的科技公司都在钉钉创新:谷歌、微软、亚马逊、苹果、三星、Adobe，不一而足。让我们以Adobe为例，它是一家知名的软件公司，已经存在了几十年，准确地说是四十年。

Adobe最近被Fast Company评为2021年全球最具创新力的公司之一。Adobe是持续改造和创新的典范。它成功地转变了整个业务模式——从本地软件转变为软件即服务提供商，并开始通过智能服务为企业客户提供新的数字解决方案。Adobe并没有放弃它的旗舰产品，但是它已经成功地扩展了它的产品组合并重新发明了产品。同时始终忠于自己的目标:制作软件，帮助任何有想法或故事的人将他们的愿景变为现实。

作为一名工程领导者，您可能希望跟随Adobe的脚步，投资于您公司的核心产品，并在多元化投资组合之前，在旅程的早期专注于特定软件的增量创新。

第三步:想象理想的未来状态

既然你已经深入了解了公司内部的创新现状，以及其他公司应对创新的方式，那么是时候设定愿景了。是时候重新想象什么是可能的了，这样你就可以在产品主导的创新方面快速跟踪公司的未来。

在设定愿景时，我鼓励领导者要大胆。不应该有任何限制或约束。我能听到你在问，‘如果实现愿景实际上是不可能的或者不可行的，那该怎么办？’这就是为什么我们有第五步:制定策略。但是现在，只要毫无障碍地梦想和想象，让你的想象力尽情发挥。

在展望未来时，工程领导者应该考虑三个因素:

  • 客户。您如何解决客户可能甚至不知道他们遇到的问题？你的产品如何让他们的生活变得更好？想想亨利·福特的名言，“如果我问人们想要什么，他们会说一匹更快的马。”
  • 员工。员工对公司及其产品有什么看法？通常，如果你花时间去真正倾听，现场的人们会更清楚什么是可能的。
  • 风景。一至三年后，该行业的前景是什么样的？

一旦你想通了，用下面的格式写下你的愿景:

在[时间期限]之前，[公司名称]将使[关键能力]能够提供[在客户或企业利益方面的切实成果]。

毕竟，正如维基百科正确描述的那样，创新是想法的实际实现，导致新的商品或服务的推出或商品或服务的改进。

步骤4:定义成功的衡量标准

我真的相信没有被衡量的事情不会做好。一旦你设定了目标，重要的是知道你将如何实现这个目标。你的愿景需要成功的衡量标准来支持。产品导向型创新战略的推荐成功衡量标准是:

  • 约会
  • 对齐
  • 顾客价值
  • 一致性

工程领导者可能希望用来衡量创新战略有效性的测量方法示例如下:

  • 在黑客马拉松上提出的想法数量
  • 使用创新时间的工程师百分比
  • 创新的OKR/关键绩效指标得分
  • 一年运送的实验数量
  • 每个团队花费的创新时间
  • 每年的黑客马拉松次数

步骤5:创建策略

这一步是绝大多数计划和调整发生的地方。这一步的目标是制定一个有效的创新战略，与愿景、公司战略、行业最佳实践和公司需求保持一致。你将确定你的战略的时间范围，并把重点放在最重要的事情上。这一点非常重要，因为虽然你的愿景可能是雄心勃勃的，但你的战略需要高度集中，并且要切实可行，才能成功执行。

在这里，我建议工程领导人确定该战略的关键支柱。根据你目前和未来的状态，你的关键支柱可能会有所不同。以下是我为你的产品导向创新战略推荐的一些关键支柱:

  • 结构。易于遵循的指南和基础设施。
  • 文化。心理上安全的环境。
  • 专长。不断学习和提高。
  • 天赋。被影响他人的欲望所驱使。
  • 协作。团队合作是梦想的工作。

第六步:投资战略计划

创新战略失败的首要原因是没有支持该战略的战略计划。因此，最后一步旨在解决问题。一个软件公司需要投资于战略计划，以保持他们的领先地位，并使执行成为可能。在此阶段，工程领导需要提出已经选定的计划，如果成功交付，将交付整体战略和成功措施。

计划列表应该按优先级排序，并共同努力实现战略中描述的未来状态。这些计划应该清楚地说明时间表和所需的投资，以及它们如何有助于整体的成功衡量。为了确保这些计划以激光为中心，重要的是要强调它们属于哪个支柱，以避免提出与整体战略不一致的计划。

创新触手可及

如果你知道自己在追求什么，创新是触手可及的。作为软件公司的工程领导者，无论你是首席技术官、工程副总裁还是工程总监，你都有责任制定一个有效的创新战略，让你的团队能够很好地执行并保持专注。我见过太多的软件公司在没有明确战略的情况下试图创新，最终结果是事倍功半和团队脱节——这是极其昂贵的。

总而言之，制定创新战略的六个步骤如下:

  • 第一步:对你的现状进行审核
  • 第二步:看看别人在做什么
  • 第三步:想象理想的未来状态
  • 步骤4:定义成功的衡量标准
  • 步骤5:创建策略
  • 第六步:投资战略计划

通过遵循上述步骤，你将能够创建一个创新战略，该战略考虑了当前的挑战、愿景和原则，以及你希望达到的未来状态。
