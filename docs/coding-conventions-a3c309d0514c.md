# c 编程语言

## SEI CERT C 编码标准

这些规则和建议的目标是开发安全、可靠和可靠的系统，例如通过消除可能导致未定义程序行为和可利用漏洞的未定义行为。

详情请访问:[*https://www . secure Coding . CERT . org/confluence/display/C/SEI+CERT+C+Coding+Standard*](https://www.securecoding.cert.org/confluence/display/c/SEI+CERT+C+Coding+Standard)

## 嵌入式 C 编码标准

Barr Group 的*嵌入式 C 编码标准*旨在通过专注于将错误排除在外的实用规则来最大限度地减少固件中的错误，同时提高嵌入式软件的可维护性和可移植性。

更多阅读请点击:[*http://barrgroup.com/coding-standard*](http://barrgroup.com/coding-standard)

## 固件开发标准

该标准的原因是确保所有公司开发的固件满足可读性和可维护性的最低水平。

更多阅读请点击:[*http://www.ganssle.com/fsm.pdf*](http://www.ganssle.com/fsm.pdf)

# C++编程语言

## Quantum Leaps C/C++编码标准

以下编码指南旨在提高软件的代码可移植性、可读性、一致性和可维护性。该编码标准的主要目标是通过促进通用性和避免误解和猜测来提高软件生产率和降低维护成本。

更多阅读请点击:[*http://www.state-machine.com/doc/AN_QL_Coding_Standard.pdf*](http://www.state-machine.com/doc/AN_QL_Coding_Standard.pdf)

## C++编程/编程语言/c++/代码/风格约定

一个指南或一组约定的使用为程序员提供了一组代码规范化或编码风格的规则，这些规则建立了如何格式化代码、命名变量、放置注释或任何其他在代码上使用的与语言无关的结构决策。这是非常重要的，因为你与他人共享一个项目。同意一组通用的编码标准和建议可以节省时间和精力，因为这样可以更好地理解代码库，提高代码库的透明度，为未记录的结构提供一个公共基础，便于调试，并提高代码的可维护性。

更多阅读请访问:[*https://en . wiki books . org/wiki/C % 2B % 2B _ 编程/编程 _ 语言/C % 2B % 2B/Code/Style _ Conventions*](https://en.wikibooks.org/wiki/C%2B%2B_Programming/Programming_Languages/C%2B%2B/Code/Style_Conventions)

# C#编程语言

## C#编码约定(C#编程指南)

编码约定服务于以下目的:

*   它们为代码创造了一致的外观，因此读者可以将注意力集中在内容上，而不是布局上。
*   它们通过基于以前的经验做出假设，使读者能够更快地理解代码。
*   它们便于复制、更改和维护代码。
*   他们展示了 C#的最佳实践。
