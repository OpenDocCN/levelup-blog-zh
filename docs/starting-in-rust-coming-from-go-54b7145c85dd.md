我工作过很多语言，并且总是对学习新的语言感兴趣。我想试试 Rust，但是没有时间。阅读到 Rust 真的要进入 Linux 内核激励我开始工作。

## 来自围棋

在过去的一年里，我一直在围棋界工作。在这一点上，我可以称之为精通，甚至很好，但我不能说我喜欢它。围棋有很多令人喜欢的地方，但对我和我的开发风格来说，也有一些不喜欢的地方。一般来说，我的问题是关于代码质量和可维护性，这是由冗余的锅炉板代码和过于开放的结构造成的。

## 我的第一次生锈

以前我已经完成了 Rust 教程，但为了继续学习，我想要一个具体的发展目标。我选择了一个我写的现有围棋 app 的端口: [cryptoport](https://github.com/nwillc/cryptoport) 。这个应用程序很简单，但是结合了多种技术:JSON、HTTP、文件 IO、数据类型、文本格式化、文本着色等等。于是我开始了这个创造性的命名为 [rcryptoport](https://github.com/nwillc/rcryptoport) 的项目。

# 我对铁锈的早期印象

## 社区

Rust 有一个强大的社区，有现成的好信息。

## 工具链

我转向我最喜欢的安装程序， [asdf](https://github.com/asdf-vm/asdf) ，找到了一个开箱即可完美运行的 Rust 插件。

*   安装没有痛苦。
*   Cargo 像宣传的那样工作，并且很好地集成到我的 IDE (IntelliJ)中。
*   这些工具总体来说构思良好，维护良好，并且在积极地改进。
*   [crates.io](https://crates.io/) 岩石。

## 语言

我喜欢铁锈。

*   它的语法虽然有点丰富，但是很难理解。
*   类型系统是强大的和编译时的。
*   它很好地支持泛型，没有大惊小怪。
*   它对代码组织和一个可行的包/模块系统有着相当好的、尽管过于简单的支持。
*   编译器完成繁重的工作并给出有用的反馈。这些信息最初有点神秘，但一旦理解就有用了。
*   编译时间是可以接受的，尤其是考虑到那里做了多少工作。
*   它有一个很好的集成测试过程。

## 具体喜欢

Rust 要求很高，如果你做得不好，它会大声抱怨，虽然有时可能会让人抓狂，但当你把事情做好时，你会有一种自信。泛型功能强大，但没有增加太多的间接和抽象。

## 挑战

Rust 的内存所有权模型严谨且执行严格。作为一个没有经验的开发人员，我多次被它绊倒。特别是，Rust 的闭包，以及它们的内存所有权规则，几乎都让我受挫。当迭代数据结构和应用闭包来做平面映射之类的事情时，我努力解决内存所有权问题。我还有很多工作要做，但是现在，我回到了更简单的循环，完成了任务。

其他挑战主要围绕寻找问题的答案。有很好的文档和一个活跃而有帮助的社区，然而，Rust 确实有独特的处理事物的方法，并且使用了一些有些独特的方法和习惯用法。有时，我被难住了，不是去哪里寻找答案，而是在什么方面，或者至少如何描述我试图找到的东西。

## 摘要

我真的很喜欢铁锈。虽然这并不容易，但我觉得这是一种你可以直接从付出的努力中得到回报的语言。对于我的第一次努力，我得到了一个干净、实用、可靠的结果。此外，它的大小是我移植的 Go 程序的 30%,速度也快得多，我在这个程序上投入了更多的时间和精力。我期待继续学习和使用 Rust。
