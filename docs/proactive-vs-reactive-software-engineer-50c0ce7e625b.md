在签署新工作合同之前，我对一家新公司的第一个印象是我的新雇主说的一句话:“在我们公司，我们不练习与同事交换工资细节。”

同一家公司的第二个亮点是我新工作的第一天，至少有两个新同事问我的薪水。不仅如此，他们还给我关于他们自己的信息。

我不得不承认。我过去一直在实行工资分成。这样做是我和同事们多次交谈的主题。我是说，为什么不了解一下你能推多少？

在某种程度上，我已经转到了另一边，这赋予了我决定其他人薪水的特权。那一刻，我意识到我对薪酬的看法只是一派胡言。

突然，我接受了原力的黑暗面。在那里，我可以看到是什么让优秀的专业人士成为伟大的专业人士。我可以看到编码技能对工资等级有很小的影响。

# 我们的判断建立在主观基础上

![](img/4776897421c73f9737512cd5efa9b1fd.png)

[Sasun Bughdaryan](https://unsplash.com/@sasun1990?utm_source=medium&utm_medium=referral) 在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍照

哇哦。我不能说上面的标题配得上诺贝尔奖，但是，我不得不带上它。虽然我们都知道这一点，但我们大多数人倾向于把自己的判断视为宇宙中的终极判断。

我们的观点基于经过筛选的事实。我们接受我们认为重要的东西，并用它来建立我们的真理。我们从来不会对某个主题提出一个完整的知识，而只是一些片断。

因此，我们只以我们认为是优势的技能来衡量我们的薪酬等级。并且，毫不奇怪，软件工程师主要认为编码技能是高工资的强有力的预测因素。

但是，成为一名出色的程序员并不是 IT 行业中最重要的技能(或者至少不是唯一的技能)。它把我们带到了最好的软件公司的门口。这一事实很明显。

> 但是，我们的编码技能在公司内部并不重要。

我们在同一个公司主要是编码技能差不多。是的，我们可能会发现一些天才在为国有公司工作，但或许我们可以期待在一些软件巨头中看到他们。

每当我们听到某个同事赚的钱比我们多，却没有更好的编码技能，我们都会觉得不公平。这种感觉让我们一直感到沮丧。

我们的编码技能可能是相同的，甚至我们比我们的同事是更好的编码员。我们可能有更多的经验。我们可能在修复错误、调试和算法优化方面有巨大的天赋。

尽管如此，他们每月仍能比我们多挣 20%、30%或 40%。这可能会造成伤害和挫折。日复一日，我们觉得自己受到了虐待。我们抱怨更多，不开心，开始找新工作。

但是我们对自己诚实吗？

*   我们多久会等一个人完成他们的工作，这样我们才能继续做我们的事情？
*   我们多久抱怨一次与业务或技术相关的决策？
*   我们多久为我们的工作定义和解决一次依赖？
*   我们多久评价一次经验不足的同事？
*   我们多久向老板提出一次问题？
*   我们对团队的透明度如何？
*   最重要的是，我们多久去找老板要求加薪一次？

# 积极主动很重要。

![](img/5109fd7dff671b8c29697ef9273edb11.png)

马库斯·斯皮斯克在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上的照片

许多[书籍](https://facilethings.com/blog/en/five-books-on-productivity)谈论主动性。我们也可以在网上找到许多博客。当我们翻阅文献时，我们可能会弄清楚积极的含义，特别是与被动的头脑相比。

我们将通过例子和生活的情况。我们会读到一些可能突然抓住我们的经历，作为对我们生活的描述。所有这些故事都是关于谁拥有控制权。

> 反应性思维模式会放弃对自己生活的任何控制感，有时会将这种控制感委托给其他实体。他们不断对新的事件做出反应，毫无准备，感到惊讶。
> 
> 积极心态掌控自己的生活。他们审视自己的处境和环境，这有助于他们理解和预测未来，以便他们能够以最佳方式做出反应。

正如我们所见，两种心态都对事件做出反应。毕竟，违背伟大的 T2·艾萨克·牛顿爵士的法律是不可能的。不管我们的环境中发生了什么，我们都会对它做出反应，不管我们是否愿意。

但区别在于我们准备得有多充分。如果我们有一个被动的心态，我们会放弃对结果的所有思考，行动只会打击我们，而没有我们适当的反应。

我们最终可能会失望，会问“这是怎么回事？”或者“为什么那会发生在我们身上？”。我们可能会惊讶地站着，寻找一个有罪的人，并把责任推给除了我们之外的任何人。

> 老实说，如果要责怪谁，那就是我们。

## 童年的回忆

当我还是个孩子的时候，我非常害怕修理工。我把他们视为可怕的人，不道德，说谎，令人厌恶。每当公寓里有什么东西坏了，我都准备好迎接新的创伤。

原因是(惊喜，惊喜)——我的父母。每当我们有任何问题，他们会安排一些朋友或家人的熟人的工作。为什么？因为它们很便宜。但是，也是业余的。

因此，每次“修复”要么是暂时的，要么是未来某个重大事件的开端。每当一个新问题出现，我们都会经历更多的恐慌和失望。

这些是这本书中反应性思维的例子——最糟糕的例子。尽管意识到这个家伙可能会搞砸我们的房子，我的父母还是一遍又一遍地重复着同样的动作。

*但是，他们该怎么办呢？*

第一步是承认，在经历了这么多糟糕的经历后，我们可能会成为不专业的修理工的受害者。但是，这是否意味着我们不应该叫人来修理我们的损坏呢？当然不是。

继续抱着同样的希望，我们将有一个不好的经历也不是一个选择。所以，我们应该接受错误发生的可能性。

然而，这并不意味着生活在持续的恐惧和恐慌中，等待事件的发生。这意味着——让我们在精神上和身体上做好准备，为这样的结果把我们放在更安全的地方。

几年前，我的公寓发生了巨大的变化，许多砖瓦工、水工、维修专家等等，完全重新设计了我的公寓。

我已经从父母那里吸取了教训。我付给他们更多的钱，但是我和他们都签了合同。所有的都有保证书。是的，我们后来有一些问题。但是这并没有花费我的金钱和时间。

我知道我无法对未来的事件做出绝对的预测。没人能。我已经接受了这种事情可能发生的可能性。人类会犯错。修理工也是人。

相反，我决定尽可能避免这类问题。我想定义在我能预测的所有结果中涉及的所有各方的责任。

自然，我不能考虑地震，因为我没有住在地震区。但如果是这样的话，我会在我的合同中加上这一点。

坐以待毙并期待积极的结果不是我的选择。解决方案再简单不过了。

# 一个积极的软件工程师做什么不同？

![](img/c5c1b936983a466694a710ecefd8c239.png)

由[克里斯·莱佩尔特](https://unsplash.com/@cleipelt?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的照片

几年前，主动性这个话题吸引了我，并让我一直保持着兴趣。一旦我在公司的级别提高了，我就开始从一个更客观的角度来观察主动性。

如果我说我生来就有这种技能，那我是在撒谎。这是一个漫长的过程，包括自我质疑、阅读、接受治疗、评估过去的错误。必须有多次失败和突破。

这些年来，我一直在审视自己过去的决定。有些是好的，但有些是可耻的。他们中的一些人真诚地告诉我，为什么我不应该得到和我同事一样的薪水。

我们多久会等一个人完成他的工作，这样我们才能继续做我们的事情？

我们听到“我被阻止继续”的频率有多高。我每天都听，很多次。这是我日常生活中最讨厌的事情。我是说，那是什么意思？我没有说我被封杀了十几年。

*   如果我们依赖某人的工作，承认我们在等待它是我们唯一能做的事吗？
*   我们能和那个人坐在一起，帮助他们更快地实现目标吗？
*   我们能不能和那个人谈谈接口、方法签名、API JSON 方案，这样我们就可以继续并行工作了？
*   如果环境不稳定，能否自行调查？
*   如果第三方 API 文档不完整，我们是否可以继续实现一个更抽象的方法，直到澄清为止？
*   我们能换个任务吗？
*   如果所有的可能性都用尽了，我们能不能至少写一些单元测试，重构一些代码，让它比原来更好？

那么，我们等待别人完成他们的工作到底意味着什么呢？我们拒绝控制自己的行为。

**我们多久抱怨一次与业务或技术相关的决策？**

很多时候，对于一些关于技术或功能需求的决策，我们会说，“这太愚蠢了”。在某些情况下，整个团队都站在我们的立场上，挫折感与日俱增。

*   然而，当解决方案来自外部时，我们会问自己为什么没有人对我们的判断有信心吗？
*   在从外部得到决策之前，我们是否在团队内部营造了共同决策的氛围？
*   在判断我们的想法更好之前，我们有没有实施过？
*   我们争论的基础是我们喜欢什么，而不是商业人士认为对他们最有价值的东西吗？
*   我们对我们从事的项目、市场和客户的愿望有完整的了解吗？
*   我们是否有实际证据证明我们的解决方案更好，影响了其他人的核心信念，以及为什么我们的想法可能对他们更有利，而对我们没有好处？
*   如果我们不做一个实际的例子来证明我们是正确的，为什么会有人考虑我们的解决方案呢？

那么，我们抱怨别人的决定到底意味着什么呢？我们拒绝控制自己的行为。

**我们多久为我们的工作定义和解决一次依赖？**

有时我会听到，“我们无法完成它，因为看起来验收标准的关键点还没有完成”。是的，就像这个星球上除了我们之外的每个人都知道我们到底需要什么，所以我们不能靠自己。

*   如果我们的任务或用户故事中缺少一些验收标准，谁应该是在工作之前意识到这一点的最佳候选人？
*   如果我们，作为技术人员，不能意识到一些微服务、端点、技术文档是不完整的，我们为什么要期望业务人员完成呢？
*   当我们意识到我们的任务依赖于其他未记录的任务时，我们该怎么办？
*   我们是自己记录，还是把它交给信息比我们少的人？
*   我们是否与对这种依赖性负责的各方进行了沟通，或者我们是否将它留给某人以某种方式来解决？
*   我们是否问过自己，为什么我们接受了任务却没有意识到这种依赖？
*   毕竟，我们应该做些什么来避免将来出现这种晚实现的情况？

*那么，我们因为惊人的依赖性而停止执行到底意味着什么呢？我们拒绝控制自己的行为。*

## 我们多久评价一次经验不足的同事？

每个团队由不同背景的人组成。很难找到具有相同经验和生产力的人。但是，当我们与效率较低的同事一起工作时，我们经常会产生内心的挫败感。

*   然而，我们有多少次尝试去帮助经验不足的同事呢？
*   我们多久坐在他们旁边，给他们一些建议或指导？
*   我们的评论在代码评审中听起来如何？
*   我们在交流中完全避免使用判断性词汇了吗？
*   我们有没有推荐一些对他们有益的好的文学作品、课程和工作坊？
*   我们是否意识到，在团队之外，没有人关心谁个人做了什么，而只关心团队是否交付。
*   我们是否意识到，如果我们不帮助我们的同事，就没有人能做到这一点？

那么，我们批评经验不足的同事到底意味着什么呢？我们拒绝控制自己的行为。

## 我们多久向老板提出一次问题？

很多时候我们会把一些问题“委托”给老板。在某些情况下，这种授权是可以理解的，但在某些情况下，我们这样做是因为我们把老板和父母混为一谈了。

*   首先，我们是否将问题的解决方案与问题一起带来，并且我们只需要执行它的许可？
*   如果我们只是带来了一个问题，我们是否意识到在那个时候，我们就成了老板的一个问题？
*   我们的老板负责解决我们和同事之间的冲突吗？
*   我们的老板负责我们的笔记本电脑、开发环境、更新和部署的稳定性吗？
*   我们的老板是否应对团队内部的误解和错失有效解决问题的机会负责？
*   当我们把问题委托给老板时，我们期望最终的决定是什么？

*那么，我们把问题委托给老板到底意味着什么？我们拒绝控制自己的行为。*

## 我们对团队的透明度如何？

我记不清有多少次我做了一件事，花了几个小时，才意识到我的同事过去已经经历过一些问题。尽管这种情况引发了巨大的愤怒，但这也是我的错误。

*   我们多久在团队内外分享一次我们的知识？
*   我们多久询问一次是否有人有过类似问题的经历？
*   我们有多少次因为在公共假日休假申请被拒绝而失望，仅仅是因为一个迟到的通知？
*   我们多久宣布一次我们做了什么和我们刚刚做了什么，以便其他人，以不同的专业知识，可以继续下去？
*   当我们不理解某件事时，我们会问多少次问题？
*   为了不显得愚蠢，我们经常避免提问吗？
*   我们有多少次诚实地说某事可能比预期的要花更多的时间？
*   我们有多少次说我们不能做要求我们做的事情，因为我们没有这种技能？
*   我们真的认为这个星球上有人能读懂我们的思想吗？

*那么，我们不与同事分享信息到底意味着什么？我们拒绝控制自己的行为。*

## 我们多久去找老板要求加薪一次？

我觉得这是最有趣的一个。大多数时候，我们团队或公司的每个人都知道我们沮丧是因为我们没有赚到我们想要的那么多:每个人，除了我们的老板。

*   如果我们把钱给一个没有负面反馈的人，我们会给他们更多吗？
*   我们能预测别人对我们影响他们的行为的看法吗？
*   唯一正确的做法是知道某人的感受，但不要问所有人，这有时可能有数百个。
*   出现在老板面前要求加薪是唯一的办法吗？
*   在提出请求之前，我们能做些什么？
*   我们如何提高我们在公司的影响力？
*   除了薪资谈资，我们还应该在哪里腾出我们的杠杆点？
*   我们了解我们所在行业、我们的职位、我们所在地区的工资水平吗？
*   将来我们能给老板提供什么，作为加薪的补偿？
*   我们是否明白，首先，我们需要承担新的责任，然后我们得到新的薪水？
*   如果情况不同，在意识到我们无法完成新的职责后，我们会接受降低工资吗？

那么，我们对自己的薪资愿望保持沉默到底意味着什么呢？我们拒绝控制自己的行为。

# 结论

在这篇文章中，我们刚刚触及了主动性和反应性的话题。越来越多的文献出现在市场上，带来了新的细节和建立这样一种生活的方式。

这两种心态都意味着对事件的反应，但是积极主动意味着为此类事件做准备。这种练习不是持续的恐惧和焦虑，而是接受可能的结果，并准备好减轻最终的问题。

积极主动是一种很好的性格特征，能给私人生活带来很多品质。在职业生活中拥抱它也让我们受益，不仅仅是金钱，也是日常工作中的乐趣。
