当你想写一本书时，我首先想到的应用程序是谷歌文档或微软Word。但是经典应用默认不支持代码片段。谷歌文档在他们的市场有免费的扩展，但是我不确定在你的文档中使用它是否安全。

理想情况下，我想要一些简单，快速，离线支持。

看起来你可以以每月5.99美元的价格购买尤利西斯，然后毫无问题地写书。这款以作家为中心的应用程序拥有我们写书所需的一切，支持代码片段，并且可以将文本导出到epub或pdf。

但作为一名软件工程师，你可能已经有了一个写书的理想应用。是VS代码！差不多两年前，我看到了一篇文章作者 keypressingmonkey 关于他如何将VS代码转换成writer工具并使用markdown作为源语言。那天我认为这是一个有趣的想法，但没有认真考虑。今天我重新考虑了一下。

使用VS代码用markdown编写技术电子书的想法是最好的选择。Markdown支持代码片段，语法简单。此外，这种语言可以在任何文本编辑器中编辑，并导出为流行的电子书格式。

VS Code是一个免费的代码编辑器，可以有效地处理markdown文件。此外，将VS代码与Ulysses进行比较，最大的好处是您在编写代码块时可以获得完整的代码。

电子书章节怎么写？

现在我们决定与markdown合作，并找到了一个更好的编辑器。下一步是什么？我们需要写一个实质性的单个降价文件吗？

是的，单个文件可以工作，但是很难在页面之间导航，并且每个新页面的性能都会变慢。

按章节分割文件要好得多。只需创建一个文件夹并放入。md文件一个一个。文件名应以章节号开头，并以零开头，例如01-Preface或02-Introduction。我建议在图书编辑过程中用数字来控制文件的顺序。关于这本书的编纂，我们稍后会详细讨论。

这种命名的利弊

优点

  • 你总是看到你的章节的顺序
  • 很容易打开任何图书部分进行编辑
  • 小文件带来最佳性能

缺点

  • 如果你决定在书的中间添加一个新的章节，你需要重命名所有进一步的文件

这就是为什么最好从一开始就计划好你的目录。

我建议将图像存储在单独的资产文件夹中。然后，要在markdown文件中使用它，您需要使用一个相对路径。它将在VS代码预览版和书籍编译后正常工作。

如何制作。epub，。pdf，或。用pandoc的mobi文件？

Pandoc是一把瑞士军刀，可以让我们将markdown转换成流行的电子书格式。它是可以在任何操作系统上运行的免费软件。

在Mac电脑上，你可以通过brew使用命令brew install Pandoc来安装它。苹果M1支持。

对于Windows或者Linux，你需要从官方仓库下载安装程序。

如果你打算出售pdf文件，你还需要安装LaTeX

brew install librsvg python homebrew/cask/basictex。

安装后，您已经可以使用长的终端命令来构建您的图书了。然而，为了简化这个过程，我建议在book文件夹的根目录下创建一个Makefile。和编写命令来构建感兴趣的图书格式。

例如，要构建epub和pdf，Makefile应该如下所示:

# Generate EPUB file
epub:
 pandoc -d config.yml \
 `find $(SOURCE) -name '*.md' | sort` \
 -o $(DIST)/$(BOOK_NAME).epub

在这两行之间，你可以提到我们使用了config.yml文件作为我们的书的配置。该文件将包含Pandoc、纸张大小、字体大小、页边距、目录等所有图书设置。

此外，这个文件将包含你的书的元数据(标题，作者，关键字，权利，语言等)。).你可以将这个元数据移动到一个单独的文件metadata.yml中，并在config.yml中链接它
