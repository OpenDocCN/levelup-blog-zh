JavaScript 是一种非常宽容的语言。编写可以运行但有错误的代码很容易。

在本文中，我们将研究分号、语句和函数之间的间距以及对象键和变量的排序。

# 分号前后的间距

分号前不需要空格，但分号后需要空格，因为我们想把不同的语句分开。

例如，下面的代码不好，因为它在语句之间没有空格:

```
let a = 1;let b = 2;
```

正如我们所看到的，在关键字`;`和`let`之间有一个空格，很难阅读 2 个`let`变量声明。

另一方面，如果我们在分号后加一个空格如下:

```
let a = 1; let b = 2;
```

然后我们可以更清楚地看到两个变量的声明。

因此，如果在同一行中有多个语句，我们应该在分号后添加一个空格字符。

在分号之后和下一条语句之前有一个空格字符是放置空格字符的最常见位置。

# 排序对象键

为了使查找对象键更容易，我们可能希望在一个对象中按字母顺序对键进行排序。

例如，我们可以编写以下代码来实现这一点:

```
const obj = {
  a: 1,
  b: 2
}
```

在上面的代码中，我们按照字母顺序对`obj`的键进行了排序。如果我们想找到对象键，这更多的是一个建议。然而，如果键没有被排序，也没什么大不了的。

# 变量排序

与对象键一样，我们也可以按字母顺序对变量声明进行排序，以便更容易找到它们。

与排序对象键一样，这更多的是一个建议，而不是一个要求。

例如，我们可以对变量声明进行如下排序:

```
let a, b, c;
```

# 块前空间

我们可能希望在块前有一个空格，这样我们可以清楚地看到函数签名和块的开头。

例如，我们可以编写以下代码来实现这一点:

```
const foo = () => {};
```

在上面的代码中，我们在箭头前后都有一个空格字符，这样我们可以清楚地看到箭头函数的每个部分。

对于传统函数，我们可以看到下面的代码:

```
const foo = function() {}
```

在括号和大括号之间有一个空格。这也使得代码比没有任何空格时更清晰。

空格使东西更容易阅读。

同样，我们可以对循环应用类似的间距，如下所示:

```
for (let i = 0; i < 10; i++) {
  console.log(i);
}
```

在上面的代码中，我们在右括号和左花括号之间有一个空格。

又来了。这让我们的眼睛能够分辨环路的不同部分。

对于`try...catch`，我们通常有如下间距:

```
try {} catch (ex) {}
```

有一点空间只是让阅读代码更容易。

# 函数括号前的空格

我们通常在左括号前没有空格，因为在右括号后有一个空格。

例如，我们通常将 JavaScript 函数定义如下:

```
const foo = function() {};
```

正如我们所看到的，在关键字`function`和左括号之间没有空格，在右括号和左花括号之后有一个空格。

命名函数具有相同的间距。例如，我们编写一个命名函数如下:

```
function foo() {};
```

在右括号和左花括号后面有一个空格字符。

# 括号内的空格

我们通常不会在括号和里面的表达式之间放空格。

例如，我们通常不写这样的东西:

```
const foo = ( 5 + 6 ) * 3;
```

上面的代码在 5 之前和 6 之后有一个额外的空格。相反，我们通常这样写表达式，不带空格，如下所示:

```
const foo = (5 + 6) * 3;
```

# 结论

通常，JavaScript 代码的间距遵循一些普遍接受的约定。圆括号和表达式之间没有空格。

在函数签名和开始的花括号之间，我们通常有空格。

在分号之后和下一个语句之前，我们通常也有一个空格字符来分隔这两个都在一行的语句。
