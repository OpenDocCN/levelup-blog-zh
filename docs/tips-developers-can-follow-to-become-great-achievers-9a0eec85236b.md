做开发者很难。我们不仅要知道技术技能，还要有很多我们擅长的软技能。

在本文中，我们将看看作为开发人员我们可以做得更好的方法。

# 重新评估我们的工作流程

重新评估我们的工作流程通常有助于我们找到更好的工作方式。我们应该在短期内使它们适应我们的需要。

# 在家工作

在家工作需要自律。我们必须通过消除诱惑和干扰来完成这项工作，比如电视、社交媒体和许多其他会让我们在家里分心的东西。

这可能很难。但它肯定会帮助我们在家里完成事情，这有助于我们在工作中生存，或者如果我们为自己工作。

# 可访问性代码

当我们写代码的时候，我们应该考虑到那些不能做健康人能做的所有事情的用户。

因此，我们在编写代码时应该考虑可访问性，这样他们就可以像其他人一样享受我们的产品。

# 兑现我们的承诺

如果我们承诺做某事，那么我们应该在承诺的日期前完成。

否则，如果我们落后了，我们应该更新每个人，这样他们都会调整他们的期望。

# 积极主动

当我们有时间的时候帮助别人总是一个好主意。如果我们帮助他们，他们都会感激的。

# 建立投资组合和在线业务

投资组合是推销自己的好方法。我们现在可能不需要它，但如果我们不需要它，它可以给我们带来额外的机会，如果我们需要找一份新工作，它们会给我们更多的帮助。

此外，我们都应该有一个公开的在线角色，这样人们就知道我们存在于互联网上。

今天的世界不仅仅局限于我们生活的地方，机会可能来自任何地方。

因此，我们应该利用这一点，这样无论机会来自何方，我们都能获得。

# 还记得我们为什么热爱编程吗

我们必须记住为什么我们热爱编程，这样当我们经历困难的时候才不会气馁。

如果我们正在建立负面的感觉，休息一下。我们必须思考我们当初为什么要从事编程，这样才能重新点燃我们的激情。

# 分享我们的知识

在任何地方分享我们的知识，包括聚会、会议和在线都是一个好主意。

教授我们的知识可以帮助他人，这样他们就可以与他人分享。它不仅帮助了我们直接教的人，也帮助了我们教过的人正在教的更多的人。

# 认识到我们的弱点

我们应该认识到我们的弱点，这样我们就可以学习一些东西来改善我们的弱点。

重要的是我们要提高他们，这样我们将来会变得更好。

# 思考大局

考虑大局总是需要的。人们使用的大多数系统都很大，所以我们必须了解全局，即使我们只做其中的一部分。

我们必须考虑几个部分来构建大图。我们必须满足商业方面的需求。

有用户端，它实际上提供了用户需要的特性。

然后是我们用来构建整个解决方案的技术。我们必须知道每个部分是如何相互作用的，这样我们才能轻松地与它们一起工作。

# 快速发展

我们必须能够快速创建正确的代码，这样我们才能交付我们的东西。这需要经验。

每个代码库都是不同的，所以当我们转到一家新公司时，我们必须从头开始学习他们的代码库，这样我们才能快速开发。

# 找到理想的解决方案

在编码之前应该找到理想的解决方案。我们必须从用户的角度考虑问题，以及我们的解决方案可能失败的所有方式。

如果特性很复杂，我们可能需要另一个人来帮助我们找到缺陷。

# 结论

当我们写作时，我们必须考虑用户和利益相关者。这意味着我们应该兑现我们的承诺，如果我们做不到，就更新它们。

此外，我们应该了解全局，这样我们的解决方案就不会出现意外。

最后，在困难时期，我们应该提醒自己为什么喜欢编程。
