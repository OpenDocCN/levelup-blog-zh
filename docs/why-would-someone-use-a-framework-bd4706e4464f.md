许多有经验的开发人员知道如何在不使用框架的情况下从头开始构建一些东西，而且解决方案运行良好。然而，从长远来看，这种方法好吗？

我将谈论我在框架方面的经验，构建绿地项目，指导人们，领导开发，以及使用一个好的框架有多重要。

# 什么是软件框架？

[来自维基百科](https://en.wikipedia.org/wiki/Software_framework)的一个定义如下。

> 在计算机编程中，软件框架是一种抽象，其中提供通用功能的软件可以由附加的用户编写的代码选择性地改变，从而提供专用软件。
> 
> 它提供了一种构建和部署应用程序的标准方法，是一种通用的、可重用的软件环境，提供特定的功能作为更大软件平台的一部分，以促进软件应用程序、产品和解决方案的开发。

换句话说，它可以是一个库、一组工具或一组建议来帮助您开发应用程序。

# 重新发明轮子

这是谈论框架时最常见的争论之一。你不需要重新发明轮子。

当构建一个新项目时，几乎所有需要解决的核心问题都不是您的项目所独有的。我说的是身份验证、密码管理、电子邮件确认、会话、支付关口、数据库配置……如果您必须从头开始实施所有这些，您将“永远不会”真正开发对您的公司重要的东西。你只会一遍又一遍地重复发明轮子。有许多库可以用来解决这些问题，一些框架会处理大部分这些问题。

作为一套可用工具的例子，在 npmjs.com，几乎所有的东西都有一个库。Rest API？快递。DB？猫鼬。认证？护照。饼干？快速会话。

要想成功，我们的想法必须独特，我们的解决方案也必须独特。如果你正在开发别人已经做过的东西，你可以去谷歌看看是否已经有适合你的解决方案。作为开发人员，我们通常只需要将那些库、工具和框架与一个独特的想法联系起来，就可以实现我们的目标。

# 寻找新同事

如果你在寻找 NodeJS 开发人员，大多数求职者都有使用 [Express](https://www.npmjs.com/package/express) 或[mongose](https://www.npmjs.com/package/mongoose)的经验。然而，他们中几乎 0 人会有使用你定制的`my-company-express-like-wrapper-for-automated-password-checking`内部库的经验。

如果你正在使用流行的库和工具，你将很有可能找到一个候选人，他对这些工具、一般项目结构有很好的理解，并且在加入你的公司后会很快变得富有成效。

但是，如果您在系统中依赖您的定制解决方案，新员工可能会发现很难完成工作，因为他们需要再次学习新的东西，这可能需要几周甚至几个月的时间！几周甚至几个月后，员工会熟悉你的工具，然后他就可以变得有效率了。然而，他并没有使用任何关于流行图书馆的知识。真是浪费时间和资源！

想象一下，刚刚学会如何使用`my-company-express-like-wrapper-for-automated-password-checking`库的新员工已经决定离开公司。现在让我们和其他人一起重复整个学习过程。

甚至不是员工不高兴，因为他没有学到一些他可以用在其他项目上的东西，而是雇主不高兴，因为很多时间被浪费在学习如何使用`my-company-express-like-wrapper-for-automated-password-checking`图书馆上。

从长远来看，从`my-company-express-like-wrapper-for-automated-password-checking`迁移到`express`和`mongoose`并做一些重构可能会节省很多时间，因为大多数候选人已经熟悉了这些库是如何工作的。

你认为`my-company-express-like-wrapper-for-automated-password-checking`是在解决一个以前没有解决的真正重要的问题吗？如果是的话，考虑将这个库开源，你甚至会从中发现一些好处。人们甚至会开始捐款。！你甚至节省了开发和头脑风暴的费用！

# 测试、维护和社区

好的框架通常是开源的，因此任何人都可以更深入地研究代码，也可以贡献，可以自由修改。

社区通常在维护框架和建议框架发展的正确方向方面起着重要的作用。如果一个框架正在被成千上万，甚至上百万的用户使用，那么这个框架很有可能不会很快消亡。贡献者通过在这些平台上讨论代码、提交问题或提交拉请求来提供帮助。任何人都可以做到，你也可以！

当你开始面临一些问题，并在 StackOverflow 上寻找解决方案时，拥有一个围绕该框架的非常活跃的社区将是一个好处。

大多数框架都经过了很好的测试，如果社区喜欢与最新技术保持同步，这是一个好迹象！例如，许多 npm 库现在增加了对 TypeScript 的支持，或者通过自动化单元测试工具和 CIs 进行测试。

# 想的少，有安全感，有结构

乍一看可能不那么明显，但这是一个巨大的红利。框架给你一个结构。如果你遵循结构，你不应该担心打破任何东西。通过遵循这种结构，你可能会发现更容易实现结果。一些有经验的开发人员能够在没有任何非核心库的情况下构建一个项目(我并不是说这是一个好主意)，但是对于一个普通人来说，这将是难以承受的。框架也允许经验较少的开发人员构建复杂的系统。通过使用正确的工具，您可以轻松地构建复杂的系统。

# 安全性

安全性可能永远不会完美(我也要表达一下我的想法)，但它每年都在变得越来越好。框架在这里，所以你不必考虑跨站点脚本，敏感信息泄漏，SQL 注入，…

通过使用正确的工具，您可以提高系统的安全性，保护您的数据，并保护您的客户。

# 结论

框架、库和工具将会对你的日常工作产生巨大的影响，同时也会对你正在努力实现的一些长期目标产生影响。框架不仅影响你，还会影响整个公司以及雇佣新的有经验的人的能力，或者及时实现你的目标。在选择一个框架时，有很多事情需要考虑，所以不要担心花时间去阅读它们。会有回报的。
