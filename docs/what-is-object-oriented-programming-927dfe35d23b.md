# 什么是面向对象编程？

它是一种软件范例，围绕数据/对象而不是功能和逻辑来组织软件设计和开发。这可以通过主要关注类和对象的概念来实现。

*   **类** 用户定义的数据类型，充当对象、属性和方法的蓝图。
*   **Object** 定义了更多数据的类的实例。

# OOP 的概念是什么？

当谈到面向对象时，有四个主要的概念。

*   抽象
*   包装
*   遗产
*   多态性

# 抽象

抽象是显示本质属性并隐藏其余属性的概念。这可以通过使用 ***接口*** 来实现。然而，接口不是抽象类。

## 连接

接口只包含方法的定义，而不包含实现。他们可以有多重继承。抽象方法被声明但没有被实现。接口用于将来的增强，不需要修改。

## 抽象类

抽象类是不能用于创建对象的受限类。因此，要访问它，它必须由另一个类继承。

抽象类不能被实例化。它可以定义和实现方法。子类可以实现或覆盖父类中定义的方法。它也可以包含变量。

抽象方法是无体方法，只能在抽象类中使用。主体是从派生的子/子类继承的。因此，在访问它时，我们需要使用 override 关键字。

一个类只能继承一个抽象类，但可以使用多个接口。抽象类可能有一个构造函数。

## 抽象 vs 密封

密封类防止类被扩展或继承。它不能被声明为抽象的。

当 sealed 关键字用于方法时，这可以防止方法被派生类重写。

# 包装

它的概念是将方法和数据成员放在一起，形成一个单一的单元。从而控制对其数据和方法的访问级别。

为了实现这一点，我们使用 ***访问修饰符***

*   **私有**——只有它所在的班级才能访问
*   **Public** —可从项目内的任何地方访问
*   **受保护的** —只能从该类及其派生类中访问
*   **内部** —只能在其组件内部访问(同一项目)
*   **受保护的内部** —可在派生类或同一程序集中访问
*   **Private Protected** —可在同一程序集中的类或派生类中访问

# 遗产

当一个类从另一个类派生时，创建共享属性和/或方法的层次结构。

四(04)种主要的继承类型:

*   **单继承**
    A 类是父类。
    类 B 是从类 a 派生出来的，因此，继承了它的特性和行为。
*   **多重继承** A 类和 B 类是父母。
    C 类从 A 类和 b 类中派生出两种特性和行为。
    在 ASP.NET、C#、VB、F#、Java 中不支持
*   **多级继承**A 类是父类。
    B 类派生于 a 类
    C 类派生于 B 类。
*   **层次继承**
    A 类是父类。
    B 类和 C 类从 a 类派生
    D 类和 E 类从 B 类派生
    F 类和 G 类从 C 类派生。

# 多态性

这个概念指的是一个变量、函数或对象采取多种形式的能力。当子类从父类继承并且两者具有相同的函数名，但是它们的行为不同时，可以实现这一点。

多态性有两种类型:

## 编译时多态性

当一个对象在编译时与功能绑定时。这是通过 ***“方法重载”实现的。***

这意味着在一个类中可以有多个同名的函数，有不同的原型。即使它们具有相同的名称，它们也可能具有不同的参数或签名，并返回不同的值。

```
public function add(int a, int b){
  return a + b;
}public function add(int a, int b, int c){
  return a + b + c;
}
```

## **运行时多态性**

它是在运行时将一个对象与功能绑定在一起。这是通过“ ***方法压倒*** ”实现的。通过重写，这为从父类继承的方法提供了新的实现(主体)。这也称为动态绑定或后期绑定。

```
// Base class
public class User 
{
  public virtual void GetInfo()
  {
    Console.WriteLine("Base Class");
  }
}// Derived class
public class Client : User
{
  public override void GetInfo(){
      Console.WriteLine("Derived Class");
  }
}
```

我们在基类方法中使用了 ***虚拟*** 关键字，这样就可以使用 ***override*** 关键字用派生类中的一些附加功能来覆盖该方法。
