> 您可以构建世界上最好的应用程序，但是如果您不知道如何告诉任何人，为什么要这么麻烦呢？

成为一名成功的开发人员不仅仅是学习一两种语言和知道如何构造 git 提交。还有许多不涉及编写代码的其他技能对开发人员的日常工作至关重要。您可以构建世界上最好的应用程序，但是如果您不知道如何告诉任何人，为什么要这么麻烦呢？

有许多技能可以使一个全面发展的开发者，其中很多都与复杂的算法或系统无关。这些技能中有很多是关于“软技能”或相对简单的概念，如沟通或写作。能够在头脑风暴会议上清晰地表达想法，起草技术大纲，甚至转动螺丝刀，会让你比只吃饭、睡觉和编码更加灵活敏捷。

在这篇文章中，我想看看程序员工作中的一些元素，他们在学校或互联网上并不总是强调这些元素。其中一些可能看起来显而易见，甚至很无聊，但是我向你保证，无论是作为开发人员还是作为专业人员，它们都是成功的关键。无论你是自由职业者还是全职人员，当你在开发的世界中导航时，所有这些基本原则都是重要的和适用的。

让我们开始吧。

## 沟通

这是迄今为止你可以投入时间改进的最重要的事情。在传统的商业环境中清晰有效地交流是一回事，但是作为一名开发人员这样做是完全不同的另一回事。当你与人交谈时，你可能不得不戴许多不同的帽子。有些帽子你可能不太适应。

某一天，您可能正在向一群高级开发人员或亲密的同事展示一些非常技术性的东西，而第二天，您就必须向产品团队介绍一组新功能以及它们如何帮助客户。管理你的观众是很重要的。没有人喜欢在会议上总是惹人讨厌的坏脾气工程师，他们认为自己是房间里最聪明的人。另一方面，每个人都讨厌听一个毫无准备和草率的人说话。

> "但是如果我完全不擅长说话呢？"

别担心！很多人讨厌公开演讲，有时会发现自己很难在会议上表达自己的观点。有很多资源可以解决这个问题，尽管一开始“练习说话”听起来很傻，但从长远来看，你会发现它确实很有帮助。

国际演讲会是一个致力于帮助人们提高公众演讲能力的团体。许多公司(尤其是那些有销售部门的公司)已经有定期的演讲会，任何人都可以参加。如果你的公司比较小，或者你没有这个，你可以找一个当地的会议，开始磨练你的演讲技巧。

我强烈推荐的另一个极好的资源是参加销售课程。是的，你没看错。*我建议一名工程师参加销售课程。这些课程教给你的不仅仅是如何达成交易。这些课程中有丰富的沟通技巧，教你如何读懂别人，如何有说服力。我强烈推荐的一门很棒的课程是 [Chris Croft 在 Udemy](https://www.udemy.com/course/master-successful-selling/) 上的课程。*

## 写作

当涉及到诸如概述新功能的技术规格或记录 bug 之类的任务时，拥有扎实的写作技能是必不可少的。有时候，你必须起草一些冗长的文件，并详细描述一些事情是如何运作的，但有时，能够制作一封措辞优雅的电子邮件或消息是你的武器库中必不可少的工具。

过于简洁或者喋喋不休就像什么都没写一样糟糕。确保你尝试在细节层次和以简洁的方式传达信息之间取得平衡。

> 在解释、记录和说服方面，书面交流和口头交流一样重要。

我经常引用的说服性写作的一个例子是亚马逊的电子邮件模板。 [*亚马逊无意中发出了他们的电子邮件模板*](https://medium.com/better-marketing/amazon-accidentally-sent-out-their-email-template-33e6ecbc0875) 是由[亚伦施诺尔](https://medium.com/u/eca86857cde3?source=post_page-----ba3b56b41495--------------------------------)创作的一个奇妙的作品，详细描述了亚马逊的电子邮件模板看起来是什么样的，以及他们如何将优雅的句子和段落缝合在一起，形成一条连贯的信息。

如果你发现自己更关注更长、更多的技术文档，那么看看[技术写作](https://en.wikipedia.org/wiki/Technical_writing)是一个很好的开始。技术写作者必须掌握传达高度详细和准确信息的艺术，同时保持内容的可读性和趣味性。Coursera [在这个专业有一个很棒的课程目录](https://www.coursera.org/courses?query=technical%20writing)供你开始学习。

## 自我完善

一旦你离开学校找到工作，你就不能停止学习。一两年之内，你会停滞不前，固执己见，抗拒每一个出现在你面前的小变化。你必须接受[持续改进](https://en.wikipedia.org/wiki/Continual_improvement_process)，甚至不仅仅是在编码的意义上。作为一名工程师，为了你的职业生涯和个人发展，你必须亲自拥抱它。*工作之外也是！*

阅读最新的[技术新闻](https://news.ycombinator.com/)，了解最新的库和框架，注册[在线课程](https://www.udemy.com/)，阅读！不要找工作，坐下来。做海绵。尽你所能从你的同事那里吸取一切，尤其是那些职位更高的同事。试着去了解更多你不懂的东西。机器学习或“大数据”是不是很吓人，很神秘？去找出原因。

## 物理硬件

有很多工程师对计算机或网络硬件几乎一无所知。虽然你可能能熬过一段时间，但它会在某个时候回来咬你一口。你至少要对事物的运作有一个基本的了解。有理论知识没问题，但是没有什么可以代替用真实的装备弄脏自己的手。

“只是一个前端工程师”这种常见的借口并不能说明问题。你的代码在某个时候会在一台物理计算机上运行，所以你最好知道它是如何工作的。

## 结论

程序员的工具箱在进化，最重要的是*在增长*。部署应用、服务和硬件的方式比以往任何时候都多。您的代码在出门之前通过无数部门的几率很高。能够成功地与各种不同的观众和个性交流是成功的关键。通过沟通管理期望，为面向公众的服务起草技术文档，以及理解系统中的流量都是日常工作。

通过花时间不仅提高你的编程技能，而且提高你的软技能，你会处于一个非常灵活的位置。下一次，当你发现自己在逃避属于这些类别的工作时，挑战自己。推动自己不仅探索代码的奇迹，而且探索专业人员工具箱的其他部分。
