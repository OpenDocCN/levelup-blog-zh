## 所以你最好选择他们作为你工作的榜样

![](img/39194bec14d57822c4ec08fe260c5c92.png)

我曾经在编程生活的黑暗面呆过一段时间——我可以证明，这没有把你的日子花在快乐面上有趣。

所以今天，我想评论一下我多年来遇到的不同类型的程序员，他们给人一种普遍快乐的感觉，当然也有例外。

# 新成员

我还没见过一个新同事不充满动力的——所以从纯粹的个人、纯粹的快乐角度来看,“频繁跳槽”的哲学还是有一些优点的。

对于一些人来说，他们的六个月试用期也标志着他们的动机突然下降，有些人会保持这种状态更长时间。努力保持新鲜感，这是获得工作满足感的一个简单方法。还有一些事情，比如交换团队和角色，或者承担像指导或“知识倍增”角色这样的新责任(答对了！).

# 专家们

我曾经和一个人共事过，他是一个神谕巫师，一个近乎神话的生物。他一年到头都住在湖边的房子里，偶尔会去办公室，但当他去的时候，他会花时间向我解释基本的 SQL，当时我还是学徒，还在努力理解表连接的概念。

他还在极短的时间内重构了复杂的流程，甚至有经验的开发人员也经常带着烟从他们的耳朵里出来。

在所有这些之间，这个人生活在一个完全不同的领域，几乎远离我们其他人面临的日常戏剧——但以一种非常好的方式。就像，他是一个冷静的点，我记得有几次我差点放弃，然后给他发信息，得到了一个小时的有益的现实检查。

我们都在自己的道路上有所专长，但真正的专家是完全不可解雇的，也不认为有必要辞职。

# 兼职工人

这些年来，我确实认识一些幻想破灭的兼职工作者——但也有完全相反的人。每周工作四天会让你完全摆脱困扰我们许多人的抑郁。

这通常是家庭的事情，虽然有时不是，好的人也是最渴望在工作场所提供帮助的人，而这并不是必然的。其中一个人每周休息一天进行跑步训练，其他一些人总是有新的故事来讲述他们，他们的妻子和孩子在你上次见到他们后的三天里都做了些什么。

# 心甘情愿的工作狂

我能很好地理解这一点，因为我在这方面花了几年时间。长时间的工作，太多的责任，太少的薪水——天哪，我真喜欢我的时间。我总是有足够的事情来保持忙碌，工作让我接触到我真正喜欢工作和聊天的人，我经历了一些最酷的工作冒险，否则我会完全错过。

在某个时候，当整个公司失去控制，我的许多工作狂同事开始离开时，这一切都结束了，然后我也离开了——但在那之前，我们度过了一段非常美好的时光，即使是在最糟糕的时候。

# 男女混合的团队

典型的全男性 IT 团队和男女混合的 IT 团队之间的差异是显而易见的。所有的团队和工作动态都会改变，而且只会变得更好。你获得了一些个人经验和观点，但也有很多专业的。

工作任务和工作流程的方式发生了变化，你可能会带着一些 UX 经验(原文如此)和电子营销背景的偏见，而每一次聊天都变成了不同生活经历和背景的充满活力、从未平衡过的混合物。我明白。

我想说的是，在谈论汽车、足球和同类相食之前，一个全是男性的开发团队需要大约 3.5 分钟的时间——这在第三次之后就变得过时了。

# 在此基础上:多功能团队

在我前一家公司工作的末期，我是一个团队中的一员，他们并不真正关心严格的等级制度和其他过时的概念。这甚至不是一个真正的团队，只是人们互相吸引，因为我们在个人和职业上是一致的。

团队领导、产品负责人、工作在他们工作描述之外的人，我作为一个随机的软件开发人员，女孩和男孩，年轻人和老年人——有趣。首席执行官有时会过来，公开讨论秘密，当有工作需要帮助时，其他人会加入一个小时的轻松工作和聊天。

下班后，当办公室里空无一人时，工作生活会有所不同，但仍有工作要做，还有人一起做。当你将这些与一个在不同部门工作的团队结合起来时，你最终会得到一连串的新闻、谣言，仅仅是掌握时间和事件的脉搏，这是很难超越的。

# 目睹一切的人

我真的很喜欢和那些在我还没出生的时候就已经是软件开发人员的人聊天——他们有最好的怀旧故事。这通常是一段美好的时光，因为他们仍然记得拨号调制解调器是比美国在线光盘成为飞盘之前更好的选择(是的，那是我进入科技世界的开始)。

有些人 30 岁就精神上退休了，有些人到了实际退休年龄，却仍然对当天最新的编程问题开怀大笑。

# 外卖:快乐的程序员是存在的

显然，还有很多其他因素在影响工作满意度——但我注意到这里提到的这些因素可以产生明显的差异，即使在其他方面相似的条件下。
