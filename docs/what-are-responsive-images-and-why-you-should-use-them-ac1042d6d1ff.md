# 什么是响应式图像？

在当今世界，图像已经成为网页设计的核心。

很少会发现一个网站没有任何图片。

但是你知道吗，虽然你的网站布局可以适应设备的尺寸，但是显示的图像却不能。

无论使用何种设备，都可以通过手机、平板电脑和台式机下载相同的图像。

例如，如果图像大小为 2MB，那么这 2MB 的数据将被下载到所有设备上。

您可以决定在移动设备上显示该图像的一部分，但是仍然需要下载整个 2MB 的图像。

这不公平。

如果同一个网页有多个图片要下载怎么办？

下载大量原本要缩小显示的大图片会影响手机和平板电脑的网站性能。

如果我们可以针对不同的设备宽度使用不同的图像，比如针对移动设备的小型图像、针对平板电脑的中型图像和针对台式机的大型图像，会怎么样？

这样，我们可以避免在较小的设备上下载不必要的数据字节，并提高我们网站在这些设备上的性能。

让我们来看看如何才能做到这一点。

# HTML 中的响应图像

![](img/622835efb210a5a3a8158537d57a946d.png)

考虑上面的图像。

假设这张图片是我们需要在桌面上显示的。为了减少在较小的设备上下载字节的数据，我们可以显示如下的裁剪版本，而不是下载完整版本的图像。

![](img/b880232fc23991c6958abafaea29c7fd.png)

加里·本迪格在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的照片

我们可以在 HTML 中编写类似这样的代码，在不同的视窗中下载不同的图像。

```
<img src="racoon.jpg" alt="Cute racoon"
     srcset="small-racoon.jpg 500w,
             medium-racoon.jpg 1000w,
             large-racoon.jpg 1500w" sizes="60vw"/>
```

让我们来理解这个代码片段是做什么的。

标签负责在 HTML 中渲染图像，src 属性告诉浏览器默认显示哪个图像。在这种情况下，如果 srcset 属性不受浏览器支持，则默认为 src 属性。

srcset 属性是这里最重要的属性之一。

它通知浏览器图像的宽度，而浏览器不必下载它们。浏览完 srcset 后，浏览器决定为该视窗宽度下载和显示哪个图像。

您可能还会注意到 srcset 中每个图像大小都有一个“w”描述符。

```
srcset="small-racoon.jpg **500w**,
        medium-racoon.jpg **1000w**,
        large-racoon.jsp **1500w**"
```

上面代码片段中的“w”以像素为单位指定了 srcset 中图像的宽度。

还有一个大小属性。它通知浏览器具有 srcset 属性的元素的大小。

```
**sizes="60vw"**
```

在这里，sizes 属性的值为 60vw，这告诉浏览器图像将是视口宽度的 60%。大小属性帮助浏览器从 srcset 中为该视口宽度选择最佳图像。

例如，如果浏览器视口宽度为 992 像素，则

992px 的 60%

= 592 像素

根据上面的计算，浏览器将选择宽度为 500w 或 500px 的图像，最接近 592px，以显示在视窗上。

最终是浏览器决定选择哪张图片。

请记住，为不同视窗宽度选择图像的决策逻辑可能因浏览器而异，您可能会看到不同的结果。

为较小的设备下载较少字节的数据允许浏览器快速显示这些图像，这提高了您的网站的性能。

# 外卖食品

网站下载速度慢的最重要原因是下载图像，这是兆字节的数据。

使用响应图像可以避免获取不必要的数据字节，从而缩短网站的加载时间并提供良好的用户体验。

唯一的缺点是，我们放弃了对浏览器的完全控制，让它为特定的视窗宽度选择显示哪个图像。

每个浏览器都有不同的实现来选择合适的响应图像。这就是为什么你可能会看到不同的浏览器以相同的分辨率加载不同的图像。

放弃对浏览器的控制以根据视口宽度显示图像来获得性能优势，这是您在使用案例中可能要考虑的一个权衡。
