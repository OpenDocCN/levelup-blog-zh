获得软件开发实习机会的5个技巧

一个3倍的实习生和一个导师

你的第一次软件开发实习成功了吗？展示了你的每一点能力，交付了伟大的成果，并得到了回报？我也是，这就是为什么我要分享我的一些对我有帮助的建议！

我是托马斯， Vinted 的前端开发人员。我已经在 Devbridge 、 Wix 和 Vinted 实习过了。有些很糟糕，有些则很刺激。今年夏天，我在面试潜在的实习生，并指导了其中一位。这就是为什么我相信我处于一个完美的位置来分享我经历的痛苦和收获，关于如何作为一名软件开发实习生取得成功。

1.不要把错误变成模式

你会犯错误。事实上，你会犯一些你能想到的最简单的错误。然而，你也应该从中吸取教训，继续前进。重复同样的错误表明要么是疏忽，要么是缺乏学习能力。

你是否被告知你的 PRs 太大了？很好，记得记下来，下次把它们分开。你的导师告诉你加班是不对的？太好了，你的笔记本上又多了一课。

2.专注于学习，而不是交付

实习是有时间限制的。不太可能期望您实现复杂的第三方API集成或构建新的测试框架。即使那会在你的简历上看起来很棒，你知道什么会更好看吗？

“在实习期间，我把所有的时间都花在了学习技术和最佳实践上。我利用我学到的知识构建了公司新登录页面的框架。

很容易发现一个选择交付而不是学习的人。学习的意图会让你更加好奇，优先考虑工作的质量，而不是数量。因此，以正确的心态开始实习。

3.没有单一的“正确方法”来编码这个特性

我一直认为伟大的公司对所有代码方面的问题都有“正确”的答案。我完全错了。现在我认为伟大的公司有所有的问题。作为一名软件开发人员，你的工作是评估所有可能的解决方案，并选择最合适的一个。

我编码得越多，就越觉得编码决策首先是基于价值观的。你喜欢函数式编程吗？你可能更喜欢纯函数。性能是优先考虑的吗？您可能已经准备好牺牲代码可读性来获得性能提升。

因此，如果你的解决方案被驳回，并不一定意味着它们是错的。通常，这只是意味着做决定的人心中有不同的价值观。

4.要有竞争力，但不能以牺牲他人为代价

除非你是公司里唯一的实习生，否则你会感受到竞争。那是不可避免的。接受它，但要针对你自己，而不是别人。它能给你动力，帮助你成为最好的自己。

一个人能做的最糟糕的事情就是贬低别人。在大多数情况下，团队精神是公司成功的核心。没有人愿意和一个行动与团队的成功不一致的人一起工作。我见过实习生在反复建议不要加班后加班，从而损害了与同龄人的信任，给他们带来了更大的压力。它不会让你走得太远。

5.支持你的每一个决定

你有没有因为没有找到拆分一个大功能的方法而在你的PR里留下乱七八糟的代码？一定要写一篇评论，解释你的想法。

支持你的决定的论据可能不是最有力的。这个决定本身可能不是最好的。然而，你表达了你的思考过程并分享了你的担忧，这一点至关重要。它强调诚实和你愿意为实现共同目标而变得脆弱。很多人会称之为勇气。

结论

这些建议会帮助我在实习期间展示最好的自己。现在，我希望它们能帮助你实现你的目标:

  1. 确保你从错误中吸取教训，不要再犯
  2. 学习应该是你的首要任务
  3. 解决问题有多种有效的方法
  4. 从不伤害别人企图更好看
  5. 每一个决定都要有理由。说清楚
