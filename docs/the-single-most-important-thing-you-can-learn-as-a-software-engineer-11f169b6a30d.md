## 学会如何学习

网络上充斥着各种自助文章，比如“如何成为高级软件工程师”或“2020 年你必须知道的 10 大技术”。实际上有成千上万个。

虽然这些文章中有许多包含有用的建议，但我相信所有这些都可以浓缩成一个教训:

**作为一名软件工程师，你能学到的最重要的一件事就是学会如何学习。**

# 元技能

学会如何学习是一种元技能，这意味着它是一种能让你在不止一个领域成长的能力。举几个例子:

学习反应是一种技能。学习如何快速掌握新的 UI 框架和库是一项元技能。

学会与你的经理和同事有效沟通是一项技能。学习如何最好地发展你的情商是一种元技能。

# 学习如何学习

![](img/8aed238e6647409cced9bca685cd6afe.png)

照片由[Unsplash](https://unsplash.com/@climatereality?utm_source=medium&utm_medium=referral)[上的气候现实项目](https://unsplash.com?utm_source=medium&utm_medium=referral)拍摄

那么，*你*怎么学最好呢？是的，你。

你是从网上看文章学的吗？你会去老派学校读真正的编程教科书吗？也许您甚至会花上几个小时从 W3C 或 MDN 中搜索文档。

你在网上看视频吗？也许你有一个最喜欢的 YouTube 频道，或者在 Pluralsight、LinkedIn Learning 或 Egghead 等学习平台上有一个帐户。

你通过做来学习吗？你喜欢仅仅为了尝试一种新的语言、库或框架而构建自己喜欢的项目吗？你在像 HackerRank 或 Codewars 这样的平台上练习编码吗？

你从现场演示中学到什么了吗？你参加当地的聚会和会议吗？你的公司有午餐学习演示吗，你可以从你的同事那里学到东西？

我个人使用资源的组合。我读了很多书，写了很多东西，看了很多视频课程，还做了很多兼职项目。不过最终，你如何对待学习真的无关紧要。重要的是你找到适合你的方法。

# 学习如何学习的好处

![](img/9491e0cf0acd49cd990aaa5a4cecf2f3.png)

韦斯·希克斯在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上的照片

如果你能掌握学习如何学习的元技能，世界充满了可能性。你不仅可以学到新的技能，而且可以高效地学习。

需要学习一门新的编程语言？没问题——你知道如何快速理解这门语言的基本概念和特点。

从一家不熟悉代码库的新公司开始？没问题——你知道你需要做什么来加快速度并立即投入运行。

想转行？没问题——你知道如何培养新的能力，自学你需要的材料和技能。

学会如何学习。其他一切都会水到渠成。
