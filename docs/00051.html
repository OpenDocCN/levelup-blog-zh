React 16和麻省理工学院许可证

React继续主导前端领域，2017年提供了迄今为止最受期待的版本之一版本16 。它包括支持异步UI呈现的纤程架构。这个版本还通过提供错误边界以及许多其他特性，使得管理意外的应用程序故障变得更加容易。

令人惊讶的是，在过去的一年中，React最重要的改进不是新特性，而是对其开源许可的改变。脸书放弃了导致公司远离React的BSD许可，并采用了用户友好的MIT许可。此外，Jest、Flow、Immutable.js和GraphQL也获得了MIT许可。

核心团队和顶级贡献者包括多米尼克·甘纳维、丹·阿布拉莫夫、索菲·阿尔珀特、塞巴斯蒂安·马克贝格、保罗·奥尚尼斯、安德鲁·克拉克、程露、克莱门特·黄、大概还有弗莱尼、布莱恩·沃恩。

渐进式网络应用

我们一直在寻找解决方案，以弥合网络和其他客户之间的差距。谷歌一直是通过将网络应用转换为渐进式网络应用(PWA)来增强网络应用的运动的先锋，2017年已经看到了快速的采用。PWA利用现代浏览器技术来提供更像移动应用程序的网络体验。这提供了改进的性能和离线体验，以及以前只有移动设备才有的功能，例如推送通知。PWA的主干是一个manifest.json文件和利用服务人员的组合。

Yarn的采用改善了JS包生态系统

《NPM》自首次发行以来已经成长了不少，但它仍然缺少一些关键的功能，而Yarn希望添加这些功能。Yarn的主要贡献是包缓存、确保确定性构建的锁文件、并行操作和扁平化依赖。这些特性非常成功，以至于NPM在5.0版本中实现了它们。Yarn拥有超过10亿次下载(目前每月125万次下载)，并拥有惊人的 28，000+ GitHub明星。即使您不使用Yarn，JavaScript包管理作为一个整体也由于它的发布而得到了显著的改进。

CSS网格布局

网格终于是CSS的原生产品，浏览器正在迅速采用它。过去，网格系统是用表格、float、flex和inline-block在CSS中创建的。原生CSS网格布局擅长将页面划分为主要区域，并为内容创建列和行。通过蕾切尔·安德鲁查看https://gridbyexample.com/开始学习。

所有主流浏览器都支持WebAssembly

WebAssembly (或 wasm )现已在所有主流浏览器中推出。wasm是一种用于浏览器内客户端脚本的低级字节码格式。由于它是低级的，它拥有令人难以置信的性能，但也提供了一个 JavaScript API ，为前端开发人员提供了一个更容易的切入点。火狐最近宣布在所有浏览器中包含。

无服务器架构

Vue.js继续流行

即使React取得了巨大成功，Vue(由尤雨溪创造)仍然越来越受欢迎。该框架提供了基于组件的架构，是React的主要替代方案之一。它已经被包括GitLab在内的大公司采用，git lab使用这个框架总结了他们过去的一年。

CSS-in-JS并为即将到来的CSS圣战做准备

在我们见证了JavaScript的快速发展之后，生态系统开始稳定下来。不可避免的是，我们也将看到CSS同样无情的进步，因为它赶上了现代web应用程序的需求。2017年，主要的进步是对CSS-in-JS的显著改进和采用，其中所有样式都是通过代码而不是样式表构建的。目前还不清楚这是否是前端社区的最终发展方向，但这是当前最前沿的方法，似乎可以解决构建基于组件的应用程序时发现的许多问题。

静态站点生成

2017年，随着静态网站的回归，时间发生了扭曲。像 Gatsby 这样的框架让你能够使用React和其他现代工具来构建静态网站。不是每个网站都需要或者应该是一个复杂的现代网络应用程序。静态站点生成为您提供了服务器端呈现的好处和无与伦比的速度，因为您提供的是预构建的标记。如果你正在寻找一个好的例子，官方的React文档本身就是使用Gatsby构建的。

静态网站生成引发了另一种被称为 JAMStack 的趋势:“JavaScript、API、标记”。JAMStack使用相同的静态预建HTML文件以及可重用的API和JavaScript来处理请求/响应周期中的任何动态编程。对于免费开始使用JAMStack和静态托管来说，Netlify是一个很好的选择。 Brian Douglas 写了一篇优秀的文章，通过构建黑客新闻克隆，比较了JAMStack与服务器端渲染应用。

GraphQL爆炸，让我们重新思考API构造

GraphQL似乎正在快速占领REST的地盘，而萨梅尔·布纳甚至声称 REST已经死了。GraphQL不需要管理多个端点和获取不必要的数据，而是允许客户端声明性地定义它需要的数据，并从单个端点检索所有数据。

它变得如此流行，以至于 GitHub 已经用GraphQL编写了其API 的最新版本，许多公司正在开发产品使其对所有开发者开放，例如由 Johannes Schickling 开发的流行的 Graphcool 框架。

反应路由器4

由Ryan Florence和Michael Jackson设计的React路由器从仅仅是React的一个路由器发展成为真正的React路由器——一个简单使用React组件实现的声明式路由器。这是React团队认可的第一个版本。API已经稳定，并且 React培训团队已经声明在项目的生命周期内不会看到任何大的突破性变化。

Angular发布了v4，紧接着是v5

在臭名昭著地跳过版本3来维护SEMVER之后，Angular 4于3月23日正式发货。在版本4中，Angular团队采用了社区项目Angular Universal——它提供了一种服务器端渲染Angular应用程序的方法——作为Angular项目的官方部分。Angular Animation包也是从@angular/core中提取的，仅在需要时导入，并且视图引擎中的提前编译已针对性能进行了重构，“在大多数情况下，为您的组件生成的代码大小减少了约60%。”

第5版采用了更多期待已久的改进。由于新的@angular/service-worker包，用Angular v5创建渐进式Web应用程序现在比任何早期版本都容易得多。Angular编译器也得到了改进，在开发过程中支持更快的构建/重建，Angular路由器现在公开了所有新的生命周期挂钩，包括ActivationStart、ActivationEnd、ResolveStart和ResolveEnd。

2018年有什么期待

  • 当我们想出在基于组件的应用程序中处理样式的最佳方法时，CSS之争愈演愈烈。
  • 更多的公司采用具有统一代码库的移动解决方案，如 React Native 或 Flutter 。
  • 随着离线功能和无缝移动体验的出现，网络变得更加本地化。
  • WebAssembly可以取得巨大进步，提供更好的网络体验。
  • GraphQL继续挑战REST。
  • React加强了它的地位(是的，甚至更多),因为它不再担心许可证了。
  • Flow和TypeScript占据了更强的优势，给了JavaScript更多的结构。
  • 集装箱化对前端架构的影响变得更加普遍。
  • 虚拟现实使用像 A-Frame 、 React VR 和 Google VR 这样的库向前迈进。
  • 人们使用区块链和 web3.js 构建一些非常酷的应用程序(作者是马雷克·科特维奇和法比安·沃格斯泰勒)。
