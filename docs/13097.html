我们在编程的第一天学习if/else。然后我们忘记了其他的

问题

  • 可读性
  • 认知负荷
  • 不可预见的情况
  • 故障快速原理违反

解决方法

  1. 编写显式else

语境

如果我们提前返回If句，我们可以省略else部分。

之后，我们移除IF 并使用多态。

这就是我们错过真正案例的时候。

示例代码

错误的

function carBrandImplicit(model) {
  if (model === 'A4') {
    return 'audi';
  }
  return 'Mercedes-Benz';
}

对吧

function carBrandExplicit(model) {
  if (model === 'A4') {
    return 'audi';
  }
  if (model === 'AMG') {
    return 'Mercedes-Benz';
  }  // Fail Fast
  throw new Exception('Model not found);
}
