# 什么是链表，信息是如何存储的？

在计算机科学中，不同的数据结构被用来以允许对该数据的特定功能的方式存储数据。在存储、可扩展性和可访问性方面，每种数据结构都有自己的优缺点，因此，在解决某些问题时，一种数据结构可能比另一种更好。

链表是一种线性存储数据(即有顺序)而不线性使用内存的数据结构。让我们更深入地研究一下这个特性，因为链表的所有其他好处都源于这个特性。

当一个程序在计算机上运行时，程序运行所必需的信息存储在 ram 或随机存取存储器中(在本文的其余部分，我将把它称为“内存”)。内存存储信息类似于数组，因为它由一组索引组成，可以存储一定量的信息，在本例中是一个字节。内存是有限制的，所以使用的内存越多，可用的内存就越少。

在许多情况下，程序会在内存中线性地存储功能信息*以利用计算机的*缓存*来更快地访问数据。计算机的高速缓存用于存储内存中被调用的任何信息块附近的数据，允许计算机更快地访问所需的下一个重要数据块。*

*牢记在心；有时，在内存中线性存储数据是不可能的。在任何给定的时刻，多个程序在一台计算机上运行，并且所有程序都在竞争使用内存来运行。因此，当需要利用内存中的可用空间时，程序必须想出非线性存储功能信息的方法。*

*链表是一种数据结构，程序可以使用它来利用未使用的内存，同时仍然保持线性数据流，这就是它如此重要的原因。*

*一个链表由包含两条信息的*节点*组成:被存储的数据和对列表中下一个条目的引用。这允许程序根据需要创建节点，将它们存储在可用数据中，并维护顺序。*

```
*// Example of a node for a Linked List
class Node { constructor(data, next = null) {
    this.data = data;
    this.next = next;
  }
}*
```

*请注意，还有双向链表，其中的节点也包含对列表中前一个节点的引用。*

*每个链表包含两个特殊的节点:一个*头*和一个*尾。*链表的头是第一个节点，建立了链表的顺序。虽然不是每个链表都用一个头来初始化，但是当信息被添加到链表中时，总是会创建一个头。*

```
*// Example of a Linked List
class LinkedList { constructor(head = null) {
    this.head = head;
  } insertFirst(data) {
    this.head = new Node(data, this.head);
  } size() {
    let counter = 0;
    let node = this.head; while (node.next) {
      counter++;
      node = node.next;
    } return counter;
  } // etc....}*
```

*正如你在上面的链表的`size`方法中看到的，我们能够通过从头开始向下迭代直到到达我们想要的索引来访问链表中的信息。*

*当节点的下一个值为 *null* 时，发现链表的*尾*或链表的结尾，表示列表中没有其他节点。*

# *赞成还是反对*

*我们已经讨论了链表的一些优点，但是让我们更直接地讨论它们。*

*在某些情况下，链表是首选的数据结构。首先，链表通常优先于其他线性数据结构，如队列或数组，因为它们允许我们根据需要存储信息，而不必担心节省内存空间。数组和队列以线性方式保存信息，这意味着您需要提前设置它们的大小，即使它们没有完全填满数据。*

*请注意:数组在 JavaScript 中不是这样工作的。JavaScript 使用动态数组来存储信息，动态数组会随着自动增长而调整大小。我们稍后将进一步讨论这个问题。*

*当需要频繁操作数据时，链表也是首选。对于数组，在开始时删除或添加信息需要重新调整后面每条信息的位置，导致时间复杂度为 O(n)。*

*对于链表，这个操作可以在常量时间内更快地执行，因为我们所要做的就是重置它的头。*

*也就是说，链表也有一些缺点。*

*链表在存储成本方面被认为是昂贵的，因为需要内存来保存每个节点的顺序。这相当浪费空间，并显著增加了保存信息所需的内存量。*

*此外，对于链表来说，随机访问是不可能的，这意味着即使您知道想要访问的数据的确切位置，您仍然需要遍历链表的一部分才能到达它。与可以在固定时间内访问特定索引的数组相比，这是非常低效的。*

# *JavaScript 中什么时候使用链表？*

*正如我前面提到的，使用链表可能更好的一个主要原因并不适用于 JavaScript。在其他语言中，数组需要有预定义的长度或大小，而 JavaScript 使用动态数组。这些阵列可以根据需要扩展和收缩大小，从而在存储不确定大小的信息时无需另一种解决方案。*

*动态数组还具有按需使用可用内存的优势，并且可以按需扩展到未使用的内存。*

*记住，JavaScript 项目中链表的用例很少。老实说，我读过的关于这个主题的每一个参考资料在提供如何使用链表的例子之前，总是附带一个警告:链表在 JavaScript 中有点多余。*

*也就是说，链表通常是其他数据结构的构建块，所以你应该花时间去理解它们。基本原则总是很重要，链表也不例外。*
