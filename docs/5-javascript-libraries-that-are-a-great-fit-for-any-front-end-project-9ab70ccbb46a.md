## 最棒的是，无论你使用 React、Vue 还是 Angular 都没有关系

作为一名开发人员，你一直在寻找改进工作方式的方法，只要有可能，你就会努力让你的生活变得简单一点。一种方法是在项目中使用正确的工具。由于有如此多的工具，几乎不可能把它们都检查一遍或者试一试。

这就是为什么我们将在本文中介绍五个非常适合任何前端项目的 JavaScript 库。不管你是用 React，Vue，Angular，还是根本不用框架。这个库列表是我亲手挑选的，我在我参与的项目中使用了所有的库。

# 1.总是（德语词）

[Immer](https://immerjs.github.io/immer/docs/introduction) ，在德语中是 *always* 的意思，是一个很小的包，允许你以更方便的方式处理不可变状态。Immer 基于写入时复制机制。

根据文件，这是 Immer 的工作:

> 基本思想是将所有的更改应用到一个临时的 draftState，它是 currentState 的代理。一旦你所有的突变都完成了，Immer 将会基于草稿状态的突变产生下一个状态。这意味着您可以通过简单地修改数据来与数据进行交互，同时保留不可变数据的所有好处。

Immer 的强大之处在于，它允许您通过简单地修改当前树来创建下一个不可变的状态树。处理状态有时会有点痛苦和混乱。Immer 让这变得容易多了。

使用 Immer 时，您不需要学习任何新的 API——您可以使用您已经熟悉的 JavaScript 对象、数组、集合和映射来创建不变性。

虽然 Immer 有一点点的学习曲线，但它确实值得一试，因为它使处理状态变得轻而易举。

# 2.洛达什

[Lodash](https://lodash.com/) 通过消除处理数组、数字、对象和字符串的麻烦，使 JavaScript 变得更加简单。在 GitHub 上有超过 40k 个星星，Lodash 是一个受欢迎的库。根据其网站，Lodash 是一个现代 JavaScript 实用程序库，提供模块化、高性能和额外功能。

Lodash 可以很容易地集成到任何 JavaScript 项目中，并且是我个人最喜欢使用的库之一，因为它非常优雅。有了 Lodash，你的代码将变得更加易读和易于维护。我强烈建议你去 Lodash 看看。

# 3.玩笑

编写代码时，测试是绝对必要的。这就是为什么 [Jest](https://jestjs.io/) 必须在这个名单上。Jest 是一个专注于简单性的 JavaScript 测试框架。Jest 具有你期望从测试框架中得到的所有特性，比如代码覆盖、模仿和非常全面的文档。

Jest 允许你为你的代码编写单元测试，它有一个非常容易理解和直观的 API。如果你以前编写过单元测试，这应该很容易上手。

除此之外，Jest 还附带了另一种形式的测试:快照测试。当您使用组件时，快照测试非常有用—例如，当您使用 React、Vue 或 Angular 时。

但是什么是快照测试呢？这是医生对快照测试的描述:

> 一个典型的快照测试用例呈现一个 UI 组件，获取一个快照，然后将它与测试旁边存储的一个参考快照文件进行比较。如果两个快照不匹配，测试将失败:要么是意外的更改，要么是引用快照需要更新到 UI 组件的新版本。

Jest 是由脸书创建的，在 GitHub 上有超过 3 万颗星星，这对一个测试框架来说是相当多的。

# 4.故事书

[Storybook](https://storybook.js.org/) 当你想孤立地开发 UI 组件时，它是一个非常棒的工具。这允许您一次处理一个组件。你不需要构建一个复杂的开发栈来处理你的组件。你所需要的只是一个启动 storybook 的命令。

但是到底什么是故事呢？这是医生告诉我们的故事:

> 故事捕获 UI 组件的呈现状态。开发人员为每个组件编写多个故事，描述一个组件可以支持的所有“有趣”状态。

每个组件都有多个故事，每个故事描述一个组件可能处于的状态。这意味着你将有描述快乐之路场景的故事，也有涵盖更多特殊用例的故事。

storybook 的伟大之处在于，无论何时你在一个组件上工作，你都会立即看到由于热重新加载而产生的变化。

故事书和笑话结合起来特别好。因为 Storybook 隔离了 UI 组件的开发，所以为这些隔离的组件编写测试变得非常容易。确保为您的组件创建快照测试。当你想确保你的用户界面不会发生意外变化时，快照测试是一个非常有用的工具。

Storybook 在 GitHub 上有超过 50K 颗星星，这使得它成为开发者非常推崇的工具。

# 5.Moment.js

如果您曾经不得不处理日期或时间，那么您可能已经为格式化、验证或处理不同的时区而挣扎过了。使用 [moment.js](https://momentjs.com/) 时不会。这个库是您遇到的所有日期和时间问题的一站式解决方案。

开箱即用，它有十几个特性，比如支持国际化。您可以加载多个语言环境，并在它们之间轻松切换。

GitHub 上有近 45K 个 stars，这是处理日期和时间时最受欢迎的库。
