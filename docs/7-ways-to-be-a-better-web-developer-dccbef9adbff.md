**我在 2001 年建立了我的第一个网站。这是互联网时代的千年。**

**在那段时间里，我学到了很多东西，生活中的*和网络开发中的*。**

**现在，作为一个成功的企业主，他在数百个网站上工作过，我想与你分享我的主要经验。**

**这条给网络开发者的建议是经过深思熟虑的。它来自于惨痛的教训。我希望当我开始我的职业生涯时，有人告诉过我这些。**

# **#1 分享你所知道的，不要囤积你的知识和经验**

**我合作过的一些最好的开发人员乐于指导他们的同事。实际上，这仅仅意味着既是学生又是老师。**

**分享你所知道的意味着当有人在公开场合哀叹他们的编码问题时，不要滚动阅读。有多少次一个陌生人在 Twitter 或 Stack Overflow 上来帮助你？**

**对我来说，真正的变革者将他们的知识公之于众，与他人合作。因果报应有一种魔力——你越多产地支持他人，社区就越有可能在你需要的时候来帮助你。**

> **你越积极地支持他人，社区就越有可能在你需要的时候帮助你。**

# **#2 提高你的书面和口头沟通技巧，让别人理解你**

**直到我职业生涯的后期，我才意识到 web 开发不仅仅是能够破解逻辑难题或设计直观的 UI。**

****事实上，我会说能够*有效沟通*甚至比作为开发人员拥有天赋*更重要。*****

**良好的沟通技巧对于解释和表达想法至关重要。如果你不与客户进行深入的交谈，你怎么能真正理解客户的简介呢？如果你不能清楚地表达自己的想法，你怎么能指望别人接受呢？**

**以我的经验来看，在 web 开发项目中，很多出*错的*都可以通过利益相关者之间简单明了的对话来避免。**花时间真诚地理解他人。你会发现，作为回报，你的地位也会得到更大的尊重。****

> **Web 开发不仅仅是能够破解逻辑难题或设计直观的用户界面。**

# **#3 挑战你对他人所知的假设**

**有时候，我的客户对技术的理解比我最初期望的要清晰。在其他情况下，他们知道的要少得多。**

**我经常很惊讶。**

**最重要的是，最好的开发人员挑战他们对客户和同事的先入为主的看法。**

**web 开发人员对客户经理或营销人员嗤之以鼻是很常见的。我们认为其他人不可能知道我们知道的那么多。事实上，其他人带着大量有价值的知识。**

**如果你对你周围的人不做任何假设，并虚心听取他们的想法和问题，你将会无限地提高你的网络开发游戏。**

# **#4 学习新事物，但确保建立基础知识**

**科技筹码格局[不断变化](https://thebusinessofwebdesign.com/how-to-keep-up-with-technology-as-a-web-developer/)。我们多久会听到一个闪亮的新框架或即将到来的必备库？**

**许多开发人员将他们的知识建立在不稳定的基础上。在深入研究下一个 Javascript 或 CSS 框架之前，**花点时间学习基础知识。**学会减少对初学者工具包的依赖，并花时间钻研具体细节。**

**这将有助于你在职业生涯中建立长久的职业生涯，并让你对这一切是如何结合在一起的有一个基本的了解。通过获得对行业的全面了解，你可以更好地与同行交流，更容易学到新东西。**

**无论如何——把你的精力集中在花上，但是不要忘记给根部浇水！**

> **通过获得对行业的全面了解，你可以更好地与同行交流，更容易学到新东西。**

# **#5 不要抱怨其他开发者的工作**

**最重要的一课是——不要评判他人的工作！**

**即使是一个新手开发者也能发现几乎所有在线网站的问题。没有人是完美的。网络项目受制于充满政治色彩的决策和不断变化的优先级。**

**不要去判断，要知道在几乎所有的情况下，开发者都在尽力。和你一样，他们也在旅途中。成熟的专业人士不会抱怨他们的同行。在成为高级开发人员的道路上，要学会让你对别人和他们的工作的感觉来影响你。不要因此而过于分心——这只会让你慢下来。**

> **网络项目受制于充满政治色彩的决策和不断变化的优先级。**

# **#6 有保险以防时局不佳**

**您是否曾经因为硬件故障而丢失文件，或者因为合同疏忽而失去有偿工作？**

**如果你没有，那只能说明它还没有发生。**

**我们不喜欢想到不好的事情发生，但不幸的是，它最终会发生在我们身上。去年，我的服务器出现了一个大问题，破坏了我的本地网站。经过几天的压力，我设法恢复了大部分数据，但我吸取了教训——**有某种保险**。多亏了那次可怕的经历，我现在一天多次备份我的整个开发服务器，以防将来发生灾难。**

> **如果你以前没有经历过技术灾难，那只能说明它还没有发生。**

**这本身不会让你成为一个更好的开发人员，但是它会引导你走向最佳实践。**

# **#7 爱你的客户——这会让一切变得更好**

**在我的职业生涯中，我很晚才意识到**客户*不是敌人。*****

**多年来，我与蔑视客户的开发团队一起工作。哦，我们会如何抱怨！直到现在，我才意识到抱怨客户对我的精神健康和工作生活造成的灾难性影响。**

> **多年来，我和一些蔑视客户的开发团队一起工作。**

**保持谦逊，向你的客户学习，尊重他们的知识。有时，它们可能仍然令人头疼，但至少你会跨越更高的境界。只有真正同情他人，你的客户关系才能蓬勃发展。**

**当我写这篇文章的摘要并重读我在二十多年的 web 开发中磨练出来的建议时，似乎出现了一个共同的主题。**

**主题是谦逊。**

**Web 开发人员是有创造力的人。创造力带来了自我的巨大帮助。我最后的建议是控制你的自我重要性。毫无疑问，你可以为这个世界贡献很多，但是当别人自己发现你的这一点时，奇迹就发生了。停止耀武扬威——分享你的知识，减少你的批评，实践同情心。**

> **创造力带来了自我的巨大帮助。**

**这 7 个技巧，当结合起来使用时，将会让你的网站开发者生活更加富足和愉快。**
