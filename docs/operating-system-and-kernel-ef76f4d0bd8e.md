# 操作系统概述和内核解释

> 原文：<https://levelup.gitconnected.com/operating-system-and-kernel-ef76f4d0bd8e>

![](img/5c8dac10bb17a4eb6b424cb1a8bd3bf6.png)

操作系统是硬件资源的管理者，为这些资源提供了一个简化的、独立于设备的接口。在这个角色中，它还促进了这些资源的共享，因此它负责在现在分离的进程(或最近的虚拟机)之间提供隔离和安全性。

以一种书呆子的方式，

> “一个由一组特定的库和基础设施组成的平台，应用程序可在此基础上构建并相互交互”

**操作系统的功能** —操作系统执行三个功能:

1.  **便利:**操作系统让电脑使用起来更加方便。
2.  **效率:**操作系统可以让计算机系统资源得到高效利用。
3.  演进能力:操作系统的构建方式应该允许新系统功能的有效开发、测试和引入，同时不干扰服务。

![](img/836481b291d229736e9f9b146cd1e89f.png)

计算机系统的概念视图

每台通用计算机都由硬件、操作系统、系统程序和应用程序组成。

硬件由内存、CPU、ALU、I/O 设备、外围设备和存储设备组成。系统程序由编译器、加载器、编辑器、操作系统等组成。应用程序由商业程序、数据库程序

每台计算机都必须有一个操作系统来运行其他程序。操作系统在各种用户的各种系统程序和应用程序之间协调硬件的使用。它只是提供了一个环境，其他程序可以在其中做有用的工作。

这些功能中有很多是紧密相关的。进程需要分配给它们的 CPU 和内存。设备需要内存和中断，并且通常不会专门分配给一个进程，因此操作系统需要通过更高级别的抽象层(如套接字、VFS)来共享它们

# 核心

内核是这些被认为不可分割的部分的集合——因此得名——并且独立于任何用户/进程运行。它充当应用程序和在硬件级执行的数据处理之间的桥梁。

内核是操作系统的核心模块。它是首先加载的操作系统的一部分，它保留在主内存中。因为它驻留在内存中，所以在提供操作系统和应用程序的其他部分所需的所有基本服务的同时，内核尽可能小是很重要的。内核代码通常被加载到内存的受保护区域，以防止它被程序或操作系统的其他部分覆盖。

通常，内核负责内存管理、进程管理/任务管理和磁盘管理。内核将系统硬件连接到应用软件，每个操作系统都有一个内核。例如，Linux 内核被许多操作系统使用，包括 Linux、FreeBSD、Android 等。

内核在内核空间执行其任务，如执行进程和处理中断，而用户通常做的一切，如在文本编辑器中编写文本或在 GUI(图形用户界面)中运行程序，都是在用户空间完成的。

这种分离防止用户数据和内核数据相互干扰，从而降低性能或导致系统变得不稳定(并可能崩溃)。当一个进程向内核发出请求时，这个请求被称为一个**系统调用**。

内核负责:

*   应用程序执行的流程管理
*   内存管理、分配和 I/O
*   通过使用设备驱动程序进行设备管理
*   系统调用控制，这对于内核服务的执行至关重要

> **内核的功能**

**中央处理器:**内核负责在任何时候决定将众多正在运行的程序中的哪一个分配给一个或多个处理器。

**随机存取存储器:**随机存取存储器用于存储程序指令和数据。通常，两者都需要存在于内存中，程序才能执行。通常，多个程序需要访问内存，经常需要比计算机可用的内存更多的内存。内核负责决定每个进程可以使用哪些内存，并决定当没有足够的内存可用时做什么。

**输入/输出(I/O)设备:** I/O 设备包括键盘、鼠标、磁盘驱动器、打印机、网络适配器和显示设备等外围设备。内核将来自应用程序的执行 I/O 的请求分配给适当的设备，并提供使用该设备的方便方法。
内核通常还提供进程间同步和通信的方法，称为进程间通信(IPC)。

操作系统的其他部分包括与内核交互的驱动程序，有几层帮助程序处理用户界面、网络和位于内核和实际应用程序之间的操作系统的其他核心部分。
