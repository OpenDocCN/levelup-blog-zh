# 面试时要记住的事情

![](img/14002d5dc131c3d2a3be2b955a3ea646.png)

照片由 [Siora 摄影](https://unsplash.com/@siora18?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄

**你的 MVP 是多少**

你有责任提出问题，并为你要解决的系统设计问题定义最小可行的产品。任何系统设计问题都有许多你可以实现的特性，但是你有责任和面试官一起工作，找出面试官最关心的特性和不重要的特性。请记住，您是在一个非常严格的时间框架下工作的，所以请确保您的功能集很小，并且深入到这些功能集中。

**系统规模**

问题应该是“系统应该扩展到什么程度？”例如，您需要在数据库中存储多少详细信息，或者每秒需要处理多少请求，或者该系统预计会有什么样的延迟

**避免流行语**

假设你的面试在一两天后，你读到了这个一致散列法。不要只是去面试就开始在面试中抛出那些话。它有时可能有效，但大多数时候无效，事实上，大多数时候适得其反。因此，无论你在采访中提到什么技术概念，都要确保你对这些概念有深入的了解。

**50000 英尺视野**

在深入问题的细节之前，尝试定义问题的 50000 英尺的视角。

1.  定义所有的 API 端点。
2.  画正确的方框。
3.  弄清楚谁是这个系统的参与者。

# 系统方法

![](img/323de1476f2b45249ce036782e47dd5d.png)

**第一步(特色):**

确定特征，特征是你必须为面试官做的工作，以了解他们在那个时间点真正关心的是什么，以及他们不关心和可以排除的是什么。

**第二步(API 规范):**

接下来是定义 API。您需要确定您的服务有哪些 API，谁将调用这些 API，以及他们将如何调用这些 API

**第三步(可用性):**

思考服务的可用性。你需要弄清楚服务的可用性。例如，如果一个数据库宕机，服务是否仍然可用，或者如果整个数据中心宕机，服务是否仍然可用，您必须与采访者讨论，以了解他们可以容忍此服务的可用性中断程度。

**步骤 4(延迟):**

了解系统延迟。因此，如果是后台服务，他们可能不会太关注延迟。另一方面，如果它是一个面向客户的服务，那么很明显你希望你的系统超快，并且根据需求你可能希望引入缓存来改善系统延迟。

**第五步(可扩展性):**

现在，您设计了一个适用于 100 个用户的服务。但问题是，它是对 1000 个用户有效，还是对 100 万甚至更多的用户有效？

**第五步(耐久性):**

这对于某些面试来说可能很重要，但对于其他问题来说可能不重要。因此，事实是，数据可以安全地存储在数据库中，数据既不会丢失也不会受到损害。所以有时候说你会用 X 数据库，它会为我做所有的工作。另一方面，在设计数据库时，耐用性起着重要作用，您需要确保您的系统是耐用的。然后，我们需要有一个固体设计使用类图，建立表之间的关系。

当你遇到像设计一个停车场或者设计一个电梯系统这样的问题时，知道你将如何设计这些类，以及你将使用哪些面向对象的原则来解决那些我们有安全和隐私的问题是很有趣的

当您遇到为这样的问题设计授权服务器的问题时，安全性和隐私将扮演重要角色

**第五步(成本效益):**

无论你建议什么样的解决方案，如果它是一个经济有效的解决方案，是否有一个更经济有效的替代解决方案，你都需要讨论不同解决方案在成本方面的利弊。
