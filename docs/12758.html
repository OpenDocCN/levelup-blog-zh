kubectl是一个非常棒的命令行工具，它允许您与Kubernetes集群进行通信。

这篇文章分享了每个k8s开发者应该知道的最常用的命令。

上下文—用于访问多个集群

  • kubectl配置获取上下文
  • kubectl配置当前上下文
  • kubectl配置使用上下文
  • kubectl配置集-上下文
  • 库贝克配置删除-上下文

节点—虚拟机或物理机

  • kubectl获取节点
  • kubectl得到no -o宽
  • kubectl get no -o yaml
  • kubectl描述没有
  • 从一个节点中驱逐你所有的豆荚
  • kubectl顶级节点#按其资源利用率排序

Pods一个或多个紧密耦合的容器的集合

  • kubectl获取豆荚
  • kubectl get pods-o wide-namespace
  • kubectl获取pods显示-标签
  • kubectl get pods -l app=
  • kubectl get pod -o yaml
  • kubectl获取pods - field-selector状态
  • kubectl删除窗格

命名空间—在单个群集中隔离资源组

  • kubectl获取ns
  • ku bectl config set-context-current-namespace =
  • kubectl删除名称空间

部署—驱动副本集

  • kubectl获取部署
  • kubectl描述部署
  • kubectl删除部署/
  • kubectl apply -f 。yaml
  • kubectl日志部署/ -c

服务——一组逻辑单元和访问它们的策略

  • kubectl获取服务
  • kubectl描述svc
  • kubectl获得svc -o范围
  • kubectl获取svc显示-标签

其他命令

  • kubectl apply -f XXX.yaml
  • 库贝克应用-f ./
  • ku bectl diff-f ./my-manifest . YAML
  • kubectl rollout撤销部署/前端-修订=2
  • kubectl scale —副本=3 rs/foo
