也许你是一名优秀的 Java 程序员，拥有丰富的知识和经验。你在目前的职位上做得很好。你已经擅长一些基本的东西，如编码、数据结构和算法，以及计算机科学概念，如网络、协议、面向对象编程等。但是您觉得在使您成为摇滚明星 Java 开发人员的技能组合中缺少了一些东西。在这里，我记下了一些要点，以帮助你成为更好的 Java 开发人员。

# 1.了解你的 IDE

为了记住您对代码所做的一切，您需要 IDE/编辑器的支持。你花了很多时间在你的 IDE 上。所以如果你不了解你的 IDE，就不要用快捷键；会消磨你宝贵的开发时间。所以首先，选择你最喜欢的 IDE 并掌握它。如今，IntelliJ IDEA、Eclipse 和 NetBeans 等 IDE 是编写 Java 代码最流行的 IDE。我建议你使用 **IntelliJ IDEA** ，因为它有一些高级功能，如智能代码完成、动态代码分析、高级重构、工具和框架支持等。

您可以了解一些使您的任务变得简单的插件和一些使您的技能更上一层楼的键盘快捷键。

> 优秀的 Java 开发人员的一个最重要的特征是他们非常擅长他们的工具。他们不仅比一般开发人员知道更多的工具，而且他们非常了解自己的工具。”

# 2.了解 Java 的最新特性

如果你是一名优秀的 Java 开发人员，了解旧的 Java 特性，并且对此感到满意，我会要求你继续前进。尽管你的职业学习在某个时候变得缓慢，但如果你现在不行动，你可能会被甩在后面。从 Java 8 到 16，Java 有一个显著的变化。像 **Lambdas、Stream API、函数接口、默认方法、接口中的静态方法、记录、密封对象**等特性被新一代 Java 开发者广泛使用。

即使你不了解 Java 的最新特性，也可能在技术面试中表现不佳。

# 3.选择正确的框架

现在有很多框架，比如 Spring、Hibernate、Struts、Google web toolkit [GWT]、JavaServer Faces [JSF]、Grails、Vaadin、Blade 等等。你必须选择一个你将成为大师的框架。我建议您使用 **Spring MVC、Spring Boot 和 Spring Cloud** 进行 web 应用程序开发和 REST APIs 微服务，因为 Spring 框架支持依赖注入等最佳实践，并使您的应用程序更具可测试性，这是当今软件开发的关键要求。

# 4.掌握单元测试技能

单元测试技能决定了普通 java 开发人员和摇滚明星 Java 开发人员之间的差别。您应该花大量的学习时间来磨练您的 Java 测试技能。对于那些不熟悉 Java 和单元测试的人来说， **JUnit** 是最好的入门库。JUnit 5 的最新版本功能强大且灵活，每个 Java 开发人员都应该知道这一点。

**Mockito** 也是一个非常棒的 Java 测试框架。它是一个开源测试工具，允许在**测试驱动开发(TDD)** 的自动化单元测试中创建 test double 对象。

# 5.学习“干净的代码”

每个开发人员都可以编写有效的代码，但是如果您想成为一名摇滚明星开发人员，您必须编写可理解、可读、可更改、可扩展和可维护的代码。

> *“任何傻瓜都能写出计算机能理解的代码。优秀的程序员会写出人类能理解的代码。”*
> 
> *——*[*马丁·福勒*](https://martinfowler.com/aboutMe.html)

要了解更多关于**干净代码**的知识，我推荐你去读一读美国软件工程师兼作者 [**罗伯特·c·马丁**](https://en.wikipedia.org/wiki/Robert_C._Martin) **的书《 [**干净代码**](https://books.google.fr/books/about/Clean_Code.html?id=_i6bDeoCQzsC) 》。**

# 6.了解库和 API

Java 是世界上最流行和最成熟的编程语言，拥有巨大的社区支持。世界各地的许多开发人员夜以继日地开发开源库和 API，通过它们你几乎可以做任何事情。所以，**“不要多此一举”**。找到正确的并利用它。

# 7.了解 DevOps 工具

如今，对于一个摇滚明星 Java 开发人员来说，了解 DevOps 是不可避免的。借助 DevOps 工具的知识，如 **Jenkins** 、 **Docker** 、 **Kubernetes、**你可以为任何组织提供独特的东西，并提升你的生产力。这让你在面试中比其他人更有优势。

如果你想变得对组织更有价值，DevOps 知识可以发挥至关重要的作用。它可以促进你的职业发展。

我希望这些技巧能提升你的编程技能，让你与众不同。不要忘记分享你认为让你成为更好的 Java 开发人员的信息。
