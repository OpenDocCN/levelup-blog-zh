# C#与 Go:比较

> 原文：<https://levelup.gitconnected.com/c-vs-go-comparison-80d01cdf18f5>

## 两种语言提供的优势和长处

![](img/304b32875e1864e756649dd6cdf26236.png)

戴维·舒尔茨在 [Unsplash](https://unsplash.com/photos/SrewPUfo2c0) 上的照片

C#和 Golang 都是非常流行的编程语言，它们也有一些相似的特性，但是优势并不相似，这取决于你想用这种语言做什么。

从这两者来看，C#是微软在 2000 年推出的较老的一个，它可以用于几乎所有的东西。NET framework 在社区内非常受欢迎和喜爱。

Go 由谷歌打造，于 2009 年作为 C 的继任者推出。这种语言是专为简单性和并发性而设计的，可以处理谷歌的网络服务器，所以你可以想象它非常高效。

因此，我不得不在朝九晚五的工作中使用 C#，同时在业余时间开发围棋，我想比较一下这两者，希望能帮助你理解两者之间的异同。

## 什么是 Go 和 C#

Golang 是 Google 设计的一种开源、编译和静态编写的编程语言。

C# (CSharp)是微软设计的面向对象编程语言。

## 有什么好比较的

嗯，这两种语言非常相似，深受全球软件开发人员的喜爱，并且非常适合解决企业问题，所以这种比较应该很有趣。

Go 和 C#都使用编译器来创建本机代码，这意味着它可以跨各种平台使用。两者都有很好的垃圾收集功能，并且都源于 C 语言，所以它们在语法方面有一些相似之处。

## 开发/编码速度

Go 使用起来非常简单，文档也很棒，这并不奇怪，因为 Google 拥有最好的标准之一。设置一个项目并删除未使用的代码是非常容易的，所以你花更少的时间清理代码。

C#也很容易安装。您可能在几分钟之内就可以拥有一个正在运行的 API。Intellisense 非常棒，动态类型允许您检测错误并加快编程过程。

**赢家:双方**

## 表演

这就是事情变得更有趣的地方。Go 和 C#都编译成机器码，但是 C#编译成跨平台的二进制代码，但是然后。需要 NET Core 运行时才能运行。然而，Go 编译成与操作系统绑定的本地二进制文件，这使得它比 C#更快更有效。

当我们在性能方面比较 C#和 Go 时，我们应该注意到两者都有很好的性能水平，但 Go 实际上是为微服务和并发性而设计的。

Go 背后的主要思想是本质上的简单和轻量级，它在性能方面表现出色。Go 的编译时检查、错误处理和内存管理的效率非常高，因为它将代码编译成本地二进制文件。编程语言使用更少的资源，这都有助于更快的编译和内存管理。同时，C#可以跨平台编译，这也是它的优势，但与 Go 相比，它的速度要慢一些。

C#与。NET 框架，已经越来越快了。它提供了很好的垃圾收集和错误处理。但 C#如此强大的事实也是这种语言的一个负面因素，因为它的功能很多，而 Go 是轻量级的，总体上看起来更简单，这使得它可以运行得更快。

**获胜者:去吧**

## **可扩展性**

就可伸缩性而言，两者实际上都很棒。正如我们所说的，Go 是为处理 Google 的繁重负载而创建的，其快速和高性能的特性使其可以轻松扩展。

C#在可伸缩性方面也很棒，它可以轻松处理繁重的负载。

**赢家:双方**

## 流行

如果我们看一下 2021 年的 StackOverflow 调查,我们可以看到 58，031 个回答(专业开发人员)中有 29.81%使用 C#，而使用 Go 的数字仅为 10.51%。

这可能很正常，因为 Go 是 C 的继承者，不像我们之前提到的 C#那样通用。C#可以用于几乎所有的事情，它更受欢迎，而 Go 用于性能非常重要的地方，例如大型企业系统，所以 C#有更多的工作是正常的，但这两种语言都受到开发人员的欢迎。

**获胜者:C#**

## 社区支持

我听到了很多不同的观点，但是据我所见，你现在可能可以在 C#上找到更多的东西，因为它是一种更古老的语言，虽然没有 Go 那么快，但它仍然被许多大企业使用，所以支持是存在的。

你对 Go 也有很好的支持，我发现文档很有帮助，但总体来说 C#看起来更胜一筹。

**获胜者:C#**

## 学习曲线/易于学习

同样，这两种语言在语法上有很多相似之处，唯一的区别是 Go 是过程化的，对于刚开始编程的人来说可能有点混乱。Go 轻量级的事实可能有所帮助，因为学习曲线不应该太大。

C#是一种面向对象的语言，开始一个项目并学习这种语言真的很容易。它可能会花你更多的时间来学习，因为它的功能很多，但总体来说，它对初学者来说相当不错，可读性也很好。

**胜者:双方**

## C#优点

*   。NET 是一个很棒的框架，它有大量的库，你可以随意选择哪一个来满足你的需求。
*   这是一种设计良好的语言，微软将继续做得很好。
*   这是进入面向对象编程的一个相当简单的方法，因为你可以学习类、方法、接口等等。
*   有利于跨平台开发。
*   当谈到 Windows 开发时，它是最好的。
*   伟大的垃圾收集，你不必担心指针和内存管理，但你也可以直接管理内存，如果你想这样做(但你为什么要这样做)。
*   C#需求量很大，所以如果你想找一份软件开发人员的工作，这是一门很好的语言。
*   可读性很强
*   大型社区

## C#缺点

*   它非常依赖于。NET 框架，所以它不太灵活。
*   如果你是初学者，你可能会被大量的库所淹没。

## 去职业

*   这是一种简单的语言，旨在快速和可伸缩。
*   语法易于书写和阅读。
*   文档很棒。
*   内置的单元测试特性非常棒。
*   一个成长中的社区。
*   简单的设置和学习曲线并不可怕。

## 去监狱

*   一开始，实现接口可能有点复杂和难以理解。
*   由于轻量级的想法，您可能需要不时地编写更多的代码，因为您习惯的一些特性可能会丢失(比如泛型或方法重载，尽管最近已经引入了泛型)。

## C#有什么用途

C#很受欢迎，被用于各种各样的东西。一个合乎逻辑的答案是游戏开发，以及最近的 AR 和 VR。C#也非常适合微服务，它可以用于各种应用程序的后端，因为它允许很大的可伸缩性。

## 围棋是用来做什么的

正如我们多次提到的，Go 在并发性方面非常出色，因此非常适合处理阻碍可伸缩性的瓶颈。Go 非常适合分布式系统，被世界上许多最大的科技公司使用——谷歌、网飞、优步等等。

## 摘要

虽然这两种语言有很多相似之处，但它们在使用和实现上也有所不同。Go 是更快的一种，它关注的是效率和速度，而 C#更灵活，可以用于各种事情，我个人认为它是完全初学者的更好选择，因为你可以开发各种东西，探索丰富的功能。
