与程序员谈论SQL数据库？需要快速上手吗？以下是对你可能有的问题的一些回答！

这是什么？

为了解释什么是 SQL数据库，让我们把它分成几个部分:

最重要的是它是一个数据库。它是一个用来存储数据的软件，这样应用程序就可以很容易地访问和修改数据。

名称的 SQL 部分代表顺序查询语言。名称的“顺序”部分对于理解并不十分重要，但是知道这是一种查询语言告诉我们这是用于查询数据库的语言。换句话说，SQL是必须编写的与数据库对话的代码。用于插入新数据，更新或删除已有数据，最重要的是选择要从数据库中取出的数据。你会经常听到这些代码被称为选择/插入/更新/删除语句，或者更一般地被称为查询。

比如说；

我查询了数据库，找到受影响的数据，然后运行update语句来更正数据。

因为SQL是用于查询数据的语言，这也意味着数据库必须是关系型的。从字面上看，这意味着数据库中的数据可以相互关联，并且这种关联将会得到加强。然而，作为一个关系数据库的实际含义是，你所有的数据都将存储在表中。您可能已经熟悉了表格数据；它由一系列列和一系列行组成，每个列都有一个名称和类型；每行每列都有一些数据。你可以认为这类似于一个Excel文档。

我需要一个数据库吗？

如果你正在开发一个需要存储任意数量数据的应用程序，那么是的，你需要一个数据库。

但是，如果您不需要存储任何数据，例如，如果您正在编写一个工具，或者您正在从另一个资源获取数据，那么您可能不需要数据库。

有其他选择吗？

是的。使用SQL数据库有很多替代方案！还有许多其他选项经常被纳入 NoSQL 的保护伞下，字面意思就是非SQL，它们是不是SQL数据库的数据库。

最突出的NoSQL数据库类型有:键值、文档和对象数据库。这三者之间有细微的差别，但它们的工作方式基本相同:不是为每种类型的数据定义一个结构，任何数据都可以存储在数据库中，只需一个文本位作为关键字。

我个人从未做过NoSQL数据库比SQL数据库更有意义的项目。然而，我听到有人声称它可以使解决方案的初始开发速度大大加快。虽然我也听说随着应用程序的成熟和变化，它会导致很多数据完整性的问题。

贵吗？

看情况— 有商业产品如；微软SQL Server 、 IBM DB2、和 ORACLE数据库。这些都需要许可才能在生产中运行，但通常只对托管它们的人开放，所以如果你已经为托管解决方案付费，许可费用可能已经包括在内了！在同一个服务器上有多个数据库也是可能的，所以如果你已经有了一个正在运行的服务器，你就不需要支付额外的许可费了。

在光谱的另一端，有开源SQL数据库，如 MySQL 和 PostgreSQL 。

一般来说，不同的开发人员对给定的数据库产品的熟悉程度不同，这可能是决定使用哪个产品的最大因素。
