# 数据工程面试流程:

这是大多数公司遵循的一般流程:

1.  **筛选轮:在线测试**(1–2 道编码题:leet code Easy-Medium+2–4 SQL(Advanced))。
2.  第二轮:Face2Face 面试:DS Algo + SQL Advanced + Spark Basics。
3.  第三轮:DS Algo + SQL Advanced +分布式系统设计**或** Spark +项目讨论。
4.  第四轮:最后一轮(项目讨论/设计模式知识+ Spark + Scala/Python + Hive 问题)。
5.  第五轮:人力资源讨论。

**数据工程师 vs 软件开发人员面试:**

作为一名数据工程师，你不必专注于难懂的 Leetcode 问题。此外，对问题进行编码更像是数据工程工作，而不是核心算法问题。

*省省力气，只准备 LC 容易和中等。*

**进来可怕的高级 SQL + Spark:** 作为一名数据工程师，编写复杂的 SQL 查询一定是你的强项。这意味着不仅仅是 INSERT、DELETE、WHERE 语句，您还需要了解以下内容:

1.  窗口功能
2.  子查询
3.  递归子查询
4.  通用表表达式
5.  使用连接回答问题。

*从* HackerRank(做题全)，Leetcode(免费的就够好了)学习和练习高级 SQL。我很快会发布 SQL +面试经验。敬请关注或关注我的李/Medium。

# 数据工程师编码回合看起来像这样:

1.  **DS & Algo 问题** : LC Easy / medium:经常涉及读出数据和使用字典进行处理。
2.  **SQL 问题**:2–3:肯定会被问到窗口函数和高级连接的工作知识。
3.  **Spark 编码**:将上述 SQL 代码转化为 spark/ word count 类型的问题。必须知道:map，flatmap，reduce，reduceByKey，joining，collect_list/collect_set 和 wordcount 类型问题。
4.  **Spark 概念**:与通用 Spark 和优化相关的概念。
