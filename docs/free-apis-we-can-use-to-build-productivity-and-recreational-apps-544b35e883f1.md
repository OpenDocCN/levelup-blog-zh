在软件开发领域，熟能生巧。所以要尽可能多的找方法练习编程。有了免费的公共 API，我们可以通过创建使用这些 API 的应用来练习编程。

在本文中，我们将研究一些可以使用这些 API 的实践项目想法。

# HTML 到 PDF 转换器

我们可以使用 [pdflayer](https://pdflayer.com/) API 将 HTML 文档从文件和 URL 转换成 pdf。

我们所需要的是一个 API 键，然后我们可以用它轻松地将 HTML 转换成 pdf。

# 文件书签

Pocket API 允许我们使用他们的书签服务，通过他们的 API 来添加、修改和删除书签。

它让我们可以自动化这个过程，并通过 API 查看被书签标记的文章。

使用文章视图 API，返回整个文档的内容，我们可以使用它来显示数据。

# 待办事项列表

我们可以使用一些待办事项列表 API，如 [Todoist](https://developer.todoist.com/) 和 [Wunderlist](https://developer.wunderlist.com/documentation) 来创建我们自己的程序，以创建待办事项列表，并通过服务提供者的 API 保存这些列表。

Todoist API 可以做很多事情，比如创建项目、模板、条目、标签、注释、项目注释、文件上传等等。

此外，我们可以为 todo 任务设置提醒和截止日期。

这些都是我们需要花很大力气去做的事情。有了这个 API，我们可以事半功倍，因为我们不再需要构建一个 API 来完成所有这些工作。

Wunderlist API 也有类似的功能。我们可以用它做同样的事情，添加成员来访问待办事项列表。

我们可以创建网页挂钩来创建待办事项列表。

# 活动信息应用

像 [Eventbrite](https://www.eventbrite.com/developer/v3/) 、 [Picatic](http://developer.picatic.com/?utm_medium=web&utm_source=github&utm_campaign=public-apis%20repo&utm_content=toddmotto) 、 [Ticketmaster](http://developer.ticketmaster.com/products-and-docs/apis/getting-started/) 这样的大型活动网站都有 API，可以用来检索活动。

Eventbrite API 允许我们通过 OAuth 访问他们的 API。它为我们提供了活动容量、描述、时间表、团队等数据。

我们还可以获得组织的数据，包括其成员和角色。

此外，我们可以用它得到各个场馆的座位图。

Ticketmaster API 提供了与 Eventbrite API 相似的数据，这样我们就可以用它获得相同类型的数据。不同之处在于，我们使用 API 密钥进行身份验证，而不是 OAuth。

# 食品和饮料应用

我们可以从各种食品和饮料 API 中获取数据，比如从加拿大安大略省的政府酒类商店获取酒精数据的 [LCBO](https://lcboapi.com/) API。

它包含图像、酒厂信息、酒类连锁店的零售点、不同地点的酒类库存等数据。

如果我们想创建一个美食应用程序，我们可以使用[食谱小狗](http://www.recipepuppy.com/about/api/) API 创建一个应用程序。

API 只有一个端点。我们可以用我们输入的关键字搜索存储在 API 后面的数据库中的食谱。

它支持分页，这样就不会淹没他们的 API 服务器。

# 餐厅应用

一款餐厅 app 是发现餐厅的有用 app。像 [Zomato](https://developers.zomato.com/api) API 这样的框架让我们可以获得餐馆名称和位置、评论、菜单等信息。

我们只需要一个 API 密钥来访问他们的数据。

我们还可以获得按类别、城市、收藏、美食和地理位置坐标等各种事物组织的餐馆数据。

它还可以用来查找给定餐馆的每日菜单、餐馆信息和评论。

# 啤酒应用

有了 [PunkAPI](https://punkapi.com/) ，我们可以创建自己的应用程序来查找啤酒信息。

这个 API 不需要认证，但是对于给定的 IP 地址有一个速率限制。

它返回诸如啤酒名称、酿造酵母啤酒的原料、酿造啤酒的年份以及啤酒花和麦芽含量等数据。

它也可以返回一个随机啤酒。

# 结论

我们可以用免费的 API 做很多事情。有 API 可以把 HTML 转换成 PDF，所以我们可以做自己的文档格式转换 app。

此外，如果我们对食物感兴趣，我们可以获得关于餐馆、食物、食谱、饮料等的数据。因此，我们可以使用它们来制作自己的应用程序，以我们的方式搜索和显示数据。

最后，我们可以制作自己的应用程序来显示活动和场地数据，这样我们就可以按照自己想要的方式显示数据。
