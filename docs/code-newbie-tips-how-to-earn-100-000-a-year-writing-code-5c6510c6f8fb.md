代码还是让我害怕。

我做软件开发已经好几年了。我有商业背景。我喜欢数字。我喜欢做实验。我不喜欢的部分是电脑。

写代码一年赚 10 万美元有一个明确的公式。我不太明白所有的数学，但我明白的是。

# 不断学习

我是一个关心代码以外学习的老屁。我职业生涯的早期是躲在图书馆里度过的。

当你从错误中学习时，你开始变得更加好奇，并继续寻找更多的答案。这种好奇心让你在谷歌上搜索答案，最终导致编写代码。

最好的学习方法是去做。

因此，虽然很多学习可以来自课程，但大多数学习将发生在你陷入困境、被打断、不得不寻找解决方案的时候。你不能谷歌所有的问题。

学习代码之外的知识是我从未停止编写代码的重要原因。

不仅仅是写代码。这是关于学习如何好奇，有耐心，并使用一点点的发现来解决问题，并取得小胜利。

# 专注于一种编程语言

当你开始工作的时候，你想做好所有的基础工作，成为所有方面的专家。

但问题是这要花很多时间。我建议学习一种编程语言并坚持下去。这样你就可以学习框架、语法和解决问题的正确方法。

一旦你知道一种语言，那么你就可以很容易地切换到另一种语言。

你学习的第一种语言将是你的基础语言。

# 花时间学好一门语言

谈到代码，许多人都有点胆怯。

他们像看蛇一样看着它。这可能会伤害他们，他们的职业目标将化为乌有。

所以他们坚持学习教授他们基本编码的文档和在线课程。我开始的时候也是这样。

问题是深入学习一门语言需要数年时间。你甚至不能称自己为初学者，除非你已经学了两年的语言——即使那样，你仍然是一个初学者。

如果你真的学好了一门语言，那么你就可以在这门语言的基础上掌握多种技能，并且比那些只会说一门语言的人用更少的代码获得同样的结果。学好一门语言。

# 只考虑薪水

如果你打算写代码并赚钱，你必须用不同于一般人的方式思考。

作为一名程序员，你需要考虑薪水。这是你得到报酬的唯一方法。

你必须编写能为用户或公司带来收入的代码。当我开始写代码时，我想到了用户工资支票。现在我想到了公司的薪水。这不是一朝一夕的变化。我花了一年多的时间写代码才实现。

但是，如果你到了那里，你将开始编写代码，可以产生某人的投资回报。那你就是在写赚钱的代码，这才是你应该做的。

# 在一个没人想去的地方工作

早在 2011 年，我就学习了 Ruby on Rails。大多数编码员都讨厌它。很复杂。已经被开发商抛弃了。很慢。

都是真的。然而我学会了。为什么？没有竞争。我可以写关于这个主题的教程，因为没有人关心这门语言。

我赚了钱。

接下来，我了解了 React，这是另一个没有竞争对手的小众话题。我在网络平台上写过。然后我在自己的博客上写了这件事。这时候谷歌广告出现了。

重点是:

*   学习一个冷门话题
*   在网上写下它
*   成为这方面的权威

# 编写您会用到的代码

当我写代码的时候，我感觉自己生活在未来。我视其为超级大国。能够为自己创造未来真是太酷了。成为一名代码编写员的最好方法是开始编写你会用到的代码。

我的第一个软件产品是为网络写手设计的。现在，我为内容创作者开发的软件是我的主要收入来源。我为作家设计的第一个产品是基于我的经历。我写的代码是为了回应像我一样在这个过程的某些部分苦苦挣扎的人。

有一天，我希望能从我的写作应用程序中赚钱。目前，这只是一个帮助我学习的实验。

# 写下你想买的代码

有两种代码:你喜欢的代码和你讨厌的代码。

如果你喜欢代码，你很可能会写更多的代码。你热爱的代码变成了你可以出售的产品。

你能写的最好的代码是你会买的代码。当你写你会购买的代码时，你很好地理解了这个问题。你不是在猜测。你已经知道它的存在，你想要一个解决方案。你也会在你写的代码上投入更多的时间和精力，因为你希望看到它成为现实。

> 写下你想买的代码。

# 编写您愿意付费的代码

如果你免费编码，你很可能永远得不到报酬。

我无法告诉你这个问题发生了多少次。一个家伙加入了一个 Slack 频道，免费提供他的代码。他建立了一个社区，有 1000 名新手来测试他的代码。

然后，他在 Shopify 应用商店上推出了自己的应用程序，总共赚了 0 美元。

有几个原因。

1.  他免费提供他的代码
2.  他没有检查他的应用程序是否盈利——他只是猜测。
3.  他不知道该收取什么费用

这就是为什么如果你不写你愿意花钱买的代码，那么你将一事无成。

> 编写付费代码是了解产品定价的最快方法。

# 最后的话

重点是学习一些代码，开始一个副业，并建立一个小的副业收入，可以用来为你争取时间去做你想做的真正的工作。

很快有一天，你会到那一步。我保证。
