你的编程阅读清单上有什么？

原文：https://levelup.gitconnected.com/whats-on-your-programming-reading-list-472fc23ed9bc?source=collection_archive---------5-----------------------#2020-10-22

“我认为你需要多读书，”我的教授直截了当地告诉我。我刚刚分享了我对一篇即将发表的论文的想法，他显然对此并不兴奋。

“多读书？”我记得当时在想。我没想到会有这种反应。我走出他的大学办公室，试图理解他的评论。

所以在接下来的几周里，我读了很多书。我的知识增长惊人——我的想法也是如此，这激发了一个新的和改进的论文主题。

大约十年前的那次经历给我上了重要的一课:如饥似渴地广泛阅读你所在领域的书籍的好处。开始时，在一个班里做好一件一次性的事情已经变成了一生的习惯。

有趣的是，当我开始专业写作时，我也收到了类似的建议。我经常被告知要多写文章，多读文章。

考虑到当时对阅读的重视，你可以想象当我开始编程时，我对阅读态度的震惊。对许多人来说，这似乎是一个低优先级。

在编程的世界里，阅读包括阅读代码和阅读技术书籍。我以前在《T2》中写过为什么阅读代码很重要以及如何去做。

现在是时候着手阅读技术编程书籍了。

为什么是书？

阅读技术编程书籍是程序员应该立即开始的一个习惯。这是一个相对较小的投资，现在可以帮助你，但将来会有回报，原因很简单:你的知识会随着时间的推移而增长。

你可能会问:为什么是书？

毕竟，我们每天大部分时间都粘在电脑上，在那里我们可以接触到看似无限量的资料(而且大部分都是免费的)。

作为在线内容的制作者，我意识到在线文章和博客文章有它们的位置。博客文章可以给你一个有趣的视角，让你接触到一个新的想法。一篇技术文章或一种语言或技术的文档可以回答您的特定问题。

但是书也有它的位置。

它们满足了不同的需求:它们是长篇内容的来源，带你深入了解某个特定主题。一篇博文或者一篇技术文章可能几百字，也可能几千字。一本书几万。

当你拿起一本技术书籍时，你会对所述主题进行深入研究。你发现了细微的差别。你学习，练习，学习。你进入了行业专家的思维模式。随着你的背景开始增长，你开始以不同的方式看待事物。

在我们的领域里，不缺少可以阅读的话题:

  • 计算机体系结构
  • 敏捷软件开发
  • 算法
  • 测试
  • 编写干净的代码
  • 重构

这个清单还在继续。

即使在一切都数字化的时代，建立自己的技术资料图书馆也是值得的。所以投资一些技术书籍。使用它们。在里面写字。编写代码时参考它们可以帮助你养成良好的习惯。

如何找到时间

当然，挑战在于如何在已经排得满满的日程表中找出时间来阅读。

但是，如果像比尔·盖茨和沃伦·巴菲特这样的人——众所周知的超级读者——能找到时间阅读，我相信我们也能。

以数学家理查德·海明为例。他也想出了一个让阅读成为习惯的方法，他在他的精彩讲座 你和你的研究 中分享了这个方法。

海明不明白他的同事、数学家约翰·图基怎么会知道这么多。“这家伙显然是个天才，”海明回忆道。所以他走进经理的办公室去了解情况。

“海明，”他的经纪人实事求是地说，“如果你像(图基)那样努力，你会惊讶地发现自己知道了多少。”

海明思考了经理的评论，决定“重组”自己的生活。他不再看“无聊的杂志”，而是花时间“研究他职业生涯中的事情”

他的咖啡桌上总有一本书“随时可以阅读”他还被任命为书评编辑，所以他可以评论他读过的书。“通过这种方式，我强迫自己广泛了解计算机科学，”海明这样评价这个过程。

“我不是一流的天才，”海明解释道。“所以我干脆把其他事情(比如阅读《纽约客》)放在一边，就这么做了，”他补充道。“这不难做到。你去做就是了。”

我们能从海明身上学到什么

我们可以从海明的故事中发现一些关键点。

首先，他总是在他的咖啡桌上放一本书。换句话说，他总是看着一本书。这种视觉传达了一种善意的提醒。

如果咖啡桌不适合你，试着在你的电脑或床头柜旁边放一本编程书。也许把一本书放在你的电视机前是最好的。这样，当你伸手拿遥控器时，你就看到了你应该读的书。

第二，成为一名图书编辑似乎让海明对阅读和理解他所读的东西负有责任。

写完评论后，他搁置了一周。然后，他会问自己这些问题:“这是一篇好的评论吗？那真的能消化这本书吗？”如果没有，“你在重读这本书，写一篇更好的评论。”

在当今世界，这个过程甚至更简单:读一本书，然后写一篇关于它的博客。事实上，写作是巩固你知识的好方法。

这是很重要的一点:对你正在阅读的内容做点什么。海明选择写评论。但是还有其他的方法。

阅读时，使用我称之为“抽查的方法。这个想法是在你学习新材料时随机检查你的知识。读完一两段后，停顿一下，看看你是否能回忆起并清晰地表达出你刚刚学到的内容。

另一个想法是在你读完一章后，打一些总结要点的笔记。这个过程本身有助于回顾和强化重点。

然后，您可以用不同的方式使用您键入的笔记:

  • 制作 Anki抽认卡 ，这样你以后会不断看到信息。
  • 做一个清单。在得到关于我的变量名的反馈后，我在《代码完成》一书中读到了关于命名的内容。然后，我做了一个“变量名”清单。它挂在我书桌前的墙上。
  • 专注于你在编码过程中学到的一些具体的东西。我最近读了一些关于编写测试的有用信息。现在我拿出这些笔记，边工作边参考。

关键是要学以致用。获取你正在获取的信息，将它联系起来，并将其转化为知识。

让它成为一种习惯

在很大程度上，把阅读放在首位是一个态度问题。

你必须意识到阅读很重要。要意识到阅读的重要性，你必须看到它的好处。当事情很重要并且我们看到了好处时，我们会想办法让它发生。

有一个简单的方法可以做到这一点:从每天阅读30分钟开始，坚持一个月。这是一个小的，可管理的时间投资。然而，这足以产生影响。

每当我给我的生活添加新的东西时，我发现删掉一些东西是有用的。这样的话，我就不会因为试图适应而增加更多的压力。

月底盘点一下，看看自己学了多少。你很可能会对自己学到的东西感到震惊。你要么找到继续这个习惯的方法，要么找到花更多时间阅读的方法。

告诉我“多读书”的教授点燃了我阅读的动力，尽管最初是为了取得好成绩。但我很快看到了好处。后来，这位教授给了我一些我所收到的最好的建议，我后来才知道他的私人图书馆里有1000多本书。
