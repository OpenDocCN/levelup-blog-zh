# 你可能错过的热门 NPM 套餐

浏览器和 Node.js 应用程序都使用 NPM 包。NPM 是最大的 JavaScript 库。

在本文中，我们将看看 NPM 上流行的 JavaScript 包。

# [蓝鸟](https://www.npmjs.com/package/bluebird)

Bluebird 是一个流行的承诺库，用于做出承诺。在承诺被内置到 ES6 或更高版本之前，它很流行。

既然 JavaScript 有望成为一个原生特性，我们可能就不那么需要它了。

# [vue](https://www.npmjs.com/package/vue)

Vue 是构建前端应用程序的流行框架。

它是全功能的，所以它包括了很多东西，比如模板、格式化模板内容的过滤器、操作 DOM 的指令以及组件的使用，所有这些都在一个包中。

Vue 越来越受欢迎，用它制作简单的应用程序很容易

为了使我们的生活更方便，有许多图书馆为它而建。

因此。我们可以用它来制作应用程序。有了 Vuex 和 Vue 路由器，我们就有了状态管理和路由解决方案，可以制作独立的前端应用程序。

# [uuid](https://www.npmjs.com/package/uuid)

这个库允许我们创建各种类型的唯一 id，我们可以用它来做一些事情，比如数据库条目的唯一标识符等等。

# [类名](https://www.npmjs.com/package/classnames)

我们可以使用类名库有条件地将类名连接在一起，这样我们就可以动态地将样式应用到网页上。

它是框架不可知的，因此几乎可以用于任何项目。然而，由于在像 Vue 和 Angular 这样的框架中有一个类名操作特性，我们不应该在这些框架中使用它。

但是这个库确实可以很好地与 React 一起工作，因为没有简单的方法可以开箱即用地动态操作类名。

# [下划线](https://www.npmjs.com/package/underscore)

下划线是另一个流行的 JavaScript 助手库，它有各种方法来操作数组和对象。

它的特性集比 Lodash 小，它的一些方法有普通的 JavaScript 等价物，所以它不像以前那么有用。

# [core-js](https://www.npmjs.com/package/core-js)

Core-js 有一组 polyfill，我们可以添加它来支持浏览器中没有内置库的浏览器。

它们包括最新的数组和字符串方法，以及像`Map`和`Set`这样的新构造函数。

此外，还包括了`Promise`构造函数，这样我们就可以使用本地承诺而不是第三方解决方案。

无论 ES6 或更高版本引入了什么，这个库都有。

# [问询者报](https://www.npmjs.com/package/inquirer)

Inquirer 是一个让我们轻松构建命令行用户界面的库。

它有让我们提示答案和接受用户输入的方法。

我们可以用我们喜欢的方式格式化问题和答案，我们甚至可以用它来创建 react 用户界面来监听答案。

# [yargs](https://www.npmjs.com/package/yargs)

是一个库，通过帮助我们轻松处理命令行参数来帮助我们构建命令行界面。

它解析启动程序的命令中传递的选项，这样我们就可以使用它们，而不用自己解析它们。

# [网络包](https://www.npmjs.com/package/webpack)

Webpack 是一个流行的包，它允许我们将模块打包成可以在浏览器中使用的包。

它支持许多选项，并被 Angular、React 和 Vue 等流行框架的 CLI 所使用。

# [mkdirp](https://www.npmjs.com/package/mkdirp)

`mkdirp` 让我们在节点程序中递归地创建目录。

fs 模块没有让我们递归创建目录的功能，所以我们必须使用这个库来完成。

它内置了同步和异步版本。

# [rxjs](https://www.npmjs.com/package/rxjs)

Rxjs 是一个流行的反应式编程库。

它让我们创建用于发送数据的对象，这些数据可以订阅到其他对象。

此外，我们可以用它来调整内置方法发出数据的方式。

# [全球](https://www.npmjs.com/package/glob)

Glob 允许我们从 Node.js 程序中搜索文件。我们可以以字符串的形式传入任何路径模式，并返回与我们传入的模式相匹配的路径的结果。

默认情况下，搜索是异步进行的，但是我们也可以同步进行。

# [主体解析器](https://www.npmjs.com/package/body-parser)

这是一个让我们在 Express 应用程序中解析请求体的包。它是一个 Express 中间件，所以可以用来解析 JSON 请求体。

因为在 Express 中没有内置的方法来解析 JSON 请求体，所以我们必须使用它。

# 结论

其中一些库让我们可以轻松地构建命令行界面。

我们还有下划线让我们做数组和对象操作，这些操作没有普通的 JavaScript 等价物，也不容易做。
