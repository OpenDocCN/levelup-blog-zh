要避免的常见JavaScript错误

原文：https://levelup.gitconnected.com/common-javascript-mistakes-to-avoid-7634f95706f0?source=collection_archive---------10-----------------------#2020-05-15

有一些非常常见的编程错误，我们希望避免这些错误，以保持代码的整洁和可读性。

在这篇文章中，我们将看看它们以及如何避免每一个。

一个函数中包含太多内容

一个函数应该只做一件事，也就是它们自己所指示的事情。

如果还有别的，那就应该挪到它自己的功能里。在一个函数中有太多的东西会使它难以阅读和理解。

例如，如果有一个函数同时获取小计和税金，那么它们应该被分成两个独立的函数，因为它们是不同的。

注释掉的代码

注释掉代码是不会运行的代码。因此，在我们完成这部分代码后，它们就不应该在那里了。

一旦我们完成了，我们应该删除注释掉的内容或者取消注释。

不管怎样，最终产品不应该注释掉其中的代码。

非描述性的变量名

一旦我们忘记了自己写的代码，回头再来看，用不传达它们的意思的方式命名变量会让读者和我们自己感到沮丧。

因此，我们应该用描述变量内容的名称来命名变量。

不要写let x;，而是写类似let numApples的东西，这样我们就知道我们的变量将保存苹果的数量。

幻数和字符串

我们不应该有神奇的数字和字符串。这些值多次出现，意思相同，但在代码中没有明确解释。

例如，如果我们有以下代码:

for (let i = 0; i < 10; i++) {
  ///...
}

那我们就不知道10是什么意思了。相反，我们应该将它设置为一个命名的常量，这样我们就知道它的含义了。

例如，我们可以写:

const numApples = 10;
for (let i = 0; i < numApples; i++) {
  ///...
}

现在我们知道10实际上意味着苹果的数量。

混乱的代码格式

混乱的代码格式使代码难以阅读，所以我们应该用linter或代码格式化程序来清理它。

有大量的选择，所以我们可以使用一些像更漂亮或ESLint这样的东西来自动整理。

硬编码值

我们不应该把值硬编码到我们的代码中，尤其是当它们是秘密的时候。相反，我们应该将它们作为环境变量保存，并从中读取所有值。

有很多方法可以做到这一点。对于前端，比如Angular，React，Vue，都有存放变量的地方，把不同环境的变量存放在不同的文件里。

对于后端，我们可以使用类似于dotenv包的东西从.env文件中读取环境变量，这样我们就不必硬编码它们了。

.env文件不应该被签入，所以我们的代码库中没有秘密。

重复代码

重复的代码是不好的。如果我们改变重复的东西，我们必须改变所有重复的地方。

相反，我们应该将公共部分移动到它们自己的文件中，以便它们可以在多个地方使用。

“不要重复自己”的原则适用于任何地方。如果我们复制、粘贴并原样使用它们，那么我们应该将它们移动到一个共享位置。

不备份代码

我们应该用Git管理我们的代码，这样我们就可以有一个本地和远程的代码回购。

这样，我们可以在远程位置自动保存一份副本。此外，我们可以轻松地恢复坏代码，并轻松地从早期提交中获取代码。

没有版本控制，我们无法做到这一点。

如果我们没有备份代码，那么一旦出现问题，我们可能会失去一切。

复杂代码

复杂的代码应该简化，以便我们更容易理解。

我们还应该把它们分成更小的部分，这样我们就可以重复使用这些东西，并且每个部分都易于阅读和测试。

结论

如果我们小心谨慎，这些常见的错误是可以避免的。

把事情分成小块总是好的，就像保持事情简单和不重复一样。

注释掉的应该消失，非描述性的变量和值也应该用更有意义的东西代替。
