每个人都有一个学习计划和他们喜欢使用的资源列表。不同的计划适合不同的人，没有一个适合所有人的标准。

这绝不是加入大型科技公司的唯一资源清单。但这是我自己用来准备所有技术面试的资源列表。

快速背景

我现在是微软的工程师，之前分别在亚马逊工作了一年。我没有硕士学位，我毕业于NYU大学，不是常春藤盟校。我将很快加入谷歌，以下资源是我如何到达那里的。

是的，可购买的资源是帮助支持这个博客的附属链接。不管怎样，这些都是我用过的资源，包括可购买的和免费的。

编码资源

破解编码面试 (CTCI)

这是让任何人开始学习编码面试的最简单的书。

如果你是一个绝对的初学者，我建议你从这里开始。这些问题有非常详细的解释，用算法和数据结构的基本知识很容易理解。

编程面试要素( Python 、 Java 、 C++ )

如果你更有经验一点，这本书里的每个问题都是所有大型科技公司的面试水平。

如果你已经掌握了本书中的问题，那么你已经为一般的技术面试做好了准备。这本书不像《CTCI》那样对初学者友好，但它包括了一个学习计划，这取决于你需要为面试做多少准备。这是我个人最喜欢的书，我在大学里到处都带着。

neet code blind 75—YouTube

Blind有一个包含75个问题的列表通常足以解决大多数编码面试。这是一个非常有条理和集中的列表，列出了最基本的算法来利用你的时间。

上面的播放列表是我见过的最清晰的解释之一，如果你需要任何问题的解释，强烈推荐。

CSES习题集—任务

这些问题很难。对于没有练习过算法并且对初学者不友好的人来说真的很难。但是如果你能够完成排序和搜索部分，你将会比一般的LeetCode用户更有能力，并为你的编码面试做好准备。

如果你觉得LeetCode中等题很舒服，并且觉得CTCI的题太简单，可以考虑一下这个。

算法学习

算法简介(第四版)

这是任何人都可以用来学习算法的最普通和最好的教科书。它也是我的大学个人用来学习大多数编码问题的核心和基本算法的教科书。

第四版最近发布，仍然与麻省理工学院的学生相关。如果你需要结构和传统的课堂环境来学习，请点击这里学习麻省理工学院的算法课程。

威廉·费塞特—图论

图论确实在面试中出现过(这也是我在彭博和谷歌遇到的一个问题)。做好准备，遵循威廉·费塞特的图论解释。

图表很全面，一步一步的解释是我见过的关于这个话题的最好的解释。

CSES.fi手册

这本手册是为精通大多数Leetcode算法的人准备的。这是一个免费资源，是对CSES.fi课程的有力补充。

竞争编程第4版。

对于最有经验的算法爱好者来说，这本书将涵盖在任何编码面试中可能被问到的每个利基数据结构和算法。FAANG类型的公司通常不需要这种准备，但如果你考虑对冲基金类型的公司，这种准备可能会出现。

系统设计

《系统设计面试》(第一卷、 第二卷、 网络课程+社区)

在我看来，使用这些资源，你将为任何系统设计面试做好充分准备。每本书的图表都很清晰，解释也尽可能简单，以帮助你快速学习系统设计概念。

我个人推荐在线课程，因为这两本书的内容都很值得拥有，正是你接触到的在线社区的不和谐使得每年的订阅物有所值。这种不和谐包括模拟面试伙伴、薪资讨论以及与其他用户一起研究每个系统设计主题的过度关注。

系统设计入门

系统设计入门是关于所有系统设计的最好的免费资源。深入挖掘Git存储库，你将会学到你需要知道的关于系统设计的一切。所有这些都集中在一个存储库中，结构清晰，可为您提供有指导的课程。

Educative的系统设计面试

如果你赶时间的话，这个系统设计的快速概述是很好的回顾。阅读通常需要用户45分钟，但是你会比一般的工程师知道更多的系统设计。

读一读。如果概念不清楚或令人困惑，那可能是你还没有准备好面试的信号。

进行系统设计面试

如果您需要更深入的东西，那么您可以查看这个资源，它为您提供了世界上最大的产品的清晰的行业标准系统设计解释。通读一遍这个和系统设计面试应该足以通过任何系统设计面试。

面向对象的设计

设计模式:可重用面向对象软件的要素

不管你是否在为面向对象编程面试学习设计模式，作为这些大公司的软件工程师，你都需要了解设计模式。

这本书是当今世界上最常见的设计模式的起源，在面向对象的面试中表现出对这些模式的精通是某些大型技术公司如亚马逊的要求。

头先设计图案

上面的资源非常密集，并且用难以理解的语言编写。虽然设计模式中的原始资料很棒，但如果很难理解，就没什么帮助了。

考虑Head First设计模式，研究这些常见设计模式的简化解释。它可能不像原始资料那么深入，但是你对设计模式的理解将足以破解任何面向对象的面试。

结束语

老实说，我没有从头到尾看完所有这些资源。如果你这样做了，我相信你就不需要再为另一个面试而学习了。但是很可能我们没有时间去做，所以确保一旦你理解了以上任何一个类别的核心概念，你就可以投入时间去做下一个。

同样，这些是我使用的资源，并不包括任何其他人的学习计划。
