# 软件工程面试官试图回答的问题(第三部分)

你最近面试过软件工程的工作吗？也许是两年前，或者你现在正在面试。作为一名软件工程师技术和非技术面试的面试官，我可以给你一些关于面试官实际上在候选人身上寻找什么的见解。这是第 3 部分，所以如果你还没有通读，请查看以前的故事！

**你能很好地沟通吗？书面和口头的。当别人是混蛋时，你会大声说出来吗？**

在生活中和几乎任何行业中取得成功的关键之一是能够很好地进行口头和书面交流。想一想在面试过程中有多少方法可以测试这一点。首先，你向公司提交你的简历或申请，公司会审核你的书面文件。然后你在电话上与某人交谈，这将测试你的语言交流能力，以及你在表达和描述过去的经历以及回答问题方面的能力。最后阶段是面对面或视频通话，有人可以看到你的肢体语言是否符合你的口头交流，如果有技术面试再次测试你的书面交流。

如果你觉得你的技能需要一些爱或锻炼，那么考虑写博客或找一个笔友或和一个导师一起提高你的电子邮件写作。如果你想专注于语言沟通技巧，那么准备午餐&在你的公司或当地聚会上学习演讲，或者加入一个定期聚会的在线小组，谈论你感兴趣的话题。不断锻炼和提高你的写作和口头交流技能将会带来更多的机会，那些在这两方面都做得很好的人通常是一个组织或公司的领导者。如果你还没有导师，找一个，让他们评论你的交流，这样你就知道你可以首先关注哪些领域。

现在我们已经讨论了你的沟通技巧，让我们来谈谈什么时候使用它们是至关重要的。许多人都有过这样的经历:在一次会议中，某人要么回应另一个人，要么即兴说出一些令人不快的话。在这些情况下，房间变得安静，或者有人会对那个人大打出手。能够平息这些紧张的局面，并就他们行为的影响向个人提供反馈，可以极大地改善工作环境。我喜欢在提供反馈时使用 [SBI 模型](https://www.mindtools.com/pages/article/situation-behavior-impact-feedback.htm),这样我在提供反馈的同时，个人也掌握了他们打算如何改变的流程。对可观察到的行为给出反馈会减少很多情绪，你谈论的是事实而不是观点。你以前遇到过这种情况吗？你有什么反应？您是否向个人提供了反馈？你的经理？能够表达自己并平静局面是一种专业的沟通技巧，也是许多公司所重视的，尤其是在经理及以上级别。

总之，我们可以看到你的沟通技巧在整个面试过程中得到了利用和检验。提前做一些工作来找出任何弱点或解决问题是值得花时间的，如果你在压力下保持冷静，那么想办法在面试过程中提出来，你将成为一名出色的候选人。
