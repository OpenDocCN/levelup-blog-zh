软件开发人员关于编码的最大误区

原文：https://levelup.gitconnected.com/greatest-myths-about-coding-among-software-developers-e578b7556171?source=collection_archive---------1-----------------------#2020-10-06

流言终结者

摆脱这些误解，在软件开发中有一个加速的职业生涯。

开发人员通常对编码有很多误解，尤其是刚起步的开发人员。对这些神话的信仰限制了他们在职业生涯中学习和成长的能力。

每年都有应届毕业生加入我们的团队，他们对软件开发都有类似的错误观念。我甚至遇到过一些有多年经验的开发人员，他们有一些这样的误解。

从人们的头脑中丢弃最初的感知是困难的。一旦它们建立起来，就需要很大的努力才能摆脱它们。

因此，在你的职业生涯中，最好尽早了解有关编码的神话，这样你就可以将它们与事实区分开来。

误区一。编码很难。

流言终结者:

这在软件行业经验较少的开发人员和新手中是一个经典的神话。

我不是说编码是小菜一碟，任何人都可以从第一天开始做。但是，这也不是火箭科学。任何值得学习的东西在你开始的时候都会是一个挑战。

当你开始学习一门新的编程语言时，最初的几周并不容易。但是一旦你了解了基本面，有了动手的知识，它就会慢慢在你身上成长。

为了让你更有说服力，在我的职业生涯中，我和许多优秀的开发人员一起工作过，比如戴尔、花旗集团和美林证券，他们都没有工程背景。

这些开发人员在接受教育期间没有接受过软件工程方面的正式培训。但是他们自己学会了，随着时间的推移，他们变得擅长编码。

您可以做什么:

  • 学习你选择的编程语言中的基础。
  • 不要期望在一夜之间变得超级高效。
  • 给自己至少三到四周的时间来学习一门新编程语言的基础知识。
  • 用小代码片段定期练习。

神话2。复制粘贴是糟糕的程序员的标志。

流言终结者:

在他们职业生涯的最初几年，开发人员认为复制粘贴是一种犯罪。他们认为他们总是需要从头开始编写代码。

如果你做得对，复制粘贴代码并不是一个坏主意。

我不是说在应用程序中复制完全相同的代码。如果你这样做，你需要重新思考为什么代码不能在一个公共的方法中。

然而，当您理解这段代码在做什么以及它如何解决您的问题时，复制-粘贴是有益的。它还通过避免键入样板代码节省了大量时间。

甚至在今天，我有时会复制一部分代码，以便立刻准备好我的方法或类的框架。我希望专注于编写业务逻辑，而不是键入一些可以很容易从其他地方复制的内容。

您可以做什么:

  • 清楚地理解手头的需求。
  • 谷歌不同的可能解决方案。
  • 评估解决方案的优点和缺点，然后复制适当的代码片段。
  • 更新复制的代码以符合您的要求以及应用程序的编码标准。

误区三。你应该记住所有的内置方法以及它们在语言中的用法。

流言终结者:

大多数初学者认为他们需要记住一门语言的每个细节来编码。

除非你是Josh Long，否则你可能永远不会记住所有的内置方法——这完全没问题。我不认为即使是他也记得他需要在代码中使用的所有方法。

我们不是在1990年代，在那里你必须在记事本或命令提示符下编码。

几乎所有语言都有许多强大的编辑器。编辑器使编程更快，并有助于避免愚蠢的错误。使用编辑器的自动补全功能，您就万事俱备了。

您可以做什么:

  1. 理解内置方法的作用。那就是根本。
  2. 使用一个好的编辑器，它为你的编程语言提供了自动完成功能。
  3. 如果你不知道使用哪种内置方法，可以使用Google来了解不同的选项。

误区四。优秀的开发者从来不会有编译错误。

流言终结者:

没错— 没有编译问题的唯一方法就是永远不要写任何代码。

初级开发人员在他们的代码无法编译时会感到羞愧。

然而，编译错误几乎每天都会发生在每个开发人员身上。我与数百名开发人员合作过，他们中的每一个人都可以告诉你，代码在编译时爆炸是正常的。

即使编码了15年，我还是会被编译器捕捉到随机错误。

为了自己看，找出著名开发人员的培训视频。您将看到他们在现场编码会议中面临多少次编译问题。

你不必为每个语法问题绞尽脑汁。让编译器完成它的工作，这样您就可以更专注于实现业务逻辑。

您可以做什么:

  • 明白编译问题没什么好丢人的。
  • 编码时多注意一些小细节。
  • 通过对代码进行预演来捕捉尽可能多的错误是一个很好的实践。

神话5。如果你寻求帮助，那么你不是一个好的开发者。

流言终结者:

我觉得这是很多团队的文化问题。寻求帮助被认为是这个人没有能力做好自己的工作。

我见过有人在背后嘲笑初级开发人员，因为他们问了一个被认为很容易的问题。这让人觉得寻求帮助是不被鼓励的。

然而，我可以向你保证，嘲笑别人的人在日常工作中也会寻求帮助。

在IT行业，没有一个人可以不需要任何人的帮助就能工作。

寻求帮助表明你乐于接受反馈和建议。这是成为一名优秀软件开发人员的基本特征。

您可以做什么:

  • 当你面临一个问题时，首先试着自己解决。这一步对培养你的分析和逻辑能力至关重要。
  • 如果你仍然找不到解决方案，记下你尝试过的所有方法，并与你认为有帮助的队友分享。
  • 如果一个人不准备帮忙，不要过分逼迫他们。问问别人。

神话——成为一名优秀的软件开发人员只需要技术知识

“编码就是一切所需”“的思想阻碍了开发者的职业进步。大多数软件工程师都有这种想法，这是最大的误解。

只要看看你的团队和你的公司，就能发现有经验的软件开发人员的职业轨迹。

只有少数优秀的开发人员在职业生涯中成长为架构师，这是有原因的。
