# 如何衡量一个程序员的态度

> 原文：<https://levelup.gitconnected.com/how-to-measure-a-programmers-attitude-5882066da747>

![](img/891a689c3ae016e1ee7a06e8b725980b.png)

编程面试充满了假设的情况。AWS 推广的整个明星面试形式都围绕着创造一个适合场景的情景，让候选人描述他/她解决这个问题的理想方法。

> 明星面试的问题在于，经过一段时间后，题库会变得饱和

这种方法(以及整体的明星面试形式)的好处是，候选人的人际交往能力会得到简洁(不完美)的评估，主要是:客户关注度、主人翁精神、勇气和团队精神。

这种方法的坏处是，经过一段时间，题库变得饱和。

一旦围绕问答格式建立了一个模式，聪明的程序员也可以创造一些情景来猜测面试问题，并准备好他们的答案。

尽管我是一个普通的开发人员，在错过了一个可观的工作机会后，我还是想出了《T4》明星访谈 Q &一本电子书。

这显然是不可取的。一个候选人能够胜任这个角色的唯一充分理由，不是他足够聪明，能够提前为这个问题做好准备，而是他已经经历了所讨论的确切情况。

根据公司的规模，典型的行为面试包含 10-20 个行为问题。面对每一种情况是完全不可能的。

结果？伪造理想答案的候选人赢得了面试，进入了公司，破坏了公司文化，却在 2-3 年内为了更好的薪水而辞职。

真诚的开发者遭到拒绝，甚至没有人知道为什么。

这种事情在《明星访谈》问世之前就已经发生了，而且那时候要容易得多。我知道，因为我也成功了，尽管有时是最不适合的。

原因？这些问题太暴露了。

**举例**:

> 如果你的同事在生产部署日生病了，你会怎么做？

**理想答案**:我会逾期留在办公室完成待交付。

**现实的回答**:我会请教生病的同事，如果他平易近人的话。我会在交付日询问并评估任务。如果可以的话，我会送货的。如果不行，我会去找客户，以同事的健康状况和我对分娩缺乏了解为由，恭敬地请求延期。

> 如果客户问的特性很复杂，你会怎么做？

**理想答案**:我会说是，提供报价，然后通过参加在线课程立即学会所需技能。

**现实的回答**:我会要求一个估计时间表。在此期间，我会评估特性的复杂性。基于此，我将向客户提供我的报价。如果不是以我现在的身份，我不会拿公司的声誉冒险，做出过分的承诺。

大多数候选人都去寻找理想的答案，并且被录用了。原因往往与在场的人力资源有关，他们看到态度最好的人就跳了起来！

> 拜托，你们这些技术人员已经有机会在最后 3 轮里烧烤了，这次是我的！

技术人员通常知道这家伙的承诺是海市蜃楼。真正的考验在隔间地板上等着他。但是他们必须把雇佣的标签贴在某人的头上，而这个人已经检查了所有的盒子。

不用说，由于这种多余的面试形式，数十万名有能力的程序员最终选择了错误的工作。明星增加了一些复杂性，但它有一个结构，它可以很容易地被一个有能力的程序员玩，他已经钉了 3 个技术回合。

行为访谈和整个态度测量更多的是用心而不是用脑。用僵化的、有时间限制的问答形式来衡量是相当困难的。

# 衡量候选人态度的唯一实用方法是:

当候选人提前准备行为问题时，他更有可能会挑战态度。

> 不要把一个候选人绑在问答格式里。相反，允许他敞开心扉。

面试官必须防止这种情况发生，以展示候选人最好的和最坏的一面。

他应该为候选人提供一个畅所欲言的空间，而不是把他束缚在僵化、结构化、无所不能的问答形式中(例如:STAR)。为了提高透明度，最好能披露一般评估标准(例如，*技术技能、所有权、团队合作的例子是我们正在寻找的东西*)。)

迄今为止，在我参加的无数次面试中，唯一一次我发现向面试官敞开心扉是在我被问到我最好的项目时。

## 为什么这种策略比问答更有效:

*   会让程序员赢得面试官的信任。询问某人最喜欢的是赢得约会游戏的古老技巧，面试也不例外。
*   项目描述总是绑定技术和行为。技术主管和人力资源都可以窥见候选人的心态。
*   这将促使程序员谈论是什么让他/她被触发去创造一些令人敬畏的东西。这不仅有助于评估他/她对技术的热情，而且也是让他/她在未来某项任务中行动的关键，如果他/她被录用的话。
*   相对于固定的、可游戏的问题，你不会得到不连贯的分数，你会得到一个关于开发者沟通技巧的全面的想法，他想出点子并展示它们的能力。

## 为获得最佳效果:

*   最好不要有预设的问题(例如*挑战是什么？你是怎么克服的？相反，你应该用有意义的轻推来打断候选人(例如，*你为什么选择了 JS 而不是 TS？*或*运行了多长时间* ***npm 构建*** *？*或者*失败的时候你不觉得无聊吗？*)。*
*   要衡量程序员的所有权特质，可以问这样的问题:要求一定很高。中间有时间间隔吗？你是如何保持这种势头的？
*   要衡量程序员的客户处理技能，可以问这样的问题:*你是如何处理客户升级的？你不害怕报告你最近注意到的漏洞吗？*
*   要衡量程序员的团队游戏态度，可以问这样的问题:*你有没有接受任何人的帮助？* *你开源了解决方案吗？你是怎么和老板分享的？*
*   尽量随意一点。用乐趣点缀问题，但避免潜在的攻击性讽刺。

# 结论

随着经济衰退的逼近，软件行业终于认识到长期忠诚的候选人的价值。跳槽者最终被认定为招聘失败、态度恶劣的程序员或者仅仅是文化不适应。

当面试以现实的方式包括态度问题时，就有更大的可能性提前建立对一个伟大团队的信任。
