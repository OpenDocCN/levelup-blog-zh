5.  Flutter 的热重新加载功能使它能够提供更快的性能。有了它，应用程序就可以使用 ***arm C/C++*** 库进行编译，使其更接近机器码，运行速度更快。

8.  使用 Flutter，您可以编写代码、管理代码，并在多个平台上运行它。对于开发人员来说，这节省了时间、金钱和精力。
9.  Flutter 提供了一个可定制的分层架构，允许高度可定制的设计、富于表现力的 ui 和快速渲染。
10.  Flutter 的结构包括三层:-框架 Dart、引擎 C/C++、特定于嵌入式平台
11.  基于 Dart 的平台，处理应用程序小部件、手势、动画、插图和材料
12.  Flutter 应用程序开发依赖于单一的框架和平台来开发、部署和管理变更，而不是使用不同的平台和框架来实现不同的目的。
13.  Flutter 应用程序开发提供了为 Android、iOS 和 Google Fuchsia 定制的小部件。
14.  正是因为有了 Flutter 的即用型小部件库，开发者在使用框架时可以更快地开发 app。除了各种各样的小部件之外，它还包括动画，您可以使用这些动画来丰富您的应用程序。
15.  但是，由于 Flutter 相对较新，第三方库的数量很少。此外，Flutter 中的一些小部件只能在一个平台上使用。
16.  当发布版本的大小不符合预期时，开发人员有时会感到沮丧。
17.  Dart 是一种面向对象的编程语言，但它无法与 Java、JavaScript 或 C#竞争，因为它仍然相对较新。因此，没有多少开发者选择它。
18.  Flutter 的 3D 建模、Unity 集成和游戏引擎都有所欠缺。所以大部分广告移动平台也不支持。
19.  颤振还没有被广泛使用。尽管它受到了技术爱好者的关注，但它仍然缺乏随着时间的推移而提供的持续支持。可悲的是，Flutter 得到的唯一支持来自它的社区。
20.  根据您的开发阶段，框架以不同的方式或模式编译您的代码，我们称之为构建模式。
21.  调试模式:该模式支持在物理设备、仿真器或模拟器上调试应用程序。这里启用了断言和服务扩展。然后通过优化编译实现快速部署。
22.  在 Profile 模式下，保留了一些调试功能，足以在测试时分析应用程序的性能。在这种情况下，会启用跟踪和一些扩展。在仿真器和模拟器上，配置文件模式被禁用，因为它们的行为不能再现真实的性能。
23.  以下命令可用于编译轮廓模式:**颤振运行—轮廓**
24.  部署应用程序时，发布模式用于最小化占用空间和最大化优化。这里禁用调试、断言和服务扩展。
25.  一般来说，Flutter 应用程序由许多小部件组成。小部件是声明和构造用户界面的一种方式。
26.  在 flutter 中，窗口小部件可以分为两类:-有状态窗口小部件和无状态窗口小部件
27.  Dart 是谷歌开发的一种编程语言，用于编写 Flutter 应用程序以及服务器和桌面应用程序。通过使用 Dart，Flutter 避免了使用单独的声明性布局语言，如 JSX 或 XML。
28.  开发人员可以非常轻松地阅读和可视化 Dart 的布局，因为它是声明性和编程性的。
29.  Dart 不直接支持数组。相反，它支持复制数组、泛型和可选类型等数据结构的集合。
30.  与其他编程语言不同，它支持大多数基本编程概念，如类、接口和函数。
31.  尽管与 JavaScript 相似，但 Dart 运行代码的速度要快几倍。
32.  为了获得更好的性能并减少代码执行时间，Dart 虚拟机(VM)同时使用实时(JIT)和提前(AOT)编译器。
33.  Dart 是面向对象的编程，这使得它对于创建甚至复杂的应用程序来说都是非常可伸缩和稳定的。
34.  应用状态也可以被称为共享状态或应用状态。
35.  **main():** 该函数启动程序。Flutter 不允许我们写任何没有 main()函数的程序。
36.  **runApp():** 使用 runApp()，您可以返回连接到屏幕的小部件，作为将在屏幕上呈现的小部件树的根。这个函数在 main 函数中调用，main 函数是 app 的驱动。
37.  包是类、接口和子包的集合，支持创建模块化代码，用户可以轻松地共享这些代码。
38.  使用包可以快速开发应用程序，而不是从头开始开发。您可以使用 Flutter 中的包将新的小部件或功能导入到应用程序中。
39.  包和插件在 DartPub 上通常被称为包，只有在创建新包的时候才会对两者进行区分。
40.  为了开发移动应用，Flutter IDE 和工具需要一些插件。有了这些插件，我们可以编译 Dart，分析代码，开发 Flutter。用于 Flutter 开发的两个最流行的 ide 是:-Android Studio 和 Visual Studio
41.  在 Flutter 中，键被用作小部件、元素和语义节点的标识符。GlobalKeys 和 LocalKeys 是 Key 的子类。在小部件树中，键负责保存修改过的小部件的状态。
42.  使用键，您还可以重新组织和修改具有相同类型和已定义状态的小部件集合。键的主要用途是修改包含有状态小部件的小部件树，而不是修改完全由无状态小部件组成的树。
43.  基本上，在 Flutter 中，容器是一个能够容纳多个子部件并通过尺寸、填充和背景颜色有效管理它们的部件。
44.  每当我们想要设计一个小部件的背景时，无论是因为颜色、形状还是大小的限制，我们都可以使用容器小部件。
45.  有了容器类，小部件可以根据我们的意愿在屏幕上存储和定位。
46.  与 React Native 相比，Flutter 是一个相对较新的框架。Flutter 最初由另一家名为谷歌的巨头支持。
47.  主轴对齐是项目在该轴上的对齐方式
48.  交叉轴对齐是项目在另一个轴上的对齐方式
49.  行和列小部件可以使用 crossAxisAlignment 和 mainAxisAlignment 属性根据我们的偏好对齐它们的子部件。
50.  Flutter 是谷歌的一个免费开源 UI 软件开发包，允许开发者轻松构建跨平台的移动应用。
51.  第一次构建 Flutter 应用程序时，花费的时间比平时长得多，因为 Flutter 会创建一个特定于设备的 IPA 或 APK 文件。在这个过程中使用 Xcode 和 Gradle 来构建一个文件，这通常需要很长时间。
52.  有一个强大的工具叫做 Flutter Inspector，用于 Flutter 应用程序，它允许你可视化你的小部件的蓝图和它们的属性。

56.  Flutter SDK(软件开发工具包)使开发人员能够使用单一代码库构建移动、web 和桌面应用程序
57.  热重装被认为是 flutter 的一个优秀特性，大约需要一秒钟来完成它的功能。有了这个功能，您可以轻松快速地进行更改、修复 bug、创建 ui 以及添加功能。通过利用热重新加载功能，我们可以在一个文件中快速编译新代码，并将其发送到 Dart 虚拟机(DVM)。
58.  **热重启**:与热重装相比，其功能略有不同。在这种情况下，我们的应用程序的保存状态被破坏，代码从头开始重新编译。
59.  BuildContexts 用于在部件树中识别或定位部件。
60.  每个小部件都有自己的 BuildContext，即每个小部件一个 BuildContext。基本上，我们用它来查找对其他小部件和主题的引用。
61.  此外，您可以利用它来与小部件父级交互并访问小部件数据。
62.  使用单元测试，您可以测试一个类或方法。单元测试不检查屏幕呈现、与外部服务的交互或用户交互。
63.  使用小部件测试，您可以测试单个小部件。这确保了小部件的 UI 看起来像预期的那样，并适当地响应事件。换句话说，它确保了小部件的设计、呈现以及与其他小部件的交互符合标准。
64.  使用集成测试，您可以测试整个应用程序的关键流程。检查所有小部件和服务是否如预期的那样协同工作是很重要的。您还可以使用它来衡量和基准测试您的应用程序的性能。
65.  无论您是构建移动应用程序还是 web 应用程序，状态管理都至关重要。使用它，各种 UI 控件的状态被集中起来处理应用程序中的数据流
66.  它可以是文本字段、单选按钮、复选框、下拉列表、切换、表单等。在 Flutter 中，状态管理可以分为两种类型:-短暂状态或应用程序状态。
67.  **短暂状态:**短暂状态也称为 UI 状态或本地状态，它属于一个特定的小部件。换句话说，它是包含在特定小部件中的一种状态。通过 StatefulWidget，Flutter 提供了对这种状态的支持。
68.  **应用程序状态:**这不同于短暂状态，因为它是一种我们打算在应用程序的不同部分共享的状态，并且我们希望在会话之间保持这种状态。因此，这些类型的状态可以全局使用。借助 scoped_model，Flutter 为这种状态提供了支持。
69.  pubspec.yaml 文件也称为“pubspec ”,是在创建 Flutter 项目时包含的文件，位于项目树的顶部。这个文件包含关于依赖项的信息，如包及其版本、字体等。一个项目需要的。

77.  在异步方法完成之前，await 会中断流程。Await 一般的意思是:在这里一直等到这个函数完成，这样就可以得到它的返回值了。Await 只能与 async 一起使用。使用此功能，所有当前正在运行的功能都将暂停，直到 PF nature 完成。
