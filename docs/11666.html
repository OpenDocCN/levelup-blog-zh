软件工程最伟大(也是最糟糕)的事情

原文：https://levelup.gitconnected.com/the-single-greatest-and-worst-thing-about-software-engineering-76119034647d?source=collection_archive---------5-----------------------#2022-04-04

我写代码已经有一段时间了，我非常喜欢软件工程中的一件事。然而，你必须忍受剧烈的疼痛才能达到目的。

这是一个存在于编程世界中的奇怪悖论，但我指的是问题解决。是的，我知道这听起来很老套，但是在你认为这是一篇关于热爱软件的普通文章之前，给我一个机会来提供一个更深入的解释。

当他们说他们有多喜欢解决问题时，没人告诉你的是，压力和痛苦是作为必要的副产品加在问题解决者身上的。压力、痛苦和困惑实际上可能是解决问题的主要产物，因为不需要弄清楚一些事情，它真的是一个问题吗？这听起来有点消极，但我说所有解决问题的方法都会导致压力和痛苦，这有点言过其实。这不是真的，但是这篇文章是关于需要软件工程师真正发挥创造力的更难的问题。

这篇文章的原标题是:“如果你喜欢挑战，尽快学会编码。”我给它起这个名字是因为当面对一个非常困难的问题时，一个人会有两种反应:

  • 诅咒造成这个问题的环境，并在精神上屈服。

或者

  • 通过为发现对他或她有挑战性的事情而高兴，并为胜利做好心理准备。

如果你通常像后者一样回应，你应该立即学习编码 因为尽管我认真编程才几个月，我多年前就决定 一个人精神上不断尝试克服困难或过去经验的意愿是做任何有意义事情的关键。T13】

此外，我认为公平地说，一个人对难题的反应不是一种永久的性格特征。我认为有些人比其他人更容易成为优秀的问题解决者。那些可能会像前一种情况一样做出反应的人，现在可以在以后学会改变他们的心态成为优秀的问题解决者。所以，如果你正在读这篇文章，并且通常选择在精神上屈服，你仍然有希望。检查一下你的心态。

心态和软件工程最伟大的事情有什么关系？

这与它有着千丝万缕的联系，因为软件工程最伟大的事情就是克服障碍的喜悦和内在的感觉，但是如果一个人太愿意放弃，他或她永远也不会有这种感觉。

在软件中有一个20分钟规则，本质上是说:

  1. 试着用20分钟自己解决一个问题。
  2. 如果你仍然停滞不前，使用谷歌。
  3. 如果你仍然停滞不前，问问同伴。
  4. 最后，如果还是解决不了问题，就去问专家。

如果你占用了别人的时间或在工作，这是一个很好的协议，但对于个人项目，我会在前两步增加时间，因为现在我正试图变得自给自足，当你自己解决问题时，满足感会更甜一些。然而，我提供了那个协议来展示事情通常是如何完成的。如果你不能解决一个问题，有时它可能只是超出了你当时的知识范围，所以你不应该花大量的时间去强调它；你应该得到帮助和清晰的认识。

即使在解决问题的世界里，也有一个获得帮助的系统，因为在这样一个领域里，典型的人都愿意在精神上不断尝试。这是软件工程最伟大之处的核心。

每一天，每一个项目，每一个问题，都是成长和尝试的机会。

用这种心态对待每一个问题，你已经成功了一半。

保重。
