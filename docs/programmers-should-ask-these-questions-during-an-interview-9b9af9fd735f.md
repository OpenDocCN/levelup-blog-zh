# 程序员应该在面试时问这些问题

> 原文：<https://levelup.gitconnected.com/programmers-should-ask-these-questions-during-an-interview-9b9af9fd735f>

通过问这些问题，你可以更好地了解这家公司，提高聘用你的几率。

![](img/8ae427526ba605580176c95c7f878677.png)

作为一名软件工程师，相信大家都经历过很多求职面试。大家也在各自的专业领域总结了很多经验。当你在面试中提出与专业技能相关的问题时，你可以回答每一个问题。但我不知道你是否注意到了。面试结束时，面试官会问:“你有什么问题要问我吗？”。问了一些身边的同事朋友，大部分都会直接回答:“不用了，谢谢”。如果这就是你的答案，你有必要看一下这篇文章。你很可能会错过更多了解你面试的公司的机会。

作为应聘者，你可能会更关注自己的工作经验和专业技能能否通过这次面试获得工作机会。但我想说，面试是一个双向选择的过程。公司面试你的时候，你也在面试公司。如果公司团队管理不善，技术落后，你还想为其工作吗？今天我们就来谈谈作为一个开发者，如何通过向面试官提问来进一步确认这家公司是你喜欢的公司。

一般会有几轮[技术面试](https://medium.com/javarevisited/10-data-structure-algorithms-and-programming-courses-to-crack-any-coding-interview-e1c50b30b927)，面试角色可能会有所不同，比如软件开发人员、技术负责人、部门主管等。面对不同的角色，要问有针对性的问题。

## 向软件开发人员提问

**你有编码标准吗？**

问这个问题了解面试公司的专业性。如果他们连一个基本的编码标准都没有，你就要小心了。你面试的公司可能不追求技术管理。组里的工程师要花时间适应不同的编码风格，开发效率会受到影响。好的编码标准可以统一编码风格，提高代码的可读性和可维护性。

**你如何处理软件故障？**

在我们的职业生涯中，软件故障是不可避免的。毕竟我们不是上帝，不会写出没有 bug 的完美程序。如果开发者回答的很自信，并且有专门的措施来应对，那恭喜你。

他们能够冷静处理软件故障，并采取具体措施，保证下次不再出现。这样的公司值得加入。

相反，如果一个公司经常出现软件故障，又没有具体的应对措施，那你就要慎重考虑了，因为这个团队并不注重产品质量，很可能你加入后会有频繁生产和灭火的困扰。

**您使用什么技术堆栈？**

问这个问题可以合理确定你和公司的技术匹配度，通过面试官的回答可以知道他们是否还在使用一些陈旧落后的技术，从而判断技术团队是否对[技术](https://medium.com/javarevisited/top-5-online-courses-to-learn-mern-stack-in-depth-9947230f194)有热情。

**你如何控制软件质量？**

这个问题是为了尝试了解团队的测试过程。如果团队对单元测试、集成测试、性能测试有明确的规定，或者他们有很强的自动化测试能力，你可以认为这个团队专注于软件质量，是一个可靠的技术开发团队。

## 向技术经理提问

**你们是如何确定技术方案的？**

通过这个问题可以了解团队的技术方案设计流程。如果团队的技术方案设计过程有一定的规范，设计方案会作为详细的设计文件输出，会有方案评审，那么我会认为团队的项目管理是合格的。

**您如何衡量员工绩效？**

有经验的技术经理经常与团队成员交流，并仔细收集团队成员的反馈。他们根据这些交流和他们观察到的事实来评定员工的表现。

**做技术分享还是技能培训？如果有，多久一次？**

作为一名软件工程师，除了工资，我希望通过在公司工作来提高自己的能力。如果团队经常做技术分享，说明团队愿意通过一些方法培养员工，这绝对是吸引我的地方。

**你们团队的离职率是多少？**

团队离职率可以判断团队是否可靠。当然，跳槽是一件很正常的事情。但是，高离职率会让我保持警惕。如果这个团队的离职率比较高，很可能是这个团队的管理比较差。

## 向导演提问

公司盈利吗？

如果是，说明公司做的不错。你可以安心留在公司，不用担心公司倒闭，你很快就会失业。如果没有，可以继续问资本状况和商业模式。

**什么是企业文化？**

应聘者可以通过公司文化来判断公司的工作氛围，以及我们是否认可他们的价值观。一个优秀的企业必须有伟大的企业文化作为支撑，因为伟大的企业文化所形成的文化氛围，能够无形中凝聚公司员工，潜移默化地提高员工的积极性、主动性、创造性，从而提升企业的整体执行力。

**车队目前面临的最大挑战是什么？**

如果面试官开始说产品技术，产品开发，人员短缺等等，这很正常。如果他的回答是项目时间紧迫或者需要更多的资金支持，你就要小心了。很可能是公司管理有问题。

## 最后

面试是双向的。公司在面试你的时候，你也在面试公司。面试时向面试官提出有针对性的问题，可以让你充分了解你要加入的公司的运营情况。感谢您的阅读。
