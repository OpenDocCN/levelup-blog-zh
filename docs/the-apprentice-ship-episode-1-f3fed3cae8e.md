# 学徒(船)第 1 集

> 原文：<https://levelup.gitconnected.com/the-apprentice-ship-episode-1-f3fed3cae8e>

三个月过去了，我的学徒生涯已经全面展开。我已经完成了前两个学习阶段。爱死了。

![](img/c6e3eb1ababb8dcaddffd510ef1ca08b.png)

为了继续第 0 集的工作，我完成了我正在使用的贷款计算器应用程序。它并不完美(我在使用`background-image`时遇到了一些`::after` & `position`的问题)，但是它可以工作！它(几乎)崩溃到移动，但老实说，如果贷款金额输入太大，利率也太高，我可以崩溃我的浏览器。我只完成了剩下的两个 Codecademy 模块中的一个。我更新了简历。但除此之外，Bootstrap 主页和投资组合网站之类的东西就是没有出现。

但是我的学徒生涯终于开始了！以下是我们头两周所做的一瞥。

# 欢迎来到 Sparkbox！

我不记得我是否说过这些，但我现在工作的公司叫 Sparkbox！我和他们在一起才 3 个多月，我甚至无法告诉你在这个地方工作我有多么受鼓舞和有动力！他们是一家与一群非常酷的客户合作的机构，但目前，我没有做任何客户工作。对我来说完全没问题，因为现在我已经见证了良好的编码实践，我对我的一些项目是多么的无组织感到尴尬！

在我第一天上班之前，我收到了我的工作电脑。几天后，我也拿到了几本书。现在是十月，所以我们已经完成了[响应式网页设计](https://www.amazon.com/Responsive-Design-Brief-People-Websites/dp/098444257X/ref=sr_1_6?crid=3TRDF8MCK16KV&keywords=responsive+web+design+book&qid=1666532482&qu=eyJxc2MiOiIwLjAwIiwicXNhIjoiMC4wMCIsInFzcCI6IjAuMDAifQ%3D%3D&sprefix=responsive+web+design+book%2Caps%2C149&sr=8-6)和[实用程序员](https://www.amazon.com/Pragmatic-Programmer-journey-mastery-Anniversary/dp/0135957052/ref=tmm_hrd_swatch_0?_encoding=UTF8&qid=1666532532&sr=8-1)。我也得到一个惊喜！一个小小的欢迎包裹寄给了我，里面有一个火花盒球座、一个小笔记本和一张恶心的贴纸。我甚至还没有开始，但我已经感觉到这些人关心我，让我感到受欢迎。🥳

## 第一周

我的第一周主要是入职。因为我们是远程的-首先，它是许多变焦电话。我想称赞他们的一点是*文档*。这是我没有太多经验的事情，但它确实让一切变得不同。我们得到的每件事的说明都非常清楚，非常详细，只要我遵循所有的说明(即使我犯了一些解释上的错误)，我就在第一周内完成了所有的事情！突然，(150 封邮件后😵‍💫)我有一个特雷罗董事会，我有一个吉拉董事会，一个 Sparkbox 电子邮件，一个密码管理器，Harvest 成立了，我见了我的导师，见了财务总监，见了人事总监，建立了一个工作缩放帐户，被邀请参加公司务虚会，得到了 Slack 的设置，拍了我的团队照片，让 Lattice 准备好运行 shell 脚本。那是我碰壁的地方。

我们的一个技术主管(我想他们是工程经理)给了我们一个 shell 脚本，我们被指示在我们选择的终端上运行。这东西本来是要给我们装一堆应用的。我在机场有这么多🤏的经验，所以我肯定他给了我很好的指示，我只是跟不上。这可能是我的错——我已经安装了我的编辑器(VSCode)，我已经安装了一些浏览器，我的工作电脑上已经有了 Zoom 和 Slack 之类的东西。当然，这个脚本应该做到这一点，所以我认为这是它的一部分。他试图在 45 分钟内帮助所有的 4 个学徒，而我却迷失了。

![](img/dd3efebf04b3fc7e2f1d2d2598ac0f54.png)

那天晚些时候，我和我们“学徒圆桌会议”的一个成员聚在一起，寻求个人帮助来解决所有问题。这是我立即注意到的关于我其他同事的一件事:

> *所有的全职员工都非常理解、支持和善良。*

在第一周的剩余时间里，我们开始举行每日学徒站立会议，向我们的协调员和学徒项目经理汇报我们的吉拉卡片的进展情况。**我们还上了 CLI 和 Shell 脚本的第一课。**好像是为我而生的！它澄清了我前几天的一些困惑。我们见到了总裁和创始人，又一次找到了另一个人，他看起来很踏实、善解人意、对我们的需求感同身受，并且乐于帮助我们。

## 第二周

涵盖的课程:

*   **编辑器和 IDE 的**
*   **Git 基础知识**
*   **吉拉操作指南**

我们第二周最重要的事情是结对练习。我们负责获得每周大约 3 小时的全职配对。有一个日历，全职开发人员可以张贴办公时间，我们可以继续和他们一起争取一些时间。这真的很酷，因为我基本上得到了我需要的个人关注。我还和我的导师进行了第一次真正的一对一交流。原来，她几年前还是个学徒！

我们本周的定向学习从编辑器和集成开发环境(ide)开始。我不知道外面有多少不同的编辑。面对如此多的定制，如何选择呢？！我的配对集中在查看其他人使用什么，他们有什么插件和扩展，以及在某些情况下如何使用它们。

我们上了关于 Git 的第一课，特别是如何以及为什么在命令行中使用它。那对我来说太多了。我听说过 Git，但并不知道它和 GitHub 有什么不同。谢天谢地，我们有一个练习回购，有很多关于如何使用 Git 的说明。我一直在推代码和拉代码…嗯，不像一个专业人士，但东西少得多。

对我来说，弄清楚如何使用吉拉，何时检查它，以及何时移动牌对我来说很容易。我基本上让它一直开着，以便不断检查进度和刷新自己的任务(但我确实意识到它耗尽了我电脑的生命)。

本周实施的其他事情是制作团队介绍。团队中的每个全职开发人员都向这 4 名学徒做了 15-30 分钟的简短介绍。除此之外，我们还必须安排配对会议。这些很酷，因为它以一种不那么吓人的方式介绍了每个人。这是一个小组介绍，在这里我们可以见到全职人员，而不需要去做一些事情！我们了解了他们的背景，他们是如何进入 Sparkbox 的，他们目前在做什么，有时我们甚至有代码演示。这些也有助于打破僵局，所以一旦我们到了个人配对，我们就没有那么尴尬了。

> *别担心——还是有很多尴尬。*

![](img/15e700ce174bcc32523580b2536110c2.png)

在最初的日子里，我最大的收获是:天哪！我要学的东西太多了。我身边有一个如此出色的团队，我必须弄清楚如何真正坦诚地面对我不知道或不理解的事情。在这样一个支持仍在学习的人的环境中，这对我来说是最好的地方！有时承认我迷失了或者我不明白仍然是一种挣扎，但是我正在努力。

还有，Git 很难。🤯
