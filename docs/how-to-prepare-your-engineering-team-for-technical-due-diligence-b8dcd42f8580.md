# 如何让您的工程团队做好技术尽职调查的准备

> 原文：<https://levelup.gitconnected.com/how-to-prepare-your-engineering-team-for-technical-due-diligence-b8dcd42f8580>

![](img/3d8580e8e8be3243e53a9b672ee9e0a4.png)

如果你在一家成功的初创公司工作，你的团队将需要为融资或并购做准备。作为这个过程的一部分，你的团队可能需要与第三方尽职调查公司合作，研究你的技术，以确保对他们的客户和你的潜在资本合作伙伴或收购者来说，这是值得的投资。

在本文中，我将详细介绍我们为确保 ParkHub 在技术尽职调查方面取得成功而采取的步骤，以及这些步骤如何让我们以 1 亿美元的价格被收购。

需要注意的是，每家公司都是不同的，因此运营方式也不同。在一些公司中，只有首席技术官参与这一过程。在其他情况下，你最资深的工程师会帮助你构思故事。很可能大多数公司会介于两者之间。我的希望是给其他人提供建议，让他们能够在自己的技术框架内找到合适的方法。

**关注你的回复**

回购管理在尽职调查中非常重要，因为他们希望了解公司的技术足迹。对这些输出进行组织可以让你对它们有一个清晰的了解。这就引出了几件事，确保你能更好地为这种情况做好准备。

*   确保每个回购都有清晰的描述。不要因为太激动而没有点击“创建回购”就决定跳过这一步。这可能会导致未来的大量额外工作。
*   归档旧的、废弃的技术和示例项目，除非它们服务于你准备投入的目的。
*   使用包许可验证技术来确保每个应用程序中使用的所有包都能够在您公司的用例中使用。

**格外关注你的代码库中最重要的部分**

重要的是通过文档和流程图清晰地表达出来，以帮助指导您完成代码中最技术性的部分，尤其是“赚钱”的部分。假设您的企业接受不同类型的交易，构建定义这种行为的流程图将允许第三方尽职调查公司了解并深入研究具体问题。我们在 ParkHub 使用 [Lucidcharts](https://www.lucidchart.com/pages/landing?utm_source=google&utm_medium=cpc&utm_campaign=_chart_en_us_mixed_search_brand_exact_&km_CPC_CampaignId=1457964857&km_CPC_AdGroupID=57044764032&km_CPC_Keyword=lucidcharts&km_CPC_MatchType=e&km_CPC_ExtensionID=&km_CPC_Network=g&km_CPC_AdPosition=&km_CPC_Creative=354596046394&km_CPC_TargetID=kwd-84176206937&km_CPC_Country=9027284&km_CPC_Device=c&km_CPC_placement=&km_CPC_target=&gclid=Cj0KCQjw_4-SBhCgARIsAAlegrWzVv0tWTBMV2-e-oVYesBZZNjtYBTHNA3o6EXrlXMxYdQP4Urc4rgaAi-YEALw_wcB) ，但还有许多其他选择。

**记录您团队的发布周期**

你的团队如何交付软件？有时你的部门可能有多个团队发布不同的软件。你的团队遵循任何分支机构命名惯例吗？你在代码提交前运行测试了吗？分支机构一旦合并会怎么样？如果在部署过程中出现问题，您的回滚策略是什么？所有这些事情(以及更多的事情)都需要被记录下来，并且团队中的每个人都应该能够访问到。

**在构建应用程序时使用可重复的模式**

在尽职调查过程中的某个时候，你会被要求钻研你所有的代码或者他们选择的代码。如果你没有准备好，这绝对会让人紧张。使用可重复的文件夹结构、编码指南等。，将使代码审查变得轻而易举——不仅仅是对你，对所有跟随的人都是如此。这不仅有助于尽职调查过程，还有助于提高开发人员的满意度和交付速度。

**为你的应用程序生成测试覆盖报告**

即使您的覆盖率没有您希望的那么高，对您的团队来说，保持测试覆盖率的状态也是非常重要的。这些文档也有助于讲述关于您公司的故事，并让第三方公司了解开发团队中是否有任何差距。如果测试覆盖率不高，也许你有人员配备的问题？也许你的团队一直忙于发布关键软件，测试并不是优先考虑的事情。

**构建整体架构图**

整体架构图有助于实现几个目的。对于 onboarding，它可以帮助一个新的开发人员获得“土地的布局”,并理解他或她可能关注的地方，以及它如何影响软件的其他部分。当你带领投资者踏上每一个软件、服务、应用编程接口等的旅程时，它也可以帮助指导他们。，因为毕竟他们会要求您带他们踏上旅程，所以要做好准备。

**为您的每个回购包和 docker 映像生成安全和开源策略**

对于任何公司来说，安全性都非常重要，尤其是对于试图融资的公司。随着公司的发展，其基础架构的安全性需要不断提高。

开始保护你的平台的一个方法是确保你正在扫描你的包和图像。这将指出任何需要解决的高风险问题。请注意，这不是你唯一应该关注的安全问题。确保您也考虑到了基础设施的安全性。定期进行渗透测试将有助于识别您的平台可能存在的任何安全风险。

为您的团队开发一个开源软件策略来实现，将有助于为您的客户提供更安全的 API 平台。我们应该使用什么样的包装？我们不应该使用什么样的包装？他们的执照是什么？等等。当你创建自己的问题时，一定要问这些类型的问题。

**灾难恢复计划**

如何从数据库故障中恢复？你如何备份你的数据库？你在多少地区主持？制定一个计划来解决这些问题对你的公司来说是一个巨大的优势，也是任何成熟公司都必须具备的，无论是初创公司还是财富 500 强公司。在尽职调查期间，能够详细解释这一点并回答有关灾难恢复的问题至关重要。

**记录您团队的入职协议**

记录您团队的入职协议极其重要，因为它定义了您可以多快、多高效地招聘新员工来提供价值。记录第一天应该是什么样子，他们应该参加什么项目，他们的联系人是谁，如何设置他们的本地测试环境等。指向此文档将为您和您的团队节省大量时间和金钱。投资者关心你的入职过程，所以不要让它错过了。

**深入了解为您的团队创建的角色和职责**

这不仅对于为你的团队成员建立职业发展道路很重要，而且定义清晰的角色和职责可以让投资者了解团队是如何运作的，并发现是否有任何差距需要填补。当人们听到这些时，他们会感到害怕，但这通常不是一种“办公室空间”的情况。

**记录你的证书，确保它们是最新的。**

确保你准备好详细讨论经营你的企业所需的任何证书(如果有的话)。在 ParkHub，我们每年都需要获得 PCI 认证。

**清晰表达未偿还的技术债务**

在开发过程中经常要进行权衡，这可能意味着增加更多的技术债务，以后再偿还。重要的是要跟踪这些债务，以告知你未来的投资者你软件的真实情况。作为个人偏好，我喜欢默认尽可能以正确的方式来做，只有当机会成本权衡对扩大我们的业务最有意义时才会动摇。这个决策通常不是在真空中做出的，通常包括多个利益相关者。以这种方式做事，可以更好地管理需要支付的技术债务。

**总之**

为参加尽职调查会议做好准备非常重要，对自己的技术有信心将让你的公司获得进入下一阶段所需的资金。
