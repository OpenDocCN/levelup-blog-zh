JavaScript的优点——编码最佳实践

JavaScript是世界上最流行的编程语言之一。它可以做很多事情，并且有一些领先于许多其他语言的功能。

在本文中，我们将把JavaScript代码样式中好的部分用于格式化，并看看JavaScript的其他好特性。

代码样式

我们应该避免偶尔危险但有用的特性。

如果我们这样做，我们可以避免一大类错误。

如果我们清楚地与我们的代码交流，那么它在未来就不太可能出错。

阻碍

应该使用一致的风格，以便我们能够容易地理解它们。

例如，我们应该缩进并把每个语句放在它们自己的行上。

我们应该始终使用块来包围任何语句。

例如，不写:

if (a)  
  b( );

我们写道:

if (a){
   b( );
}

如果我们有:

if (a)  
  b( );  
  c( );

那么看起来是这样的:

if (a) {  
  b( );  
  c( ); 
}

但这意味着:

if (a) {  
  b( ); 
} 
c( );

因此，即使没有必要，我们也应该用花括号将块括起来。

JavaScript有块范围的变量，所以我们应该在第一次使用它们之前声明它们。

看起来像错误的事情

其他看起来像错误的事情包括在if语句中使用赋值:

if (a = b) { ... }

我们可能想写:

if (a === b) { ... }

Switch语句

我们应该永远不要忘记在一个switch语句中的每个case后面加上一个break。

让他们失败是个坏主意。

有时我们可能想故意这样做，但大多数时候，我们可能并不想这样做。

所以我们应该总是写:

switch (val) {
  case 1: {
    //...
    break;
  }
  case 2: {
    //...
    break;
  }
  //..
}

全局变量

应该尽可能避免全局变量。

我们可以使用内置于JavaScript标准库中的已有代码。

然而，我们不应该定义新的方法或者通过替换或添加新的方法来覆盖现有的方法。

如果我们在脚本中声明变量，我们可以在块或闭包中声明它们。

还有，我们可以考虑用模块把变量私有。

美丽的容貌

JavaScript有一些很棒的特性可以简化编写程序。

它们包括作为一级对象的函数。它们可以作为参数传递给其他函数，也可以赋给变量。

而且，它们可以有自己的属性。

这允许它们被用作回调。

动态对象可以为我们所用。

我们可以定义一次性的对象。因此，我们可以把它们当作字典使用。

对象可以直接从其他对象继承成员。

同样，数组文字对于定义数组也非常方便。

我们可以不使用任何类或构造函数来定义数组。

让pi有自己的常数也是一大特色。

在JavaScript中，一个单词不能同时用作保留字、参数或函数名。

这使得我们的代码更容易理解，因为它们不能有多重含义。

有了let和const，我们就有了块范围的变量。这样，他们总是被限制在街区内。

所有这些特性都允许我们简化代码，这很好。

结论

JavaScript有很多好的部分。

因此，我们应该尽可能多地使用它们。

它们包括对象和数组文字、块范围的变量、缩进和块的花括号。

像使用其他对象一样使用函数是我们应该利用的另一个重要特性。
