# 你没有被录用的 6 个原因

## 过度自信和技能不足

![](img/5fbb08e9a3f1bbd629d78a3fdd91c20f.png)

照片由[玛丽奥拉·格罗贝尔斯卡](https://unsplash.com/@mariolagr?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 拍摄

我刚开始的时候也是这样的人。在参加了 Udemy 课程，看了一些教程，并创建了一个普通的 todo 应用程序后，我觉得我实际上在我的旅程中取得了一些进展，离找到一份工作只有几个面试了。这是自学成才的开发人员的缺点之一，你会变成你所知道的最聪明的白痴。

没有人指导我，我只是从我的错误和成功中学习，没有方向。

我真的相信“软件工程师/开发人员”这个头衔开始失去它的真正意义，因为每个拥有 MacBook pro 和 Github 账户的人都这样称呼自己。即使他们不知道自己在做什么。

当我找到第一份工作时，我很快意识到我对软件工程的了解是多么的少。冒名顶替综合症占据了我的生活。没有大学学位或丰富的经验，我一直觉得我不配有今天的地位，即使别人肯定了我的才华和能力。

我很感激我有一个经理，他不怕因为我写 sh*t 代码而叫我出来，或者当我不小心把它推到项目的主分支时。由于性格差异，我们并不总是合得来，我不会说他是新手程序员中最好的，但他确实帮助我成为了一名更好的开发人员。

他给我讲了我不知道的，我需要学习的，我不爱听的。例如，当我第一次被雇佣时，我根本不知道什么是大 O 符号。我几乎不知道如何大声地发出 Axios 请求！

我知道他想给我最好的，因为他真的花了很多时间帮助我，并确保我理解，以便我可以变得更好。我父亲就是这样。他不会生我的气，而是生我的气，因为他觉得我的错误妨碍了我发挥最大的潜力。这和体育迷在他们最喜欢的球队的球员投篮不中或失球时对他们大喊大叫是一个道理，因为他们知道他们可以做得更好。

因此，在你对“饱和”的就业市场或招聘人员、经理或其他人感到不安之前，问问自己，“我真的是市场的资产吗？”，或者“我如何运用我的技能解决业务问题？”。或者更好的是，“我可以自信地提出 Axios 请求吗？”

软件工程 90%是解决问题，10%是编码。编写代码只是工具，但理解计算机科学、数据结构、算法和其他与你选择的编程语言相关的库和框架将帮助你脱颖而出。

我建议任何正在学习 web 或软件开发的人看一看[哈佛的 CS50 —计算机科学导论](https://pll.harvard.edu/course/cs50-introduction-computer-science?delta=0)，这是一门免费课程，它将向你解释计算机是如何工作的，这样你就可以用更有效的方式解决问题。

如果你真的想更进一步，获得那些大型科技工作，我强烈推荐 [AlgoExpert.io](https://algoexpert.io) 。它不是免费的，我不相信它不应该是诚实的。如果你**真的**想成为一名“真正的”软件工程师，尤其是作为一名自学成才的“无人指导”开发人员，这是一个一站式商店。

# 糟糕的招聘者

当你寻找第一份软件工程工作时，招聘人员是很好的合作伙伴，但是像每个行业一样，你也有一些害群之马。

我没有什么疯狂的故事，但是我知道很多招聘人员根本不知道如何招聘软件工程师。

他们会看到我只做 web 应用程序，然后给我发一份结构很差的职位描述，严格要求“10 年以上的 C++”。

有人告诉我，他们认为我懂 Java 是因为我有 JavaScript 经验。

去想想。

# 信心不足

![](img/3347586c46f1062a062f48af766a6bb2.png)

照片由 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上的 [krakenimages](https://unsplash.com/@krakenimages?utm_source=medium&utm_medium=referral) 拍摄

在意识到自己有多愚蠢之后，甚至在了解了成为一名真正的软件工程师的必要先决条件之后，不自信是另一个存在于你头脑中的恶魔。经验会减轻这种感觉，但它会一直存在。

不自信会让你失去很多钱，因为你太害怕去申请你在谷歌或 LinkedIn 上看到的那份工作。作为一名软件工程师意味着找到一个解决方案，而不知道如何立即解决它。说到你在招聘启事上看到的技能和要求的清单，这些对很多雇主来说都是很好的，但是在很多情况下，即使你没有打中每个复选框，他们也会考虑你。

这就像有人去医生的办公室，期望他或她不做任何测试就能立即给你一个准确的诊断。如果你胃疼，可能是因为你需要放屁，或者你可能患有肠易激综合征。不要因为不知道某事而感到内疚。我总是自豪地告诉人们，我只是目前不知道一些事情。

当你对自己不知道的事情诚实时，你会得到准确的指导，人们会尊重和信任你。没有哪家公司愿意花 6 位数的钱在一个假装什么都懂的开发商身上。一天下来，你会看起来像个小丑。

## 怠惰

![](img/eb00d7c86bce8694188ac3d20a544229.png)

由 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上 [Khushbu hirpara](https://unsplash.com/@khushbu_22?utm_source=medium&utm_medium=referral) 拍摄的照片

有时候你太习惯于做同一种工作，或者长时间做一个项目，你开始感到厌倦。你开始减少花在编码上的时间，因为你把它和你日常工作中单调的任务联系在一起。

非常熟练的工程师会抽出时间学习新技术，并在个人项目中练习编码。许多雇主喜欢看到开发人员在工作之余从事个人项目。这表明你真的热爱你所做的事情，你想继续做好它。我并不是说把你所有的空闲时间都花在编码上，但是开发一些小的应用程序放在你的文件夹里，或者为你真正喜欢的项目做些贡献，这对于保持你的动力是非常好的。

如果你喜欢某个电子游戏，学习如何为它制作 mods！我个人很喜欢侠盗猎车手系列，我喜欢看社区开发者的作品。你也可以通过这种方式从其他高级程序员那里学到伟大的编程原则。

无论你做什么，不要落后。技术以及人们想要和渴望的东西每天都在变化。

**但是不要忘记在做的时候要开心！**

## 糟糕的简历

我无法告诉你我因为简历格式不正确而错过了多少次面试。信不信由你，当我试图用 Photoshop 从头开始做的时候，使用自定义 PDF 文件把我搞砸了。

我知道你可以用它制作漂亮的简历，但我没有足够的技能为 ATS(申请人跟踪系统)正确格式化简历，ATS 是一个根据格式、内容和关键字帮助过滤简历的系统。

> "你会变成你所知道的最聪明的白痴"

我曾经对自己说，“当我向 ATS 提交简历时，招聘人员会看到什么？”。我也去网上做了免费扫描，这是我得到的照片。

这些机器人能够阅读你的内容是非常必要的。招聘人员每次发布职位都会收到数百甚至数千份申请，因此他们使用这些工具来加快招聘过程。想象一下，要采访 534 名开发人员，其中 80%以上的人甚至不能提出 Axios 请求！招聘人员会浪费 80%的时间去面试那些一开始就不合格的“开发人员”。

**精神错乱！**

这些机器人节省时间，让你更快被雇用！因此，确保你使用的简历工具能够针对这类系统正确编排你的内容，这样你就能被招聘者看到。

我会推荐 Zety，这是一个简历生成器，我曾用它来找技术类的工作。

## 态度恶劣

事实是，没有人欠你一份工作或职业。在一天结束的时候，世界上的每个人都会为自己的最大利益着想。这很有道理。

工程师可能是一家公司最大的投资之一。你要价超过 6 位数。他们需要知道他们将获得投资回报，因为这是他们保持活力，让你继续就业的方式。

面试时过于自信会让你听起来像个混蛋，很难共事。从我个人的经验来看，和一个混蛋一起工作只会扼杀工作文化，而且真的会影响团队的运作。我很感激在科技领域我从来不需要处理太多这样的事情，但是在其他行业工作给了我不必要的经历。这是我离开零售和酒店行业，投身科技行业的主要原因之一。

工程已经够紧张的了，没人想处理这个问题。做一个“永远正确的先生”不会让你在这个领域走得很远。

## 不切实际的薪水期望

你的第一份科技工作可能会有超过 6 位数的薪水。这些故事中有许多是真实的，但可能会误导人。

这些 6 位数的击球手中有许多是开发人员，他们能够在几个月内每天花很多时间来快速推进他们的学习。如果你没有足够的时间和纪律来实现这个目标，那就不会那么容易，而且会花更长的时间来实现你的目标。

例如，认为你可以在 3-6 个月内从“零英雄”成为 Hulu 的高级软件工程师，同时每天只花 1 小时学习，每周 3 次，这是非常不现实的。

你有时可能不得不接受较低的起薪，这并没有错。其实对于没有经验的开发者，我是非常推荐的。它降低了你的入门门槛，会让你更快地获得经验。你做的每一份工作，对下一个雇主来说就越有价值。

那些 6 位数的薪水不是随便给谁的。

# 最后的想法

软件是最赚钱的行业之一。每个人都有工作。用分析和逻辑的方法处理开发。代码是工具，解决问题是你的技能。
