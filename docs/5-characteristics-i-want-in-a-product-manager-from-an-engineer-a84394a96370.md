# 我希望产品经理具备的 5 个特征(来自工程师)

> 原文：<https://levelup.gitconnected.com/5-characteristics-i-want-in-a-product-manager-from-an-engineer-a84394a96370>

你有没有和一位出色的产品经理一起工作过，他在工作中表现非凡，一起开发产品有时很有趣而且不费吹灰之力？或者，也许你已经撞了头，或者希望你的产品经理不仅仅是“告诉你要做什么”的人？我有这方面的经验，我希望一个产品经理有 5 个特征，能够真正享受与他们一起工作，一起开发伟大的产品。

**合伙**

一个伟大的产品经理最重要的特征之一就是成为工程团队以及其他跨职能团队的好伙伴。如果这种合作没有到位，或者被打破，团队将陷入“那不是我的角色/工作”的对话，创造性的工作将会停止。

良好的伙伴关系意味着来自整个跨职能团队的意见得到重视和鼓励。让工程师尽早参与路线图计划或设计工作室会议可以增加协作，工程师应该让产品经理参与进来，权衡不同级别的实施和权衡。像这样的合作鼓励跨职能的团队建设，产品经理应该推动尽快将最有价值的产品送到用户手中。

**通信**

产品经理是信息的中心，他们经常会在会议上或通过电子邮件或其他电子通讯方式被问到问题，这并不奇怪。这需要良好的沟通技巧来描述问题并向许多不同的受众概述他们的愿景，良好的书面和口头沟通让优秀的产品经理脱颖而出。

能够向内部或外部利益相关者提出想法以获得认同是至关重要的，尤其是在产品启动时。能够在产品的原因或价值主张上指导团队的产品经理通常会带来更好的结果和来自公司内外的更多投资。我还见过产品经理共同领导或领导用户访谈，以了解最终用户和客户的需求。另一方面，良好的沟通技巧会带来更好的信息和结果。

**好奇心**

软件工程中我最喜欢的一个方面是有几乎无限多的东西要跟上或学习。行业发展迅速，最新的技术和市场机会也在不断变化。一个优秀的产品经理能够适应这些变化，并且知道这些变化是需要降低的风险，还是应该抓住的增加市场份额的机会。这是产品管理最令人愉快的方面之一，尤其是对于那些对创业感兴趣的人来说。这也符合对你正在开发的产品有真正的兴趣。构建一个没有人，甚至是产品经理，关心的产品是非常令人沮丧的。对于伟大的产品经理来说，最重要的是挖掘产品的细微差别，并推动产品发展的极限，以获得越来越大的商业和用户价值。这种好奇心和驱动力也可以延伸到其他方面，伟大的产品经理拥有所有或部分这些。

开发过程中的好奇心告诉我，产品经理正在与工程师合作，并希望了解该过程，以便他们可以知道工作如何在队列中移动，以及他们如何帮助优化该过程。有时需要另一个团队成员的提问来发现工程师已经习惯的不必要的步骤或障碍。渴望帮助改善团队、流程，帮助他人在知识或职业上成长，这是一个伟大产品经理的体现。

**倡导改变团队需求**

跨职能产品团队的需求在不断变化，目前的需求可能与一两周前不同。伟大的产品经理认识到这一点，并会努力预测努力发布新版本、规划即将到来的工作、收集用户反馈和解决技术债务的潮起潮落。

因为产品经理通常是出席许多会议的团队代表，所以对他们来说，推迟不合理的时间表并倡导结果而不是产出是至关重要的。上层管理人员很容易给团队施加压力，让他们交付一个他们的竞争对手拥有的特性，而不理解为什么这个特性在产品中有意义或没有意义。优秀的产品经理可以讨论这个问题，在承诺不必要的截止日期之前为团队留出讨论的空间，并理解外部利益相关者的动机。

**产品愿景**

产品经理的主要角色之一就是成为产品和领域的专家。这并不意味着他们马上就知道所有的事情，但是他们应该知道如何学习更多，或者谁可以为团队或他们自己回答问题。

一个优秀的产品经理可以计划路线图上的下一步，并无情地优先处理待办事项中最有价值的工作。无情地确定优先顺序是他们带给团队的最有价值的技能之一，他们可以做到这一点，而不会受到单一利益相关者或他们老板的严重影响。理解什么时候进行实验、承担风险以及从任何失败中学习是他们的专长，他们寻找机会定期从用户那里获得反馈和有效的学习。
