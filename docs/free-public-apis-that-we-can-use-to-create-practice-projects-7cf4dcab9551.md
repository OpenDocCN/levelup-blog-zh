在软件开发领域，熟能生巧。所以要尽可能多的找方法练习编程。

有了免费的公共 API，我们可以通过创建使用这些 API 的应用来练习编程。

在本文中，我们将研究一些可以使用这些 API 的实践项目想法。

# 动物照片应用

我们可以用一些 API 轻松创建动物照片应用程序。他们为我们提供了可爱动物图片的 URL，比如狗和猫，我们可以用它们来创建应用程序。

一些 API 包括 [Cat API](https://docs.thecatapi.com/) ，它需要一个 API 密钥才能使用。

[狗 API](https://dog.ceo/dog-api/) 是完全开放的，为我们提供狗及其品种的 URL 数据。

它们也是 [RandomCat](https://aws.random.cat/meow) 、 [RandomDog](https://random.dog/woof.json) 和[random fox](https://randomfox.ca/floof/)API，分别提供猫、狗和狐狸的动物照片。

[始别。网上](http://shibe.online/)为我们提供柴犬狗、猫或鸟的随机图片。

# 动漫 App

我们可以用 [AnimeNewsNetwork](https://www.animenewsnetwork.com/encyclopedia/api.php) API 创建自己的应用程序来显示动画数据。它为我们提供 XML 格式的动漫新闻。

这实际上很好，因为它让我们有机会使用库将 XML 转换成 JSON，以便在我们的代码中使用它们，从而在这个过程中学习更多关于 JSON 和数据的知识。

此外，还有 [AniList](https://anilist.gitbook.io/anilist-apiv2-docs/) API，它为我们提供了一个需要 OAuth 认证的 GraphQL。

我们都可以对这两者进行一些练习，以便为使用 OAuth 或 GraphQL 的任何工作做好准备，OAuth 或 graph QL 是分别用于身份验证和构建 API 的越来越常见的技术。

GraphQL 很有用，因为它让我们可以控制我们希望在响应中返回哪种数据。REST APIs 很难实现这种控制。

如果你是吉卜力工作室动画的粉丝，比如《龙猫》系列，还有一个吉卜力工作室 API，它可以在一个 API 中为我们提供所有的数据。

# 博物馆应用程序

我们可以构建自己的应用程序来显示来自博物馆的文本和图像。有一个[国立博物馆](https://www.rijksmuseum.nl/en/api) API，让我们可以通过一个 API 访问他们的艺术收藏数据。

它需要一个 API 密钥来访问，所以我们可以毫不费力地使用它。

还有[哈佛艺术博物馆](https://github.com/harvardartmuseums/api-docs) API，让我们可以访问他们的艺术收藏。

如果我们想要访问图标，我们可以使用 [Iconfinder](https://developer.iconfinder.com/) API 通过注册一个 API 键来访问它们的图标。

# 图书应用程序

构建一本书很棒，因为我们可以练习显示书中的数据。我们可以使用 [Google Books](https://developers.google.com/books/) API 来访问他们的大量藏书。

它还使用 OAuth，这样我们就可以练习使用 OAuth 来访问 API。

此外，我们可以从[开放库](https://openlibrary.org/developers/api) API 获取图书数据，该 API 包含 XML 或 JSON 格式的数据。我们可以更改用于访问数据的查询字符串的格式，这样我们就可以练习构建查询字符串并使用它来访问数据。

# Trello 应用

Trello 是简单易用的任务跟踪器。我们可以通过创建 power-ups 为其添加我们自己的功能，power-ups 是使用 [Trello API](https://developer.atlassian.com/cloud/trello/guides/power-ups/your-first-power-up/) 根据我们的喜好定制其功能的插件。

通过从我们自己的 Trello 帐户的仪表板设置功能，Trello API 允许我们做任何我们允许的事情。

# 谷歌分析应用

谷歌分析从我们的网站收集数据，因此我们可以跟踪即将到来的流量，并从中获得一些见解。

为了使这个过程自动化，我们可以使用 [Google Analytics](https://developers.google.com/analytics/) API 来访问显示在我们自己的程序中的相同数据。

然后，我们可以使用 API 编写自己的程序，根据自己的喜好定制 google analytics 的功能。

# 假日应用

[日历索引](https://www.calendarindex.com/) API 为我们获取世界各地的假期和工作日。

数据可以通过 API 键访问，因此我们可以轻松地获得数据并在自己的程序中使用它。

# 结论

有了这些 API，我们可以用它做很多项目来练习。如果它给我们提供了有用的功能，我们甚至可以自己使用它们。

仅仅因为它们是实践应用，并不意味着如果它们有用，我们就不能使用它们。
