# 行为面试

> 原文：<https://levelup.gitconnected.com/the-behavioral-interview-9be3a54ea01d>

![](img/3b94c2b2bccdae16938809b0afce7652.png)

行为面试是 BigTech 面试环节的最后一个环节。顾名思义，这不是技术面试。而是把重点放在你之前的经历上。你在那里表明你知道如何行动和沟通。它类似于系统设计轮，因为它对高级候选人更重要。

*你可能会想，我不需要准备就可以谈论我的经历*。你就错了。不要低估这次面试，否则你会惨败。行为面试不是一次非正式的聊天，在那你可以回忆你以前的经历，最后开怀大笑。有明确的预期。

我要结束我关于面试的系列，谈谈这种面试形式。

# 行为面试的结构

行为面试围绕开放式问题展开。网上有很多很多[样题](https://igotanoffer.com/blogs/tech/amazon-behavioral-interview)。但是，它们听起来很相似:

*   告诉我一个你影响很大的项目
*   告诉我你是如何解决多个团队之间的冲突的
*   告诉我你不得不快速做出决定的时候

请注意，这些问题关注的是你之前的*经历，而不是假设情景。理由是，通过听你已经做过的事情，更容易理解你是如何思考的。*

这些问题模糊不清。然而，这并不意味着你应该首先想到什么就回答什么。这样做是被拒绝的原因。相反，规范的建议是使用[星](https://interviewsteps.com/blogs/news/amazon-star-method)方法。虽然这是一个正确的起点，但我们可以在此基础上做得更好。请允许我解释。

# 明星是有缺陷的

![](img/6999f22c01fbe7b3349a760d931be89f.png)

这个响应可以更短

星号(*情境*、*任务*、*行动*、*结果*)是一种组织你对任何行为问题的答案的方式。你开始解释背景。然后，你谈谈你的任务。你继续你的行动，最后，你用这些行动的结果总结它。使用这种结构可以确保你讲述一个连贯的故事。

有一个问题，它与重复有关。在实践中，*情境*和*任务*经常会明显重叠。如果你严格地遵循这个方法，你最终会把同样的事情说两遍。这听起来可能无伤大雅，但是你不能无效率地使用你有限的时间。

在这些采访中，时间是你的敌人。例如，在 Meta，它们持续 45 分钟。请注意，这包括介绍和你的问题。你不会有足够的时间来谈论你做过的所有美好的事情。重复自己留给你更少的时间来提供有用的信号。会让面试官的评价更难，减少你的机会。

我更喜欢用精简版的 STAR。姑且称之为 PAR: *问题*，*动作*，*结果*。

# 使用 PAR

假设你得到了上述问题中的一个。你有备而来，想按照这个方法来回答。每个部分包括哪些相关的位？

## 问题

这一部分是关于给出上下文。你在哪里，什么时候，你的角色是什么。让它又短又脆。跳过任何与你的职位无关的细节，这些细节对于了解情况是不重要的。

之后就是描述问题的时候了。记住，你必须让不在场的人也能理解。谈谈时间表。如果你用一些 KPI 来衡量问题，这是值得一提的。

简洁点！如果你用不相关的细节让面试官厌烦，他们会感到沮丧。练习说最少可能的话，这仍然足以理解发生了什么。

## 行动

你搭建了舞台，现在是重要的时候了。你*做了什么*？不用特定领域的上下文或内部术语来解释。关注你做了什么*。行为问题不是谈论团队的时候。他们希望雇佣的是你，而不是你的团队。确保这一点没有歧义。*

你的回答不要太宽泛。对于一个复杂的项目，你做了很多事情。你没有时间去做所有的事情。选择与问题最相关且影响最大的前三项。再说一次，你在寻找正确的细节。

## 结果

结果是你总结你的答案并得出一个令人满意的结论。你做了一大堆事情。谈谈结果。有**有**要有一些结果。如果你所有的行为都没有改变，那就再找一个例子。

理想情况下，你会有可以提及的数字。你并不总是有可靠的、精确的数据，但是你可以做出合理的估计。除非你准备好证明它们，否则不要做出古怪的声明。

根据时间，考虑包括:

*   下一步:接下来会发生什么？
*   扩展:你如何利用你所做的来扩展到更多的观众？
*   学习:下次你会有什么不同的做法？

你把这三个放在一起，这就是你的答案。参见 Will Larson 的[这篇文章](https://lethain.com/star-method/)来看一个例子。

# 建立经验组合

以我的经验来看，准备行为面试的最好方法是根据你的经验建立和策划一个案例组合。外面有太多问题了。你不可能为所有人准备具体的答案。相反，我专注于拥有涵盖多种行为的强有力的例子。这样，你可以根据情况调整他们，而不是匆忙地当场想出一些独特的东西。

![](img/4cc145450fecbcbd6682e87e481663a3.png)

成功的未来组合

亚马逊领导原则是一个很好的基础。它们涵盖了很多领域，所以你可以考虑你以前工作的很多不同方面。我有一个表格，里面有我职业生涯中精心提炼的例子。我通常遵循相同的方法来扩展这个表。

首先，我写下我的答案的草图。只要足够让我知道我想涵盖的所有要点。带着你的笔记去面试是可以的，但是不要直接照着笔记读完整个答案。一旦箱子准备好了，就该抛光了！你的第一个版本不可避免地会很烂。你可能做得太久了。或者没有把结果说得足够清楚。无论如何，你都需要坚持不懈地迭代。从你信任的人那里获得额外观点的反馈。

最后，测试一下。为自己或他人大声朗读该场景。做一次模拟面试，或者在真实的面试中使用。当你的经历保持不变时，你展示它的能力将会大大提高。

## 一些额外的想法

不要瞎编。老实说，你很可能会被抓住，面试失败。即使你通过了面试官，你也在为失败做准备。

试着在你的职业生涯中涵盖一段相当长的时间。人们希望看到你获得了广泛的经验。同时，你最近的作品是最有价值的。这更容易记住，也更能代表今天的你。你最有力的例子应该来自过去几年。

你应该准备足够多的例子。你可能会在每次面试中遇到行为问题。预计会有几个。一遍又一遍地谈论同一个场景是没有好处的。

最后，不要仅仅局限于积极的经历。我们都有失败的时候。公开谈论你的失败是展示自我反省和成长的好方法。你学到了什么，你会有什么不同的做法？不用说，不要通过责怪别人来面对失败。即使这碰巧是真的，你也向试图衡量你贡献的人展示了非常关心的一面。

# 衡量你的成果

让我们明确一点:你不能改变你的经历。你可以做得更好。可以展现自己的反思和学习能力。如果你认为你的例子太谦虚，不要气馁。

如果你没有有影响力的例子，想想如何随着时间的推移达到目的。用这个来指导你的工作。如果你正在解决工作中的问题，深入了解它的真正影响。寻找能够反映变化的指标。如果你工作出色，你迟早会表现出来的。如果你不知何故还没有燃烧你所有的动力，[这段视频](https://www.youtube.com/watch?v=PJKYqLP6MRE)来自杰克逊·加巴德。
