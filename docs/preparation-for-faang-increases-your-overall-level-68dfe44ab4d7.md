# 准备 FAANG 增加你的整体水平

> 原文：<https://levelup.gitconnected.com/preparation-for-faang-increases-your-overall-level-68dfe44ab4d7>

![](img/7ffdd565d37703b0a2c330abe437a18f.png)

在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的 [ThisisEngineering RAEng](https://unsplash.com/@thisisengineering?utm_source=medium&utm_medium=referral)

我们都知道通过工作面试是一项特殊的技能。大多数时候，我们会遇到一些不适用于软件工程师现实生活的问题。我经常听到一个建议，即使你不是在找新工作，也要每年参加 3-4 次面试。把准备工作面试作为你的日常工作值得吗？

中小型公司的面试与 FAANG 公司的面试有很大不同。问题通常与某个特定的编程语言有关，背 1000 种搬起石头砸自己的脚是那么的没用。很高兴，每个月做 1-2 次面试官就足够了，可以记住 SOLID、CAP、ACID、事务隔离级别和其他类似的问题。这种准备是否提高了你的整体水平？不会吧。

但是 FAANG 的采访呢。它们非常简单，每个人都知道他们在准备什么。

算法题可能不一样。但是，当然，在现实生活中，你是否会使用 Morris Traversal 或 EERTREE 是值得怀疑的。所以我不会试图去记忆我面对的每一个算法。不然过了一个月左右我就忘了。但是对他们中的一些人来说，记住一个关键的想法是很容易的，而且最有可能的是，你将能够在面试时重新发明它。

当我在 leetcode 解决了大约 400 个问题时，我意识到即使我不记得一些用过的算法，我仍然记得所有的模式，所以为新问题找到正确的方法需要更少的时间。此外，在这次旅程之后，您可能不会想要编写复杂度为 O(n)的产品代码，因为这很容易实现 O(n)复杂度。

系统设计甚至更适用于现实生活，尤其是如果你在工作中使用微服务架构。构建可扩展解决方案的能力是任何令人兴奋的职位的必要组成部分。而且是高级或更高级别与中级的区别。

这里的问题是，你必须在工作或宠物项目中尝试，才有你的意义和体验。死记硬背 RabbitMQ 和卡夫卡的优劣，很奇怪。但是一旦我在我的项目中使用了它们，区别和目的就变得清晰了。作为一个好处，我立刻明白了为什么选择 Avro 比选择 JSON 更好以及其他隐藏的细节。

因此，每个后端开发人员都应该学习如何构建可扩展的架构及其组成部分(负载平衡器、队列、NoSQL、监控等)。).但要深入了解它，不如自己动手做点什么，而不只是看大量的相关书籍。

最后，行为面试也被称为文化适应。看起来只有亚马逊的领导原则与市场有显著不同。所有其他公司都分享了想法。

我想说，准备这次面试是提高你作为团队领导能力的一种方式。冲突管理、促进小组讨论、反馈、决策——所有这些都是团队领导的日常事务。你读得越多，你就越能理解公司在某些情况下对你的期望。

并不是所有的工程师都知道如何成长。他们中的一些人(像我一样)需要一个计划来帮助以舒适的节奏提高技能。从这个角度来看，我会说 FAANG 面试准备做得很好。是马拉松，不是春天，但这种方式的副作用也是甜的。
