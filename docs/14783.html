想开一个企业工程博客？这里有6个主题

原文：https://levelup.gitconnected.com/want-to-start-a-corporate-engineering-blog-here-are-6-topics-to-start-with-ecdb46edc137?source=collection_archive---------18-----------------------#2022-12-23

回复这个帖子的大多数有经验的开发人员在在公司工程博客上了解到更多信息后申请了一份工作或加入了一家公司。u/implebably wrong here写道，“找到一家公司的博客帖子就像你在某家公司的面试官找到一个候选人的公开GitHub回购。”写博客是的苦差事，加上，还有做事，告诉人，规模化。如果你面临招聘的挑战，你的团队需要更多的人了解他们正在做的工作。

每个团队需要一个博客。如果你的团队还没有开始写博客，并且不确定该用什么样的想法来前进，这篇文章包含了6种类型的文章作为开始。它将主要涵盖工程博客(如标题所示！)，尽管这种结构可以应用于团队编写的各种行业博客。将这些类型视为起点，而非终点。

这些写作结构对于那些刚刚开始或重新启动他们的工程博客，却不知道下一步该写什么的团队也很有用:

1.面试和招聘流程是什么样的

自然地，你主动或被动联系的求职者会想知道面试过程是什么样的？“你可以预见到这一点，然后写一篇关于他们可以期待什么，以及他们如何才能成功的博文。会有多少回合？面试官会问什么样的问题？

示例:

  • Shopify的技术面试流程:期待什么以及如何准备
  • 我们如何改进Flipp的面试流程
  • 如何准备你的不和谐面试
  • 在GitLab面试是什么感觉:一窥招聘过程
  • 招聘人员和求职者获得更好招聘体验的5种方法

Fly.io也把这个放在他们的文档里。

这篇文章也可以包括团队如何看待一个特定职位和这个职位的理想候选人的信息。

示例:

  • 揭开网飞后端工程师面试的神秘面纱
  • 我们面试时寻找什么

你也可以写下你的团队对面试和招聘的总体看法；比如蜂巢这里有个有趣的段子。

2.我们如何建造___

这些帖子展示了工程文化和过程的样子。这个想法是公开构建:共享最近发布的一个特性的过程。选择一些你的团队感到兴奋的事情，趁记忆犹新的时候写下来！即使只有几个星期也为时已晚；人们的背景会发生变化，记忆会变得模糊。任何阅读本文的人都将了解更多细节，例如，您如何部署前端web应用程序，谁做决策，有多少团队参与，等等。

示例:

  • 我们如何构建Fly Postgres
  • 我们如何构建Hydrogen:一个用于构建定制店面的React框架
  • 我们在网飞如何编写代码
  • 我们如何建立虚拟账户
  • 创建概念API

此外，解释为什么你的团队建造了一些东西可以展示类似的思维过程，这也是求职者想要了解更多的内容:

示例:

  • 我们为什么要建账本
  • 为什么我们在硅谷之外建立核心工程团队
  • 我们为什么构建Cloudflare Workers

3.分享您的技术堆栈

向读者展示你正在使用哪种技术。

示例:

  • 为新Facebook.com重建我们的技术体系
  • Trello Tech Stack 2016
  • 优步工程技术栈，第一部分:基础

4.为什么你的团队做出了一个技术性的决定

分享您对特定技术的看法和决策过程。通常情况下，你实际上已经把这些话写在备忘录、便笺或帖子里了，你只需要把它们编辑在一起，然后放到博客上。

示例:

  • 为什么我们选择Kafka作为Trello插座架构
  • 为什么我们要离开云
  • React Native是Shopify手机的未来
  • mirrord的carcina化(或者:我们为什么要用Rust)
  • 我们使用Kotlin多平台移动设备快速跟踪我们的应用程序开发
  • 为什么不和谐正在从去锈化

5.文化和生活方式

展示在公司工作的一天是什么样的，以及特定角色的责任和范围是什么。

示例:

  • 三个后期制作工程师的一天
  • 生活中的一天:incident.io的数据分析师
  • 内容分析工程师的一天

或者，也突出一个在该公司工作的团队成员(面试或第一人称)；伟大的工程师希望和其他伟大的工程师一起工作。

示例:

  • 网飞的数据工程师——采访凯文·威利
  • Ben Fathi:我为什么加入CloudFlare
  • 我叫约翰·博恩——这就是我加入Aha的原因！

6.福音传道

分享您的行业建议和观点，展示您开发的最佳实践。

示例:

  • 明智事故管理启动指南
  • 无服务器服务器
  • 富裕简单星系的守护者
  • 保持网站平稳运行的8个网站监控最佳实践
  • 数据科学如何影响战略&不一致的创新
  • 开发React原生应用的最佳实践
