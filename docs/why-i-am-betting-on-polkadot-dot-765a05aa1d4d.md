# 团队与发展

正如营销是把一个狗屎硬币项目带到月球上一样重要，需要一个伟大的开发团队来提升一个项目，如 Polkadot。加文·伍德博士是以太坊的联合创始人，并创建了 Solidity，Proof-of-Authority consensus 和 Whisper，他在 2016 年与罗伯特·哈伯迈尔和彼得·查班一起创立了 Polkadot。

奇偶技术和 Web 3 基金会正在研究 Polkadot，他们开发了一个叫做**衬底**的框架来简化区块链的开发。

[Substrate](https://www.parity.io/substrate/) 是一个框架，允许开发人员创建他们自己的区块链，而无需关心复杂的细节，如可伸缩性、互操作性和治理问题。它运行在 Web 汇编中，使用 Rust 和 JavaScript 实现。

> 用 Polkadot 联合创始人 Robert Habermeier 的话来说，Substrate 是“一组库，用来做所有写区块链时令人讨厌的事情。”

# 可量测性

![](img/2cbfbdf0e266c880a2fb4b4004c6331c.png)

由[马克-奥利维尔·乔多因](https://unsplash.com/@marcojodoin?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的照片

可扩展性是任何区块链网络甚至任何初创企业成功的一个重要因素。分散式网络中的可伸缩性包括每秒事务数(TPS)、网络带宽和数据存储。对于 TPS，业内有一个共同的基准，大约 24000 TPS，由 VISA 设定。

但是，未来的区块链网络不仅要处理日常交易，还必须提供通信、流内容，并提供各种需要更高 TPS、存储和带宽的在线服务。

> 互联网已经解决了可扩展性问题，因此将永远是大多数不太在意隐私的人的首选。

让我们看看 Polkadot 是如何解决可伸缩性的。

Polkadot 有一个支持独立区块链网络的多链框架，称为 [**副链**](https://wiki.polkadot.network/docs/en/learn-parachains) 。这些副链由称为 [**中继链**、](https://wiki.polkadot.network/docs/en/learn-architecture#relay-chain)的主链协调，中继链通过一组验证器为整个平台提供一致性。

> 副链是使用底物框架创建的独立区块链网络。这些网络可以有自己的令牌、治理和特定用例，以吸引开发人员和特定应用程序的项目。

将有大约 100 个副链插槽将被拍卖和租赁。由于这些副链是它们自己特定应用的特定区块链，它们可以并行运行。这些副链将在它们的顶部容纳几个 DAppsDeFi 的 parachain 将拥有 DeFi 项目，NFT parachain 将托管游戏、数字资产等。**这种三层架构提供了可扩展性。**

但是还是有一些问题。

*   首先，副链槽有限
*   第二，这些插槽在经济上并不适用于所有人。

**输入** [**副线程**](https://wiki.polkadot.network/docs/en/learn-parathreads) **。**

副链享有与副链相同的技术和安全优势，但具有不同的经济模式。Parathreads 的固定注册费远低于获得 parachain 插槽的成本，但他们必须通过经济手段竞争每个区块的加入，类似于以太坊目前的工作方式。

# 互用性

![](img/6ddc634b6f58d17ef283241207f75a48.png)

照片由[普里西拉·杜·普里兹](https://unsplash.com/@priscilladupreez?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄

区块链网络中的可扩展性问题引发了另一个问题:**互操作性**。*如何？*

随着区块链网络的发展，它会吸引更多的开发人员和项目，因为安全性和联网效果都有所提高。但是因为任何网络中的资源都是有限的，所以在这样的系统中运行的费用增加了，主要的例子是以太坊煤气费用。这意味着不可能有一个*‘赢者通吃’*的区块链网络。

其次，每个区块链平台，如 Cardano、Polkadot、Ethereum 和其他平台，都有自己的哲学，这可以从它们自己对共识算法、经济激励和治理的实现中看出。因此，开发者会选择一个符合他们价值观的网络。

> 因此，区块链网络必须具有互操作性，才能相互通信，并与中央世界进行通信，以传输各种数据。

让我们看看 Polkadot 是如何解决这个问题的。

在波尔卡多，有两种桥。[基于基板的桥](https://wiki.polkadot.network/docs/en/learn-bridges)促进了 prarachain、parathreads 和其他使用基板构建的区块链之间的双向通信。奇偶校验技术引领着这座桥梁的发展。

Web3 基金会正在资助为波尔卡多建造跨链桥梁的项目。 [**夹层**](https://interlay.io/) 正在建设一座从比特币到波尔卡多特的无信任桥梁，这将为比特币在波尔卡多特生态系统中打开 DeFi 机会。Snowfork 正在为以太坊搭建一座桥梁，以促进资产和数据的跨链转移，从 ERC-20 代币到智能合约。还有其他项目如[**ChainX**](https://chainx.org/)[**Bifrost**](https://thebifrost.io/)[**Darwinia**](https://darwinia.network/)和 [**离心机**](https://centrifuge.io/) ，为各种解决方案搭建自己的桥梁。

还有一些其他加密货币项目试图连接分散和集中的世界。一个主要的是链环。

# 可持续性

![](img/3890f29fa627574ab23e0fce01f343a6.png)

[kazuend](https://unsplash.com/@kazuend?utm_source=medium&utm_medium=referral) 在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍照

拼图的最后一块是可持续性。*你如何让一个区块链网络自给自足，超越某一群开发者和创始人？*

对于一个分散的网络来说，可持续性比你想象的要重要得多。人们利用这些系统来逃避中央集权的独裁统治，因此设计良好的治理模式、经济激励和安全是必不可少的。

为了安全，Polkadot 有提名者、验证者、校对者、渔夫和各种各样的协议。让我们简单讨论一下。

[**提名者**](https://wiki.polkadot.network/docs/en/learn-nominator) 选举确认者来保护中继链。假设你有一些点令牌，你想把它们用于工作，但你不想建立一个节点。你可以选择几个验证器，然后标上你的点。奖励/惩罚将由你和验证者共同承担。

[**验证者**](https://wiki.polkadot.network/docs/en/learn-validator) 通过验证校对者提供的证据来保护中继链并提供共识。最多可以有 1000 个验证器，你需要 170 万点(3400 万美元)才能成为一个验证器。

[**整理器**](https://wiki.polkadot.network/docs/en/learn-collator) 为中继链及其特定的副链维护一个全节点。他们收集 parachain 事务并创建一个块候选。这些基本上都是处理每笔交易的矿工。

[**渔民**](https://wiki.polkadot.network/docs/en/learn-availability#fishermen) 寻找不良演员和无效状态转场。他们报告副链的任何不当行为，如果被证明是正确的，他们将获得代币，但如果被证明是错误的，他们将面临失去股份的风险。

波尔卡多特的治理非常复杂，所以我会尽量保持简单。

> 波尔卡多特有一个管理网络的委员会，每一项改变都必须经过他们。理事会可以有 24 个成员，每个成员需要大约 1100 万个点(2 . 2 亿美元)

任何人都可以向委员会提出解决方案，但有几个障碍。提议者必须把一些点作为债券，如果提案被否决，所有投赞成票的人，将失去他们的债券给财政部。任何委员会成员都可以否决这个提议，因此你失策了。万一它通过了，如果技术委员会认为它对网络有害，你猜对了，你的点就没了。

这些治理问题困扰着许多 POS 链，Polkadot 也不例外。我不喜欢 Polkadot 的治理模式，明显偏向富人。这将使链条更加集中，风险更大。无论如何，我认为 Polkadot 有一个了不起的团队和聪明的想法，可以使网络成功。
