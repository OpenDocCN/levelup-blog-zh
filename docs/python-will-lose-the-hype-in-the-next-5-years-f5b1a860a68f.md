由于数据科学的普及，Python 编程得到了大肆宣传和关注。截至目前，2021 年，我们有两个著名的编程语言 Python 和 R，分别用于数据科学和分析。但是，如果我们回到 2016 年，我们有一个著名的数据科学建模编程语言，Python。

事实上，Python 也用于 web 开发，它的 Django & Flask 框架在许多行业中被广泛使用。但是，我们有许多其他编程语言，如 Java、Javascript 及其流行的库 React，可以有效地进行相同的 web 开发。而且，今天我们也有 flutter，它在 web 开发任务中得到了很多关注。

Python 编程从来不是为了 web 开发。Python 的主要用途是统计工作和数据科学。这就是为什么今天，我们有大部分的数据科学家在他们的工作中使用 Python。甚至谷歌的员工也使用 Python 进行机器学习工作。是的，他们也使用 C++，但是大多数时候，Python 被使用。

2021 年的今天，像脸书、Snapchat、微软和谷歌这样的大型科技行业正在投入合理的时间来构建无代码自动化系统。构建无代码机器学习应用的一些最新例子有 [Google AutoML](https://cloud.google.com/automl) 、 [MakeML](https://makeml.app/) 、 [Fritz AI](https://www.fritz.ai/) 、 [Spark AR](http://sparkar.facebook.com/) 和 [Lens Studio](https://lensstudio.snapchat.com/) 。在这里，AutoML 来自谷歌，Spark AR 来自脸书，Lens Studio 来自 Snapchat 社区。

> 现在的重点是创造一个环境，让非技术人员也可以建立、训练和利用完整的机器学习模型。

人们将只需要业务逻辑和数据来制作他们自己的模型。他们不需要任何 python 编程经验就可以开始。

# Python 不是程序员的选择

Python 简单易用，但速度较慢。而且，我们确实不能做任何事情使它像 C 一样快，原因是 **Python** 是一种解释型语言，而 C 是一种编译型语言。

解释代码总是比直接机器代码慢**，因为实现一条解释指令比实现一条实际的机器指令需要更多的指令。**

**你可能参加过很多编码比赛，成功构建了逻辑部分。尽管如此，当你最终提交代码时，你已经超过了时间限制，这是非常令人沮丧的。**

**所以，很明显，一个想要成为优秀的有竞争力的程序员的优秀程序员，绝不会比 C 或 C++更喜欢 Python。**

# **无代码环境的适应**

**数据科学家的未来责任将是很好地理解业务和数据。他们不会担心数据清理、特征选择、模型选择和数据建模。完整的数据科学管道将实现自动化。哪种算法是最好的，哪些特性最适合你的算法？所有这些都将是自动化管道的工作。

数据科学家的工作就是要在更好的理解业务需求的基础上，去理解流水线，去提供更好的数据，去观察机器提供的输出。我们手动编写 python 代码并运行它们所花费的时间在未来几年将不复存在。**

**让我们来理解一个场景。当我们谈到物体检测时，这是深度学习的一部分。为了训练一个简单的面罩检测，我们必须遵循以下步骤**

*   **收集面罩数据**
*   **标记图像数据**
*   **应用图像增强过程**
*   **根据图像和您的机器配置配置对象检测模型**
*   **培训模式**
*   **推理**

**到目前为止，我们必须遵循所有这些过程来建立您的第一个人脸面具检测模型。**

**今天，我们有一个由许多公司提供的工具，如 Fritz AI，Roboflow，它可以在无代码环境中完成相同的任务。唯一需要的是图像。只需上传面具的样本图像，并做图像标签，让您的定制训练模型准备就绪。**

> **机器学习的目标是让人类生活更简单，那么；如果自动化工具可以自动化手动编码任务，为什么我们还需要花时间写代码。**

# **最后的想法**

**我们已经看到，随着我们向更加自动化的策略发展，Python 是如何慢慢失去其依赖性的。我们讨论了为什么 Python 不是程序员的选择，在 web 开发领域，我们有很多像 React 和 Flutter 这样比 Python 快得多的竞争对手。

然后，我们讨论了我们如何慢慢转向无代码环境，这意味着制作数据科学模型不需要任何 python 知识。尽管如此，只有商业知识和数据才是数据科学家的关键优势。**
