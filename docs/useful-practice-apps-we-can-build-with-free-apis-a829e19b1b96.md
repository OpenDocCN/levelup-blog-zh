在软件开发领域，熟能生巧。所以要尽可能多的找方法练习编程。

有了免费的公共 API，我们可以通过创建使用这些 API 的应用来练习编程。写完实践应用后，我们甚至可以花钱买一些 API，写出更有用的应用。

在本文中，我们将研究一些可以使用这些 API 的实践项目想法。

# 电话号码验证器

我们可以使用 [numverify](https://numverify.com/) API 来验证电话号码，以确保它们是有效的。

现在，我们不必担心拨打可能没人使用的电话号码。

它可以处理来自 232 个国家的电话号码，并且可以告诉我们电话号码所在的电话公司和位置。

# Git 应用程序

大型 Git 主机都提供了自己的 API，让我们可以自动完成需要它们完成的任务，比如拉/推代码、合并代码等。

还有 [Bitbucket](https://api.bitbucket.org/2.0/users/karllhughes) 、 [GitHub](https://developer.github.com/v3/) 和 [Gitlab](https://docs.gitlab.com/ee/api/) ，都可以让我们用它们做很多 Git 操作。

它们都非常安全，因为身份验证是使用 OAuth 完成的，授权与使用 UI 操作 Git 存储库是一样的。

# 网站/ RSS 聚合器

使用 [import.io](http://api.docs.import.io/) API，我们可以聚集来自各种网站和 RSS 提要的数据，并从一个来源获得它们。

它支持 CORS，因此我们可以直接从我们的前端应用程序访问数据。使用此 API 需要 API 密钥。

用 import.io 聚合的数据在 JSON 中返回，这样我们就可以马上使用这些数据。

我们可以用它来抓取网站和 RSS 源，并根据我们的意愿提取数据。作业可以动态运行，也可以通过使用 API 设置计划来按计划运行。

# 查找其他网站上使用的技术的应用程序

有没有好奇过我们访问的网站上有哪些技术？我们可以使用[让我们验证](https://github.com/letsvalidate/api) API 来创建一个应用程序，展示我们选择的网站上使用的技术。

它不需要任何认证，因此我们可以立即使用它们。

# API 测试应用程序

我们可以使用 [Postman](https://docs.api.getpostman.com/) API 轻松地向 API 发出我们自己的请求。它让我们通过使用 Postman API 编写程序来编写 API 测试脚本。

Postman API 可用于发送查询字符串、头，并获得响应正文和头。

# 字典应用

有了各种 API，我们可以用各种字典 API 创建自己的字典 app。

有[韦氏词典](https://dictionaryapi.com/) API、[牛津](https://developer.oxforddictionaries.com/)字典 API、 [Wordnik](http://developer.wordnik.com/) API 和[单词](https://www.wordsapi.com/) API。

只要我们限制我们的查询并注册一个 API 密匙来使用它们，它们都是免费的。

它们提供了单词的定义、发音、单词类型和更多数据。

对于单词 API，它还提供了单词的同义词。牛津字典 API 也有一个词库，单词的翻译，句子 API 来获取给定单词的例句，等等。

如果我们付费，那么我们可以访问更多的东西，比如每天更多的 API 请求

# 文件转换应用程序

[Cloudmersive 文档和数据转换](https://cloudmersive.com/convert-api)应用程序让我们编写程序来使用 API 在各种格式之间转换数据。

它们包括将 HTML、Office 文件或照片文件等常见格式转换为 PDF 格式。

我们还可以用它将多个 Office 文档合并成一个。

API 可以转换的其他数据格式包括 CSV 到 JSON 和 XML 到 JSON。

Cloudmersive 还开发了其他 API，如电子邮件验证 API、自然语言处理 API、OCR APIs、条形码扫描 API 等。

它可以获取图像，然后用 OCR API 将它们解析成可读的文本。条形码扫描器 API 获取条形码的图像，并从条形码返回数据。

# 结论

有很多 API 可以用来做各种事情，比如转换文件、条形码扫描、OCR 等等。

我们还可以使用 API 来自动化 Git 操作，比如从 repos 推送、获取和提取代码。

最后，我们可以用各种 API 验证像电话号码这样的数据。
