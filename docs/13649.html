2022年面向所有开发者的10个特别GitHub回购

采访资源，构建你自己的X，伟大的公共API列表，等等

GitHub是共享各种技术、框架、库和各种集合的头号平台。但是庞大的数据量也带来了寻找最有用的存储库的问题。因此，我决定整理这十个为所有软件工程师提供巨大价值的奇妙仓库的列表。它们都有很多GitHub明星，强调了它们的相关性、受欢迎程度和有用性。

他们中的一些将帮助你学习新的东西，一些将帮助你建立酷的东西，所有这些将帮助你成为更好的软件工程师。

1.建立你自己的X

GitHub明星:16.6万

这个奇妙的资源库基本上是一个关于如何构建你自己的技术的教程集合。这里有如何构建命令行工具、操作系统、搜索引擎、3D渲染器等等的例子。

你曾经想要创造你自己的编程语言吗？或者你自己的码头工人或饭桶？那么这就是适合你的仓库。

2.免费编程书籍

GitHub stars:25.5万

尽管这个资源库的名字是免费编程书籍，但它提供的远不止这些。它有许多不同的语言版本，包含免费在线课程、互动编程资源、习题集和竞争性编程、播客和编程园地等部分。

但是大部分都是关于编程的书籍。这真是一个了不起的收藏。

3.哦，我的Zsh

GitHub stars: 15万

Oh My Zsh 是一个社区驱动的开源框架，用于管理Zsh配置。Zsh既是交互式shell，也是许多开发人员使用的强大脚本语言。

哦，我的Zsh带有强大的插件和漂亮的主题来定制你的Zsh设置。安装并运行它需要一些工作，但是网上有很棒的教程和其他开发者的例子，可以帮助你找到适合你的设置。

4.编码面试大学

GitHub明星:23.4万

这个资源库是一个为期数月的学习计划，目的是成为像亚马逊、谷歌或脸书这样的大公司的软件工程师。它是为刚接触软件工程(需要CS知识)的人准备的，也提供了如何学习成为可靠性工程师或操作工程师的建议。

作者最初创建这个回购作为个人待办事项列表，以跟踪他的学习过程。在连续几个月每天学习8到12个小时后，他终于在亚马逊找到了他梦想中的工作，成为一名软件开发工程师。

编码面试大学将帮助你准备在谷歌、微软、脸书或类似公司的技术面试。充分利用它。

5.Gitignore:一组。gitignore模板

GitHub明星:13.9万

这个库顾名思义:有用的.gitignore模板的集合。对于每一个作为GitHub库建立的新项目，必须有一个.gitignore文件来过滤上传的内容。该文件的内容因项目和语言而异。repo包含几乎所有语言或框架的模板，比如Rails、Python、Perl、Laravel、Java等等。甚至还有一个Fortran模板！

6.系统设计入门

GitHub明星:19.8万

对于软件工程师来说，这是一个很好的资源库，它将帮助你学习如何设计大型系统。这将帮助你成为一名更好的工程师。repo为这个广泛的主题提供了一个有组织的资源集合。

因为系统设计通常是许多公司技术面试过程的必要组成部分，所以该存储库还可以通过学习指南、如何进行面试的建议、带解决方案的面试问题、用于交互式学习的Anki抽认卡集以及交互式编码挑战来帮助您准备这些面试。

7.公共API

GitHub明星:20.9万

公共API是一个很棒的免费API列表，您可以将其用于您的项目和应用程序。涵盖商业、动漫、动物、新闻、金融、游戏等各种话题。

有像猫或狗的API给你猫或狗的图片。

但是也有一些更有用的，比如Gmail API或者Google Analytics API。

这真是一个广泛的收集，所以去看看你自己。

8.命令行的艺术

GitHub stars: 11万

了解如何使用命令行经常被开发人员忽略，但是它有助于提高工程师的生产力和灵活性。这个库包含了在Linux上使用命令行的有用注释和提示。有专门针对Windows或macOS的章节，一般来说，提示适用于其他基于UNIX的操作系统。它既适合初学者，也适合更有经验的人。

虽然存储库似乎不再被主动维护，但它仍然提供了非常好的提示，可以帮助您使用命令行。而且随时可以叉回购，自己维护。

9.JavaScript算法和数据结构

GitHub明星:15.1万

这个库包含了许多流行的JavaScript算法和数据结构的例子。每个例子要么标记为初学者，要么标记为高级，以表示难度。示例包括哈希表、堆、队列、堆栈、数学、字符串、集合等等。

10.开发者路线图

GitHub明星:21.2万

最后一个知识库包含一组图表，展示了为了在2022年成为前端、后端或开发-运营工程师，可以采取的不同途径和采用的技术。虽然最初看起来有点令人不知所措，但它是一个有用的指南，说明了在这个快速变化的行业中什么是可能的和需要的。

回购每年都会更新，以反映生态系统的变化。

我希望你和我一样发现这些库很有用，并且可以利用它们成为更好的软件工程师。感谢阅读！
