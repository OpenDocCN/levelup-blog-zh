如何确定你的下一次系统设计面试

系统设计面试通常是高级软件工程师及以上级别的面试过程的一部分。这些面试是开放式的讨论，要求候选人设计一些东西，通常是受欢迎的产品或服务。例如，你可能被要求设计Twitter、YouTube或Google Docs。

当然，30-60分钟对于设计这些系统来说是远远不够的。真正的产品是由成千上万全职工作的开发人员多年来构建的。

那么，在你的系统设计面试中，你应该做些什么呢？我们可以把它分成四个步骤:

  1. 提出问题并确定范围
  2. 创建高层次的设计
  3. 深入研究几个组件
  4. 总结并讨论进一步的改进

下面我们逐一考虑一下。

提出问题并确定范围

在系统设计面试中，你可能犯的最大错误是没有先问问题就直接进入设计。因为系统设计面试问题故意含糊其辞，所以您需要与面试官合作来确定设计范围，并验证您可能有的任何假设。

你可能会问一些或所有以下问题:

  • 哪些特征最重要？(例如，设计Twitter并非易事，所以面试官可能会告诉你只关注新闻提要。)
  • 日活跃用户有多少？(重要的是要知道这是一个面向小型初创公司的v1产品，还是一个拥有数百万用户的成熟产品。)
  • 我能否利用现有的云提供商，如AWS、GCP或Azure？(有时他们希望你从头开始构建一些东西，但大多数时候，你可以说你将使用现有的基础设施，如负载平衡器(ELB)、自动扩展组、文档存储(S3)、消息队列(SQS)、缓存(Redis)或CDN (CloudFront)。
  • 我们正在处理哪些类型的数据？(最好能知道你是在处理文本，还是在存储图片或视频作为服务的一部分。数据类型和数据结构可能会影响您对数据库的选择。)
  • 我们需要考虑【X】吗？(面试的开始都是关于建立范围，所以问你的面试官关于范围内或范围外应该考虑的各种问题。通常情况下，他们会让你从简单的方法开始，然后在面试快结束时，如果有时间的话，再去处理较难的问题。)
  • 我可以假设[X]吗？(让你的面试官知道你的假设。这将确保你们俩在同一页上。面试官也可能会根据你之前的假设在这里为你提供额外的澄清。)

一旦你觉得你对这个问题有了很好的理解，你就可以继续进行高层次的设计了。

创建高层次的设计

在第二步中，你应该和你的面试官一起设计一个高层次的系统。现在不要太深入地研究单个组件。相反，坚持对系统如何工作的高层次概述。此时，没有必要进入您的数据库模式或者您将实现什么API端点。

一开始就保持高水平，你可以和面试官一起验证你的想法。你可以从面试官那里获得一些想法，并听取他们的意见。在系统设计面试中，你应该像合作者或同事一样对待你的面试官。毕竟，如果你得到这份工作，你很可能会和他们一起工作。

重要的是要记住，即使这次面试是关于你的技术设计技能，它也是关于你的沟通技能。

你能证明为什么你使用一种类型的数据库而不是另一种吗？你能解释一下你是如何让你的应用变得可扩展的吗？当谈到一致性、可用性和分区容忍度时，你能讨论一下CAP定理中的折衷吗？你想过如何处理系统故障吗？当你的面试官给你反馈或者让你考虑一个不同的方法时，你的反应好吗？一个好的面试官会在整个面试过程中寻找所有这些信号。

在这一步结束时，你应该有一个你和面试官都同意的系统。

深入研究几个组件

现在是时候深入了。同样，你正在设计的系统很可能是一个流行产品的复制品，所以不可能在你面试的短时间内涵盖所有内容。问你的面试官下一步想关注哪里是个好主意。或者，建议几个你想更深入讨论的领域。

例如，如果设计Google Docs，当多人同时编辑同一个文档时，你会如何处理冲突？

或者，在构建YouTube时，当用户上传新视频时，实现消息队列将如何帮助解耦您的系统组件？

您已经有了概要设计，因此在面试的这一部分，您可以展示您对基础架构如何工作的了解程度。

总结并讨论进一步的改进

在面试的最后部分，如果你有更多的时间，你可以讨论进一步的优化。你可能会讨论如何处理各种边缘情况，设计中的任何性能瓶颈，或者如何随着应用程序的增长进一步扩展系统。

这是展示你体贴的好时机。

结论

以下是系统设计面试的四个步骤:

  1. 提出问题并确定范围
  2. 创建高层次的设计
  3. 深入研究几个组件
  4. 总结并讨论进一步的改进

在面试中遵循这个模板将会帮助你不迷失方向。祝你下次面试好运！
