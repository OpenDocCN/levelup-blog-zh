学习你不需要学习的东西——成功开发职业的秘密

原文：https://levelup.gitconnected.com/every-developer-should-learn-what-they-dont-need-to-learn-f9d57d1567bb?source=collection_archive---------14-----------------------#2020-07-15

软件和Web开发是一条职业道路，为开发人员提供了探索多种语言和技术的机会。根据需求和行业趋势，比较容易估算出我们需要学习什么，才能在这个发展旅程中大放异彩。但是，

学习我们不需要学习的东西，是有价值的，因为它提高了我们的生产力，更重要的是让开发人员安心。

因此，在这篇文章中，我将分享一些关于开发人员不需要学习的东西、和一种方法的建议，这些方法可以让我们逐渐成长，并作为开发人员脱颖而出。我们开始吧，

1.开发人员不需要学习编程语言的所有知识

没错。不必学习或记住编程语言中每个方法都有的每一点语法、概念、方法或参数。我们最需要知道的，是关于他们的存在。即使是高级开发人员也会参考并每天花费大量时间在网上寻找解决方案。有一个非常流行的原则叫做'80–20 '原则，它指出

这不是学习80%的东西，也就是20%的时间被使用的东西，而是学习20%的东西，也就是80%的时间被使用的东西。

所以，只学习那些构成技术基础的概念，慢慢地你会在你的基础上建立起来。注意— 研究在你作为开发人员的工作中起着至关重要的作用，但是请记住，

在网上寻求解决方案是合理的，但不从中学到任何东西是不合理的。

2.开发人员不需要精通每一种编程语言

大多数开发人员在他们的职业生涯中不会使用超过2-3种编程语言(我们没有计算任何 库 ， 框架 ，以及 工具 在那个 )。尝试学习过去被证明设计良好，现在稳定，未来将主宰的语言。我强烈推荐学习这一类的 Javascript 和 Python 。

精通多种技术肯定会通过提供一个从不同角度看问题的机会来提高您对编程基本原则的理解，但这绝对不是必须的。记住，

一个优秀的开发人员不是只知道多种语言和技术的人，而是一个能够熟练有效地利用他们所拥有的技术来编写和组织代码的人。

注意— 如果你想了解更多关于组织和编写代码的方法，我推荐你阅读我的文章“组织和编写代码的艺术:范例”。

3.开发者不需要学习新推出的语言和技术。

在当今时代，随着技术的进步，新的库和框架一次又一次地进入市场。成为一名开发人员仅仅意味着要致力于终身学习。但是，

与当前行业保持同步并不意味着学习流行的库和框架，而是要了解它们。

开发人员应该知道这个库或框架解决了哪些现有的库或框架没有解决的问题。开发者应该意识到这种技术会胜过竞争对手的情况，以及它存在的目的。

新的库和技术没有那么成熟，通常需要几年时间才能被社区广泛接受。因此，开发人员永远不应该怀疑现有技术，直接去学习新技术，因为如果新的竞争者被证明是成功的，它仍然需要几十年才能被取代。现在一个可行的解决方案，

成功的开发人员职业的秘密是:总是喜欢基于项目的方法

基于项目的 方法简单地说就是在需要学习的基础上学习东西。在手头项目的基础上学习和切换技术。因为在一天结束时，与能够处理需要不同技术的多个项目的开发人员相比，开发人员在使用一种特定语言方面表现出多少专业知识并不重要。

此外，如果某项技术过时了，不拘泥于特定的语言会降低你失业的几率。
