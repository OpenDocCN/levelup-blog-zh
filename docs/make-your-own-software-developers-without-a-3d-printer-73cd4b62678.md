# 为什么要自己做开发者？

![](img/3829bef00a8c177756b603bb3f3269ec.png)

照片由[阿兹甘·米耶什特里](https://unsplash.com/@azganmjeshtri?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 拍摄

2021 年，英国有近 50 万(实际上是 448435 人)年轻人获得了第一学位，其中 39690 人学习计算机科学，56380 人学习工程和技术，15960 人学习数学。这是一个人才辈出的地方，在一个技能竞争非常激烈的就业市场中，一个“自己成长”的模式比雇佣一个昂贵的熟练开发人员更值得考虑。然而，这不是一条容易走的路，也不是一条捷径，但是如果我们要有专业的开发者，这是一条行业需要承诺的路。

建立内部培训计划的一些原因:

*   公司对大量开发人员的需求很大，这是公开市场无法满足的。
*   产品或公司使用小众语言或应用程序，很难招募到具备这些技能的人。
*   中高级开发人员认为公司、产品或市场领域并不令人兴奋。
*   市场部门需要特定的技能和正规培训，可能是一个受监管的行业。
*   该公司享有很高的声誉，所以人们会呆很长时间，这意味着所有的开发人员都处于高级别，没有人处于更低级别的职位。
*   该公司希望从初级职位开始，提供具有明确发展路径和晋升可能性的长期职业前景。

# 让学生成为开发者的计划

有各种类型的方案可以使用。以下是我参与的项目，其中最重要的因素通常是成本、你需要多快展示收益和 ROI(投资回报)。

*   暑期实习——也就是学生在校期间的一份假期工作。
*   一年实习计划(1-1)——大学学位课程期间的一个过渡年。
*   **1 至 2 年学徒/研究生计划**——在全日制教育结束后实施。

关键是要考虑你为什么要做这个计划，它的设计以及对双方的好处是什么。如果目的是提供原始人才，让学生毕业后有可能回来就业，我喜欢一年制实习计划。2 年计划是一项巨大的投资，因此公司希望初级开发人员在公司呆一段时间，但是，你也需要现实一点，几年后很大一部分人可能会跳槽。

# 招聘和选拔

面试学生的发展职位是很难的，因为他们几乎没有经验可以借鉴或谈论。技术测试很难以标准的方式进行，因为学生们经常使用不同的编程语言。

那么，你如何着手选择合适的人呢？

答案是有一个共同的方法来寻找资质、学习能力、团队工作能力、解决问题的逻辑方法、对细节的关注，以及最重要的对技术的热情。

# 培养

虽然在职培训是有用的，但是对于一个更正式的计划来说，有必要为开发者提供一个结构化培训的计划和定义。培训可能包括以下内容:

*   公司一般入职。
*   第三方培训公司的在线自学培训。
*   来自外部第三方培训公司的特定主题的培训课程(在线或面对面)。
*   与直接角色和技能相关的内部培训课程，以及产品培训和市场意识。
*   关于特定主题或一般领域的书籍。
*   团队介绍和入职。

# 经验

目的是有一个大纲，如何让新开发人员接触到广泛的经验，以便他们能够很好地解决未来的问题和项目。这可能包括:

*   拜访客户。
*   参观其他部门。
*   与不同团队的工程人员一起工作，例如支持、开发和研发。
*   与具有不同技能和角色的工程人员一起工作，例如技术文档、开发、测试、自动化测试、集成和开发运维。
*   学习如何学习一门新的编程语言。
*   学习如何加入一个团队并快速上手。
*   学习如何完成一个项目并离开一个团队，以便有效地移交所有信息。

我最喜欢的方法结合了以上几种:

*   定期轮换到一个新的团队和项目，理想情况下，项目是非常不同的，并且使用不同的编码语言。
*   在高级开发人员的监督和指导下，进行大量非常短的、通常每月一次的 R&D 调查，创建原型，并完整地记录调查结果，作为执行简报。

# 评价

要判断某人的表现如何，有一种预定义的方法来衡量和提供绩效反馈是很有用的。这可以是非正式的每周一对一和更正式的季度评估的组合。评估可以是单个项目，也可以是多个项目的组合:

*   经理的非结构化评估。
*   特定技能的正式评分和分级。
*   将他们的工作与其他学生和现有的初级/中级开发人员进行比较。
*   对特定工作的评估——文档或软件。
*   给一组人的期末演示。
*   来自关键个人或与此人互动过的团队的反馈。
*   培训课程结束时的正式测试或测试结果。
*   完成一系列培训模块后的正式外部认证，例如来自微软、Azure 等的认证。

如果一个人被安排了一年的工作，那么他的教育背景可能会对他们的评估有所要求，或者他们可能需要一份公司的正式绩效评估，这有时可以作为他们学位的一部分。

有一个长期的导师(类似于大学的导师)可能是有用的，新开发人员可以不断参考职业建议、技术问题或项目问题。通常，这些导师可以在开发人员完成他们的正式培训后建立持续的关系。

# 结构化职业道路

有一个伟大的培训计划是没有意义的，只是为了让个人完成它，成为一个初级开发人员，但是没有为未来的发展勾勒出前进的职业道路。在职业生涯的开始阶段，关键是让个人参与进来，定期给予反馈，并通过频繁(但幅度较小)的加薪来展示进步。这种逐步增加的薪水会让初级开发人员更开心，并防止他们把新获得的技能带走。有一套正式的职称来配合工资级别是展示前进路线的好方法。拥有助理、初级、中级、高级、技术主管、负责人等头衔和级别表明公司致力于员工的持续发展。公开的职位描述，包括相应级别的技能和行为细节，有助于进行结构化讨论，并确定需要成长和改进的领域。

技能发展不应停止。能够使用在线资源，如 pluralsight、Udemy 或其他类似系统，显示了对工作人员的投资和赞赏。在事物不断变化的技术世界中，灌输一种持续个人发展的文化是至关重要的。

# 我个人的学习

一旦你建立了一个计划，在接下来的几年里重复就容易多了，适当的调整。

与 T2 当地的大学或学院建立牢固的关系是值得的。

完成一年实习的人很可能会在他们受教育的地方**传播好消息，促进你的生意。这有助于吸引未来的学生加入该计划。**

通过指导初级员工，高级开发人员提高了他们的人际交往能力，并从观察新手如何使用最新技术解决问题中受益。

一个团队中有一名实习学生是好的，但两人往往效率较低，因为当他们遇到问题时，他们更喜欢相互交流，而不是与他们的导师交流。他们也可能停留在“学生”模式，而不是专业人士的工作模式。

一次有几个分班学生，但在不同的小组，为他们提供了一个支持结构，并给予**规模经济**。这也是有帮助的，因为这些导师将能够相互合作。

一些学生开始认为他们什么都知道，并发现很难接受建议。好的很快就会改变和学习。

学生们认为，他们必须从第一天开始**编码**才能被认为是有用的，这需要反复强调信息，以确保他们专注于学习广泛的主题并获得软技能。

学生通过犯错(在安全的环境中)比被阻止犯错学得更好。

永远不要让学生或实习生接近生产环境。

记录代码和**编写紧凑的文档**和规范是比编码更难学习和提高的技能。

学生们通常不喜欢在支持团队中工作，直到他们自己做了，并且意识到这有多难。

极少数计算机科学专业的学生不会编程，但是细心的指导可以帮助他们找到一个不同的有回报的职业，他们可以成功。

一些计算机科学的学生是如此令人惊奇，以至于在第一周结束时，他们发现了用他们从未见过的语言编写的遗留代码中的问题，并且能够向敬畏的高级工程师简明地解释解决方案。

拥有硕士和博士等更高学历的人并不保证会有更好的表现或结果，但是他们会增长知识，这是有利的，通常是在非常特殊的领域。

那些资历比学位(A-levels，HND 等)低，但充满热情和激情的人也有可能做得很好。
