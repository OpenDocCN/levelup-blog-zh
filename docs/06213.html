通过编码面试

原文：https://levelup.gitconnected.com/ace-the-coding-interview-3405dbcc443?source=collection_archive---------6-----------------------#2020-11-03

在过去的五年里，我参加了上百次编码面试。我以面试官的身份主持了其中的大部分面试，在其中的15个面试中，我是一个紧张的求职者。

一开始，我害怕编码面试。在时间压力下解决算法问题不是我认为的好时机。但是，随着我作为面试官在谈判桌上积累了更多的经验，我意识到，关键在于你是否解决了问题，这是而不是和。这很大程度上与你如何解决问题有关。

在这篇指南中，我将回顾一下我所学到的能决定面试成败的一般技巧。

我们开始吧...

分析并阐明需求

通常会给你几个句子的要求来开始。很多时候，人们会马上开始编码。候选人最常犯的错误之一是，在没有讨论问题以验证他们的假设的情况下，过早地编写代码。

一个好的开始方式是用你自己的话向面试官重复这个问题。一旦你确定你理解了问题，花点时间想想边缘情况，澄清在每种情况下应该发生什么。你可以问的一些常见问题:

  • 我需要担心无效输入吗？
  • 这是我需要报道的边缘案例吗？
  • 这组输入的预期行为是什么？
  • 在这种情况下，我应该如何处理错误？

即使是最简单的问题，你通常也能找到一些问题来问。如果有大量的边缘案例需要讨论，那么值得将它们作为注释记录下来，以便您记得稍后测试它们。

用几个样本输入来检查问题总是一个好主意，以确保你的输出和面试官期望的输出是相同的。

一旦你理解了这个问题并发现了一些边缘情况，你就应该对未来有所了解。此时最好用文字描述一下你提出的解决方案。如果你能想到多种解决方案，和面试官一起讨论，权衡利弊。一起探索问题空间可以让面试官很好地了解当你接到要求时，你是如何思考和沟通的。毕竟，将需求分解成代码是我们每天都在做的事情。

有些问题有多个部分，当您解决一部分时会引入这些部分。每次给出一个新的需求，你都应该重复寻找边缘案例和提出问题的过程。此外，请留意您需要的新功能将如何影响您目前已有的功能。你需要重构什么，它将如何修改现有的行为？

应用干净的代码原则

你的代码可读吗？你是硬编码价值观吗？你的变量名和函数名有意义吗？你写的是小函数还是整个解决方案都在一个大函数里？

面试官会留意你的代码中的一些东西。有时候时间压力下写生产级代码很难，走一些捷径也没问题。运用你最好的判断力；一定要让面试官知道“我在做X，所以我可以做得很快，但如果我承诺生产，我会做Y”。

即使快捷方式很明显，或者命名很糟糕，因为你很匆忙，也要提及任何使代码更混乱或更难阅读的东西。记住面试官对你的编程能力一无所知，除了你在这一小时展示给他们的。面试官可能会认为你在面试中使用的快捷方式反映了你的工作方式。

增量测试你的代码。这将帮助您发现并纠正语法问题。如果它是一个多部分的问题，当您解决和测试新的需求时，确保您的测试仍然通过前面的部分。

不要过度设计。虽然牢记干净的代码实践很重要，但不要太沉迷于这些原则而忽略了整体问题。当务之急是解决问题；干净利落地解决它会为你赢得额外的分数。

关于编码语言的注意事项 : 你对你正在使用的语言感到舒服吗？有些人喜欢使用不同于他们日常工作中使用的语言。如果你这样做，确保你的面试语言是最新的。你不应该在面试时谷歌如何循环数组。大多数编码面试都可以让你查找文档，但是你应该熟悉基本的数据结构和操作。表现出对你选择的语言的精通能给你加分！

有条不紊地调试和排除故障

当你遇到错误和bug时，不要惊慌。这是一个展示你专业调试技能的好机会。如果有编译错误，慢慢来，阅读stacktrace，缩小出错的范围。如果没有错误，但是有意外的输出，请采用结构化的方法进行故障排除。有条不紊地打印报表，帮助你找到问题所在。一个好的方法是打印出解决方案中的内部数据结构和其他关键变量。你要避免没有任何韵律或理由的零星印刷的东西。

积极交流你的思维过程

如果你在面试中沟通不畅，即使你解决了实际问题，也会给面试的其他部分蒙上阴影。我所在的每家公司都有一个部分来评估应聘者的沟通技巧。个人不写代码，团队写。面试官想确保你是他们愿意与之共事的人，而这很大程度上取决于你的沟通能力。

简而言之，在整个面试过程中传达你的想法。面试官应该能够跟随你的思路，就好像他们是你的乘客一样。当你到达问题的要点时，你需要做决定，或者如果你遇到了障碍，通过你的谈论你如何决定走哪条路，以及你如何疏通自己。

积极的沟通也大有帮助。当你遇到一个特别困难的障碍，你似乎找不到前进的方向，试着不要沮丧。以积极的态度去接近它，当你需要帮助的时候寻求帮助。

当你编码时，这样的交流可能不是你每天在工作中做的事情。这会感觉不自然和尴尬。和朋友一起练习问题，同时说出你的想法是提高这方面的一个好方法。

虚拟面试沟通技巧:不要打断面试官。在虚拟面试中，这是很难坚持的。一旦清楚面试官已经说完了，就试着开口说话。在线通话中的轻微延迟会造成不和谐的中断。

改进的最终想法

这是做采访时要记住的很多事情。和任何技能一样，练习是提高的最好方法。

练习我描述的不同方面。和朋友或同事进行模拟面试，让他们按照上面的标准给你打分。

采访你的朋友。当你在桌子的另一边时，更容易发现错误和需要改进的地方。这个过程可以帮助你在面试中关注自己的优势和劣势。多年来，作为面试官进行面试一直是让我成为更好的面试者的关键因素。

做 leetcode 题。熟能生巧。做leetcode的目的不是记忆解决方案，而是识别不同类型的问题。当您看到一个新问题时，您将能够对其进行分类，并利用您的知识来了解要走哪条路，哪些边缘情况可能是一个问题，您需要询问哪些问题，以及您需要解决哪种性能影响。
