Javascript类型与类型脚本类型

  1. Javascript类型是动态类型，而Typescript类型是静态类型。
  2. 动态类型是可以在运行时改变的类型，而在静态类型中，我们需要根据需要定义类型。
  3. 在动态类型的情况下，我们无法控制它们，从而导致意外的行为，而在静态类型中，我们定义这些类型只是为了控制我们的代码流。

类型推断

类型推断意味着typescript本身通过确定赋值的类型来标识变量或常数的类型。如果您将鼠标悬停在函数的任何变量名上，以防您没有给它分配任何类型，您会经常发现这一点。

类型分配

我们可以在类型声明期间赋值的同时显式定义类型。例如

let x: number = 5;

这里的“x: number”就是我们所说的类型赋值。

这里需要注意的是，当你只是声明变量或常量，而没有给它赋值时，你需要使用类型赋值。例如

let x: number;

在继续下一步之前，我们应该知道的另一件重要的事情是，如果我们使用const并将初始值赋给它，而没有指定该const的类型，则typescript推断将假定该const的值为默认值。举个例子，

const x= 5;

当你在IDE中将鼠标悬停在这个常量x上时，它会显示类似这样的内容，这意味着这个x的类型是5

const x: 5;

作为类型的功能

在Javascript中，我们可以将一个函数指向一个变量例如:let combineValues = add其中add是一个函数。现在，如果我们调用combineValues()，那么它将执行我们调用的函数add()，但有一个问题，我们可以在Javascript中为变量赋值，它不会抱怨，但当涉及到typescript时，它会对你大喊大叫。

因此，为了解决这种糟糕的情况，typescript有一种叫做“函数作为类型”的东西，它提出了两种类型的解决方案，一种是灵活的，另一种是更严格和简洁的解决方案。让我们逐一检查两者。

let combineValues: Function = add;

在上面的例子中，现在typescript知道有一个变量combineValues，它持有add()，目前我们可以假设它有两个参数a、b，这个add函数返回这两个值的和。

作为开发人员，我们知道它将返回一个数字作为输出，但对于上面的示例，typescript只知道它只是一个函数，而不知道这个函数返回的是什么类型。这导致了另一个问题，你可以给任何有问题的函数赋值。

因此，为了解决这个问题，typescript有一个更严格和简洁的方法，让我们看看它是什么。

let combineValues: (a: number, b:number) => number;

现在，只有那些有两个number类型参数的函数和一个返回number的函数才能赋给combineValues变量，如果我们试图赋给任何其他类型的变量，typescript会抛出编译错误。

未知类型

unknown是typescript中的一种类型，我们过去常常在不确定要获取或要存储什么值时使用它。它听起来可能与“任何”相似，但这两者是有区别的。

在任何情况下，TS将放弃其检查，并告诉你做任何你想做的事。但是在未知的情况下，typescript将继续检查我们作为值传递的任何内容。

这意味着在没有检查类型的情况下，你不能给一个未知类型的变量赋值。
