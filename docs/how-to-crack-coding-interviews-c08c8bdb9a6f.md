# 如何破解编码面试

> 原文：<https://levelup.gitconnected.com/how-to-crack-coding-interviews-c08c8bdb9a6f>

所以，2020 年也没那么糟糕！经过大约 3 个月的集中准备，我能够为几家软件公司——Google、Rubrik 和 Goldman Sachs——安排编码面试。我有大约 3 年的工作经验，并从过去的失败中吸取了教训，我很高兴这次我决定做有计划的准备。所以，我在这里写了一篇文章，概述了我准备编码面试的策略，这可能对其他人也有用。我试着给尽可能多的有用资源添加适当的链接。

首先，花点时间(至少一个周末)进行研究是非常重要的——为面试寻找合适的学习资源，并为接下来的几周制定一个计划。这个计划可能还包括你希望申请的公司名单，以及申请的顺序(是的，顺序很重要！).

# 面试过程

大多数软件公司的面试流程大致包括以下步骤:

> 简历筛选—电话面试—(虚拟)现场面试—录用决定—录用通知

(虚拟)现场面试通常有 4-6 轮(取决于公司)，包括编码和行为两轮。编码测试旨在测试你对**数据结构和算法**的了解，在某些情况下，还包括计算机科学基础知识。对于高级职位，公司还包括几轮测试你的系统设计知识(高级/低级设计)。行为回合通常是为了确定文化契合度。

> 为机会做好准备却没有机会，好过有机会却没有准备。

# 在采访之前

准备编码面试的关键是大量的编码练习。但是在开始之前，最好先温习一下对面试特别重要的理论概念，看看主题方面已经解决/未解决的问题。这种方法与我们过去准备学校/大学考试的方式非常相似。我发现下面两本书对我的备考非常有用。这些书的好处是它们也讨论了解决问题的模式。

## **编码实践**

**有几个网站提供面试准备练习题。我更喜欢 LeetCode，因为它有一个惊人的公共集合，收集了所有三个难度级别的顶级面试问题——简单、中等、困难。我的建议是从这个精选的列表开始你的编码实践(根据你过去的编码实践/当前的舒适程度，从容易或中等开始)，然后根据你的薄弱主题转移到其他相关问题。最重要的是，练习在纸上或纯文本编辑器/Google docs 上编写代码(尽量减少错误)。**

> **[https://leet code . com/explore/interview/card/top-interview-questions-medium/](https://leetcode.com/explore/interview/card/top-interview-questions-medium/)(*链接到 medium 类别，因为*medium……)**

**准备面试有时会令人厌倦，尤其是当你不得不应付朝九晚五的工作时。嗯，在我准备的日子里，我经常看著名的情景喜剧《生活大爆炸》来放松心情。**

**回到面试，在手边保留一份最常见的数据结构和算法的列表(就像[这张](https://docs.google.com/document/d/13QBp7VCNcNL3ZNCxTs01V18uVV3LOXSDmCPOBZgq1ec/))是个好主意。这可能有助于你在实践和实际面试中快速想出新问题的解决方案。此外，不要忘记准备一个简短的自我介绍(就像[和](https://interviewgenie.com/blog-1/2017/7/30/how-to-answer-tell-me-about-yourself))，因为许多面试官在进入编码问题之前会给你一个自我介绍的机会。**

**除此之外，你还应该能够向面试官解释(如果需要的话)你简历中的前 2-3 个项目。选择最好的，因为这是一个展示你过去工作的机会，特别是突出你在这些项目中的工作所产生的技术影响。最后，列出 4-5 个问题(比如这些问题)在面试结束时问面试官。**

****一些免费的建议**
1。把事情写下来以便更好地学习，可以是保存已解决问题的 excel 表格、文档，甚至是手写笔记。
2。彻底了解你将在面试中使用的编程语言(例如 C++/Java)，至少是库函数。
3。学习基本的面向对象编程概念，如果时间允许，能够用简短的注释编写模块化和干净的代码(面试官喜欢模块化代码！).
4。行为回合很重要！准备行为问题并不需要花很多时间，但是它们可能会对你的申请产生很大的影响。使用[星形方法](https://en.wikipedia.org/wiki/Situation,_task,_action,_result)思考[常见行为问题](https://www.themuse.com/advice/30-behavioral-interview-questions-you-should-be-ready-to-answer)的答案。**

## **公司特定准备**

**我相信花一些时间去研究你将要面试的公司(它的产品，文化等等)。)，非常有助于推动你与面试官和招聘经理的对话。说明你真的很有兴趣加入他们公司。再一次，回到我们的学校/大学考试方法，这是一个好主意，看看最新的面试形式和公司问的问题类型，看看你面试的职位。这增强了你的自信，因为你知道(在某种程度上)你将进入什么样的领域(相信我，惊喜对面试来说真的很糟糕！).**

> **最新面试问题链接: [**LeetCode 讨论**](https://leetcode.com/discuss/interview-experience?currentPage=1&orderBy=newest_to_oldest&query=) ， [**CareerCup**](https://www.careercup.com/page?pid=google-interview-questions)**

**最后，在一个免费平台上进行几次模拟面试(比如 Pramp )。或者更好的办法是，请你的朋友(在你将要面试的公司工作)参加模拟面试，因为他们可以提供有价值的反馈，因为他们过去也经历过同样的过程。**

**嗯，这看起来像是面试前的大量工作，但是相信我，如果你设法完成了其中的大部分，你将能够在几乎所有即将到来的编码面试中使用它。**

# **在采访过程中**

**第一条规则是*而不是*在面试中强调出来(难道*不能强调*这就够了吗！).第二是对如何进行面试有一个粗略的想法。以下是有条不紊地解决面试问题的几点建议(摘自《T4 编程面试曝光》一书)。**

1.  **确保你理解了问题——问题被故意弄得模棱两可/开放式的。提出正确的澄清性问题，并陈述你对问题范围的任何假设。**
2.  **选择一个简单而又好的例子/测试案例，这可能会导致关于如何解决一般问题的见解。**
3.  **关注你将用来解决问题的算法和数据结构。想出几个方法，继续和面试官谈论你的思考过程(真的很重要)。**
4.  **向面试官解释你的最佳解决方案——他/她可能会要求更有效的解决方案。在找出一个完整的解决方案之前，抵制开始编码的诱惑是非常重要的。**
5.  **编写你的解决方案，同时向面试官解释你在做什么。确保您编写的代码简洁明了，具有良好的变量命名，可能还带有适当的输入验证。**
6.  **通过跟踪测试用例(来自步骤 2)来验证您的代码是否工作，以消除小错误。接下来，检查你的代码中所有的死角和特殊情况(如果时间不允许，至少口头告诉面试官)。**
7.  **最后，谈谈你的解决方案的*时间和空间复杂度*。**

**应聘者被面试问题难住是很常见的。与其放弃，不如表现出对这个问题的兴趣，并继续尝试解决它——回到另一个例子，尝试不同的数据结构，或者可能使用您正在使用的编程语言的高级功能。几乎所有的面试问题都有简短(且优雅)的答案——如果你开始写大量代码，你可能会走错方向。**

# **面试结束后**

**该是你申请的公司根据你的面试反馈做出招聘决定的时候了。根据公司的不同，这个过程可能需要几天或几周(毕竟，这是一个非常艰难的决定！).你的招聘人员可能会回复以下情况之一——公司决定暂时不处理你的申请。在这种情况下，你应该向招聘人员寻求一些有价值的反馈(这样你就可以在将来改进这些方面)和冷静期(重新申请前的持续时间)。**

**另一种情况是公司决定向你发出邀请。好了， ***祝贺*** 的走到这一步，但是还剩下一步——薪资谈判。这很大程度上取决于你目前的薪酬，以及你可能向公司提供的任何竞争性报价。在任何情况下，你都不应该羞于询问基于你的研究的一些合理的数字(例如，关于[水平。参考消息](https://www.levels.fyi/Salaries/Software-Engineer/India/))。最后，你会收到录取通知书。唷！**

**这基本上概括了整个过程，但最终，重要的是你加入了你最喜欢的公司。**

> **面试很难，但是对面试过程的了解和心中的计划会让面试变得容易得多！**
