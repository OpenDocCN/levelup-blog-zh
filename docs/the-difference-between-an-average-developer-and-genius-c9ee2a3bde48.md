## 你就不一样了。

开发人员的生活并不容易，不完全是键入代码，但开发人员是问题解决者，也是不断学习的个体，他们需要不断升级。

仅仅学习一个新的 Python 或 JavaScript 框架/库不会让你变得更好。这只是一半，对于另一半，你需要在框架之外的事情上取得进展。

库会不断变化，新的 API 会不断出现，但有些东西会一直保持不变，你可以做一些改变，让自己更上一层楼。

我所说的另一半，是正确的心态、态度和性格，这对于一个开发者来说是非常重要的。以某种方式解决问题或完成任务将决定你的表现，从而决定你的整体价值。

在这篇文章中，我列出了一些重要的事情，它们将帮助你直接进入进步的道路，成为一流的开发人员。

# 每天写代码

在某方面做得更好的最好方法就是练习，除此之外别无他法。

一个孩子只有通过尝试走路才能学会走路。同样，对于开发人员来说，进步的方法是每天坚持写代码。永远不要忽视小习惯的复合效应。

再多的书或教程都不足以让你变得更好，只有通过练习你才能提高。

> "知识没有价值，除非你把它付诸实践."—安东·契诃夫

你学过面向对象编程吗？那么现在是做一个融入 OOP 的项目的时候了。

如果你只是不停地阅读书籍或观看 youtube 上关于 OOP 的内容，那么你将永远在学习 OOP，而永远不会使用它，通过整合你对这个概念的理解，你将达到最大化。

只有通过练习，我们才能巩固我们所学的知识。最重要的是，通过将我们所学的付诸实践，我们会遇到障碍和错误，这会让你变得与众不同。

这种设计模式在开始时似乎很难实现或者至少很难理解，但当你在多种不同的情况下实践了这种模式之后，它就会变得像“hello world”。因此，坚持练习，你越快卡住，你学得就越快。

# 编写干净的代码

简而言之，干净的代码是易于理解和易于更改的代码。

坏代码也可以运行并执行所需的任务。但是如果代码不干净，它会让一个组织崩溃。每年由于维护糟糕的代码而浪费了无数的时间和资源。

如果你写干净的代码，那么你就是在帮助你未来的自己和你的同事。有了这样的代码，你就降低了你正在编写的应用程序的维护成本，并且修复 bug 也变得更加容易。最重要的是，添加新功能将会更加容易。

有了干净的代码，你就可以享受工作的乐趣。最终，你会让参与这个项目的每个人的生活变得更轻松。

你可以很容易地将自己与普通的开发人员区分开来，只需编写更简洁的代码。

如果一行代码引入了不必要的复杂性，并使其他开发人员的生活变得一团糟，请不要炫耀它。如果需要，可以多加三行，但要使代码易于理解。像某人正在读一本书一样写作。

# 别死记硬背了

如果你想学习新的东西，唯一的方法就是更好地理解它。为了更好地理解它，你需要在每一行代码中问为什么。请停止死记硬背。那不行！相反，开始质疑每一条该死的线！

> “我没有特别的天赋。我只是强烈好奇。”
> 
> ——阿尔伯特·爱因斯坦

如果你想把新概念融入你的血液，那么永远不要停止问为什么和怎么做。

请问:为什么这里用的是' let '而不是' const '，为什么这里用的是回调，为什么我需要异步使用一些东西。只要不停地问“为什么”,你就能在自己的领域成为天才。

# 永远不要停止学习

技术世界每天都在快速变化，如果你不想在这条快速变化的道路上落后，那么你必须不断学习新技能。不断升级，不断努力，永不满足。

一个好的开发人员理解更新的重要性，并且聪明地处理事情。

你不一定需要学习每一个新的库或者新的技术。没人有那么多时间把它们都学会。相反，应该这样对待它。

学习你认为对你的领域工作很重要的库，但是在表面水平上，仅仅触及它的表面。然后在以后的项目或任务中需要用到它的时候再深入研究。通过这种方式，你将永远拥有基本的知识，然后你可以很快地将它应用到你的工作流程中。

# 帮助你下面的人

帮助其他比你年轻的开发人员。通过这种方式，你会有更多的社交机会，最终提高你的领导能力。当低年级学生向你寻求帮助时，尽量不要说不或感到焦虑。相反，如果你有时间，帮助他们或指导他们。

此外，你不一定要非常擅长帮助比你级别低的开发人员。即使在同一水平上，你也总能从自己的角度增加一些东西。

敞开心扉去帮助那些你可能不擅长的事情，你可能最终会从他们面临的问题中学到一些有价值的东西。

大三学生通常会问一些非常奇怪的问题，回答这些问题会让你的概念得到锻炼。这些都是你可能从来没有想象过或者思考过的问题。

> 如果不能简单的解释，说明你理解的不够好。
> 
> ——阿尔伯特·爱因斯坦

# 做你梦想中的项目

个人项目是学习新的框架和技术的好方法，这些你可能在工作中没有体验过。

开发者是拥有超能力的人。我们不一定需要雇人来做我们的梦想项目，我们可以自己做。

如果你不喜欢 Instagram 上的一些功能，好吧，用你想添加的功能做一个克隆。用更好的功能制作自己的 Instagram。这点燃了激情学习的欲望。因为你在创造一些有价值的东西，而且这些东西也贴近你的心。

对于你个人的项目，你是产品经理，你是开发者，你是设计师。这里有很多东西需要学习。

更重要的是，如果你正在开始你的开发生涯，或者第一次学习编码。制定个人计划可以增强解决问题的整体信心。

# 现实和诚实

对你知道的和不知道的要真实诚实。自命不凡永远不会让你有所成就，这就像欺骗自己。你不懂 JS 里的一些概念，承认吧。这样，只有你会努力改善它。

发展或学习的第一步是承认你不知道某事。只有接受自己不知道某件特定的事情，才会产生学习的欲望。

你认为如果有人相信他们知道关于那个概念或库的一切，他们会学习吗？

请不要假装。什么都不知道并没有什么不好。没有人什么都知道。我们都在不断地学习。我知道做比说难。

# 为开源做贡献

为开源做贡献会让你接触到工作中可能永远不会发生的场景，从而限制你的感知。这是达到下一个水平的最好方法之一。

您将学习如何以分布式方式运行项目。你可以看到世界范围内使用的最新尖端技术。

你可以与你的领域之上和之下的人的代码互动。开源贡献有很多东西要学。不仅如此，这也向未来的雇主表明你是一个自我驱动、充满激情的开发者。

有人可能会想，既然没有金钱回报，为什么还要为开源软件做贡献。

是的，你的贡献不会得到报酬，但是你将获得的知识和经验是无价的，你建立的关系是有价值的。你遇到的人可能有很多项目或事情需要做，或者他们的雇主可能会招聘空缺职位。这是获得新工作或自由职业的一种方式。

# 寻找更快的方法

努力工作不休息被高估了。这种态度只会让你变成一个工作狂，然后抱怨生产力低下。只有更聪明地工作，才能在不过度工作的情况下完成任务。

一个好的开发人员会不断寻找更好更快的方法来完成任务。他/她使用所有的工具、扩展和 API 在短时间内构建应用程序。

> “大多数人说是智力造就了伟大的科学家。他们错了:这是性格。”
> 
> ——阿尔伯特·爱因斯坦

在这篇文章中，我们看了一些开发人员可以做的事情，以达到下一个水平。

谢谢你付出宝贵的时间并坚持到最后。我希望这能给你的生活增加价值。谢谢你坚持到最后。你们都是很棒的人，发动引擎吧。
