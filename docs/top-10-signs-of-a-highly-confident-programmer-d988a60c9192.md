# 高度自信的程序员的 10 大标志

> 原文：<https://levelup.gitconnected.com/top-10-signs-of-a-highly-confident-programmer-d988a60c9192>

## 你是一个自信的程序员吗？

![](img/dc2e10aa70f090978feefc376a0ad14d.png)

图片来自[皮克斯拜](https://pixabay.com/?utm_source=link-attribution&utm_medium=referral&utm_campaign=image&utm_content=5382775)

在你的工作场所，你会遇到一些有魅力的程序员，他们知道如何有效地处理各种情况。他们能够平等地解决技术和人员问题。对他们来说，每个新问题似乎都是令人兴奋的挑战。

这些程序员无论做什么都散发着自信。他们的自信不仅有吸引力，还能帮助他们在职业生涯中奋斗。公司里的每个人都喜欢和他们一起工作。也许有些人也有点嫉妒。

让我们来看看高度自信的程序员的十个标志，并从他们的行为中学习。获得这些特质可以帮助你提高自信水平，并在你的软件职业生涯中茁壮成长。

# 1.没有自我怀疑的余地

自我怀疑在软件开发人员中相当普遍。通常，这是对这个人缺乏信心的象征。在这个快节奏的技术世界中，一个开发人员需要学习的东西太多了，很容易患上冒名顶替综合症。

自信的程序员不会让自我怀疑阻碍他们的进步。自我怀疑一般源于不安全感。不断拿自己和别人比较的程序员受害最深。但是不断的比较对这个人的自信弊大于利。

自信的开发人员不会让不安全感压制他们的表现，而是努力改进他们缺乏的地方。他们问问题、接受培训、参加会议等。，要跟上技术的步伐。

他们也明白你不必学习世界上所有的东西。作为一名程序员，你需要知道足够多的知识来高质量地完成你的任务。凭借积极的态度和毅力，他们始终保持着帮助他们全力以赴工作的士气。

**你能做什么**

*   不要拿自己和团队中的其他人比较。比较滋生不安全感和自我怀疑。
*   找出你擅长什么，并进一步磨练你的技能。会帮助你增加自信心。

# 2.他们依靠反馈茁壮成长

如果有人在他们的代码中发现缺陷，缺乏自信的程序员会变得愤怒。如果他们的合并请求因评审意见而被拒绝，对自己的能力缺乏信心会让他们觉得受到了侮辱。但是一个自信的开发者把这些事件当做软件开发活动中的正常过程。

这些自力更生的开发人员明白反馈是提高他们绩效的最有力的工具。他们知道他们不能自我识别自己的缺点。因此，他们欢迎各种反馈。与此同时，他们分析输入的信息，并丢弃那些无助于他们更好工作的信息。

很多时候，你会发现他们主动向同事或经理寻求反馈。如果经理向他们提供改进反馈，他们不会认为这是针对个人的。他们知道他们不可能总是正确的。因此，他们总是乐于倾听和学习。

**你能做什么**

*   对每一个反馈都保持欢迎的态度。在你决定放弃任何输入之前，做一个公正的分析。
*   虚心从错误中学习。你学得越多，你就会变得越自信。

# 3.他们不玩指责游戏

软件开发是一个团队游戏。没有团队中其他人的帮助，任何个人都无法独自超越并成功完成一个项目。而一群人在一起工作，必然会出错。

自信的开发人员知道当项目失败时，这不是个人的错。他们不会试图为错误寻找替罪羊。在团队游戏中，团队一起赢或输。

他们也不让团队中的任何人玩指责游戏。高度自信的开发人员完全专注于解决问题，从事件中吸取教训，并继续前进。这有助于保持团队团结，作为一个有凝聚力的单位朝着一个统一的目标努力。

**你能做什么**

*   要明白在指责游戏中没有人会赢。因此，把你的注意力集中在解决问题上。
*   对你的开发伙伴感同身受。如果有人犯了错误，帮助他们从后果中恢复。

# 4.他们承认了自己的错误

当一个自信的程序员犯了一个错误时，他们不会试图隐藏它们。他们知道错误是工作的一部分。他们把错误视为学习新事物的机会。因此，他们承认自己的错误。

当你努力工作时，有可能在一两个地方出错。如果你的团队和经理足够成熟，那么他们会明白犯错误是正常的，只要你不重复同样的错误。

一个自信的开发人员知道，当出现错误时，自省是必要的。犯错是可以的，但是忽视它，从事件中什么也学不到是不行的。因此，他们一定会征求队友的意见，看看下次如何做得更好。

**你能做什么**

*   当你犯错误时，不要为自己辩护。接受错误，面对后果，并从中吸取教训，以免将来重蹈覆辙。
*   如果你需要帮助来纠正错误，请寻求帮助。知道何时寻求帮助是对开发人员有信心的标志。

# 5.他们不喜欢出风头

软件行业竞争非常激烈。每个人都想充分利用现有的机会，在职业生涯中成倍增长。在自我宣传成就的竞赛中，人们通常会忘记他们是在团队中工作的。他们的成功并不是个人的。

这是一个自信的开发者脱颖而出的地方。他们明白只有当团队中的每个人都做好自己的工作时，项目才能成功。没有一个角色比其他角色对项目更重要或更有价值。他们也真诚地相信，该表扬的地方就会给予表扬。

因此，你永远不会发现自信的开发人员拥抱聚光灯。他们总是从团队的角度谈论成功。此外，他们尽最大努力远离不健康的竞争。当团队中有人对他们的工作表示赞赏时，你会发现他们真的很开心。

**你能做什么**

*   无论团队成员的贡献有多大，都要欣赏和认可他们。
*   明白你不能脱离团队独自成长。永远把团队的需求放在第一位。

# 6.他们给别人灌输信心

不仅仅是自信的开发者对自己的能力有信心；他们也善于校准他人。他们知道如何让人们对自己的工作感到舒服。他们从不试图贬低任何团队成员的想法或努力。

对自己的能力高度自信的程序员喜欢把类似的信念灌输给别人。如果你伸出援手，他们永远不会说不。同时，他们也会帮助你磨练你的技能。他们不怕让你知道他们成功的秘密。

如果你的团队中有这样一个成员，你就会知道在困难的情况下，他们让其他人感到多么舒服。当发生冲突时，经理们更愿意向他们求助。就好像他们有解决所有问题的办法，有他们在，团队就没什么好担心的。

**你能做什么**

*   总是袖手旁观你的团队在困难的情况下，并在他们需要的时候帮助他们。不要害怕为你的团队表明立场的不利后果。
*   不断支持队友做到最好。激励他们克服对失败的恐惧。

# 7.他们明白知识就是力量

自信的开发人员知道，在软件职业生涯中，知识就是力量。这不仅仅是技术专长的问题。他们试图从周围的每个人那里获得尽可能多的经验，而不管对方的工作角色。

一个自力更生的开发人员明白，要在软件事业中取得成功，他们需要不断学习。他们不怕尝试新的想法。他们认识到在取得重大成就的道路上，失败是不可避免的。就好像他们对获取知识有着贪得无厌的欲望。

同时，他们也不害怕分享自己的知识。你会发现他们在帮助每个人。他们知道传播知识比囤积知识能获得更多。团队中知识越丰富的人，每个人互相学习并在职业生涯中共同成功的机会就越大。

**你能做什么**

*   每天或每周花些时间学习新的东西并实践它。随着时间的推移，少量但持续的学习将在很大程度上积累你的知识。
*   毫无保留地与同事分享你的知识。你分享的越多，你获得的知识就越多。

# 8.他们不害怕升级

在软件开发中，紧迫的期限和频繁变化的需求是非常标准的。这些情况导致未兑现承诺和执行质量差。团队成员通常不愿意在这种事件中承担责任，害怕失败会影响他们的绩效评估。

但是，自信的开发者是不怕责任的。当需要时，他们是第一个投入战斗的人。解决复杂的问题或处理紧急的优先任务让他们感到兴奋。

通常自信的开发人员很乐意参与升级的练习，因为他们知道他们可以从中学到最多。由于他们精通处理压力情况，领导们也更喜欢提名他们去做高风险的项目。这有助于拥有高度自信的开发人员获得可见性和快速的职业发展。

**你能做什么**

*   如果你是团队的新成员或初级开发人员，请跟随擅长处理升级的高级开发人员。向他们学习在发布或重大事件管理呼叫期间如何处理交火的技巧。
*   如果你是一个有经验的开发人员，请准备好处理升级。只有通过处理压力情境，你才能增强自信。

# 9.他们是会议中安静的人

一个自信的开发者永远不会在会议或电话会议上不合时宜地发言。他们对沉默的力量有着极好的理解。他们知道什么时候该听，什么时候该说。

在团队会议中，人们试图实施自己的想法是很常见的。一般我都见过想证明自己的人，试图通过打断别人来更多的炫耀。通常，这些人会将讨论引向与预期目的完全不同的方向。

然而，你可以很容易地在会议中发现一个自信的开发者。他们是最受关注的人。当轮到他们时，或者当人们询问他们的意见时，他们会发言。他们不相信通过压制他人的声音来获得优越感。自信的程序员帮助那些不愿意在小组讨论中发言的人创造空间。

**你能做什么**

*   仔细听会议中的讨论。只有当你有相关的东西要贡献，或者你的意见被要求的时候，才说话。
*   在讨论中为其他成员提供机会，鼓励他们提出自己的想法。

# 10.他们喜欢直截了当地说话

你经常会看到人们对上级的想法或要求说是，即使他们不同意。大多数情况下，他们害怕失去领导的青睐。这些人不愿意表明立场。他们宁愿违背自己的意愿，也不愿把自己的观点摆在大家面前。

一个自信的开发者不怕直言不讳。他们不会说领导想听的话来取悦管理层。他们礼貌但坚定地说出自己的想法。然而，如果决策与他们的想法不一致，他们不会自负。

对自己能力高度自信的开发人员不会试图隐藏他们对某个主题的无知。每个人都明白，没有一个程序员能够学会技术或功能的每一个方面。

如果一个自信的程序员不知道正在讨论的话题，他们会主动提问以了解细节，而不会为此感到羞愧。

**你能做什么**

*   学会如何在专业上同意不同意见。当你的观点与你的上司或经理的观点不一致时，不要点头或保持沉默。
*   不要用含糊的回答来掩盖你的无知。如果你不知道某个话题，接受并提出问题来澄清你的疑问。

# 最后的想法

这些是自信程序员的一些重要标志。自信不是一种要么全有要么全无的特质。你可能在某些方面很自信，但在其他方面缺乏迹象。然而，如果你想提高你作为一名开发人员的自信水平，你可能想在你觉得你能做得更好的特质上努力。

您可以决定您希望在多大程度上与这些标志保持一致。这不是有人定义你是否自信的问题。它是关于在你的职业生涯中成长，享受你正在做的事情，并实现你为自己设定的所有目标。
