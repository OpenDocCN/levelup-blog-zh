# 开发人员对经理的五大误解

> 原文：<https://levelup.gitconnected.com/5-epic-misconceptions-developers-have-about-managers-5d37013a9ed1>

## #4.管理者真正做的是什么？

![](img/5472d78627f94c5c2c1feeb776fe8d2b.png)

信用:Canva

在任何职业中，人们对他们的经理有误解是很常见的。软件开发领域也不例外。关于管理者做什么和不做什么，有很多神话和刻板印象；以及他们擅长什么和不擅长什么。

结果，大多数开发人员最终对他们的经理抱有不切实际的期望。他们认为他们的经理应该能够解决他们所有的问题。当他们的期望与现实相符时，他们会失去信心，并开始将工作场所的所有问题归咎于经理。

但事实是，管理者并不是创造奇迹的人。

在本文中，我们将探讨开发人员对其经理的五种常见误解。了解这些要点可以帮助你设定现实的期望，并与你的经理建立信任关系。

# 1.我不必突出我的工作。我的经理应该知道我在做什么

这是开发人员中常见的误解。他们认为他们的经理应该随时自动知道他们在做什么。他们不需要被明确告知他们的工作。

开发人员觉得——我已经在努力交付了。这足以展示我的贡献。

事实是，你的经理不是一个读心术者。让他们了解你的工作是很重要的。如果你对自己的成就感到自豪，那么就努力突出它们。这将有助于他们了解你给团队带来的价值，并为他们提供所需的信息，以便他们对你的未来发展做出决策。

# 2.我的经理负责我的职业发展

你的经理在你的职业发展中确实发挥了作用。但最终，还是要由你来掌控自己的事业。

你必须积极主动地设定目标，规划通往成功的道路。您的经理可以提供指导和支持。但是他们不能让事情为你发生。

你负责安排一对一的会议，并与你的经理讨论你的抱负。请求对您的表现进行反馈。问问他们你能做些什么来获得下一个级别的角色。

如果你表现很差，那么是的，经理有责任告诉你。但是他们没有必要逼你超越自我。如果有人这样做了，你应该感激他们。

# 3.他们控制谁得到多少远足

经理们确实对员工的加薪幅度有发言权。

但是，他们不控制增量周期的资金。大多数公司根据公司业绩和市场条件，在评估周期内分配明确的预算。

每个经理都需要坚持分配给他们团队的资金。只要不超出团队预算，他们当然可以提高或降低员工的加薪幅度。但是他们不能给每个队员无限制的徒步旅行。

# 4.管理者真正做的是什么？

大多数开发人员认为，如果有人不编码，他们就不是“真正”在工作。在我职业生涯的开始，我也曾这样想过。

经理的主要职责是领导和激励他们的团队。他们需要确保团队的生产力和效率。所有成员都在为同一个目标而努力。

他们还需要解决团队内部的任何冲突，向员工提供反馈，并帮助他们实现职业发展。

因此，虽然经理可能不会直接参与团队的日常任务，但他们在确保团队成功方面发挥着至关重要的作用。

# 5.他们因发号施令而获得高薪

> 管理者又是做什么的？他们写的代码从来没有他们的开发者写的多。他们只是在工具中批准休假和请求。他们向他们的高层领导做报告。为此，他们得到如此丰厚的薪水！在那种情况下，我应该成为一名经理。

这是对管理者最常见的误解之一。人们认为经理们光是发号施令就能赚很多钱。

现实情况是，大多数经理责任重大，工作压力很大。他们的职业发展取决于他们团队的成功。他们需要身兼数职，同时兼顾多项任务。他们需要在做这些事情的同时兼顾自己的职业发展。

然而，这仍然不意味着所有的管理者都拿着巨额的薪水。影响他们工资的因素很多，比如团队规模、公司预算、经理经验等。他们的绩效评估不仅包括个人绩效，还包括整个团队的表现。

# 最后的想法

总之，重要的是要记住，经理只是普通人，和其他人一样面临着同样的挑战。他们不是奇迹创造者，也不能让一切都完美。

一旦你消除了这些误解，你就可以和你的经理建立良好的关系。它可以帮助你与他们开诚布公地讨论你的期望。这最终会对你的事业有所帮助。

永远记住——你的职业是你的责任。你的领导可以是一个巨大的资源，为你的成长提供必要的支持。但是你需要尽一切努力成为一名成功的软件工程师。
