# 关于如何增加你作为开发者的价值的非显而易见的提示

> 原文：<https://levelup.gitconnected.com/non-obvious-tips-on-how-to-increase-your-value-as-a-developer-e01e9b1e6324>

软件开发人员通过阅读编程书籍、博客或参加在线课程来学习，以有效解决项目中出现的技术问题。某个开发人员每天都变得越来越有价值，因为他或她能够解决的任务越来越复杂。作为一名开发人员，增值的过程可以像任何其他过程一样进行优化或改进，以下是对此的建议:

# 1.阅读至少 10 篇关于你在日常工作中遇到的新技术方面的文章

在你的工作日里，当你执行任务时，你可能会在你的项目中遇到一些新的技术问题。它可以是任何东西:非聚集索引、saga 模式、深度优先搜索算法的实现、JWT 令牌等。一旦你遇到一些你以前没有太多经验的东西，尽可能多地阅读这方面的文章和书籍章节，直到你觉得你对它了如指掌。例如，如果你在你的项目中遇到一个装饰模式，尽可能多的搜索关于它的信息，从不同的角度学习这个模式:

*   装饰设计模式有哪些用例？
*   实现装饰模式的优点和缺点。
*   基于继承的实现与基于聚合的实现。
*   装饰模式和面向方面编程的区别。
*   装饰器与代理、适配器模式有何不同？
*   检测装饰循环的技术。

学习你在现实世界项目中遇到的很多东西的方法很有效，因为**你可以立即将你学到的东西付诸实践**。比方说，在学习了理论之后，你明白了一个模式的实现可以被另一个模式改进或者完全替代。你现在就可以做出改进。这会让你的学习过程更加有效。

# 2.成为您客户的合作伙伴

作为开发人员，您实现客户要求您实现的特性。这显然是你应该做的，但是**不要只是盲目地实现你得到的**的需求。想想为什么需要这个功能，它是否足够适合客户的业务。一旦你开始这样做，你可能会发现一些用户故事的需求，甚至是产品的整体方向有很多不准确的地方。让您的客户了解您的顾虑，并提出可能的改进建议。

努力成为你当前工作领域的专家。不要只关注项目的技术方面。这种做事方式会让你从实干家变成客户伙伴。

# 3.开始分析开源项目的代码

开源项目不仅对初学者来说是实用知识的极好来源，对有经验的程序员也是如此。你需要做的就是在 Google 上为你的编程语言找到这样的项目，并开始分析它的源代码。这个练习将让你理解如何应用设计模式、依赖注入机制、算法等等，因为你将看到它在现实世界中是如何使用的。

**一个开源项目类似于一本编程书籍**，我打赌你一定读过很多，但是它包含的实践知识比理论知识多得多。

# 4.写你的第一篇技术文章

试着就你擅长的话题写你的第一篇文章，如果你还没有的话，把它放到网上。它将帮助你在你选择的主题上构建你现有的知识，学习新的东西，并从读者的反馈中学习更多。能够组织你的想法将有助于你清楚地向你的队友或客户解释某些技术细节。

如果你还没有写文章，这个过程对你来说可能是一项艰巨的任务，但是至少试着开始写吧。就我个人而言，我在写任何新文章之前都会拖延，但如果我强迫自己写哪怕是第一段，我就会停不下来，直到文章发表。

**我不是要求你每天都写博客，请只写一篇你熟悉的文章**。你不会失去什么，但它可能会改变你的生活。

# 5.了解您的技术堆栈的性能技巧

许多现代应用程序由一组框架、库和编程语言组成，其中的每一个都可能由于误用而导致性能问题。这种技术堆栈的每个组件都有关于如何正确使用它以避免应用程序中出现严重性能问题的指南。这些指南通常被称为性能提示，你可以很容易地在谷歌上搜索它们，例如*“JavaScript 性能提示”*或*“C #性能提示”*。学习性能技巧是一种相对快速的方法，可以在不深入学习所有技术的情况下，找出如何避免应用程序中的大多数性能问题。

# 6.学习做琐碎工作的另一种方法

开发人员，即使他们有时有复杂的任务，也必须每天执行许多琐碎的任务，例如添加新的 API，实现新的存储库方法，等等。编程的好处是它是一门相当复杂的学科。对于我们开发人员来说，这意味着几乎所有的编码任务，除了初级的，都可以用不同的方式解决。例如，至少有 [5 种方法来实现单例](/5-ways-to-implement-the-singleton-design-anti-pattern-in-c-68bb664c31f2)，[克隆一个对象](/5-ways-to-clone-an-object-in-c-d1374ec28efa)，或者[实现一个存储库模式](/5-ways-to-implement-repository-pattern-in-c-e12565e4d4a2)。

即使你认为你知道解决某个编码问题的好方法，也要花些时间寻找替代方法来解决它。对你所有的解决方案进行权衡分析，然后选择最适合你特定情况的方案。

# 7.对你当前的项目进行审计

我相信你在从事一个复杂的项目，这个项目肯定存在一些问题。例如，您的项目可能有糟糕的文档，使用不适当的分支策略，或者缺少单元测试，等等。

您可以做的是对您的项目进行全面的审计，以确保它遵循软件开发最佳实践，从命名约定和编码标准到持续的交付和部署过程。

这里有一些提示供你开始:

*   检查是否有广泛的项目文档，包括编码标准、本地环境设置、持续集成过程、源代码管理最佳实践、非功能性需求、开发人员测试策略、第三方库等。
*   如果你的项目有正确的[技术债务管理](https://medium.com/codex/technical-debt-management-best-practices-for-software-engineers-871a315ac812)策略？
*   开发人员知道并遵循[代码评审最佳实践](https://medium.com/r?url=https%3A%2F%2Fesashamathews.medium.com%2Fcode-review-best-practices-in-software-engineering-7ed39b83ef0b)吗？
*   静态代码分析是构建管道的一部分吗？
*   单元测试是构建管道的一部分吗？
*   项目中是否有定义良好的开发人员测试流程？
*   单元测试是否完全覆盖了业务逻辑？
*   项目是否使用合适的数据库变更管理策略(基于状态或基于迁移)？
*   是否有针对自动化部署的自动化回滚程序？
*   项目的性能关键部分是否有性能测试？
