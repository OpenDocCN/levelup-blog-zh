酏剂中的意外列表

原文：https://levelup.gitconnected.com/unexpected-lists-in-elixir-f5b6629e89d3?source=collection_archive---------27-----------------------#2021-06-07

嗯——当时对我来说是意想不到的，尽管是语言的一部分。

我一直在 Phoenix 中整理一些东西，其中一个页面一直抛出一个它找不到item_path/3的错误。这很奇怪，因为模板中的生成代码有四个参数。

同样的事情发生在我闯入IEx去戳它的时候:

iex(1)> Routes.page_path(conn, :index, user_id: user.id, item_id: item.id)
** (UndefinedFunctionError) function AppWeb.Router.Helpers.item_path/3 is undefined or private. Did you mean one of: * item_path/4
 * item_path/5

熟悉长生不老药的人已经能发现问题所在…

在药剂中，key: value, key: value形成了所谓的关键字列表。我没有想到的是，最后两个参数会被自动转换成一个关键字列表:

iex(2)> i one: 1, two: 2
Term
  [one: 1, two: 2]
Data type
  List

去掉关键词，一切都很好。

iex(3)> Routes.item_path(conn, :index, user.id, item.id)
"/users/1/item/2"

至少等到下一个bug。:-)最近写JavaScript和复习Python的时间太多了我觉得！
