“开源”指的是公众共享、修改或访问的任何代码。它是编程的一部分，允许社区通过报告问题甚至帮助他们解决 bug 来为团队或个人构建的工具做出贡献。

开源软件在今天的软件开发中被大量使用。它们提供的主要好处是它们提高了软件开发的速度，因为开发人员不必重新发明轮子来执行一些操作。还有更多好处，继续看下去。:)

已经开发了许多开源软件、组件和工具，更多的还在开发中。除非你试图实现的是非常独特和罕见的，否则很可能已经为它创建了一个开源工具。

这减少了制作逻辑或手工函数的时间，因为你可以在任何你想去的地方下载和使用这个工具。

让我们看看使用开源组件的一些最好的理由，然后继续讨论您可能会有的犹豫。

# 开源组件的优点

# 1.强大的功能

以一个数据库为例。如果你在应用程序中唯一与日期相关的操作是将日期转换成秒，那么下载日期库可能没有意义。这不必要地增加了生产规模。但是假设你做的都是和日期相关的事情，一个好的日期库会给你很多特性(比如把日期转换成秒，把小时转换成分钟，把分钟转换成可读格式，等等)让开发变得更容易。

# 2.性能卓越，久经考验

一些开源组件是由个人开发的，而另一些是由团队开发的。对于后者来说，他们需要聪明的头脑聚在一起有效地构建工具。此外，这些工具还经过质量性能测试，确保所有边缘案例都被发现并得到解决。当您使用这样的工具时，您将受益于它们带来的性能，以及它们可以被视为最佳解决方案的事实。

# 3.安全性

当工具变得流行并被许多开发人员使用时，对于这类工具的创建者来说，安全性就成了一个非常重要的因素。对于这样的工具，发现安全漏洞的速度更快，因为在使用这些工具时，每 20 个人中就有 1 个人可能经历过这样的缺陷。

通过使用这样的工具，有一个高水平的安全保证，因为团队(以及贡献社区)只要一发现安全问题就不断地解决它们。

# 4.社区

开源工具是社区驱动的。除了对修复代码问题的贡献之外，社区还致力于文档、教程和论坛，以确保这些工具以最简单的方式得到解释。有了社区的这些贡献，这样的工具变得易于使用和无缝适应。

# 5.性价比高

许多性能良好且安全的开源工具都是免费使用的。自己创建解决方案可能需要时间，也可能需要有成本的资源。通过使用开源工具，你可以享受上面列出的所有好处，而且它们是免费的。你什么都不用付钱。下载/安装使用即可。

# 开源软件的缺点

# 1.生产规模的增加

它们最终可能会增加代码的生产版本大小。有时，您只需要开源库提供的一个工具、函数或组件，但是通过安装该库，您可以获得其他您可能不需要的工具，从而增加最终的构建规模。

然而，这是有解决办法的。其中之一是在生产构建文件的生成过程中使用工具来消除未使用的变量或[死代码](https://en.wikipedia.org/wiki/Dead_code)。

# 2.安全漏洞

易受攻击的开源组件将您的应用程序暴露给攻击者。手动检查这些工具是否有安全缺陷可能很费时间。然而，有像 [WhiteSource Bolt](https://www.whitesourcesoftware.com/free-developer-tools/bolt) 这样的安全测试工具，您可以将其集成到您的开发环境中，或者构建管道来检测和解决此类漏洞。

# 3.可能很陡的学习曲线

当这些工具没有很好的文档时，情况就更糟了。一些工具可能有很高的学习曲线，因此它们实现目标的方法可能不那么直接和简单。在这种情况下，开发人员必须在学习该工具或放弃它之间做出选择。

# 结论

在本文中，我们讨论了开源工具的一些缺点，解决这些缺点的一些解决方案，还讨论了为什么应该在项目中使用开源组件。重新发明轮子并不总是一个坏主意，因为你可以精确地控制你想要的东西是如何工作的，但是有了上面解释的好处，你现在应该有更多的理由不重新发明它。
