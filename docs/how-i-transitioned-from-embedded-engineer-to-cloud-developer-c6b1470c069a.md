## 这并不容易！

我从嵌入式程序员做起。在我职业生涯的前五年，我在软件的最底层工作。我主要用 C 编写代码，构建协议栈、固件和媒体中间件。

我的几个同事创办了一家互联网公司，尽管我对他们开发的技术缺乏经验，但我还是决定加入他们。我很高兴我做出了这个决定。

我在学习应用程序和后端开发时遇到了很多问题。

# 引擎盖下发生了很多事

作为一名 C 程序员，你必须自己用很少的开源库来编写所有的代码。对 Ruby 和 Python 这样的语言的介绍让我很不舒服。

我想知道如何在包含整数的数组中添加一个字符串。我知道这些语言是如何工作的，也知道它们的目的，但在我的潜意识中，有些东西还没有准备好接受这一点。

我花了相当长的时间来接受现实并继续前进。现在我喜欢这些语言，每次我必须为小任务编写脚本时，我都会使用它们。

# 必须学习一个全新的生态系统

固件和云中的工作方式完全不同。学习一两种语言是一件容易的事情。了解生态系统中的一切是如何协同工作的并不容易。

要学的东西太多了，我一步一步来。我从写 web scrapper 开始，慢慢开始写应用代码，然后让自己熟悉 CI/CD 和云基础设施。

# 缩放比例

然后是每个云开发者都想解决的问题，将应用扩展到成千上万的活跃用户。

在我早期的职业生涯中，我曾在 KBs 内存系统上工作过。我们曾经把代码优化到不能再优化的程度。我开始用云软件做同样的事情。在几周内，我们可以减少内存占用、CPU 利用率，并重新设计一些子系统以提高效率。

有一段时间，我们在白天解决这些问题，并在下午 6 点后实现新功能。

# 包扎

从嵌入式软件开发到云软件的旅程在一开始并不容易。随着时间的推移，你会意识到计算机科学的基本概念和基础可以应用到任何地方，只要你改变应用的方式。

我注意到所有软件领域都有类似的模式，无论是固件、手机应用还是云软件。每个领域可以有不同的方式来解决这些模式，但这些模式是可以发现的。
