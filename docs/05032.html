通过终端更新GitHub知识库

版本控制是开发过程中一个绝对关键的T2组件，每个程序员都应该熟悉。版本控制是一个日志记录系统，它记录了代码的历史，以便以后可以使用整个进程。版本控制允许开发人员编写、更改、删除和编辑新的和现有的代码，同时保留我们文件的所有不同“版本”的详细记录。特别是GitHub使用了版本控制系统“Git”，这也是我们在本帖中要重点介绍的。

本地与远程

版本控制系统主要有两种类型:本地和远程。本地系统指的是存储在你的电脑上的任何信息。任何时候你执行一个单独的项目或者一些实用的快速编码，本地存储就足够了。然而，一旦协作成为可能，本地版本控制就不再起作用，因为您很少会和同事使用同一台计算机。

一个分布式版本控制系统是一个所有用户拥有整个存储库(项目服务器)的完整副本的系统。该项目现在可以独立于任何网络或连接进行，因此无论他们在哪里工作，同事们都可以访问彼此以前的工作和版本更新。一旦变更完成，新的工作被“推”到存储库中，现在所有的合作者都可以看到。

叉子和“Git克隆”

当使用GitHub的版本控制系统时，需要注意的第一个概念是所谓的分叉。分叉指的是对存储库中已经存在的东西进行个人复制。假设您的同事编写了一个快速脚本来加载、清理和保存项目数据到Pandas DataFrame。如果您只是想熟悉这个脚本，并在本地将它应用到您自己的数据中，那么您只需分叉存储库，然后就可以访问它了。您可以通过单击存储库右上角的“fork”来派生存储库。

  1. 创建一个文件夹来存储分叉的材料
  2. 点击“分支”按钮创建一个副本到您的个人帐户(您现在可以编辑和更新)

现在，我们有一个分叉的副本，但我们需要下载到本地。

3.将GitHub存储库URL复制到剪贴板

4.在bash终端中:git clone <URL>

这将自动将在线存储库中的所有内容下载到您的本地计算机。使用终端中的ls查看内容。注意:您的终端必须在本地工作文件夹中运行，以便记录版本历史。

Git状态

Git将跟踪我们所做的所有更改，因为我们已经在本地下载了代码。在我们的终端中输入git status将返回一个彩色编码的更新，显示活动文件夹正在进行的更改。如果返回的文本是红色的，这意味着我们仍然需要将更改添加到我们的存储库中(敬请关注)。如果它是绿色的，这意味着更改已经被添加并准备保存。

Git添加

我们通知我们的存储库，我们将通过向存储库“添加”变更来更新版本。键入git add将会使这些更改成为Git日志的永久记录。要添加当前点的所有更改，请添加一个句点:git add .

Git提交

我们的更改现在已经添加到日志中，但是它们还不是永久的。我们必须提交新版本的代码到我们的仓库。这是这个过程中最重要的一步。语法是git commit -m "<insert commit command>"。这个命令告诉Git提交所有的更改，我们必须包含的消息(在引号中)将作为一个标题，告诉查看版本历史的人在这一步发生了什么。这些信息应该是总结该步骤的命令性陈述。示例:git commit -m "Import Raw Data and wrap to DataFrame"现在，当查看存储库时，任何人都可以看到我在那个特定步骤中做了什么。

Git推送

我们必须完成的最后一步是“推动”。推送意味着我们将本地代码和提交上传到GitHub库。GitHub的托管存储库是我们的“源”或“远程”，我们必须通过git push用我们的新变化更新源。终端命令示例:git push origin master

瞧啊。通过执行这几个简单的步骤，我们现在已经完成了克隆、编辑和重新上传存储库的整个过程。整个过程通过GitHub记录下来，代码被永久记录。现在任何合作者或好奇的观众都可以看到它。
