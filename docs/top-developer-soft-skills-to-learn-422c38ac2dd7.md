如果你想成为一名成功的开发人员，你需要学习的不仅仅是写代码。

在本文中，我们将学习一些软技能，以便成为一名成功的开发人员。像任何工作一样，软技能也很重要，因为我们可以做技术性的工作。

# 使不同

我们必须对不同的事情持开放态度。能够学习不同的东西，摒弃过时或不好的知识是必须的。

因此，学习工作之外的东西很重要，以备不时之需。因此，如果我们现在正在开发 Rails，我们还应该学习 Node 和 Express。

当我们主要使用面向对象语言时，这也符合学习函数式编程的一般原则。它们并不相互排斥。

例如，JavaScript 既有函数概念，如语言中包含的高阶函数，也有面向对象的概念，如对象、原型和继承。

这样，我们可以对更多的人有用，也增加了不同的视角，丰富了我们对不同事物的思考。

如果我们学会了不同的做事方法，我们就不会那么小心眼了。

电脑的事情变化很快，所以我们必须适应，这样我们就不会在不久的将来变得一无是处和无法就业。

# 敏捷

敏捷意味着我们应该为随时变化的需求或项目做好准备。

我们的思想不能停留在任何地方，这样我们才能理解用户对软件产品的用例。

此外，我们必须理解客户，并将他们的反馈融入到我们的产品中。

因此，最好一次只做产品的一小部分。这种方式即使是功能变化，我们也不必从头再来。

这也使得实施任何事情变得更容易管理，因为我们不会被任何事情淹没。

# 神入

有同理心很重要。由于大多数软件项目都是基于团队合作的，软件是用来服务客户的，所以我们最好能对用户感同身受，因为他们为我们编写的软件付费。

任何反馈都应该被整合到产品中，如果它们是有意义的，并且有很多客户需要的话。

我们还应该同情我们的队友，因为他们工作努力，可能会在某个时候接触到彼此的代码。

这意味着我们应该编写干净的代码并交流变化。如果队友的建议有意义，我们应该采纳。

总是试着从他们的角度看问题，以及为什么他们希望我们按照他们的方式改变。

# 沟通

沟通包括口头和书面。口头交流包括口头更新和结对编程。

如果我们说话，我们应该清楚而自信地说话。还有，我们应该多听少说，不要打断别人。

这些只是我们应该遵守的一些基本礼仪，这样每个人都有机会发言和被倾听。

此外，我们不应该主导任何对话，以便每个人都有机会发言。

对于书面交流，我们应该把一切都写下来。像更新任务的状态和描述这样的事情很重要。

此外，记录如何设置和运行的指令也很重要。

在我们的代码中，我们应该编写它们，以便它们是自文档化的。所以变量名应该传达它们的意思。这也适用于其他东西，如函数、类或任何有名字的东西。

每个函数应该足够简单，这样它们就只做一件事，正如它们的名字所表明的那样。

# 协力

团队合作很重要，因为大多数软件项目都是由团队完成的。

这是无可避免的。我们不仅与开发人员合作，还与设计师和不同的项目团队合作。

我们必须与他人合作才能成功。与他人合作愉快会让工作变得更加愉快，如果我们不合作的话。

# 结论

如果我们想成为或继续成为一名成功的开发人员，拥有一个多样化和开放的心态是很重要的。

软件总是在变化，所以我们必须适应它们。除了我们现在做的或以前的工作，我们还应该对其他事情持开放态度。

团队合作很重要，因为大多数开发人员都是团队工作。因此，与他人合作愉快比不合作更令人愉快。

最后，沟通很重要，因为我们必须在团队中互相更新和帮助。此外，擅长口头和书面交流也很重要。
