过去的五年里，我一直在 T4 从事软件开发工作，其中有一年半的时间我在柏林度过。像许多开发人员一样，出国工作是一个梦想，大多数人通过在另一个国家攻读硕士学位并在那里工作来实现他们的梦想。

对我来说，这个故事有点不同。我不想继续学习，因为我对典型的教育过程缺乏兴趣。

对许多人来说，移居国外没什么大不了的，但对我来说，这是一个梦想。从孩提时代起，离开这个国家到国外生活的梦想就激励着我，在我脑海的某个地方，我知道我会以某种方式实现它。

# *但是等等，让我原路返回*

这一切都是在我 2015 年完成 B.Tech 之后开始的。最终应用过去 18-20 年获得的所有知识的一年。这是我第一次作为员工接受职业发展培训和电子邮件写作课程。

***2015–2016***

*   我的第一份工作非常有趣，我遇到了很多好人。办公室聚会，庆祝我们甚至不知道自己取得的成就，但是谁会拒绝聚会呢？
*   在那里工作了近 8 个月后，我意识到我没有从我所做的工作中获得足够的知识。这已经成了一种惯例，我像许多其他人一样讨厌惯例。我决定换一个。我开始准备。
*   我在找资源学习的时候，偶然发现了这本书叫[破解编码面试](http://www.crackingthecodinginterview.com/) **。作为一名新人，我没有多少接受采访的经验。读这本书帮助我了解了很多关于面试和面试官期望什么。**
*   我的主要重点是学习我在大学期间跳过的内容。我拿到了课程用的书，并阅读它们来强化我的背景。
*   我申请了很多公司，得到了其中一些公司的回复，最终参加了面试并得到了一份工作。

***2016–2018***

*   新环境完全让我感到自豪，我感觉非常好。这是我人生的转折点。我从事的是我从未听说过的技术和平台。我得到了用不同编程语言工作的机会。我从事 Android 和后端开发。
*   我有机会接触来自世界各地的人。我和不同的团队在不同的城市从事不同的项目。
*   但是我们都知道生活中的变化是唯一不变的。我决定继续前进。

# 准备工作

这是我决定申请出国的时候。我开始在 LinkedIn、Glassdoor 和其他平台上寻找工作。我把找工作的范围缩小到我有关系的城市。这进一步把范围缩小到了柏林。我最初的想法是申请创业公司，然后从那里开始工作。

我开始准备。

我专注于三件事:

## **编码和数据结构**

当你申请一份软件开发人员的工作时，这是显而易见的。你应该能够用你熟悉的语言解决算法问题。我花了大约一周时间在 HackerRank 上完成简单的挑战，尽可能多地解决问题。我开始只关注简单的小问题，然后是中等的问题。

## **架构设计/数据建模**

这是面试过程中我最喜欢的部分之一。在我看来，这一点对任何开发者来说都非常重要，因为从技术上来说，设计决策可以决定你的应用/产品的成败。我阅读了来自[网飞](https://netflixtechblog.com/?gi=fff89c96328c)、 [Airbnb](https://medium.com/airbnb-engineering) 、 [Spotify](https://medium.com/spotify-labs) 等的科技博客。这对我理解大型创业公司面临的问题帮助很大。

## **以前使用的项目/技术**

这是非常明显的，主要涉及到谈论你所做的工作的长度和宽度。证明自己是一个潜在的队友。这一部分的问题可以是具体的，比如你简历上的技能，也可以是关于一个项目的。其中一个问题可能会问你过去项目中的一个关键情况，以及你是如何处理的。

## 要关注的技术堆栈

后端工程师通常会关注:

1.  服务层: ***Spring Boot*** 同 ***科特林***
2.  数据库层: ***Postgres***
3.  部署配置: ***Terraform，Docker*** 和 ***Kubernetes***
4.  部署:了解 ***AWS*** 或任何其他云提供商。

# 面谈

在得到几家公司的回应后，他们会发出一个带回家的编码挑战，你通常有两天时间完成。问题陈述之一可能如下。

“我们希望有一个 restful API 来处理我们的统计数据。创建 API 是为了计算最近 60 秒的实时统计数据。每次进行交易时都会调用这个 API。它也是这个 rest API 的唯一输入。另一个 API 返回某个时间段的所有交易"

解决方法是这里的[和](https://bitbucket.org/maniish_jaiin/transaction-service/src/master/)。

提交编码挑战并解决它之后，会安排一些面试。我经常做的一件事就是阅读关于这家公司和他们正在做的工作的信息。我查看他们的博客帖子，了解公司最近的进展。

有一系列的面试环节，从技术环节开始，更侧重于提交编码挑战。

在那之后，还有文化交流和对公司首席技术官或副总裁的一般采访。所有的采访可以在一天内进行，也可以持续几天，这取决于采访者是否有空。

这就是柏林许多公司的一般面试流程。

在我来这里之前，我已经接受了许多采访。这绝对不容易，需要大量的准备工作。但是如果你愿意付出你所有的努力，这当然是值得的。
