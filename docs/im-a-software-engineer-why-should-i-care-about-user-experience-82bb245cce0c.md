T 关于软件工程中的用户体验(UX ),已经有了很多讨论，这是由网络的广泛传播和智能手机应用程序的引入推动的，从商业到社交，从娱乐到银行，改变了所有事情的处理方式。

然而，UX 本身并不新鲜。自 20 世纪初以来，人们就开始研究产品的可用性，最初是作为一种通过让工人更容易使用工具来优化工厂工作的方法。它很快扩展到消费品的设计，使它们更适合用途，因此对潜在客户更有吸引力。UX 是可用性的一个超集(或者子集，取决于观点),也考虑了情感。

在软件工程的上下文中，UX 涵盖了最终用户与系统的交互，不仅包括他们是否能完成他们设定的任务，还包括他们在整个过程中的感受。我们可以说 ***UX 让用户回到我们的产品，不是因为他们不得不回来，而是因为他们想要*** 。这是留住客户的支柱，也可能是你今天使用智能手机上一半应用的原因(另一半与生态系统锁定和内容可用性有关，但这些是改天的话题)。

尽管如此，我和一些工程师一起工作过，他们并不真正关心 UX，或者甚至不知道它是什么。

是时候解决这个问题了。要成为一名伟大的软件工程师，你需要关心用户体验。

# UX 和后端工程

不管你是前端，后端，还是全栈工程师。最终，**无论你做什么都会影响最终用户**，不管用户界面和底层代码之间的抽象层次如何。

API 的设计反映了如何在前端收集或显示数据。导入/导出过程生成的文件格式可能容易使用，也可能不容易使用。还不信服？考虑批处理操作的进度报告。谁需要知道我们走了多远，做得怎么样？谁需要修复输入数据中的错误？

理解用户体验对后端工程师和前端工程师一样重要。记住我们说的是体验，不是界面。这让我想到了第一个常见的误解。

# UI 和 UX 不是一回事

*用户界面* (UI)和*用户体验*只有一个共同点:“用户”二字。这就像软件工程和软件测试之间的共同点一样多，我认为我们可以同意这些学科彼此之间有很大的不同。

**UX** 是关于用户与你的产品的互动。就像我们在前面的段落中看到的那样，这是关于他们如何从 A 到 B 以及那是什么感觉。体验是否顺畅？错误情况是如何呈现和解决的？旅程的先决条件和可能的结果是什么？这些都是 UX 设计要回答的问题，这样用户在整个系统中的旅程就尽可能的顺畅。

**UI** 另一方面是关于数据如何呈现给用户以及如何从用户那里收集数据。传统上，这包括产品的视觉设计，以及呈现给用户的错误和警告消息的样式。随着语音驱动界面(想想 Siri 或 Google Assistant)的出现，这正在演变为包含更广泛的范围，但最终它仍然是用户和产品之间的最终界面。UI 考虑的例子有排版、颜色和设计语言。

说完这些，让我们回到 UX 及其对软件工程师的重要性。

# 你应该关心 UX 的一些基本原因

## 原因 1:未使用的代码是浪费时间

如果你写了世界上最好的系统，但是没有人使用它，你已经浪费了你的时间。软件工程不是脑力劳动，它是关于创造一些有用的东西，人们可以用它来实现他们的目标。

如果用户决定不尝试你产品的一些功能，因为它们很糟糕，你就浪费了时间去实现它们。

甚至在我们开始考虑代码的长期维护之前，这就是事实，在长期维护中，未使用的特性可能会妨碍重构活动。

## 原因 2:共同语言是效率的关键

很可能，你的公司有一个专门的团队为你的产品定义 UX。这很好，因为它从工程中去掉了很多用户的猜测，但是如果你能与这个团队很好地沟通，它会更好。

对 UX 及其术语表有一个大致的了解，对于减少 UX 设计师和工程人员之间的沟通不畅和误解大有帮助，有助于更有效的周转时间和整体更好的软件。

## 原因 3:软件架构考虑

对软件使用方式的良好理解必然会引导你产生一个更适合目的的解决方案。

软件架构通常是平衡算法和现有方法的一般化和特殊化的微妙行为。理解预期的用户旅程将有助于您缩小系统的预期使用范围，并更好地适应实际的错误情况。

这并不是说您可以忽略边缘情况，但至少您对常见情况和例外情况的认识应该变得更加准确。

这是你应该关心 UX 的三个简单原因。我并不主张工程师应该成为 UX 专家，这本身就是一种专长。我只是说，对这种实践的基本理解可以帮助您编写更好的软件。

# 如何更好地了解 UX

像往常一样，网上有很多资源，从 YouTube 视频到 Udemy 和 Coursera 程序，当然还有相关的媒体帖子和出版物。这里有一些资源可以帮助您开始:

[**用户体验基础知识**](https://www.usability.gov/what-and-why/user-experience.html)**——**美国卫生与公众服务部对该主题的简短而甜蜜的介绍，以及大量的额外资源供进一步阅读。

[**UX 基础知识**](https://developers.google.com/web/fundamentals/design-and-ux/ux-basics)**——**一篇来自谷歌开发者博客的关于 UX 设计过程的伟大文章。这是对该主题的一个很好的介绍，因为它使用适当的术语涵盖了它的基本原理。

[**用户体验原则和流程介绍**](https://www.coursera.org/learn/user-experience-design)**——**美国密歇根大学在 Coursera 上开设的 20 小时课程，涵盖了人们如何感知信息，以及如何据此进行设计等有趣的话题。这也是 UX 研究和设计专业的一部分，其中包括关于这一实践的更深入的主题。

[**交互设计基础**](https://www.interaction-design.org/) **—** 这个网站是一个关于 UX 的信息宝库，包括一个很棒的[介绍](https://www.interaction-design.org/literature/topics/ux-design)和大量免费资源(文章、电子书等)。).

UX 工程师:我们是什么？最后一种资源有点不同。一个来自 UX 工程师的帖子解释了他们工作的来龙去脉。这提供了一个有趣的视角，让我们深入了解那些让 UX 成为他们职业生涯核心的工程师们的工作。
