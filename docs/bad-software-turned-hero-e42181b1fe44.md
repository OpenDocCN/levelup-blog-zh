# 坏软件变成了英雄

> 原文：<https://levelup.gitconnected.com/bad-software-turned-hero-e42181b1fe44>

## 这是代码库中的技术债务，也是架构中有争议的部分。从它诞生的那一天起，它就是被抛弃的代码。然而，它的存在为数字产品的生存和成功创造了条件。这是一个糟糕的软件组件成为技术转型之旅的英雄的故事。

## **以前的世界不好**

变坏之前的世界是一个变化的世界。有问题的软件——那个后来被坏人占据的软件——已经存在多年了。然而，它属于一个在这些年里重要性急剧增长的应用程序。它的重要性已经上升到了一个点，这个应用程序，曾经是不起眼的，开始被称为“关键任务”的数字产品。

所以世界要求这个数字产品进化。几乎是突然之间，新功能的积压就形成了。对于这个世界的团队来说，这都是积极的消息，除了他们可能无法以世界需要的速度交付这些功能，或者*甚至需要*它们。

问题是团队在处理巨石柱。他们的软件和架构由三个巨大的不可分割的块组成，通常被称为独石。这些东西携带起来很重，很难打碎。更糟糕的是，这些都是被技术债务困扰的巨石。

![](img/587c47c84846a7e73f44dc58437331c3.png)

一堆巨石

这支队伍有多强并不重要。如果有更多的人加入到团队中来也没关系。举起或打碎巨石几乎是不可能的壮举。

## **存在有多糟糕**

这个团队很清楚，整块巨石需要让位于一组尺寸更小、可塑性更强的材料。当团队在他们的数字产品中构建新功能时，他们可以单独制作、重塑和润色这些作品。

所有这些实际上都不是一个新的挑战。许多世界中的许多团队以前都经历过同样类型的挑战。多年来，他们给这些产品起了反映其规模和在软件栈中位置的名字，如微服务、纳米服务和微前端。

然而，回到那个可能会被认为是坏事的世界，一次性解决这种转变是一项过于复杂的任务。如果不够仔细，仅仅对其中一个模块进行简单的更改，就有可能破坏这个任务关键型产品。改变这些巨石，把它们变成别的什么东西，被认为是极其危险的，也是不可能的。

因此，团队没有更换巨石柱，而是选择了替换它们。然而，分阶段逐步过渡仍然是一个目标。

需要某种东西来允许这种逐渐的转变。这种东西可以在旧的部件被替换的时候把新的部件粘在旧的部件上，而不需要改变旧的部件，也不需要破坏新的部件。一些允许数码产品的飞机在飞行中被重建的东西。

会有不好的事情发生。

![](img/e50f0697c6c78d1400cc7a6a52db0ae8.png)

坏在它的街区，建筑形式

Bad 被认为是一个整体。然而，与这个软件堆栈中的其他块不同，它们生来是好的、小的和简单的，然后成长为巨大的，坏的从一开始就被有意设计为一个大的坏块。一个是被创造出来的，服务于一个短暂的目的，然后被处理掉。这并不意味着亲切。人们并不指望它有什么魅力。它是为它的使命而设计的，没有别的。

不像其他的巨擘，技术债务不会像癌症一样在坏账中生长。它从出生那天起就在巴德的身体里。

当它来到这个世界时，Bad 只是简单地将自己插入现有的巨石堆中，坐在那里，实际上什么也不做。这是两块巨石之间的新胶水，只不过它们已经很好地连接在一起了，并不真的需要粘合。

![](img/c2e137ab31d2f702649d757becf7cedd.png)

Bad 插入了巨石堆中

在其生命之初，坏是坏软件和架构中有争议的组件。它只能被视为恶棍。

它辉煌的时代还没有到来。

## **活得多惨，怎么成了英雄**

Bad now 出现在世界上，静静地潜伏在古老的巨石之间，团队开始了他们的转型之旅。

第一站是更换烟囱底部的大石块。

![](img/461e440aec49e9c5e7af4b1dd4e62edf.png)

基地是第一个目标

按照计划，该团队生产了新的、更小的、更具延展性的材料来取代基础整块材料。

但是，在转型过程的这个阶段，他们如何用新的部分替换底部的一块巨石，同时保持顶部的另外两块巨石完好无损地悬挂在原处呢？这怎么可能按计划逐步实现呢？

顶部剩下的巨石甚至不会与新的、更小的、有延展性的部件连接，这些部件是用来代替底部的巨石的。那些被设计的形状和尺寸不适合顶部的旧的大块。他们并不想为了实现这种连接而牺牲自己的设计。损害这些部分的设计会破坏改造的目的。此外，另一种选择——改变任何旧的巨石，使其与新的部件连接起来成为可能——将违背一开始设定的风险规避原则。

该团队希望在转型之旅中大胆尝试。但是，与通常大胆的冒险态度相反，该团队想不惜一切代价避免风险。他们的旅程可能吗？

嗯，不可能是英雄航行的大海。就在那段关键的旅程中，Bad 掌舵，让团队航行在不可能之海。

坏人是使这一渐进的、避免风险的、大胆的旅程成为可能的英雄。

![](img/e2b2c8b477a16b07749c9aa3ff93e8e3.png)

如何让不可能变成可能

但是，它是怎么做到的呢？

Bad 的顶部表面与顶部剩余整体堆栈的底部完美匹配。所以在旅程的这个阶段，坏继续保持自己与它紧密相连。顶部的巨石不受干扰地经历了基础的替换，完全没有意识到脚下的世界正在改变。

更重要的是，Bad 的底部表面最初也与将在第一阶段改造中被替换的基础整体顶部完美连接。但随着转变的发生，Bad 逐渐改变了表面的形状，停止了与基础整体的连接，并开始与团队生产的新部件连接。Bad 最英勇的行为是允许新的碎片安全进入，并一点一点地接管基础巨石。直到有一天，底部的巨石最终消失了，新的碎片完全控制了底部。

## **糟糕的结局**

成功接管基地后，该团队此时已准备好进入转型之旅的最后阶段，目标是两座顶级巨石。

![](img/4c404985a8ef8340a744e84f9d34eb16.png)

在那个阶段，Bad 继续帮助他们的数字产品保持活力，将原本不兼容的旧头部和颈部连接到新的躯干上。

然而，坏本身并不意味着长寿。该团队设计了新的部件，旨在取代两个顶部的巨石，以某种方式连接取代底部巨石的新部件，而不是坏部件。他们以一种可以绕过坏的方式来设计新的部件。

我们的英雄乐于成为多余的人。它为自己促成的革命感到成功和自豪。

Bad 在一次非常成功的转变之旅的最后一天平静地离开了人世，因为团队把新的顶部部件放在了适当的位置，并移走了剩下的两块巨石。

![](img/c25c6c840dddaf7b7f398b3ddf9f70e6.png)

接下来的日子里，团队交付新功能的能力有了令人难以置信的增长，最终导致了他们数字产品的成功。

在我们英雄的墓碑上写着团队留下的以下话语:

> R.I.P 光荣坏了。你将被铭记，因为你短暂一生的技术债务随着我们永远的感激而结束。
