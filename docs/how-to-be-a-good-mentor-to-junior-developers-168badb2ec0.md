# 如何成为初级开发者的良师益友

## 指导初级开发人员不是一件容易的事情，如果做错了，弊大于利

![](img/6a1537c600b7a2e4536d5c8e329ced83.png)

弗朗西斯科·加拉罗蒂在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的照片

如果你在软件开发方面有些经验，并且你在一个团队中工作，有可能在某个时候你不得不成为一个经验不足的开发人员的导师。

指导不是负责和命令人们做某事。辅导应该关注成长。一个好的导师应该专注于让自己不被需要。

在这篇文章中，我想根据我的个人经历和我在其他成为导师的人身上看到的东西，提出一些我的指导建议。

# 1.设定现实的期望

设定期望值是非常重要的一步，但不幸的是，这一步并不容易。你必须知道你对初级开发人员的期望是什么，否则你很可能会在错误的方向上浪费大量时间。

期望值不能太低或太高。你应该清楚地说明你期望一个低年级学生最终能够做什么，或者他们应该能够扮演什么角色，我说的“最终”是指他们不再需要那么依赖你，并且可以自己有效地学习。

尽量避免类似“*我希望大三学生成为优秀的开发人员*”的事情。这种说法没有明确的方向，太笼统了。相反，你应该问问自己什么样的特征能定义一个好的开发人员，这样你就有更具体的目标去努力了。

与下级一起决定最终目标是很重要的，在他们想要的和团队需要的之间应该有一个很好的平衡。

# 2.不要拿学弟学妹和自己比

你是他们的导师，因为你更有经验，所以他们比你慢是很自然的。他们不会做出和你一样的决定。

不要说“*我能做到，因此，你也应该做*”，因为不，他们不能马上做你做的事情，因为如果他们能做你做的事情，你为什么还要指导他们？

不要把他们的学习速度和你自己的相比。重要的是他们以团队可以接受的速度学习。

将他们与你相比，你冒着摧毁他们自信的风险，因为他们可能会觉得期望值太高。

# 3.给你的学员自由

自由至关重要。没有自由，初级开发人员将对自己失去所有的信心，没有人希望这样。

你应该让他们自己做决定。他们会犯错误，这是不可避免的，但是，犯错误是一个非常宝贵的经验。每个人都会犯错误，要求初级开发人员不犯错误是不人道的。

为了允许他们犯错误，他们需要做一些时不时犯点错误是可以接受的事情，所以尽量避免给下级做系统中关键部分的工作。

授权学员做决定。我不是说你应该积极地告诉他们，他们现在可以做任何事情，因为，事实是，他们不能，这就是为什么他们需要指导。

最重要的是，不要为他们做所有的工作，这对任何人都没有好处。

# 4.不要吹嘘你自己

指导的目的不是让你吹嘘你的技能和成就，所以，除非你被问到，否则最好不要过多地谈论它们。

低年级学生可能会把自己和你相提并论，向你看齐。如果你不断告诉他们你能做什么，他们可能会不知所措，开始怀疑自己。

你不应该表现得像“无所不知、人人敬仰的人”。

# 5.注意他们正在做的好工作

如果你不想让他们失去动力，就应该给予积极的反馈。

当你看到一个学弟学妹取得进步或者做了好事，告诉他们。

当一个人在学习并尽最大努力提高时，他们希望自己的努力能被人注意和欣赏。

想象你站在他们的立场上，想象你在学习上付出了很多努力，只是为了让你的导师说“好吧”然后走开。感觉不太好，是吗？所以不要没有任何反馈就走开。

# 6.对你的学员要真实

负反馈也很重要。你不想粉饰他们。诚实是最好的策略。

如果他们没有达到团队的期望，你应该对此保持透明。你撒谎越多，你的学员发现真相的时候就越不舒服。没有人喜欢被骗说做得很好。

尽量不要表现得刻薄或消极。

# 7.要耐心

有耐心并不意味着你愿意给初级开发人员他们想要的那么多时间，毕竟，你雇佣他们不仅仅是为了让他们成为一名学生。

只要他们的进度符合你的团队的期望，那就没问题，如果不符合，那就该分道扬镳了。

有时候你可能只是心情不好或者发生了让你心烦的事情，在这种时候很难保持耐心。如果你因为某种原因无法联系到他们，你应该告诉他们，不要无缘无故地抨击他们。

# 8.检查他们

没有人喜欢独处，尤其是初级开发人员。低年级学生可能不明白他们需要做什么。重要的是把他们引向正确的方向，最好不要为他们做太多的工作，也不要给他们留下自由的空间。

初级开发人员可能对他们的技能感到不安全，并且因为害怕成为团队的负担而犹豫问问题。不时检查他们，问他们是否一切都好，或者他们是否需要任何帮助。提醒他们寻求帮助是可以的。

不要抛弃他们，因为那将是你的文化的一个非常坏的例子。你不想让他们觉得团队合作在你的工作场所无关紧要，每个人都要靠自己。

如果你没有注意到他们，也没有检查他们，他们最终会失去对你的信任。

你也不想过多地检查他们，因为这可能会给他们留下这样的印象:他们自己什么也做不了，总是需要你的帮助，这也会扼杀他们的动力和自信。

# 9.与他们相关

如果你想让下属在你的团队中有宾至如归的感觉，你应该了解他们的经历。人们希望被理解，如果不被理解，他们会开始感到被疏远。

在某些时候，你处在他们的位置，知道他们现在正在经历什么。人们喜欢分享他们的经历，所以当一个晚辈告诉你他们的经历时，你应该反映出来，当他们说类似“*哇，这部分真的很难处理*“不要像”*嗯，对我来说很容易做到，这没什么*，因为如果你说这样的话，他们可能会开始对自己的表现感到糟糕，并避免在未来与你分享，因为你不关心他们的经历。相反，你可以这样说“*是的，这很难，我明白你的意思*”，因为这样你认可了他们的经历，他们也觉得被理解了。

与他人交往有助于与他们建立更好的关系，这在团队中非常重要。

# 10.不要创建垂直关系

什么是纵向关系？在这种关系中，某人被置于他人之上，比他人更重要。

你不想给人留下这样的印象:你的决定更重要，你对这件事有最终决定权。你应该尊重初级开发人员的意见。认真对待他们的话，如果他们没有意义，解释原因，而不是说“必须这样，因为我有更多的经验”。

在一个职能团队中，成员们感到同等重要，并且可以自由地表达自己。

向你的学员展示你关心并尊重他们的意见。

# 11.给出建设性的建议，并给出清晰的解释

解释很重要。作为导师，你的任务不是盲目地告诉你的学员该做什么。

你的任务是解释你的建议。他们为什么要照你说的做？你应该能够清楚地解释为什么你在做你正在做的事情。尽量避免基于观点的建议，因为你无法用实际例子来支持这些建议。

如果你不解释你的决定，低年级学生将不会理解他们应该做什么，如果没有这种理解，当他们应该学习什么、如何学习和为什么学习时，他们将开始试图记住他们需要做什么。

缺乏理解会导致自我怀疑，这会阻碍后辈的发展。

做导师不是一件容易的事，如果做得不好，会在人与人之间造成不好的工作关系，最终会浪费很多时间和金钱。

重要的是要给学员足够的自由，让他们自己行动，让他们有能力做决定，欣赏他们正在做的工作，而不要粉饰他们。

找到正确的平衡是一个好导师和好领导的关键。如果没有一个好的导师，初级开发人员可能需要很长时间来提高，而且代价可能很高。
