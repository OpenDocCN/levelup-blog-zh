# 如何免费运营一个网站

> 原文：<https://levelup.gitconnected.com/how-to-run-a-website-for-free-398df73cfd56>

## 并且以安全、可靠和可扩展的方式实现

![](img/dd9859aabc22d459ebd62e868f1ef1ce.png)

每个人都想要一个网站，但是没有人愿意为此付费。我们生活在一个有趣的世界，不是吗？每个人在必须使用某样东西的时候，都希望所有东西都是免费的。然而，当他们必须努力实现这一目标时，每个人都希望为他们的努力获得大量现金。好吧，除了这一点，事实证明，当涉及到拥有一个在线网站时，你实际上可以免费拥有它。

在这篇文章中，我将只专注于运行网站，托管它，而不是其他任何事情。**如果你想要一个域名，这显然是不可避免的——尽管成本极低**，而获得免费模板就像在谷歌或 DuckDuckGo 上搜索*“免费网站模板”*一样简单。那是你自己肯定能弄清楚的事情。

然而，似乎不太直接的是托管部分。许多小网站和网络应用程序的所有者没有意识到的是，虚拟主机不需要付费，它完全可以是免费的。不知道这一点，许多人决定放弃他们的网站早期由于财政拮据，放弃了潜在的建立自己的在线随着时间的推移。太可惜了。

## 在 Firebase 上托管您的站点

很少有人知道的是，Firebase 是谷歌的产品，而且已经有好几年了。虽然出于种种原因，我并不特别喜欢谷歌，但 Firebase 是一个例外，多年来一直是一个发展迅速的平台，文档也有了很大的改进，实际上它是 Amazon Amplify 的替代产品。

[](https://medium.com/codex/the-sole-reason-why-all-google-products-are-inferior-to-apple-ec58955a20ec) [## 所有谷歌产品不如苹果的唯一原因

### 不，这不是一个苹果粉丝的故事，它远不止如此…

medium.com](https://medium.com/codex/the-sole-reason-why-all-google-products-are-inferior-to-apple-ec58955a20ec) 

如果你想知道为什么我不推荐 Amazon Amplify，这其实可以归结为一件事——易用性。就我的经验而言，Firebase 在让你的网站或应用快速、轻松上线方面远远胜过亚马逊的替代品。你所需要做的就是用谷歌账号登录 Firebase 并创建你的项目，这是一个非常下一个完成的过程。

做任何事情，尤其是托管一个网站，都需要文档。虽然这确实需要一些技术知识，**即使是那些完全不熟悉命令行的人，如果他们给自己分配一天时间来完成所有事情并发布他们的站点**，这仍然是完全可行的。那些只具备最低限度的软件开发或命令行技能的人将在几分钟内上线。

那么，为什么**是免费的**？嗯，Firebase 提供了一个有一定使用限额的免费层，这对于中小型个人项目甚至更大的原型来说是非常慷慨的。我已经在 Firebase 上托管了[我的网站](https://attilavago.com)好几年了。

另一个好处是**速度和正常运行时间**。虽然大多数托管公司将保证 99%-99.9%的正常运行时间，但当事情发生时，需要一段时间才能恢复。将你的网站放在谷歌的基础设施上意味着你的网站宕机的可能性更小，恢复时间更快，而且由于他们的分布式云，从世界任何地方访问你的网站都将是最快的。

如果你突然发现自己想要用一个更复杂的网站或网络应用程序来扩大自己的影响力，Firebase **也是一个潜在的可行选择。**它可以根据您的需求进行扩展**，当然，当您决定进行扩展时，您会发现自己不得不转向付费层级。**

首先，[这里是官方文档](https://firebase.google.com/docs/hosting)，如果你在 YouTube 上搜索 [*“如何在 Firebase 上托管一个站点”*，你还会找到一个很长的视频列表来帮助你完成这个过程。](https://www.youtube.com/results?search_query=how+to+host+a+site+on+firebase)

## 在 GitHub、GitLab、BitBucket 上托管您的站点

软件工程师传统上非常熟悉的另一个工具是版本控制工具。然而，这并不意味着其他人不能充分利用这些工具。虽然我没有看到普通的 Jane 和 Joe 为他们的小型个人网站创建 Git 分支、拉请求和合并内容，**在这些平台上的公共项目的*【隐藏】*好处之一是能够在它们上面托管您的网站**，并使它对任何访问者都是活的，就像您在传统主机上一样。

当然，现在有些人可能会说，没有所谓的*【免费午餐】，在某种程度上，这是真的。虽然之前提出的解决方案——Firebase 有一个你不能超过的配额——GitHub、GitLab 和 BitBucket 将限制它们允许作为直播网站托管的文件。那也全是静态文件，所以 HTML，CSS，JS，所以只有前端代码。这对普通用户来说意味着**你不能在上面托管像 WordPress 网站**这样的东西，尽管 Firebase 也是如此。*

*我为您整理了一个简短的文档列表，针对所有三种解决方案:*

*   *在 BitBucket 上免费部署和托管您的站点。*
*   *在 GitHub ( **非常流行**)上免费部署和[托管你的站点。](https://pages.github.com)*
*   *在 GitLab ( **流行**)上免费部署和托管你的站点。*

***请注意，对于以上任何一种情况，在 YouTube 上搜索视频都会给你一个长长的列表，你可以从中进行选择，引导你浏览。***

## *为什么这一切都有意义…*

*传统的托管公司从那些根本不需要托管服务，只需要域名的人身上赚了很多钱。他们的许多客户也是个人、自由职业者和小公司，他们只追求一件事——可靠、专业的在线形象。他们想把他们潜在的关系、客户和顾客发送到一个合法的网站，验证他们是谁和他们做什么。因此，这些解决方案是完美的。*

*更好的消息是，任何网站建设者，无论是网络服务还是你电脑上的软件，只要能把网站输出到代码中，都可以很好地托管在这些解决方案中，完全不需要任何成本。同样的道理也适用于下载并编辑过的模板或你让朋友快速拼凑的东西。*

> *底线是，如果你不得不付钱，那么花在定制设计和一些轻量级网站开发上比托管更有意义。*
