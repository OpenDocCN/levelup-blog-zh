# 称自己为高级开发人员所需的 7 个品质

> 原文：<https://levelup.gitconnected.com/the-7-attributes-you-need-to-call-yourself-a-senior-developer-141fa46c11f8>

## 没有特定的界限可以跨越

![](img/50b6d565cd59f474d35e00001c6d1a84.png)

[作者](http://www.arnoldcode.com)和 Canva.com 制作的图片

经验是关键，但还不够。那么不可避免的秘密是什么呢？

我在生活中遇到过一些开发人员，他们中的一些人无疑已经老到可以被称为高级开发人员，而一些高级开发人员缺乏被称为高级开发人员的主要属性。

最近我写了一篇关于[为什么初级和高级这样的术语对一个项目不好](https://betterprogramming.pub/the-terms-junior-and-senior-developer-needs-an-overhaul-heres-a-better-model-a6e9c0ae49cf)并且应该用另一种尺度来衡量。尽管如此，高级这个词包含了一些我们或多或少都知道的基本原则。

但实际上，你什么时候成为高级开发人员？有没有过线？
*我不记得醒来，有人告诉我:“就是今天！”你知道吗？*

## 我认为阈值在这种情况下是无效的

由于这些条款没有正式规定，每个人的衡量标准都不一样。在你目前的公司有这个头衔并不一定意味着下一个公司会用同样的尺度来衡量。

这取决于你能提供什么，至少是我在过去几年里通过与许多开发者的交谈所看到和听到的。

*如果没有具体的要求要完成，那怎么办？*

在与其他经验丰富的开发人员交谈时，我收集了一些高级人员共有的属性。此外，在过去的 8 年里，我有自己成为一名开发人员的道路。

# 任何高级职员都应该具备的品质

不要强迫我把它们都拥有。我确信你应该有其中的大部分。

## 1.经验是你立足的基础

这是迄今为止最重要的一个。

没有经验，你就没有机会拥有任何其他可以证明你是高级开发人员的品质。难道不是这样吗？

如果你看到了地平线，但从未走过，你怎么知道那里有什么？走过从一个想法到公开发布的整个过程。拥有数百名用户或多年开发的工作应用程序。初级开发人员只能看到，但无法体验完整的开发过程。

一个学长可以。为什么？因为他一生中至少做过一次。这就像他们的第一次性感时刻一样铭刻在他们的脑海里。

一个已经看到一些的开发者可以期待一些。经验是未来决策的驱动力。

## 2.编码是琐碎的，做出正确的决定变得至关重要

一旦你知道了一门语言或一个技术栈，你就进入了元编程。

这种语言进入了你的 DNA，你像写文章一样写下代码。你会大量考虑在架构层面上解决问题。*哪种设计模式最能解决这个问题？哪种架构最适合该公司想要解决的业务案例？*

你的工作主要包括头脑风暴、制图、阅读、比较和评估，以看到编码之外的东西。执行工作系统，而不是工作方法/功能。

在*值得信赖的*高级开发人员开始编码之前，他们会做研究。一旦解决方案是防弹的，他们就开始钻研代码。

尽早检查你的选择。初级者倾向于直接从代码开始，看看他们已经走了多远。就在后面，“如果有效，那就是好的。”

## 3.对自己的行为负责

经验把你带到了这里。你有能力做决定。

即使结果证明他们是错的，你也能无所畏惧地承担责任。作为一个高年级学生，你知道复杂的结构和 VCS，所以改变它。如果它坏了，你知道你要做什么。

低年级学生可能害怕改变复杂结构中的一行代码。然后，我已经经历过了，像一只无头鸡一样跑来跑去，尖叫着求救，感到绝望。

## 4.你打败了邓宁·克鲁格效应

你*知道*你不知道的。

你已经成功击败了邓宁克鲁格效应，知道你面前的是什么，并能为此负责。

说到采取行动，打破一些工作代码。您在运行应用程序之前就知道结果了。因为你知道你不知道的。有足够多的迷因显示了初级开发人员如何成功地编写代码，并自信地认为自己是母鸡派对上的唯一男性。

当然，学长不是超人，什么都知道。他们还需要谷歌一些琐碎愚蠢的问题。有时候我甚至会谷歌“如何在 javascript 中生成固定 2 的数字”。但这不是重点。低年级学生不知道他们面前有什么，因此也不知道他们需要学习多少。

高年级学生看到他们已经完成的事情，并意识到未来的事情。因此，如果你有诚实承认你不知道的属性，你就更近了一步。

## 5.知道如何在各种情况下交谈

不管你是否不得不拒绝一项任务，协商更多的时间，或者要求重构的资源。你大声说出来。

你为准则说话，因为没有人这样做。最后，你是在为团队和产品说话。你是源代码质量的重要支柱。你可以把自己看成源代码巡查员。这是我们给一个老同事起的名字。他检查了每个提交并重构了它。有时我们没有意识到我们的代码，因为他已经把代码重构得非常好了。

此外，如果你教授或指导低年级学生或其他开发人员，你会有一些东西可以分享。传播知识是只有先进的人才会做的事情。

如果你知道你在说什么，而且另一个人也明白，你就离被称为高级开发人员更近了一步。

## 6.接受错误是你的

没有人免于失败。即使是专家也会犯错。

如果你有勇气承认它们，你可以从中学习。而且也没有错。我承认的每一个错误都严重影响了我作为开发者的学习。最好的学习效果有那些让我感到惭愧。

羞耻是一种过了一段时间就消失的感觉。留下来的是对自己做错了什么和可以做得更好的问心无愧。

每一位老板，甚至是和我有合作关系的公司，总是乐于及早发现错误。所以在生产前有足够的时间修复它。你也许可以掩盖错误，但真相总会浮出水面。

*有没有你。*

## 7.深度学习

知识是好的。知识固然很好，但远远不够。

深入了解一个技术栈也不能让你被称为高级开发人员。相反，如果你能传播你的知识，教导他人，并被他人视为解决他们问题的人，那么你就真的成功了。

至少我和很多资深开发者谈过，我认为。

# 最后的想法

**我坚信“初级”和“高级”这样的术语不太适用。**

没有什么大不了的事情需要跨越。更要命的是，每个公司都有叫你学长的标准。我曾经申请了一份他们不接受我作为大四学生的工作，对此我很沮丧。

现在你知道了 7 个特质，你可以用它们来说服那些怀疑你是学长的人。
