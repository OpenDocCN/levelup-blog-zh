2018 年，HackerRank 发布了其[学生开发者报告](https://research.hackerrank.com/student-developer/2018)。它发现超过 65%的新开发人员是自学的。

随着新资源的出现，这一比例可能会增加。有了所有这些资源，选择可能是压倒性的。如果你想开始编码，你从哪里开始？

这里有五件事可以帮助你。

**#1:从为什么开始**

有创造事物的编码，也有获得工作的编码。

没有对错之分。区别在于动机。

内在动机是执行一项任务，因为它对你有好处。外在动机是因为外在因素而完成一项任务。这些因素包括避免惩罚或获得奖励。

许多研究表明内在动机会带来更积极的结果。

想想你的理由。为什么要开始编码？对自己诚实。你正在设法得到晋升吗？你想转行吗？要不要打造下一个大 app？

您的答案可以决定学习哪种编程语言，以及它需要的承诺。

例如，如果你的梦想是创建一个操作系统，你可以从正规的计算机科学教育中受益。这将教会你复杂的主题，如算法和数据结构。

如果你是一名希望进入技术领域的专业人士，编码训练营可能是一个不错的选择。

如果你想建立网站，YouTube 教程可以帮助你。

原因可能多种多样。

**#2:从小处开始**

一旦你找到了原因，你就可以准确地找到学习哪种编程语言。

虽然没有单一的“最佳”编程语言，但有些语言比其他语言更用户友好。HTML 和 CSS 被认为是进入编码世界最容易的入口点，但是它们的用途是有限的。除了设计网站，他们做不了什么。

对于涉及支付系统或数据库的交互式网站，你需要了解 Javascript 和 SQL。移动应用开发者使用 Swift 或 C 制作 iOS 应用。视频游戏开发商求助 Unity 来实现他们的想法。

还是决定不了自己自学的编码语言？大多数专业人士建议学习 Python、C#或 JavaScript，因为它们提供了最大的灵活性。最重要的是，开始吧。

一旦你熟悉了一门编程语言，你就能更快地学会下一门。就像学一门乐器。

**#3:尝试在线课程**

如果你想更好地控制你的自学编码，在线课程可以帮助你。

然而，有许多班级教授同样的语言。很难找出哪些是值得你花费时间和金钱的。

如果你不介意练习独奏，可以去看看[的奥丁项目](https://www.theodinproject.com/paths/foundations/courses/foundations)。这个免费选项汇集了一些最好的开源内容，用于将编码新手变成自学的程序员。

安吉拉·于博士的 Udemy 课程也是不错的选择，从零到精通的课程也是不错的选择。甚至全日制训练营和计算机科学学生也报名参加这些课程来补充他们的学习。

无论你完成了多少门课程，许多初学者都发现很难应用他们的知识。为此，请查看来自 [Watch 和 Code](https://watchandcode.com/p/practical-javascript) 的免费实用 JavaScript 课程。它围绕着一个你不断迭代的项目。

**#4:保持一致&奖励自己**

要具体，要一致。每天一个小时的学习，胜过一天 15 个小时。如果你能把编码融入早晨的日常事务中，那就去做吧。

此外，设定每周目标，让自己负起责任。研究表明，有意识地设定目标的人更有可能实现目标。那些额外努力写下这些目标的人最有可能完成任务。

你希望在月底能够做什么？你将如何安排你的日子来实现这个目标？

把这个写下来。

你也可以奖励自己。比如，在你的房间里放一块白板，每天编码的时候划一个叉。在不破坏链条的情况下，尽量在白板上画出更多的 x。

“你不需要全票就能赢得选举；你只需要多数。”

“如果你为不良行为或不良习惯投了几票，这并不重要。你的目标是赢得大部分时间。”—詹姆斯·克利尔

在一周结束时，如果你得到了七个十字，奖励自己。这就像你自己的忠诚计划。也许你可以买本书或者存起来买些特别的东西。

**#5:造东西！**

自学编码，很容易卡住。有人称之为“教程地狱”。在这里你可以一个接一个地看教程，而不会弄脏你的手。很多新手都会这样。

构建项目与学习阶段并不分离。最好是两者结合。这就是为什么我推荐以代码跟踪项目为特色的学习资源。(确保跟着代码走，而不只是看着！)

练习的一个方法是考虑可以添加到您已经编码的项目中的特性。用你获得的知识做点什么。

不一定要花俏。最重要的是你写代码，遇到错误，找出调试的方法。实践巩固知识。一个完成的项目胜过看十个教程。

谷歌是你的朋友。如果你不知道为什么你的代码坏了，就去寻找解决方案。你可能不是第一个犯错误的人，互联网上的某个人可能已经为你的问题找到了解决方案。只需将你的错误信息“复制并粘贴”到 Google 中，在短语两边加上一对引号，然后按回车键。

如果你还有问题，你可以在开发者论坛上发表你的问题，比如 Stack Overflow，Reddit 的 programming subreddit，或者 GitHub。

无论你完成了多少次编程研讨会，或者学习了多少种编程语言，你的编程技能都将在你的编程项目中得到证明。

虽然你的个人项目不需要非常雄心勃勃，但它应该是你想要全天候工作的事情。你要不断改进和扩大它的范围。

**外卖**

自学如何编码是你能做的最有回报的事情之一。

这里有一个让你行动起来的五个步骤的快速总结；

1.  **从为什么开始。“有活下去的理由的人几乎可以忍受任何事情。”—尼采**
2.  **从小处着手。一旦你学会了一门语言，你就能更容易地学会下一门。**
3.  尝试在线课程。按照自己的速度学习。
4.  **保持一致&奖励自己**。建立你自己的忠诚度计划。
5.  **造东西！**
