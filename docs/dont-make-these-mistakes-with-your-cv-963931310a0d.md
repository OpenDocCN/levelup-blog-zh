# 不要在你的简历上犯这些错误

> 原文：<https://levelup.gitconnected.com/dont-make-these-mistakes-with-your-cv-963931310a0d>

![](img/05a03bdabbd92a710c6ec75ae0444a79.png)

克里斯蒂安·埃尔富特在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的照片

啊，是的，又一年慢慢结束了。这意味着项目计划即将开花结果/失败。也许人们开始感受到改变的呼唤，心里想着 2023 年将是他们离开每天徘徊的地狱的一年。

或者，也许他们的公司被一个古怪的亿万富翁收购了，他决定根据去年写的[行代码](https://twitter.com/parismarx/status/1587966900516147200)——顺便说一下，这是一个病态的度量标准——裁掉一半的软件工程团队。

不管是什么原因，我们公司收到了大量的申请。因为我是团队中最资深的成员之一，所以我有幸和我的老板一起筛选所有的简历。这个过程被证明是非常痛苦的。

所以我想和你分享一些。主要是为了娱乐。但是谁知道呢？也许这里会有一个洞见。

# 背景

请允许我给你一些背景。

我的公司总部在德国。因此，我抱怨的一些事情在美国等其他国家可能并不相关。

此外，我们试图填补的职位是一个高级 Python 开发人员。因此，我们想快速确定能够展示相关经验的人，并邀请他们参加面试。

事实证明，这并不容易。

# 言贵简洁

我先这么说吧:我不喜欢看应用！这让我无法做真正的工作。

这显然是有必要的。我们的工作量正在增加，我们总是可以使用一些新鲜的肉。然而，当我不得不花 5 分钟以上的时间阅读简历时，我会有点不耐烦。

同样，我需要的只是你在过去 3 年中对 Python 的一些模糊印象。你进入了下一轮！

> 简历不是让你被雇佣的。它会让你被邀请去面试。

然而，我得到的是那些一丝不苟地写着他们参与过的项目的申请。一页又一页。在最糟糕的情况下，人们甚至不清楚他们在其中扮演了什么角色。

有些我超过了，因为它还不足以阅读。他们还添加了当前雇主的评估。用 8 号字体写了两页关于他们是多么伟大的员工的散文。至少我认为这是这么说的，因为我懒得去读它。

> 你想看起来很专业吗？然后，尊重阅读你的申请的人的时间。

# 光谱的另一端

也有截然不同的简历。

一个刚从大学毕业的家伙申请了这个职位，并投来了一份一页的简历，其中只包含了他刚刚完成学士论文的信息。

令人愤怒的不是他申请了一个几乎没有任何工作经验的高级职位。这可能是因为我们的招聘人员实际上是第一个接近的。不，这是令人愤怒的，因为这份简历没有任何内容。

“如果简历太长就不好，但如果简历太短也一样不好。看起来这个家伙永远不会快乐”可能是你现在的想法。所以请允许我详细说明我在简历中寻找什么。

# 你过去的经历

如果你想申请使用语言 X 的开发人员职位，那么根据你过去的经验，弄清楚为什么你是合适的人选。突出你用 X 解决的问题，你用过哪些库等等。

此外，对最近的工作要更加详细，对过去更长时间的工作要更加粗略。上一次相关的工作经历是在 15 年前，这并不真正令人信服。

如果你刚从大学毕业，这也是适用的。你可能没有任何工作经验，但希望有一些编码项目，你可以说出你使用 x 的地方。

既然我们已经确立了我希望看到的东西，那就让我们继续一些我不希望看到的东西。

# 告诉我一些关于你的事情

在德国，用一些关于你自己的话来结束你的简历是很常见的。

我的意思是列出了一些人们在空闲时间做的活动。我认为这里的想法是给简历背后的人一个印象。

然而，将人性这样复杂的东西压缩成五个名词似乎是徒劳的。

> 嗯，这份简历没有包含任何相关的工作经历，但她喜欢表演传统的波兰舞蹈这一事实很吸引人。我们绝对应该给她一个机会。

拜托，看在上帝的份上。如果你在考虑加入“观看网飞”的话…

不要！

真的，我是认真的。

作为一名员工，你认为哪种情况会让你更有吸引力？

*屏幕变得模糊，接下来我们看到的是一个软件工程团队的负责人陷入了沉思*

“我真该死。在我面前的是世界上有史以来最好的软件工程师团队。他们中的每一个人都有自己的专业领域，他们补充了其他主题的广泛概述。

他们是活生生的软件开发。醒着的每一个小时都花在写作、阅读或思考代码上。

见鬼，他们中的一些人甚至学会了清醒梦，这样他们就可以在做梦的时候继续解决工程问题。

但是没有一个人能给我推荐在网飞看什么。我越来越厌倦重新观看 IT 人群。"

从上面传来的喇叭声把他从思绪中扯了出来。他对这些超凡脱俗的旋律的来源感到困惑，于是走近窗户。

他看到的是云正在散开。他眯着眼睛适应天上的光辉。最后他能辨认出一个形状。这是万物的创造者向他伸出的手。

对于一个局外人来说，这一幕会让人想起“亚当的创造”。

但与图片中不同的是，手里拿着什么东西。神圣的东西。来结束他单调的狂欢。

这是你的简历。

*恶声恶气，回到现实。*

如果这是你希望的情况，那么祝你好运。

# 我真的很想在{在此插入姓名}工作

不确定这是不是德国以外的东西。但在这里，我们通常会在简历旁边附上一封信，让申请人对申请理由给出更详细的见解。(这在一定程度上有所改变，但我们经常会遇到这种情况)

我见过的大多数信件都有两个共同的特点。

一方面，阅读它们是一种享受。他们雄辩地解释了申请人的动机。为什么你的公司是理想的工作场所，如果有机会和你一起工作，他们会多么激动。

有时这些信是如此的感人，以至于我充满了深深的感激。因为我实际上是在为这家出色的公司工作。

要是没有第二个特点就好了:编写得如此一般化，可以适用于任何公司。

我明白为你申请的每家公司写一封独特的信是不现实的。不过，如果你决定添加一封求职信，那么至少要有一段是公司独有的。例如，参考工作描述中的内容。

然而，更好的是——尽管这会让那些求职信的代笔人热泪盈眶——不要为此费心。

# 最糟糕的还在后面

你知道我最讨厌看简历的什么吗？

当人们自己判断他们的技能时。甚至似乎有网站为你创建了这些概述。

只需选择你熟悉的技术，然后用 0 到任何数字来表示你的技能水平。

除了有时候它们的格式(字体和配色方案)如此可怕，以至于我一看到它们就害怕。他们也是绝对的 BS。

可能有足够多的研究表明，我们无法客观地判断自己的技能。明显倾向于高估自己的能力。

因此，这些概述只是毫无根据的说法。没有后援一文不值。

另一方面，如果我看到他们在过去的五年里一直作为 Python 开发人员工作。明确命名他们一直在从事的项目，以及所使用的库/技术。成为判断技能更可靠的依据。

底线:抛弃这些概述。它们只是噪音(让我的眼睛流血)。

# 结论

如果你像我对一些简历一样对这篇文章没有耐心，直接跳到最后，我想重复一下要点。

> *过去的经验是关键。*

列出你过去所有与你想申请的职位相关的经历。简洁而精确，去掉所有的噪音。

面试是你展示自己独特个性(以及你的网飞推荐信)的地方。

请在评论区告诉我你是如何处理简历的话题的，或者也许你处于类似的位置，并且在过去读过一些简历。然后也可以随意分享你的观点。
