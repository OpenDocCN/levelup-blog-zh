为什么编码标准很重要，以及如何衡量开发速度

软件工程

熟悉一个新项目需要多长时间

在我最近的一个项目中，我和我的团队被要求估计在我们的软件中实现一个新特性所需的时间。我们做到了，但是实施用了10天，而不是预计的3天。

我们对这个结果不满意，并试图找出评估和实际工作之间的差距的原因，我们注意到我们忽略了我们的一个开发人员需要的“培训期”,他是第一次参与这个项目。这不是我们错误估计的唯一原因，但是我想集中讨论这个问题，并回答我们项目经理的问题:“熟悉一个新项目需要多长时间？”这个问题是基于我从我作为开发人员、技术领导者和招聘人员的14年职业经历中的成功和错误中所学到的。

TP方程

我将训练周期(TP)视为以下等式:

TP = A+ B + C

其中:

  • A =项目源代码与编码标准之间的差距
  • B =开发者知识和编码标准之间的差距
  • C =开发人员学习技能(开发人员速度)

通过标准，我的意思是:编码风格指南、代码约定、干净代码原则、命名约定、遵循设计模式等。

A + B: 项目离标准越远，您就越应该对TP进行估计。换句话说，如果你的软件遵循编码标准，你只需要确认新的开发者正在掌握它们。

C: 如果等式中的第一个缺口(参数A)和/或第二个缺口(参数B)大于预期值，则第三个参数“学习技能”至关重要。

您可能认为缺少了一个参数:“代码文档”。我在很多项目中看到过；文档不是最新的，或者写得不好，或者很长，没有人想看。在这种情况下，我同意Martin Fowler的观点，源代码必须是文档或自我文档的主要来源。

显影剂速度

仍然有一个合理的问题:“如何衡量我的开发速度？”

这是一个好问题。开发速度不仅仅是速度，而是释放开发人员的创造力，将开发人员的想法快速敏捷地转化为软件，以支持客户和业务的需求。开发者速度意味着让开发者能够:

  • 富有成效地建设
  • 合作
  • 规模创新

根据我作为软件工程师多年的经验，我可以说，当我们想要评估某人的学习技能、新项目中的集成速度和开发速度时，有四个基本因素需要考虑:

  1. 热爱学习和探索(持续学习)
  2. 脆弱和谦逊
  3. 解决问题
  4. 与他人合作的能力(社交技能)

1.持续学习

「持续学习是持续学习新技能和知识的过程。[……]它包括自我激励和接受挑战。持续学习也可以是组织内部的，也可以是个人的，比如终身学习。”——持续学习

创新、尝试新流程或熟悉新项目都需要学习。当开发人员不热衷于持续的学习过程时，掌握新技术仍然是困难的，并且理解一个外国源代码需要很多时间。为了获得新的知识、想法和技能，他需要能够挑战自己。学习必须建立在灵活、按需和持续的基础上，以便有助于这种尖端的性能。

2.弱点

我读过的关于脆弱和学习之间关系的最好例证来自布琳·布朗。在她的书《勇敢无畏:脆弱的勇气如何改变我们的生活、爱、养育和领导方式》中，她写道:

“如果你想要一种创造力和创新的文化，在这种文化中，市场和个人层面都接受可感知的风险，那么首先要培养管理者的能力，培养他们对团队脆弱性的开放态度。这也许有些矛盾，但这首先要求他们自己也是脆弱的。这种认为领导者需要“负责任”和“知道所有答案”的观念既过时又有害。它对其他人的影响是他们知道的更少，他们比。这是我听过的规避风险的秘诀。羞耻变成了恐惧。恐惧导致风险厌恶。规避风险扼杀创新。”

3.解决问题

解决问题的过程是一个持续的活动，在这个过程中，我们利用我们所知道的，去发现我们所不知道的。它包括通过产生假设、测试预测和达成满意的解决方案来克服障碍。它有三个基本功能:

  1. 寻找信息
  2. 产生新知识
  3. 做决定

解决问题的前提是我能为自己的学习承担一些责任，并能采取个人行动来解决问题，解决冲突，讨论替代方案，并专注于思考。

创造性解决问题的本质是答案无处不在。。。也许我们甚至可以问问我们的学生。~ 马丁·宾克斯

4.与他人合作的能力

与他人合作是指与他人有效互动、合作和处理冲突的能力，以完成任务和实现共同目标。这包括与他人合作，例如同事、经理、员工、客户和承包商。

为了有效地做到这一点，需要:

  • 理解集团的文化、规则和价值观并在其中工作。
  • 联合规划和决策。
  • 谈判和妥协。
  • 表达自己的观点和想法，尊重他人的观点和想法，包括不同背景的人。
  • 在角色方面要灵活——知道什么时候扮演领导角色，什么时候寻求团队合作。

结论

优化培训周期不是遵循编码标准的原因，而是促进开发人员之间交流的原因。正如鲍勃叔叔所说:

编程是一种社会活动。程序员的第一要务是与其他程序员交流。

如果您的项目由于某种原因未能遵循通用的风格指南或惯例，那么您的开发人员的速度可能会改变游戏规则。

热爱探索、解决问题的技能和与他人合作的能力不仅是评估团队成员的速度和增加成功招聘机会的参数，也是可以在您的组织内提高绩效、员工满意度和在当今全球市场中保持竞争力的因素。
