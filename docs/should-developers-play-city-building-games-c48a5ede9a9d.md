# 开发商该不该玩造城游戏？

> 原文：<https://levelup.gitconnected.com/should-developers-play-city-building-games-c48a5ede9a9d>

我通常不玩电脑游戏，因为我的日程很紧，但当我玩的时候，我会玩策略。我利用复活节假期尝试了一个非常流行的城市规划游戏。我立即意识到许多策略和思维方式可以移植到我的软件开发经验中。

玩城市规划游戏能提升你的软件开发职业吗？

我将在本文中尝试回答这个问题。

# 膨胀

城市规划游戏的主要目标是建立一个城市，并将其扩展到合理的范围。城市的区越多，由于设计错误的积累，它就越难发展。换句话说，在一个有问题的城市解决问题可能会很有挑战性。

软件产品大同小异；目标是将它们销售给实际数量的客户。架构错误会延迟应用程序的开发。在过去，一些公司选择重写他们的产品，而不是修复它们。

城市，尤其是美国的城市，都是建立在网格模式上的。这简化了距离计算，并允许人们预测城市扩张。软件开发人员也使用模式——我们称之为设计模式。

# 复杂性

城市本质上是复杂的；市长需要考虑到人、建筑、交通、工作场所、水、电。我提到过医疗保健、警察或消防吗？管理一个城市的政府又如何呢？

产品也很复杂吗？因为他们需要许多人的合作，他们往往会变得复杂。现代公司包括产品经理、开发人员、设计人员、测试人员来构建软件。更不用说支持、销售或管理人员，他们也参与其中。

规划有必要吗？我会说是的，作为一个城市规划师是一个合法的职业机会。我可以对软件架构师说同样的话。如果你想控制复杂性，就做计划。

# 统计数字

没有任何指标，几乎不可能知道一个城市正在发生什么。需要建多少学校？嗯，那要看你所在城市的年龄人口统计了！

产品也需要度量，尤其是跟踪错误。了解性能也是合理的。更不用说健康的服务实例了。

度量可以告诉我们是否同时遇到了多个问题。

# 城市生命周期

人类、建筑和交通工具都有明显的生命周期。在生活中，一切都在不断变化。随着年龄的增长，我们需要不同的服务。一个设计良好的城市需要提供上述服务。

每个产品都有其生命周期。产品生命周期管理是一个商业术语；理解它可以帮助公司做出正确的决策。

应用程序中的操作可以触发一个可以被分析的循环。有些流程在完成前会经过多个服务。也许它们可以被简化或加速？

# 交通

没有适当的交通管理，任何城市都无法存在。如果救护车不能及时赶到，堵塞的道路对受伤者来说可能是致命的。人们应该能够在多种上班方式中进行选择。

当处理交通问题时，你应该总是寻找可能的改进，比如替代路线。理解道路堵塞的原因也很重要。

互联网流量也差不多，只是我们分析的是数据包而不是车辆。控制应用程序使用所产生的流量至关重要。不管怎样，谁想用一个慢的产品呢？

# 资源

在现在的城市里，一切都围绕着钱。以至于城市借钱来兑现对市民不兑现的承诺。产品世界有一种美丽的对称，不是吗？

一个城市需要赚钱才能稳定，才能负担得起必要的维护。产品需要赚取现金来让他们的公司修复漏洞并保持服务器在线。

# 模拟

我所知道的每个城市规划游戏都是模拟的。这意味着游戏只能作为真实世界的模型。这个模型的准确性取决于游戏创作者的创新和你电脑的计算能力。

我们，开发者，怎么称呼一个项目的模拟？

测试。

# 研究

我相信没有适当的研究，你不可能在城市规划游戏中获得好的结果。互联网上有很多关于如何解决常见问题的资料。有些人甚至为他们玩的游戏的每个方面都制定了指导方针。

我可以毫无疑问地告诉你，大多数开发人员都会就他们的专业问题咨询他人。有些人使用互联网，有些人使用书籍。我想每个程序员都应该知道如何熟练使用搜索引擎。

# 摘要

*玩城市规划游戏能提升你的软件开发职业吗？*我在本文开头就问过那个问题。

我的回答是——看情况。有经验的开发人员可能会以他们开发软件的方式玩这种游戏，先计划再编码。其他人可能倾向于冲进去，只是碰到一个障碍就退出。

总的来说，我相信你应该总是考虑尝试新事物来扩展你的视角和技能。
