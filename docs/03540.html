我们可以使用的基于JavaScript的GitHub Repos

原文：https://levelup.gitconnected.com/javascript-based-github-repos-we-can-use-a72e86d14c5d?source=collection_archive---------15-----------------------#2020-05-14

许多JavaScript库和框架都托管在Github上。

在本文中，我们将看看Github上我们现在或以后可能会用到的顶级库、框架和完整程序。

vuejs / vue

毫无疑问，Vue.js是创建新的前端应用或增强现有应用的最佳前端框架之一。

它是渐进式的，所以它可以和其他Vue库一起包含在脚本中，这些库也可以用脚本标签添加。

此外，我们可以使用Vue CLI制作新的应用程序，它有许多工具可供我们创建应用程序，并按照我们想要的方式构建它们。

图书馆的生态系统和使用它来构建应用程序的用户正在有规律地增加。

沉香/ Motrix

Motrix是一个用JavaScript编写的下载管理器。它支持通过HTTP、FTP、BitTorrent、Magnet等从各种来源下载。

它可以在Windows、macOS或Linux上运行。

panjia Chen/vue-admin-template

一个简单的Vue.js管理模板，我们可以添加到我们的应用程序。我们可以将它克隆到我们的系统中，并开始根据我们自己的需要进行定制。

zloirock / core-js

Core-js是一个为JavaScript标准库中的特性提供多种填充的库，比如新的数组和字符串方法等等。

包含的方法包括像from和flat这样的数组方法。此外，该库中还包含了类似数组的对象，如集合和贴图。

它还包括自己的polyfill for promises。

除此之外，最近几年JavaScript版本中包含的几乎所有东西都在这个库中实现。

然而，polyfills比内置在浏览器中的标准库要慢，所以我们应该只在我们的应用程序面向没有这些功能的浏览器时才使用它。

优赞/万特

该存储库拥有轻量级组件，可用于Vue.js应用程序。

它支持babel-plugin-import、自定义主题、国际化、TypeScript和服务器端呈现。

根据文档，有很多组件可用，包括幻灯片、按钮、单元格、图标、图像、布局容器、滑块、单选按钮、选择器等等。

vuejs / vuex

Vuex是Vue应用的首选状态管理解决方案。它允许我们为Vue应用程序创建一个中央数据存储，以便我们可以在任何组件之间共享数据。

这一点很重要，因为它使得组件之间共享数据比用props传递数据要干净得多。

mozilla / pdf.js

js是一个PDF查看器库，我们可以将它嵌入到前端JavaScript应用程序中。

它正在积极开发中，所以我们可以肯定新的功能和修复将被添加。

ember js/ember . js

Ember.js是一个JavaScript前端框架，不如Angular、Vue或React流行。但它仍在积极维护，并有一个活跃的生态系统，包括为它制作的库和使用它来构建应用程序的用户。

它也不难学，所以我们可能想用它来创建我们的应用程序。

zeit / hyper

Hyper是一个用HTML、CSS和JavaScript编写的终端程序。

因此，它也是一个跨平台的程序。我们可以在Windows、Linux或macOS上使用它。

它也有各种插件来扩展它的功能。

有扩展可以打开与当前标签目录相同的额外标签。

此外，有一个添加搜索，另一个添加更好的键盘快捷键导航。

zeit / next.js

Next.js是一个基于React的框架，它让我们可以轻松地创建静态站点和服务器端呈现的前端。

它有很多React没有的特性，比如数据获取、静态文件服务、路由等等。

为了使创建一个新的Next.js项目变得容易，有一个专门为它设计的CLI程序。

辛德索胡斯/eslint-plugin-unicorn

这是一个eslint插件，它给ESLint增加了更多的规则来增强它的林挺能力。

默认情况下，ESLint有很多规则，但它们可能不适合所有环境。所以这个插件给了我们另一个选择，那就是我们想要使用的林挺规则。

结论

我们可以用Next、Vue和Ember构建前端。

此外，我们可以构建完整的程序，如Hyper terminal程序，它是用HTML、CSS和JavaScript构建的，可以在任何地方工作。

PDF.js允许我们轻松地向页面添加PDF viewer小部件。
