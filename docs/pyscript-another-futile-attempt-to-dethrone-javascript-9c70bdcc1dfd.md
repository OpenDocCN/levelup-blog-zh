软件工程师感到无聊时会发生什么？嗯，这是个棘手的问题。软件工程师永远不会感到无聊。证据就是大量的编程语言、库、框架、编程范例、黑客和未完成的 GitHub repos。在这个冗长的软件工程尝试列表中，最新的一个是 PyScript ，它诞生于无聊、好奇和激情的健康结合，在 PyCon 2022 上宣布。显然，我们这些软件工程师没有被疫情、俄罗斯入侵和迫在眉睫的经济衰退分散足够的注意力，我们必须想出另一种解决方案来应对浏览器中 JavaScript 的*【不可接受】【统治】。*

*撇开讽刺不谈，作为一名软件工程师，我认为有必要更客观地看看王蒙杰在 PyScript 上的 38 分钟演讲中宣布了什么。**我会推荐看。**不管你同意不同意，都不是不被告知的借口，38 分钟还不到《实习医生格蕾》的一集。它也将——希望——阻止这篇文章结尾的激烈评论。*

## *耸人听闻的媒体认为 PyScript 做了什么…*

*我实话实说。我没有偶然发现 PyScript，因为我是一个认真的 PyCon 参加者。远非如此。我几乎不看 WWDC 的会议，我是一个苹果迷。说实话，**虽然我喜欢编程，但我真的没有时间让自己跟上一门语言的每一个主要发展**。根据我最近一篇关于 if…else 语句的文章，这意味着要跟踪十种语言，你知道吗？生命太短暂了。我宁愿在凤凰公园盯着一只在湖里游泳的鸭子，也不愿拖着自己去参加另一个编码会议。*

我偶然发现 PyScript 的全部原因是耸人听闻的媒体，它在媒体上就像在 YouTube 和其他任何地方一样流行。**我还有一种不可思议的能力，可以从眼角看到标题，上面写着某种语言突然消亡**，因为*用你能想到的任何愚蠢的理由来填补空白*。你猜怎么着？显然，JavaScript 现在因为 PyScript 而走向灭亡。因此，就像任何一个有责任心的兼职写作的软件工程师会做的那样，我调查了一下。他们称之为研究。我只是称之为满足先决条件，让我的科技文章在不知不觉中成为单口喜剧。糟糕的单口喜剧。

几十个 YouTube 视频和文章已经在宣称 JavaScript 的消亡。一些人认为这是世界末日，另一些人对此感到兴奋，但是两个阵营似乎都没有真正理解王蒙杰声明的真正含义。

一个阵营认为 PyScript 本质上使 JavaScript 变得毫无意义，我们应该抛弃它，用我们的 HTML 编写 Python。用一种老套的软件工程师口吻**不是因为你需要，而是因为你能，因为你能，所以你应该**。哦，亲爱的上帝，现在就杀了我吧！整个态度揭示了对 JavaScript 和 JavaScript 工程师的恶意、腐烂的仇恨和嘲笑，好像与 JavaScript 相比，Python 是一种“*真正的”*语言。突然，秘密泄露了，Python 所谓的优势被一劳永逸地证明了，它被称为 PyScript——JavaScript 棺材上的最后一颗钉子。

另一个阵营稍微好一些，但同样幼稚。不幸的是，这是数量惊人的软件工程师的特征，他们抓住任何新的闪亮的编码替代品的机会，准备抛弃多年的经验和学习。**我保证 GitHub 上一半未完成的项目是开发者分心的受害者，而不是别的。这些知识库的许多作者也是现在像跳跳虎(来自 Winnie the💩)，对所有他们可以放弃的未来项目感到兴奋，完全忘记了他们在过去三年中放弃的另外二十几个项目。*** 

## *PyScript 实际做的是…*

*任何花了 5 分钟时间尝试在本地编写 PyScript“Hello World”应用程序并花了 38 分钟观看演示视频的人都会发现，事情并不像前面提到的阵营让您相信的那样非黑即白。*

*当我查看 PyScript 网站时，**我有一种直觉，我以前见过这个**。不是 PyScript 本身，而是在浏览器中运行另一种语言，这当然促使我尝试一个“Hello World”应用程序。它以我预期的方式工作，但我需要证据，而获得所需背景的最简单方法是观看[王蒙杰的主题演讲](https://anaconda.cloud/pyscript-pycon2022-peter-wang-keynote)。大约 10 分钟内，我听到有人称自己为*“一个他妈的天才”*，因为我是对的。不，我真的不会谦虚。在不到 5 分钟的时间里，不用看任何在线或其他地方的文档，就能弄清楚 PyScript 是如何工作的，这让我感觉有点特别。只是根据以前的经验做出的一些有根据的猜测。是的，伙计们，**它使用 Web 组装，正如我所想的那样。在 12:17 有一个很好的幻灯片解释了这个**。*

这里有很多东西需要打开，但我不会打开，因为王蒙杰在这方面做得比我好得多，毕竟这是他的孩子，但有几件事需要强调:

*   PyScript 将 Python 带入了浏览器，它并没有取代 JavaScript。就像你已经可以在 WASM 中运行 C 程序一样，你现在可以用 Python 做同样的事情。
*   PyScript 得益于底层的 Web 组装架构，实现了 Python 和 JavaScript 之间的双向通信。实际上，Peter demos 的一些例子说明了这样做的好处。
*   PyScript 对于以真正无服务器和可移植的方式处理大量数据非常有用——所有这些都在一个 HTML 文件中。虽然你也可以构建一个待办事项应用程序，但它的好处只适用于那些因为不懂 JavaScript 而想用 Python 来构建的人。
*   PyScript 仅仅是为更多的软件工程师启用浏览器，而不是让浏览器摆脱 JavaScript 及其工程师。
*   PyScript 并不打算消除 JavaScript 解决方案和创新，而是将其包装在 PyScript 中。

在本地玩我的 PyScript“Hello World”应用程序，并以开放的思想和心灵观看视频——同样重要——使我能够超越第一印象*哦，上帝，不是另一个 JavaScript 杀手*。

我不一定同意彼得说的每一件事。标签肯定是不需要带回来的。这是一个可及性的噩梦，应该永远死去。我也不同意 Python 是为了可读性而设计的，或者他所说的*“适合你的头脑”*。它肯定不像 C、Dart 或 JavaScript 那样适合我。虽然确实很容易学，但我认为在这一点上，如果你想学的只是语言的语法，那么 90%的语言都很容易学。

> 然而，PyScript 确实带来了软件开发的一个重要方面——浏览器的包容性。

虽然我是 JavaScript 的粉丝，但我将是第一个承认粉丝也来自于在我的职业生涯中使用它最多的事实，如果我们可以有一个浏览器，允许更多的软件工程师为一个安全、高性能和可访问的 web 做出贡献，而不管语言如何，我们可能应该这样做。正如 Peter 所说，**世界上只有 0.3%的人口是软件开发人员。那就是 2500 万软件开发者**。数量少得令人难以置信，能为浏览器编程的人就更少了。让这个数字增长不是威胁，而是机遇。* 

## *我自己的问题的答案是什么…*

*所以，我们到了。虽然 PyScript 还在 Alpha 中，但它已经出现了，并且很可能会激发一些非常酷的项目。但是，除此之外，它的未来会是什么样子，可能介于王蒙杰的希望和耸人听闻的文章所声称的之间。不过现在，我只回答标题中我自己的问题。*

*   ***又一次试图废黜 JavaScript 的徒劳之举？不，PyScript 从来不是这样的。JavaScript 在浏览器中很受欢迎，因为它是所有主流浏览器中唯一的本地语言，而且几十年来一直如此。当然，它的受欢迎程度一直在增长，因为它背后有一个充满激情(而且往往很激进)的社区。理想的情况是，仅仅“废除”一种或另一种语言的概念应该从编程中彻底消失。我们应该专注于为人类构建伟大的、可访问的解决方案，而不是“君主化”语言。***
*   *PyScript 想表达什么观点？简单的一个。浏览器中可以运行的语言越多越好。它支持更简单的架构和更快的开发时间，同时还包含了更多不同的编程技能。*
*   ***纯天才？没有。我也不是真正的天才。我认为网络组装是天才的一点。其他人现在可以在它的基础上进行构建。虽然这是 Python 的一个聪明的解决方案，但它不是革命性的。***
*   *还是一种无用的时尚？肯定不是没用。我认为软件开发的最佳经验法则是*“为正确的工作使用正确的工具”*，PyScript 是另一个工具，拥有更多工具通常是一件好事。这也不是一种时尚。我可以看到 PyScript 的大量直接应用，随着我们的 web 应用越来越依赖于更接近金属的解决方案，它的使用将会增加，但它仍将是专门用例的工具。不要指望所有的新网站和网络应用程序都开始用 PyScript 编写。*

> *总结总结，我们有了一个新的编程工具。让我们欣赏它的本来面目，不要用另一场无用的竞争来玷污它的价值。*
