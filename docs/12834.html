我们为未来的自己购买债务。是回报的时候了

TL；DR:不要在你的代码中留下TODOs。修理他们！

问题

  • 技术债务
  • 可读性
  • 缺乏信心

解决方法

语境

我们在代码中遇到了TODOs。我们数了数。

我们很少解决它。

我们开始欠技术债。

然后我们支付债务+利息。

几个月后，我们支付的利息比原来的债务多。

错误的

public class Door
{ 
    private Boolean isOpened;    public Door(boolean isOpened)
    {       
        this.isOpened = false;
    }          public void openDoor()
    {
        this.isOpened = true;
    }    public void closeDoor()
    {
        // TODO: Implement close door and cover it
    }      }

public class Door

        this.isOpened = true;

结论

我们可以数托多斯。

大多数棉绒都这样做。

我们需要政策来减少它。

如果我们使用TDD，我们马上写下缺失的代码。

在这种情况下，TODOs只有在进行深度优先开发以记住开放路径时才有效。
