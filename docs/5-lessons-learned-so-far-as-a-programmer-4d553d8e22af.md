# 作为程序员的 5 个经验教训(到目前为止)

> 原文：<https://levelup.gitconnected.com/5-lessons-learned-so-far-as-a-programmer-4d553d8e22af>

![](img/716a705017619b9072263375234e0ebe.png)

编程是一项非常值得开发的技能。但是，作为一名自学成才的程序员，我发现学习过程不必要地模糊和复杂。我花了很多时间来弄清楚该学什么(以及什么时候学)，该使用哪些资源，以及如何学习大量的技术主题。

所以我总结了我从自己的旅程中学到的五个教训。我希望每一课中的思想和资源将为您提供一条更直接的学习和提高编程水平的途径。

# 1.学习编程的基础知识。

在我开始学习编程时，一条建议塑造了我的整个学习轨迹。“不要做我做过的事情，”一个程序员警告我。“不要专注于技术。”

在我对编程一无所知之前——甚至在我键入一行代码之前，我就收到了这个建议。我只知道我想学习编程，但是这位程序员的建议真的让我思考我该如何着手。我不再专注于技术，而是专注于编程的基础，尽管我当时不知道这些是什么，也不知道我该如何着手。

就这样，我开始了学习编程基础的间接追求。这是间接的，因为我必须弄清楚什么是基本原理，使用什么资源，以及如何着手学习大量的技术信息。

学习过程并不迅速。这也不容易。但我可以告诉你:学习基本面是必不可少的。如果我必须从头开始，我会从基础开始。有几个原因。

首先，当你学习基础知识时——包括解决问题、算法、数据库、面向对象编程和数据结构——你就建立了一套可转移的技能，这些技能将在现在和未来几年对你有所帮助。这是因为基本面不会像技术那样过时。史蒂夫·麦康奈尔(Steve McConnell)清楚地表明了这一点:

“你经常听到人们说软件开发知识有 3 年的半衰期:你今天需要知道的知识有一半会在 3 年内过时，”他解释道。“在与技术相关的知识领域，这可能是正确的。”

然而，麦康奈尔很快指出，“软件工程原理”没有“三年的半衰期”正如他所说，“这些软件工程原则可能会在职业程序员的整个职业生涯中为其服务。”

因此，当变化发生时，你更能适应。在编程这样一个快速发展的领域，变化是必然的。

你也会觉得准备更充分，因为你已经建立了坚实的知识基础，这是基本面很重要的另一个原因。例如，你会更有信心，因为你知道为什么以及何时使用递归。

第三，扎实地掌握编程基础可以帮助您更快更容易地学习技术。如果有的话，学习过程会更顺利一点，至少这是我的经验。

不要误解我的意思:构建应用程序和使用新技术很有趣。此外，我们需要在日常工作中使用技术。但是——这是重要的一点——如果你有坚实的编程知识基础，你会更有效地使用这些技术。同样，如果一个篮球运动员去体育馆练习投篮和运球技巧，他会打得更好。

我学习编程基础的迂回方式启发我创建了[程序员金字塔](https://programmerspyramid.com/)。这是我希望拥有的工具。这是学习基本面的直接方法。它包含您需要学习的主题和学习这些主题的资源。

# 2.学习方案。

学习计划是我做过的最好的编程决定之一。它会在两个方面帮助你。首先，它会让你真正熟悉递归。

在 Scheme 之前，递归是我只在必要时才使用的东西。老实说，我很害怕。但是我用计划克服了恐惧。那是因为 Scheme 中没有循环。所以你到处都用递归！

随着你对递归的熟悉程度增加，你的自信也会增加。你不会害怕递归问题，而是会拥抱它们。久而久之，我发现自己在寻找使用它的机会。

学习计划也会教你变得足智多谋。Scheme 是一种非常简单的语言。没什么大不了的。然而，您仍然可以用有限的工具集编写一些非常简洁的程序。学习如何少花钱多办事很有价值。

我最喜欢的一个方案资源是《小阴谋家[](https://www.amazon.com/Little-Schemer-Daniel-P-Friedman/dp/0262560992)*这本书。我不仅喜欢它的内容，还喜欢它的教学风格。它将概念分解成最简单的部分，然后向您提出问题。因此，你在给出信息之前就建立了联系，这有助于你学习内容。*

*Brian Harvey 和 Matthew Wright 的《简单方案》是另一本很棒的书:有大量的实践问题，解释也很清楚。 [Lab 10](https://inst.eecs.berkeley.edu/~cs61a/fa20/lab/lab10/#coding-questions) ，来自伯克利的精品课程 CS 61A:计算机程序的结构和解释，是获得 Scheme 一些额外练习的好方法。*

# *3.解决数学问题。*

*当我刚进入这个领域时，我问了经验丰富的程序员一个问题:“如果你重新开始你的编程之旅，你会做些什么不同的事情？”*

*我不止一次得到这样的答案:“我会花更多的时间学习数学。”*

*所以我就这么做了。当我开始学习编程时，我在重新学习数学概念和解决数学问题。出于三个原因，我今天还在坚持这种做法。*

*首先，解决数学问题是提高逻辑思维能力的另一种方法。其次，它提供了解决问题的实践，但在不同的背景下。我认为解决数学问题是一种交叉训练的精神形式。*

*例如，一个跑步者可能每周跑四次，其他时间通过骑自行车或游泳进行交叉训练。他们仍然在推动他们的心血管系统，但以不同的方式。解决数学问题也是如此:你以一种新的不同的方式提升了你的思维能力。*

*第三，编程中经常会出现基本的数学概念。因此，如果您已经有一段时间没有使用指数、倍数或除数等概念了，那么就有必要重温一下。*

*为此，我强烈推荐 [*解题艺术*](https://artofproblemsolving.com/) 系列作为数学资源。你会发现自己在实践中学习，而不是去记忆公式和定义:你会解决大量的问题。结果，你将开始发现模式并建立联系，这是你在解决编程问题时将用到的关键技能。*

*这个系列包含适合不同数学水平的人的书。如果你需要一个起点，可以考虑桑德尔·莱霍奇基和理查德·鲁斯奇克的《解决问题的艺术，第一册:基础知识》。*

# *4.研究别人的代码。*

*我已经在我的博客上的许多文章中提到了研究他人代码的重要性，理由很充分:这太重要了。*

*我们可以向很多优秀的程序员学习。在此过程中，您将:*

*所以在你解决了一个问题之后，再去研究别人的代码。这是低挂的水果，更多的人应该利用。*

*有很多方法可以做到这一点。我最喜欢的方法之一是运用本·富兰克林用来成为更好的作家的一种学习技巧。富兰克林会研究一篇来自他所欣赏的出版物的文章，并对每句话中的“情绪”做出暗示。过了一段时间后，他试图用自己给出的提示重现这篇文章。*

*富兰克林的技术应用于编程时是这样工作的:*

*   *解决一个问题。*
*   *找一个解决过同样问题的程序员。*
*   *研究他们的解决方案。通过在编辑器中编写注释来总结每一行代码。*
*   *过一段时间后重新解决问题。使用您输入的注释作为提示来指导您前进。*
*   *将您的新解决方案与您研究过的解决方案进行比较，并做出任何必要的更改。*

*LeetCode 和 Exercism 是用于此练习的很好的资源。两者都提供了一系列的问题，并且有你可以研究和学习的社区解决方案。AlgoExpert 是我另一个喜欢的解题平台。它为每个问题提供一个或多个解决方案。但是真正的好处是伴随每个问题的视频解释。它包含所提供解决方案的概念性概述和演练。*

# *5.向前支付。*

*从一开始，我就对编程领域的慷慨感到惊讶。因此，许多程序员愿意回答问题，审查代码，并创建免费内容。我们所有人都从这种慷慨中受益，我鼓励你们继续这样做。*

*我们每个人都有自己的贡献，即使你是这个领域的新手。其实我觉得刚起步的人有一个真正的优势:初心。书籍、文章或课程的作者经常会忘记初学者是什么样子；他们离得太远了。所以他们假设了很多，做出了巨大的飞跃，把苦苦挣扎的学习者远远甩在身后。*

*因此，如果你刚开始是一名程序员，或者如果你是一名经验丰富的程序员，正在学习一门新的学科或技术，那么我鼓励你分享你正在学习、正在做或正在努力解决的问题。还有其他人也有同样的问题，他们可以从你学习 Git 的经历中受益，比如，你是如何解决一个问题的。或者你做了什么来学习 c。当你开始帮助别人时，你也将帮助你自己。*

# *还在学习编程*

*最重要的是，永远不要停止学习。当我听到人们谈论学习编程就像是一次性的事情时，我很难过——因为它不是。编程的领域是巨大的。你可以在一个主题或算法上花费数年时间。因此，当人们把编程视为一次性的事情时，他们就限制了自己的知识和技能，从而限制了他们的潜力。*

*所以我对你们的挑战是体现卡罗尔·德韦克所说的“成长心态”不断学习。不断变好。这也是为什么我在这篇文章的标题中加了“到目前为止”。我还在学习。我还在成长。而且只要我在编程，我还在[学习编程](https://amymhaddad.com/learning-to-program-is-a-lifelong-pursuit)。*
