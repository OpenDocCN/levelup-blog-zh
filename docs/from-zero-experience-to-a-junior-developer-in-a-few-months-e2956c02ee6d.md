## *为有抱负的软件开发人员构建雇主需要的技能的简单指南*

许多有抱负的软件工程师担心他们没有雇主期望在求职者身上看到的经验和知识。有时候确实如此，但没必要绝望。获得初级开发人员角色所需的技能并不难。

这里有一个简单的策略，可以在 6-9 个月内培养在软件开发就业市场上竞争的技能:

*   第一步。掌握一门流行的编程语言。
*   第二步。熟悉语言生态系统。
*   第三步。熟悉现代软件工程实践。
*   第四步。建立一个小的项目组合。

现在让我们来看看每一步。

# 第一步。掌握一门流行的编程语言

选择一种具有以下特征的语言:

*   各种各样的免费学习资源，
*   一个大型的开发者社区，
*   工具、库和框架的多样化生态系统，
*   大量的工作机会。

这将是你未来建立所有高级软件工程技能的基础。

## 1.1 选择正确的编程语言

如今，Javascript 是显而易见的选择，因为它符合上面提到的所有标准。Swift、Kotlin、Python、Ruby，或许还有 Go 都是其他安全的选择。

像 C#和 Java 这样的语言可能也是一个合理的选择。两者都有庞大的开发者社区和大量的学习材料。然而，它们的生态系统对于新手来说可能太大了，熟悉它们可能需要时间。然而，如果你已经花时间学习 C#/。NET 或 Java，并且乐在其中——继续前进。

新兴和小众编程语言(Rust、Elixir 等。)可能学起来很有趣。然而，想用它们来构建商业软件的开发人员的工作机会可能会更少。例如，Rust 开发人员的工作机会数量与 Javascript 就业市场的规模无法相比。

## 1.2 掌握编程语言

您需要熟悉该语言的语法和结构，并学会应用它来解决编程问题:

*   阅读文档以理解语法和概念。
*   做一些教程来练习使用这种语言。
*   读几本这方面的书。

这三项活动都是必不可少的，最好同时进行。

文档应该给你一个语言特性的完整描述。如果你跳过这个直接跳到教程，你的知识就会支离破碎。更糟糕的是，你将意识不到这些差距。

教程将帮助你通过编写代码来应用你的理论知识。为了提高效率，要做尽可能多的涵盖语言特性的教程。平行阅读参考和做教程*。这样你会更快学会这门语言。*

通过阅读一本或两本关于你所选择的编程语言的书，你会学到最佳实践、技巧和诀窍，并看到语言参考之外的例子。换句话说，你会知道如何有效地使用语言。要挑选最好的书，用你选择的语言查找推荐给初学者的书，然后从列表中选择几本来读。

一本好书可能要花几个星期才能读完，但它能教会你的东西，比你自己花半年时间探索语言特色所学到的还要多。

## 1.3 学会写可读的、惯用的代码。

书籍和教程会再次帮助你。《可读代码的艺术》是这方面的一本优秀书籍。

此外，用你正在学习的语言阅读流行的开源项目的源代码。您可能会遇到许多关于构建代码、命名变量等的好主意。

# 第二步。熟悉语言生态系统。

现在，通过熟悉用于以您选择的语言构建软件的库、框架和工具来扩展您的知识。

## 2.1 熟悉流行的开源库

有经验的软件工程师在构建商业应用程序时会避免重复发明轮子。相反，他们使用开源库来解决常见问题并加快开发速度。

其开源生态系统的规模是选择 Javascript、Ruby 或 Python 等流行语言进行学习的另一个原因。使用这些语言中的一种，你创建软件的速度会比使用新兴的或特殊的语言更快，在这种语言中你可能需要自己做更多的工作。

让我们以 Javascript 为例。为了熟悉它的生态系统，创建一个流行的 npm 模块列表，并练习使用它们。不需要把每一个都学透。阅读模块文档和完成几个教程可能就足够了。

## 2.2 学习一两个流行的框架

专业开发人员通常使用框架构建商业应用程序。为你的编程语言挑选几个流行的并深入研究它们。框架文档、一两本书/课程和教程会对你有所帮助。

## 2.3 熟悉开发人员工具

学习使用流行的代码编辑器(比如 VS Code)和一个广泛的插件生态系统。找到流行的插件来更快地编写和管理您的代码。

学习使用 Git 和 Github。您需要了解如何处理分支(提交、合并、重新构建、恢复变更)以及处理提交历史。在这个阶段，除了最基本的，你不需要任何东西。网上有很多教程，你可能已经做过一些了。

(在高层次上)熟悉林挺代码、漏洞扫描器、代码质量扫描器等工具。

学习使用适合您的语言的调试器或代码编辑器中的调试器。如果你正在学习 Javascript，学习在浏览器中使用调试器。当你开始编写真正的应用程序并排除故障时，它会派上用场。

注意，对于任何广泛使用的语言来说，开发工具的数量都是巨大的。对他们中的许多人非常熟悉是没有意义的。你所需要的只是对流行的那些的概述。在这个阶段最好覆盖更多的地面，而不是挖得更深。

## 2.4 加入开发者社区

成千上万的其他开发人员使用与您相同的语言、库和工具，您可以从他们的经验中学习。

第一件事是找到并加入在线社区:像 Stackoverflow 这样的网站，像[https://dev.to/,](https://dev.to/,)这样的社区，社区聊天，虚拟聚会，谷歌群组等等。

如果你能参加你所在地区的离线聚会，那是一个结识像你一样的人和已经走在你前面的人成为伟大软件工程师的机会。

# 第三步。熟悉现代软件开发实践

首先，熟悉自动化测试的工具和实践，尽可能使用测试驱动开发。一开始可能会很难，但最终，你会越来越流利。

## 3.1 CI/CD

尝试使用一个或两个 CI/CD 平台。有些人有免费的计划，允许学习为项目建立 CI/CD 管道的基础知识，以便为每个分支运行测试套件，并将合并的 pull 请求部署到生产环境中。

## 3.2 代码审查

获得一些代码评审的经验。阅读一些关于如何进行代码审查的文章，并尝试自己审查您的拉请求。如果你认识同行的其他人，建议为彼此做一些代码审查会议。看看用你的语言写的流行开源项目库中的 PRs，注意一下评论者在那里发表了什么评论。

## 3.3 结对编程

如果可以的话，尝试与他人结对编程。开发人员经常在实际项目中使用这种技术来解决复杂的问题、调试问题以及在团队成员之间传递知识。

## 3.4 敏捷开发实践

最后，熟悉敏捷软件开发。如今，大多数公司都使用这个过程来开发软件。再次，读一些文章或一本书或观看一些课程。

当在你的项目中工作时，实践敏捷过程的原则和实践。例如，为你的项目建立一个任务清单，评估每一项任务，为你的项目组织一个敏捷委员会(可能使用 Trello)，区分任务的优先级，计划你将在一周内完成哪些任务。

# 第四步。建立一个小的投资组合

一些雇主希望没有工作经验的求职者在简历上有一个小的项目组合。这些项目不必过于引人注目；他们只需要证明候选人具有相关的软件开发技能。

让我们来看看你有哪些选择来建立这样一个投资组合:

*   自己做一个项目
*   和朋友一起做项目
*   当自由职业者
*   为开源项目做贡献

## 4.1 自己做一个项目

这个选项是最简单的。然而，你的学习机会是有限的，因为没有人会检查你的代码并指导你，但是你仍然会学习。

为一个项目想一个简单的想法。它可以是一个基本的博客或待办事项应用程序。如果你没有主意了——去产品搜索，挑一个你喜欢的主意，然后实现最少的功能。

建造一些微小的东西。即使你正在开发一个 ToDo 应用——你也将不得不像复杂应用的开发者一样解决同样的问题。当范围较小时，您完成和发布项目的机会就较高。如果你正在构建一个 web 应用程序——托管在一个云托管提供商上:AWS、GCP 或 Azure。试着把它放到免费层，这样你就不用花钱了。如果你开发了一个移动应用，也可以考虑发布它。

## 4.2 与朋友一起做项目

如果你有类似软件开发经验的朋友，和他们聊聊。你们也许可以一起建造一些东西。在我看来，最好从第一天开始就创造一些你可以开源的非商业性的东西。

好处是每个参与者都可以互相学习，你可以回顾代码，结对编程，并使用其他的团队实践。

缺点是参与者缺乏经验。你可能很难取得进展，也可能很快遇到人际关系问题，因为很少有人知道如何在团队中工作，如何处理相互冲突的意见。

这个选项仍然比前一个好，因为它允许你发展和练习更多的技能。

## 4.3 自由职业者

即使没有什么经验，你也可以在自由职业者市场上找到工作。

缺点是:

*   你无法控制你将在项目中做什么。
*   你可能不得不使用写得很差的应用程序，并且不会学到很多东西。
*   你会把宝贵的时间花在寻找和申请项目上，而不是学习上。

优点是:

*   你可能会得到各种各样的项目来练习不同的技能。
*   你将学会与客户合作，区分工作的优先次序，评估等等。
*   你可能会发现一些长期项目，有一个开发团队会帮助你一路学习。
*   你的客户会为你的工作付钱。

## 4.4 为开源项目做贡献

为开源项目做贡献是一个通过代码评审和讨论向更有经验的开发人员学习的机会。此外，这是一个与真正的客户一起开发真正的产品就绪软件的机会，他们免费使用它的事实是无关紧要的。

这个选项的好处是:

*   你会学好你选择的编程语言。
*   您将使用软件开发实践:测试、代码审查、CI/CD 等。
*   你将学会优先考虑和计划工作。
*   你将学会在团队中工作。许多初学者不知道如何做到这一点，因为他们从未从事过真正的项目。

棘手的部分是选择合适的项目加入。

一方面，你可能想加入一个受欢迎的、维护良好的项目，有一个成熟的代码库和经验丰富的贡献者。然而，你可能经验太少，无法做出有意义的贡献。因此，你不知道从哪里开始，或者其他贡献者可能会跳过你的 PRs。另一方面，一个简单的项目可能更容易加入，但可能代码不够完美，维护也很差。

所以，仔细挑选一个开源项目来贡献。如果有投稿人指南，请查看。它可能会澄清从哪里开始:修复 bug、更新文档、添加测试用例等等。这听起来可能没什么意思，但这是软件开发不可或缺的一部分。

想象一下，你已经发布了你的开源库，以自由职业者的身份参与了几个商业项目，并且和一个朋友一起构建了另一个 ToDo 应用来组织你的工作。总之，对于申请初级开发人员角色的人来说，这将是一个很好的投资组合。

# 外卖食品

如果你有一个现实的计划并坚持下去，在几个月内获得初级开发人员的技能并不太难。

对你来说，最好的第一步是学习流行编程语言的基础知识。然后你就可以熟悉它的生态系统:库、框架、开发者工具。

接下来的步骤将是尝试现代软件开发实践，并做一些真实的项目来创建一个投资组合，以展示您的技能。你可以选择:自己或和朋友一起做一个项目，做自由职业者，为一个开源项目做贡献。

一旦你处于最后一步的中间，你就可以开始准备工作面试，与招聘人员交谈或直接联系潜在的雇主。

祝你的第一份软件工程工作好运！
