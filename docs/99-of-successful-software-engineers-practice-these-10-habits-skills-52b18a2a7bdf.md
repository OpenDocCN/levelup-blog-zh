工程师就像雪花，[没有两个是一样的](https://www.nationalgeographic.com/science/2007/02/no-two-snowflakes-the-same/)。

曾经形成的每一片雪花都是由不同的原子以独特的结构组成的。这类似于软件工程师，无论学习路径多么相似，没有两个人的开发技能和能力是完全相同的。

然而，即使有不同水平的才能，升到顶端并被认为是最成功的人是那些建立了坚实的习惯和技能基础的人。这使它们能够更长时间地接近峰值效率，并在更长的时间内持续获得更高质量的输出。

多年来，在参与软件工程项目和管理其他人的过程中，以下特征在最成功的人当中脱颖而出，获得了这个头衔。这些是技能和习惯的集合，可以提高产出的效率和质量，可以帮助任何人达到下一个水平。

# 1.找到有效管理自己时间的方法

为了保持高效率，最好的软件工程师既要编码，又要像他们自己时间的管理者一样工作。

大多数软件工程师被单独留下来完成困难的开发任务，这些任务需要一些深入的思考和计划，以便获得有利的结果。这意味着分心和注意力不集中的可能性很高。

一名优秀的工程师知道如何避免这些干扰，以及如何最大限度地利用他们在“区域”*中的时间，在那里他们产生最多的输出，并且最接近峰值效率。*

# *2.遵循编码标准*

*你应该总是以一种逻辑和一致的方式构建你的代码。*

*如果你所在的团队有一个共同的标准可以遵循，那很好。如果没有，那么你应该形成自己的个人标准。遵循一个标准提供了一个框架，随着时间的推移，可以在输出中实现更高的效率和一致性。*

*个人标准还可以帮助您识别您过去编写的代码片段，并且当需要时，在需要审查、更改或重构时，帮助您重新学习特定的代码。这一点很重要，因为在足够长的时间后，你*将*忘记你自己代码的复杂性。当回顾您过去的编码项目时，有一个熟悉的风格来阅读将有助于加快重新学习的努力。*

# *3.尽早并经常与你的团队和老板交流状态*

**世界是一个相互联系的地方，没有工程师在真空中工作。**

*在任何正在进行的开发中，总会有其他人被认为是利益相关者。优秀的软件工程师认识到这一点，并努力确保每一个利益相关者，无论多么相关，都被告知并跟上任何开发的最新状态。*

*沟通是良好团队合作的基础，最好的工程师总是努力让每个人都了解最新情况。*

# *4.找其他人学习*

*总有更多的东西需要学习。*

*不管一种语言或方法有多好，总会有新的东西在某个时候出现。作为一名软件工程师，成长的最好方式之一就是跟随那些已经铺平了道路的聪明人的脚步。*

*这并不总是需要导师-学员类型的关系。团队中的其他人、博客作者、经理和开源项目都可以为你自己的职业生涯提供知识和价值。这里的目标是永远不要停止学习。*

# *5.尽早并经常检查东西*

**现代的版本控制基本上是无限的，易于使用，并且很容易完成。所以用吧！**

*没有任何理由因为没有备份而丢失工作。如果你在日常工作中把它放在优先位置，有足够的工具可以让这项工作变得简单。*

*许多现代程序会自动保存并不断备份。如今，丢掉大量工作比以往任何时候都难，但这仍然是可能的。养成经常存钱和经常存东西的习惯对保持持续进步和高效率很重要。*

# *6.平衡工作和个人生活*

**工作是永无止境的，企业总是能够得到比你能给予更多的东西。**

*专注于一个大项目是很容易迷失的。如果你放任不管，它会耗尽你所有的精力。但是人类的大脑需要休息和刺激，而不是电脑屏幕、键盘和更多的咖啡，以保持最高水平的运行。*

*周末的存在是有原因的。花时间去散步，看一场戏，或者遛狗。当你回来的时候，工作会在那里，随着时间的推移，你将能够通过平衡工作和个人生活的方法获得更高的平均产出。*

# *7.继续创建个人项目*

*任何一个人都不可能在所有领域都跟上技术的快速发展。*

*即使你正在办公室从事被认为是尖端的工作，仍然有许多你没有探索或学习的技术前沿。*

*个人学习和独立项目是在你正常的朝八晚五的工作所不包括的领域进一步深造的一个关键方面。这让你跟上许多其他技术领域的速度和相关性，否则你将没有机会学习。*

# *8.学会准确估计自己的努力*

**业务脱离了日期和时间表。**

*不管你的开发方法有多敏捷，在某个时候会有人问这样的问题:“*什么时候准备好？**

*通过了解你自己的工作习惯和单位时间内你能实际完成多少产出，这个问题会更容易得到一个准确的答案。诀窍是在你被推到整个团队面前之前达到这种理解。了解你自己的能力和局限可以让你更自信、更准确地回答这个问题。*

# *9.记录您的代码*

*随着时间的推移，你对为什么用那种方式编写函数的记忆会逐渐消失。*

*将来有一天，回过头来看旧代码，需要同样的努力去理解是你还是别人写的。为你的代码编制文档提供了一种方法，使得任何人，包括你自己，都可以直接查看、维护或修改软件。*

*让文档成为一种习惯不仅可以随着时间的推移保留意图和意义，而且有助于更好、更快地可视化和解决复杂的问题，而不仅仅是通过代码。*

# *10.学习如何对自己的工作进行质量保证*

*没有人比编写软件的人更了解软件的优点和局限性。*

*在把新的特性和功能交给其他人评审之前，对它们进行自己的 QA 检查是开发稳定和完整软件的一个非常有价值的工具。*

*学习和实践像测试驱动开发(TDD)这样的技能，并理解如何有效地对你自己的代码进行 360 度的审查，寻找边缘案例，将会使你在输出质量方面远远超过其他人。不管这个过程看起来有多无聊或耗时，学习如何有效地 QA *将*让你成为一名更好的软件工程师。*

*作为一名软件工程师，你越早培养成功的习惯和技能，你的事业就会越成功。记住要坚持阅读和学习，永不放弃！感谢阅读！*
