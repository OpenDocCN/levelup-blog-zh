# 关于

如果您正在尝试学习 Next.js，或者只是想立即开始您正在从事的新项目，初学者工具包是一个很好的起点。只要浏览一下源代码，你就会学到很多东西。我尝试使用最佳实践，并保持代码的整洁和有序。如果你是一个经验丰富的开发人员，它应该非常容易阅读和理解。否则，如果你是初学者，看一下不同技术的文档可能是明智的——如下所述。

# 特征

## Next.js

Next.js 是一个 **React** Next.js 处理 React 所需的工具和配置，并为您的应用程序提供额外的结构、特性和优化。

## GraphQL

GraphQL 是一种 API 查询语言，也是一种用现有数据完成这些查询的运行时语言。GraphQL 为 API 中的数据提供了完整且易于理解的描述，使客户能够准确地要求他们需要的东西，使 API 更容易随时间发展，并支持强大的开发工具。

在这个模板中，GraphQL 将从 SpaceX 自己的 GraphQL API 中获取任务、发射和更多内容。

## 样式组件

利用带标签的模板文字(最近添加到 JavaScript 中)和 CSS 的强大功能，样式化组件允许您编写实际的 CSS 代码来样式化您的组件。它还消除了组件和样式之间的映射——使用组件作为底层样式构造再简单不过了！

## 框架运动

成帧器运动是 React 的一个生产就绪的运动库。利用 Framer 背后的力量，它是团队的最佳原型工具。简单的声明性语法意味着您编写的代码更少。更少的代码意味着你的代码库更容易阅读和维护。

## 故事书

Storybook 是一个 UI 开发工具。它通过隔离组件使开发更快更容易。这允许您一次处理一个组件。您可以开发整个 ui，而不需要启动复杂的开发堆栈、将某些数据强制输入数据库或在应用程序中导航。

# 最后的想法

这个堆栈对于为 SEO 和性能非常重要的 web 创建生产就绪的应用程序非常有用。

编写模块化和可重用的组件总是有用的。手边有一个设计系统，用 Storybook 开发你的组件，将大大提高效率，同时也减少错误和不一致。
