## 如何在工作中有效并尊重地使用 Slack

![](img/af5b6dca521f59abe2bca1b5794e48cc.png)

由[斯科特·韦伯](https://unsplash.com/@scottwebb?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的照片

Slack 是沟通的救星。它使组织消息、保留可搜索的消息历史、异步通信以及让每个人都参与进来变得简单。

懈怠也可能是永无止境干扰的生产率噩梦。

有效使用 Slack 的关键是每个人都承诺拥有良好的 Slack 礼仪。让我们来看看一些准则，看看这意味着什么:

# 默认为公共频道，然后是私人频道，然后是直接消息

信息共享是 Slack 的妙处之一。当您使用公共渠道传达公告、讨论想法、提出问题和解决问题时，渠道中的每个人都可以参与。

另一方面，如果你只依赖直接信息，你发信息的人现在觉得有义务回应。如果你在问一个问题，这个人可能现在不能帮你。他们可能很忙或者在度假。他们可能不知道答案，或者可能不是帮助你的最佳资源。

当你在公共渠道提问时，你现在有很多人可以提供帮助。感兴趣和有空的人可以加入进来，那些不能帮忙的人可以继续他们的一天。

在公共渠道提问，也让这些讨论在以后的 Slack 中可以搜索到。一周后，有人可能会有和你一样的问题，或者遇到和你一样的问题，在 Slack 中进行对话并给出解决方案有助于他们找到所需的答案，而不必再次进行完全相同的对话。

有时，消息需要更加私密，或者只发送给一小部分人。例如，一组经理可能需要讨论他们直接下属的薪酬。或者一组面试官可能需要讨论他们对特定候选人的反馈。对于这种敏感信息，私人渠道非常有用。

当你有私事要和某人讨论，或者确实需要某个人的帮助时，直接信息也可能很重要。

重要的规则是尽可能默认使用公共渠道。然后在需要的时候使用私人渠道，最后在有保证的时候使用直接信息。

# 不要使用`@here`或`@channel`，除非你的信息需要所有人立即关注

如果你的信息是针对频道中“每个人”的公告，那么使用`@here`或`@channel`会很有诱惑力。但是，请记住这个通知会分散注意力。`@here`通知当前在线的每个人，`@channel`通知频道中的每个人，甚至是离线的人。

你关于最近员工时事通讯的信息真的需要每个人立即停下手头的工作去查看吗？大概不会。(其实肯定不是。不要成为那样的人。)

另一方面，如果这是一个工程渠道，而你的产品应用程序在生产中出现了问题，你需要所有人都参与进来，那么一个`@here`提一下就更合适了。这是一个很好的例子，说明你真的需要打断每个人的话，以立即引起他们的注意。

所以，对于 99%的信息，一般的经验法则是:不要使用`@here`或`@channel`。

# 使用线程

Slack 是同时促进多个正在进行的对话的一个极好的工具。事实上，一个繁忙的空闲信道可能在任何时候都有几个活动的对话。出于这个原因，Slack 提供了线程，您可以在“回复”部分对通道中的任何给定消息做出响应。

使用线有助于保持你的渠道干净。想象一下，在一家繁忙的餐馆里，多组人坐在一张桌子旁，隔着桌子互相大喊，进行着三次单独的对话。信息会被遗漏，很难跟上对话。让我们避免懈怠。

相反，把你的信息串起来。

# 不要在没有包括你的信息的其余部分的情况下给别人发“嘿”

最后，我个人最讨厌的。你有没有收到过同事简单写着“嘿”的信息？

你坐在那里一分钟，等着看是否有更多的信息。但是没有。你可以继续做你正在做的事情，同时再等几分钟，看看他们是否会发送后续消息。但是——没什么。

所以，你会很自然地回答“怎么了？”然后，最后，他们给你发来他们真正的信息:他们需要你的帮助。

别做那种人。异步通信的美妙之处在于，消息接收端的人不需要立即回复您的消息。相反，你可以提前发送你的全部信息(“嘿，你有空帮我做[x]？”)等他们能回应的时候再回应。

只发“嘿”而不发其他信息是在浪费每个人的时间。

# 摘要

通过遵循这些指导方针，我们可以成为更有效的空闲用户*和*更加尊重彼此的时间。简单来说:

*   默认为公共频道，然后是私人频道，然后是直接消息
*   不要使用`@here`或`@channel`，除非你的信息需要所有人立即关注
*   使用线程
*   不要在没有包括你的信息的其余部分的情况下给别人发“嘿”

现在出去合作吧！
