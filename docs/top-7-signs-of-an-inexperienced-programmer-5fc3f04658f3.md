# 缺乏经验的程序员的 7 大标志

> 原文：<https://levelup.gitconnected.com/top-7-signs-of-an-inexperienced-programmer-5fc3f04658f3>

## 了解这些迹象，这样你就不会在你的软件开发生涯中犯同样的错误

![](img/de0202e63dc0e0f9df6850478fdc4776.png)

图片来自[皮克斯拜](https://pixabay.com/?utm_source=link-attribution&utm_medium=referral&utm_campaign=image&utm_content=5011953)

软件工程方面的经验随着你从事该行业的年数而增长。这种说法在一定程度上是真实的。但是，一份工作的年限并不能让你默认获得合适的经验。

我见过一些人表现得像个业余爱好者，即使他们已经在自己的岗位上工作了很多年。他们几乎不学习软件开发的基础知识。在最初的几年后，他们的成长停滞了，但是他们不明白他们做错了什么。

与此同时，我曾与只有几年经验的开发人员一起工作，他们显示出惊人的增长潜力。他们拥有正确的态度和避免不称职行为的知识。

基于开发人员表现出的某些特征，你可以很容易地发现谁有经验，谁没有。让我们深入探讨一下缺乏经验的程序员的七个迹象，每个软件工程师都应该知道这些迹象，以避免犯下可能阻碍他们职业发展的类似错误。

# 1.创建大量拉取请求

您是否曾经收到过代码评审请求，其中有太多的更改，以至于您不想评审它？是的，这正是没有经验的开发人员所做的。他们将在一个单一的拉请求中集合大量的变更。最重要的是，他们希望你优先考虑他们的代码审查。

我也看到许多高级开发人员有创建大拉请求的习惯。他们会在没有反馈的情况下编码几天。当您最终审查他们的代码时，他们可能已经围绕它构建了完整的功能。因此，您给出的任何评论都需要进行重大修改。

当我收到这样的拉请求时，我的第一反应是将它返回给开发人员，将它分解成更小的、逻辑上分开的 PRs。我经常只是在我找到的第一个问题中添加评论，然后将它发送给开发人员。如果我感到难以置信的慷慨，我会要求他们建立一个电话，并审查现场代码。

## 您可以做什么:

*   创建较小的拉取请求。作为一个好习惯，永远不要在没有检查的情况下离开一天的工作。
*   永远不要签入不能编译或破坏构建的代码。

# 2.写意大利面条代码

没有经验的开发人员会写出与漂亮代码完全相反的东西。他们的代码将会杂乱无章地散布在代码库中。

如果你试图阅读意大利面代码，你会不断地迷失其中。你会忘记你从哪里开始，你在找什么，以及代码到底想做什么。

有了经验，开发人员应该知道如何规划他们的编码。除非是简单的功能，否则先把你的理解和流程写在纸上。做一次演习，将它从头到尾可视化。一旦你非常清楚这些变化，那么就从实现部分开始。

如果你不遵循上面的过程，你会在阅读你自己的代码时感到痛苦。对你自己和整个团队来说，排除故障或增强你作为代码编写的那块拼图是很困难的。

## 您可以做什么:

*   在开始实现它之前，要对它有一个清晰的理解。为了对需求有一个清晰的概念，你可以问尽可能多的问题。
*   保持你的代码简单且结构良好。你的队友应该能够阅读代码并理解它的预期用途。

# 3.试图同时处理很多任务

没有经验的开发人员不知道从哪里开始一项任务，如何进行，以及何时结束。他们试图同时解决很多事情。他们不知道如何将一个大任务分成更小的逻辑部分，以使其易于实现。

如果你给他们分配了一项任务，他们会立即投入到编码中，而不会向你核实他们是否理解了你的要求。他们也不会和你一起回顾他们的进展，以确保他们在正确的轨道上。只有当他们认为已经完成时，他们才会回复你。到那时，你只能祈祷你的要求得到准确的实现。

缺乏经验的另一个标志是，这样的开发人员同时插手太多的事情。他们会从不相关的功能中挑选任务，自愿解决生产问题，并承诺帮助团队中的其他人。

最终，这些开发人员没有完整地交付任何承诺的任务。这种态度在大多数时候可能是善意的，但结果对团队来说是灾难性的。最终，团队损失了大量时间，不得不在战争状态下完成所有任务。

## 您可以做什么:

*   专注于运输小。将你的作业分成更小的逻辑块。弄清楚它，然后交付尽可能小的工作功能块。
*   一次接受一项任务并完成它。只有在前一个任务按要求交付时，才提交新任务。

# 4.充满傲慢

傲慢是没有经验的开发人员的致命弱点。他们太自以为是了，以至于不知道自己做错了什么。你对他们的代码或演示给予反馈；他们会把这当成对自己能力的个人评价。

很多大一新生炫耀自己的傲慢，多半是因为自己的无知。他们刚从大学毕业，还不明白职业世界与他们在大学里学到的完全不同。聪明的人实际上保持沉默，并对学习企业文化表现出浓厚的兴趣。

不仅仅是新生——一些傲慢的开发人员已经在软件行业呆了几年了。这可能是由于他们的一些专业成就，或者他们还没有和比他们更聪明、更有才华的人共事过。

无论是哪种情况，傲慢的行为都清楚地表明这样的开发人员缺乏正确的经验。他们的自我蒙蔽了他们，使他们无法学习正确的职业道路。最终，没有人喜欢和傲慢的团队成员一起工作。一旦增长放缓，傲慢的开发者就会将失败归咎于他人。

## 您可以做什么:

*   在你的方法上要谦虚。礼貌对于在软件开发中建立成功的职业生涯大有帮助。
*   尊重每一个人，不管他们的头衔是什么。不要因意见分歧而争论。

# 5.不从错误中吸取教训

我一直认为反馈机制是软件开发人员最有效的工具之一。反馈有助于我们了解自己的缺点以及如何改进。聪明的开发人员知道如何利用反馈来提高他们的生产力。

根据他们对建设性反馈的反应，你可以很容易地发现一个没有经验的开发人员。他们永远不会接受对他们表现的任何改进意见。他们甚至认为代码评审意见是针对个人的。

很多年前，我们有一个队友给我写了一封很长的电子邮件，告诉我应该如何审查代码。他被我对他公关的评论激怒了。他的主要论点是我不应该担心编码标准，因为他知道如何编码。他希望我只是检查代码是否满足功能需求。

如果开发人员因为评审意见而感觉受到了侮辱，这是一个明显的信号，表明他们没有从他们的经历中学到任何东西。他们继续以无能的态度年复一年地工作，不知道为什么没有人重视他们的贡献。

## 您可以做什么:

*   对每一个反馈都保持积极的态度。你可以选择接受哪个，丢弃哪个。但是在你决定放弃它之前，给它一个公正的评价。
*   虚心从错误中学习。没有人总是对的。利用学习来提高你的绩效。

# 6.将工作时间花在个人任务上

总有一些团队成员在上班时间做他们的私人工作。他们会浏览社交媒体，浏览在线购物网站，或者玩游戏。

我们有一个团队成员，曾经在上班时间炒股。这对他的交易产生了负面影响，因为他总是关注当天交易的情况。其他团队成员对此行为表示担忧，因为他们必须付出额外的努力才能在截止日期前完成。

当经理警告上述开发商时，他改正了自己的做法几天，但又回到了交易中。最终，由于他的这种行为，公司不得不让他离开。

这种行为是不道德的，并且清楚地表明开发者缺乏经验。如果你对帮助你谋生的职业是真诚的，那将是最好的。

## 您可以做什么:

*   将你在办公时间的个人任务减少到最低限度。如果你不得不花几个小时去处理不可避免的个人问题，你可以得到经理的允许。
*   你可以利用休息时间查看你的社交媒体。把你的午餐拿到桌子上，如果你想的话，可以在午餐时间做股票交易。

# 7.经常跟在被炒作的技术后面

缺乏经验的开发人员的一个标志是他们如何在被宣传的技术背后运行。你会发现他们总是在谈论下一件大事。一旦市场上出现一种新的时尚，开发商就会抛弃以前的时尚，去追赶最新的潮流。

没有经验的开发者也掌握了做教程的艺术。毫无疑问，教程是有用的学习工具。但是仅仅跟随没有任何实际用途的教程是浪费时间。这可能会给人一种虚假的成就感，但是对知识的真正考验是在现实世界中使用它。

你很少会看到开发人员使用宣传的技术或从教程中获得的知识来实现任何新的东西。他们这样做只是为了满足他们的自我。此外，许多没有经验的开发人员由于害怕错过机会而陷入这个陷阱。

## 您可以做什么:

*   花时间和精力去学习那些你可以在工作场所或个人项目中实际使用的技术。
*   使用从教程中学到的知识并进行动手实践。你将从自己实现的东西中学到比跟随任何教程更多的东西。

# 最后的想法

缺乏经验的程序员由于效率低下而降低了整个团队的生产力。他们对待工作的不正确方法使他们错过了在高回报的软件职业生涯中成长的机会。

在职业生涯的早期就知道自毁的态度并避免它们是明智的。你越是养成上述行为的习惯，在你职业生涯的后期就越难摆脱。
