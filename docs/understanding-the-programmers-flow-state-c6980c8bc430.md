在那篇文章中，我并没有完全解释为什么开发人员会变得烦躁，并且如此厌恶被打断。程序员通常是他们可中断时间的最佳判断者。当他们休息一会儿跳上 Reddit 时，这是因为他们在精神阁楼上清理了一些小空间，知道他们可以在那个特定的时刻进行短暂的逃离*。*

看看我上面的图片。我让她扮演程序员的角色。她凝视着太空。像这样在野外抓到一个开发者是很平常的事。她不是“无所事事”她不是“简单地凝视太空”虽然她的眼睛可能会漫不经心地跟着一些远处的活动——一个正在倒咖啡的行人，墙上的一只蜘蛛，或者一辆经过的急救车辆——但这些都没有真正打断她大脑中复杂的心理过程。

如果你是一个猎人，突然向你的目标靠近，这时你应该慢慢深呼吸，尽量发出最小的声音。

程序员的心流状态是他们最有效率并且完全沉浸在他们试图解决的问题中的精神状态。进入那种状态需要一点时间，突然打断他们需要他们回到那一点。

# 简单的类比

我们大多数人都去过热闹的餐馆。我们大多数人可能都遇到过在工作中表现出色的服务员。

观察服务器进入他们的流状态:

*   他们很快接受了你的订单
*   他们问了所有正确的问题
*   他们回答了你所有的问题
*   他们带来了您点的菜，并将每一道菜正确地放在正确的人面前
*   他们没有洒任何东西

没什么大不了的，对吧？这需要一点记忆和技巧。特别是，最优秀的服务器甚至不需要写下任何东西。

现在，想象一下全流程的服务器:

*   服务员不只是照顾你的桌子。他们负责一些桌子。
*   你向他们要一些番茄酱。
*   看到你的服务员在另一张桌子上安排了一个新的聚会，仍然没有做任何记录
*   他们的另一桌要求结账
*   您对面的小组正在退回一些项目进行更改
*   哎呀，就在你后面有一个溢出物
*   你注意到一个朋友在独自吃饭，所以你招手让他们过来和你一起吃。现在，服务器必须将他们的设置移动到您的设置。从心理上讲，服务员必须注意到这一变化，以便在用餐的剩余时间里使用。

服务器在心里平衡每个表的状态。谁点了什么？谁准备好接受检查了？我还要多久才能让新客人入座？五星级的服务员不漏一拍，在餐厅里游刃有余。

就像服务员一样，程序员对变量、状态、方法调用和各种编码逻辑进行同样的心理指挥。这是一种精细控制的平衡行为。

# 中断

想象一下，通过法律强制他们休息 10 分钟，让服务员离开他们的流程。很有可能，十分钟后，他们侍候的一些桌子还在用餐。谁想要番茄酱？还是烧烤酱？哦，我忘记给他们支票了吗？糟糕，我没有机会告诉他们，他们的信用卡被拒绝了。

此外，在这次强制休息中，其他人被要求代替他们的桌子。您可以很容易地看到，这个新服务器不知道前一个服务器的每个表的状态，并且必须将所有内容“加载”到它们的短期内存中。更有可能的是，这个服务器有他们自己的表，他们在头脑中有效地管理着这些表。增加额外的桌子需要额外的体力和脑力。

类似的情况也发生在开发层。一个完全流畅的程序员会被打断，去帮助一个“只有一个简短问题”的同事或下级。好吧，提供帮助的开发人员会试图记住他们所有的工作，同时“加载”他们队友工作的问题状态。一旦辅助完成，回到全流程状态取决于开发人员的技能。在某些情况下，其他开发人员的问题没有得到完全解决。现在，提供帮助的开发人员，无论他们如何努力，都不能简单地忘记他们队友的问题。现在，这个问题也在他们的潜意识中增加了精神压力。

就像服务器从休息中返回一样，他们可能会也可能不会记住每个表的状态。然而，由于一段时间过去了，每张桌子*的状态已经*改变了，当他们回到地板上并赶上时，将需要做出一些调整。

# 回到全流量

回到全流量所需的时间取决于:

*   取决于服务器(或开发人员)的技能。有些人，无论是天生的还是训练有素的，在管理和记忆心理过程方面更胜一筹。
*   当它被中止时国家的复杂程度。对于餐厅楼层的服务员来说，如果所有的桌子都是简单的无人点餐，那就很容易从休息中恢复过来。对于陷入一些乏味的、重复的、没有实际逻辑的任务中的开发人员来说，从中断中恢复会很顺利。

为了确保你圈子里的开发人员满负荷工作，尽量减少干扰。也许可以和他们一起研究一个系统，研究如何以及何时进入他们的大脑体操。
