因此，如果您现在正在寻找前端开发人员的工作，那么您已经找到了正确的文章，因为这篇文章将告诉您需要准备的所有信息！

# 你应该期待什么样的问题？

每个公司都有自己的面试策略，但最终，大多数公司都属于同一类别，不管你是去 GAFAM 还是小型科技公司面试都没关系。这些是你将被采访的最常见的话题:

# **Javascript 问题**

作为一名前端开发人员，你显然应该已经了解了 Javascript，但是你也应该理解它是如何工作和运行的。不管你是否在日常工作中使用 Angular/React/Vue，你都需要**了解基本知识。**

**一些示例问题可能是:** A .解释什么是事件循环
B .解释什么是承诺
C .闭包和上下文问题(什么是“这”)
D .箭头函数和常规函数之间的区别是什么？
E .实现 Array.prototype.map/reduce
f .什么是去抖/节流功能？执行它们。
G. Javascript 原型继承问题

**一些有帮助的资源:** [5 个典型的 JavaScript 面试练习](https://www.sitepoint.com/5-typical-javascript-interview-exercises/)
[学习高级 JavaScript](https://johnresig.com/apps/learn/)
[JavaScript 花园](http://bonsaiden.github.io/JavaScript-Garden/)
[口才 Javascript](https://eloquentjavascript.net/)

# 算法

作为一名前端开发人员，你不太可能需要遍历二叉树(当然也有例外)。但尽管如此，算法问题在面试中还是很受欢迎的(尤其是如果你的目标是在 GAFAM 工作的话)。

1.**数组或对象相关问题**

例:
1。反转一根弦
2。写一个函数来检查一个单词是否是回文。给定一个字符串，返回最常出现的字符。

2. **BFS/DFS 问题(遍历一棵树)**

例:
1。编写一个解决迷宫的函数(迷宫是一个多维数组，值为 0 —墙壁，1 —地板，2 —出口)。该函数接收一个起始坐标，并应返回您是否可以走出迷宫。

3.**链表**

例:
1。反转链表
2。从链表的末尾开始查找第 n 个节点

你可能也应该更新一下关于递归和迭代的区别以及时间和空间复杂性的知识。

**一些有用的资源:** [40 个问题、解决方案和解释](https://medium.com/siliconwat/algorithms-in-javascript-b0bed68f4038) [JavaScript 算法和数据结构大师班](https://www.udemy.com/course/js-algorithms-and-data-structures-masterclass/)——(注意:这无疑是 Udemy 有史以来最好的课程，尤其是如果你没有计算机科学学位，或者如果你需要复习算法和数据结构)。
[Codewars](https://www.codewars.com/) (或者任何其他类似的网站，比如 HackerRank 或者 LeetCode)。

# 解一道算法题要注意什么

A.不要急着解决，要求几分钟思考是可以的。

B.当你开始解决问题时，强烈建议你谈谈你在做什么，即使你不确定你的想法。考官通常希望了解你的思维过程。并不总是需要达成一个完美和可行的解决方案。如果你不习惯大声说话，或者只是不习惯，那就在家通过解决一些例题来练习。

C.如果可以的话，先用文字或者伪代码把自己的行动计划写出来。这将向面试官展示你并不急于解决问题，你可以做好计划。

D.如果一开始就卡住了，不要慌！从简单的事情开始。想想投入和产出。解决边缘案件。

E.注意你的解决方案的代码有效性。如果你明白复杂并不是真的好，那就大声说出来，和考官一起努力，达成更好的解决方案。

F.问问题完全没问题。一个好的考官会努力帮你找到解决方案。

G.通常期望你在白板/纸上解决这类问题。既然这不是我们通常写代码的方式，你可能会发现这样想更难。这又是一件值得事先练习的事情！

# 反应/角度/Vue(原位)分配

有些地方会要求你使用特定的框架/库来构建某种简单的应用程序。根据公司的不同，它可以是家庭作业，或者你必须在公司的办公室里完成。如果是后者，带上自己的笔记本电脑，这样就可以使用自己的 familair IDE 和操作系统。

在大多数情况下，你会得到某种公共 API，并被要求以某种方式使用它。

这样做是为了观察候选人编写实际代码的方式，因此需要注意以下几点:

A.保持你的代码整洁。使用缩进，提供有意义的变量名，删除不用的代码。

B.使用给定框架的最佳实践。

C.思考类似于:
1。发送到服务器的去抖请求。
2。键入< 3 个字符时阻止服务器调用。
3。注意边缘情况，比如 API 中缺少信息。

D.你的 CSS 技能也很重要！熟练使用 Flexbox(不要使用 float 来对齐容器😉).

# 逻辑问题

我承认。我自己真的很不喜欢这种类型的问题。我不明白他们怎么能帮助任何人评估你作为开发者的价值。但是一些公司仍然会问他们……
你真的不能**为这些做准备**(也许阅读一些常见问题？)，但只要知道他们确实存在。

**举例:**
你站在一个有三个电灯开关的房间里。这些开关都对应着附近房间里的三个不同的灯泡，你看不到里面，但你只能进去一次。当所有的灯开关都处于关闭位置时，你如何找出哪个开关连接到哪个灯泡？

# 谈论你自己

最后但同样重要的是，有一部分你需要谈谈你过去的经历。

如果你申请的第一份工作是开发人员，确保有一些你感兴趣的项目，或者为一些开源库做贡献。

准备好讨论细节。比如你为什么选择库 A 而不是库 B？

如果这不是你第一次参加竞技表演，你需要谈谈你在工作中负责的一项功能或产品。**在细节上**。面试官也可能想听听你个人对团队的贡献。

你应该和一个朋友练习这个对话，这样你就有反馈了！如果你有所准备，事情会简单得多。

# 最重要的是:一次失败的面试并不能定义你！

记住，一次失败的面试并不能说明你是一名多么优秀的开发人员。正如我在开头提到的，**面试是一门独立的技能**。在你的日常生活中，你有 StackOverflow 和同事供你支配，你不需要在一张纸上写代码。面试不是你习惯做的事情，所以**你需要**尝试几次来掌握它是如何做的。准备学习，但是**没通过也不要**为难自己。试着**从每次面试中学习一些**的东西，最终，你**会成功**！
