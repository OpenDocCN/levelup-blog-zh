# 自动警报系统

AWS 于 2008 年率先提供云计算基础设施即服务，此后一直主导着市场。它正在以极快的速度推出新的服务，并创建自己的计算堆栈，旨在提高效率，并将节省的成本传递下去。这个计划不太可能改变，因为亚当·塞利普斯基将回归 AWS 担任首席执行官，安迪·雅西将接管亚马逊，接管杰夫·贝索斯。

AWS 已经远远超出了云计算和存储领域。如果基于 Arm 的处理器成为数据中心的标准，行业可以感谢 AWS 的引力，[它推出了第二代 Graviton 处理器和基于它的实例](https://www.zdnet.com/article/aws-graviton2-what-it-means-for-arm-in-the-data-center-cloud-enterprise-aws/)。如果成功，Graviton 和 Nitro 抽象层可以成为 AWS 在云战争中的差异化优势。

AWS 的主要优势仍然是其服务的广度和深度，根据最新统计，其服务涵盖计算、存储、数据库、分析、网络、移动、开发人员工具、管理工具、物联网、安全和企业应用程序等领域超过 175 项。

"*毫无疑问，作为市场领导者，AWS 经常在开发者功能上胜出，这是因为它的先发优势带来了广泛的服务。正如 CSS Insight 的 McQuire 所说，AWS 在将其规模转化为客户的经济利益方面也做得很好，尽管有时云的成本可能会令人望而却步*。

尽管 AWS 的增长率相对于竞争对手有所放缓，但其收入基数却要高得多。几乎没有证据表明 AWS 没有获得企业 IT 云支出的更大份额。AWS 与 VMware、开发人员、生态系统和大型企业客户群建立了[混合云合作伙伴关系，以保持领先地位。](https://www.zdnet.com/article/vmware-cloud-on-aws-gains-workloads-but-challenges-remain/)

# 微软 Azure

微软 Azure，加上微软的软件即服务努力及其在企业中的足迹，使该公司成为 AWS 的强大第二名。尽管 AWS 占据主导地位，但微软在“云优先”首席执行官塞特亚·纳德拉的领导下取得了进展，建立了自己的庞大全球云网络。当企业挑选首选云供应商时，微软将是一个选择。

微软 Azure 于 2010 年推出，被认为是最好的云服务平台之一，这要归功于微软这家科技巨头提供的一整套可用服务。微软 Azure 提供了一系列适用于各种行业的解决方案。你所有的商业需求都会被考虑在内。这导致了更适合需求的包。

微软往往是 C 级高管的热门选择，他们与该供应商有着长期的关系，知道他们可以在一个地方消费大量的企业计算需求，从生产力和企业软件一直到他们的开发人员的灵活云计算资源，只需一只手就可以搞定。

对于计算，虽然 AWS 的主要产品是围绕其 EC2 实例，但 Azure 的计算产品是围绕其虚拟机(VM)，以及云服务和资源管理器等其他工具来帮助在云上部署应用程序，以及 Azure 自动缩放服务。

# 谷歌云

谷歌云平台及其 Anthos 平台正在努力打入数字化转型预算。与此同时，云提供商正在考虑在其关键垂直领域进行扩张，如零售和金融服务。

随着年收入接近 160 亿美元，谷歌云平台一直在赢得更大的交易，有甲骨文老将托马斯·库里安的强势领导，并被视为 AWS 和微软 Azure 的坚实抗衡。Kurian 似乎正在建立一种类似甲骨文的模式，瞄准它能够取胜的行业和用例。

谷歌还经常因其在开源技术，特别是容器方面的深厚专业知识而脱颖而出，这要归功于它在 Kubernetes for orchestration 和 Istio service mesh 的开发中发挥的核心作用，这两项技术正迅速成为行业标准技术。

# 其他人

其他值得一提的还有 IBM Cloud 和阿里云。IBM Cloud 一个平台，主要关注以下云计算服务:IaaS(基础设施即服务)、Saas(软件即服务)和 PaaS(平台即服务)。这是市场上更符合预算的定价模式之一，而且它完全可定制，可以进一步削减成本。

与此同时，阿里巴巴借助一批企业合作伙伴迅速扩张。阿里巴巴能否在中国以外扩张，还有待观察。无论是哪种情况，阿里巴巴都有很长的路要走。

目前，AWS 在市值和提供最广泛的功能和成熟度方面继续领先。

但似乎微软已经开始弥合这两者之间的差距，并将继续这样做，继续投资建设 Azure 云平台，并进一步计划加强与其内部软件的联系。

然后是谷歌，在它的新领导下，它可能成为一个更重要的企业竞争对手。它已经在某些客户身上取得了良好的进展，特别是在 Kubernetes 和机器学习专业知识方面，但要证明自己是一个可行的企业选项，还有很多工作要做。
