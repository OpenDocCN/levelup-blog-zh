# 表情符号翻译器

表情符号是图标，用来表示人们为了表达自己的感受而在脸上显示的表情。

他们有这么多可能大多数人都不知道他们的意思。

因此，我们可以做一个表情符号翻译应用程序，这样我们就可以把表情符号翻译成英文。

表情符号是标准的 Unicode 字元集的一部分，因此我们可以制作字典或地图，将使用者输入的表情符号字元转换回英文。

我们的表情符号翻译应用会接受表情符号的输入，然后通过复制字符串来创建翻译，然后将里面的表情符号映射回它们的英文对等项。

为了让它更吸引人，我们可以添加一个 Clear 按钮来清除输入值。

# 模因发生器

我们的应用程序可以让用户输入文本并添加图片，从而从中创造出模因。

为了让这变得简单，我们使用 HTML 画布来实现这一点。要以允许我们添加文本和图像然后下载输出的方式来操纵画布，我们可以使用像 Konva 这样的一些库来使其更容易。

Konva 还允许我们像任何照片编辑器一样添加控制柄来转换对象，所以这是 Konva 的另一个很棒的特性。

# 打字练习应用

制作一个让用户练习打字的应用程序也能让我们练习制作应用程序。

我们从用户那里获取键入的输入，然后将输入的文本与我们要求用户键入的内容进行核对。

我们可以这样做:拆分输入的字符串，并对要求用户键入的文本执行相同的操作，然后检查每个单词是否以相同的顺序输入。

这些问题可以在我们喜欢的任意多个问题之间轮换。我们可以在应用程序中对它们进行硬编码，也可以创建一个 API 来随机抽取问题。

我们甚至可以通过添加一个 UI 来添加或删除问题，从而使它变得更为华丽，以便我们可以轻松地更改问题集。

此外，我们还必须通过将会话开始以来经过的时间除以键入的单词数来计算用户的键入速度。

因此，我们需要一个计时器来计算自用户开始键入以来经过的时间。

然后我们会得到一个用户可以使用的不错的打字练习应用程序。

# 刮网器

我们可以创建一个网络刮除，以获得公共信息，我们想从一个网站。

这样，我们就不必手动搜索数据。

我们可以用它来搜索一个或多个网站。它搜索的数据可以是任何不为人知的东西。

在这个过程中，我们学习了如何解析 HTML 和 XML。

# 税收计算器

我们可以制作自己的应用程序来计算我们在获得工作报酬后必须缴纳的税款。

它让我们设定税收规则，并增加每份薪水或工作的收入。

然后，我们可以使用税务规则，根据我们从工资支票中收到的总额来计算我们必须支付的税额。

# 交易寻找者

我们可以用类似易贝 API 的东西构建一个交易查找应用程序，让我们搜索产品的价格。

其他零售商如沃尔玛、百思买和其他零售商也有 API 来获取他们的价格和库存数据。

我们只需要调用这些 API 来获取数据并在屏幕上显示出来。

在我们的应用程序中，我们可以添加一个搜索框，通过使用搜索关键字向这些 API 发出请求来搜索商品。

如果我们可以通过这些 API 找到我们想要的东西，我们也可以从网站上抓取数据并查看它们。

如果我们在抓取，那么我们应该抓取得太快，这样我们的 IP 地址就不会被我们抓取的网站屏蔽。

我们可以构建许多对我们使用和练习 web 开发技能有用的东西。

一个打字导师很容易建立和有趣的使用。此外，我们可以构建自己的表情符号翻译器，将表情符号翻译成英语。

我们还可以构建一些东西来帮助我们进行采购和纳税，比如交易查找器或税收计算器。
