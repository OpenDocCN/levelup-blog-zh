远程工作越来越受欢迎，在过去的几周里，对它的需求呈指数级增长。

以下是使用远程员工和远程团队的一些不明显的原因:

*   公司可以在全世界寻找最优秀的人才。他们可以在邻居之外寻找新的人才。
*   公司希望将人们安置在世界不同地区的市场中。让人们接近客户，并与他们在同一个时区是有意义的。

目前，冠状病毒正迫使许多公司让员工在家工作。突然之间，许多雇主和雇员都在想如何应对这种不寻常的情况。

公司总是希望提高生产率和利润。根据我作为远程工作者的十年经验，我可以说，并不是每个远程团队都在有效地工作并实现高生产率。世界上充满了对远程工作一无所知的公司和组织，当病毒迫使他们进行远程工作时，他们正在遭受痛苦。

公司需要有正确的实践来提高远程工作的效率。

# 远程团队的最佳实践

有很多实践可以改善远程工作。在本文中，我将介绍前三名。

1.  积压的工作
2.  异步通信
3.  及时反馈

# 积压的工作

**积压订单**确保人们需要新工作时有地方可去。

许多软件开发组织知道如何使用积压工作和可视板来组织工作。待办事项是组织想要完成的工作的优先列表。backlog 需要可以从任何地方访问，像 Github boards 和 Trello 这样的在线工具非常适合。

这些工具不仅仅适用于分布式团队:许多现场团队也在使用它们。视觉板对每个人都是有用的工具。对于远程团队来说，拥有这样的工具更为重要，因为在分布式环境中，人们不能像在协同定位环境中那样依赖面对面的交流。对于远程团队来说，四处询问的做法并不奏效。

应该对待办事项中的所有项目进行计划。

人们需要能够相信他们从待办事项中提取的任务代表了组织想要完成的工作。人们不想把时间投资在不需要的东西上。没有人喜欢把时间浪费在无用功上。

团队中的每个人都应该熟悉待办事项中的任务。

联合计划是确保每个人都熟悉任务的一种方式。团队可以将 backlog 集合起来作为团队练习。在 Scrum 中，整个团队定期聚在一起计划项目下一次迭代的任务。在[冲刺计划会议](https://www.scrum.org/resources/what-is-sprint-planning) [，](https://www.agilealliance.org/glossary/sprint-planning/#q=~(infinite~false~filters~(postType~(~'page~'post~'aa_book~'aa_event_session~'aa_experience_report~'aa_glossary~'aa_research_paper~'aa_video)~tags~(~'sprint*20planning))~searchTerm~'~sort~false~sortDirection~'asc~page~1)),)之后，团队成员对任务了如指掌，能够在迭代的后期开始工作。

我已经看到 Scrum backlog 规划方法在构建工作待办事项方面工作得很好。

不要太执着于这里的“积压”这个概念。从中吸取的最重要的东西是**需要有一些机制来确保人们总是知道下一步该做什么**。你和你的团队可以一起决定什么是最适合你的工具。

# 异步通信

编程需要高度的专注。如果你不想破坏程序员的注意力，你就不应该打断他的工作。达到紧密聚焦的“心流状态”需要时间，而干扰会损害聚焦。事实上，频繁的干扰可能会阻止一个人永远达不到心流状态。他的注意力从来没有深入到达到“深度工作”和高生产率的状态。

近年来，开放式办公室变得不那么受欢迎了。公司已经意识到，在开放的工作环境中不断的交流和互动会扼杀生产力。知识工作者和程序员一样，需要私人空间来集中精力。

也就是说，并不是所有的工作都是在心流状态下发生的深度工作。知识工作者需要交流来共享信息，并同步他们的工作以实现共同的目标。这就是异步通信的用武之地。

**当你发送一条消息而不期望立即得到响应时，就会发生异步通信。**以电子邮件为例。几个小时后，我打开并回复了邮件。

**同步沟通发生在你发送消息，接收者阅读并立即回复。**会议或水冷器讨论是同步沟通的例子。

同步交流可以提高工作效率，因为当人们从繁重的工作中休息时，他们可以集中精力。人们应该能够按照自己的节奏在深度工作和交流之间交替进行——这将是高生产率的秘诀。

公司应该鼓励异步通信。这应该成为公司标准工作文化的一部分。像 Slack 这样的工具对于异步通信来说是极好的。要在 Slack 这样的工具中实现有效的异步通信，需要遵循以下规则:

*   人们不期望他们的问题能得到立即的回答。
*   当有人问他们问题时，人们不会觉得有必要马上回答。

还应该注意到，异步通信可以使人们在不同的时区工作成为可能。并非所有的消息传递都需要在时区重叠时发生。您可以在另一个时区的收件人睡觉时发送一个问题，然后在第二天早上醒来时收到答案。

# 及时反馈

有时候，开发人员会遇到这样的情况:在从别人那里获得一些进展之前，他们无法在一项任务上取得进展。在工作继续之前，需要做出决定，需要回答问题，或者需要购买一些新工具。

你不希望你的开发者长时间被卡住。你要赶紧帮他(或她)，让他有空继续工作。

异步通信意味着你不需要立即回答人们的信息和问题。另一方面，你不应该花太多时间来回答和反馈。长时间的延迟可能会令人沮丧，而且，由于显而易见的原因，延迟对整体生产力来说是灾难性的。

# 总结

要在分布式团队中取得成功，最重要的事情是:

1.确保你有一个坚实的积压工作随时可用
2。促进异步通信
3。提供及时的反馈
