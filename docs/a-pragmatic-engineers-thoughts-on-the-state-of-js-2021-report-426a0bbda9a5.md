虽然我不是统计学家或数据工程师，但我爱、爱……爱、爱、爱数据和统计。我说过我喜欢统计数据吗？是的。好的。难怪每年当 JavaScript 的状态结果出来时，我都会放下手头的工作。我会停止编码，我会停止看苹果电视上的任何最新消息，我会停止在 Bumble 上刷卡，吃汉堡，喝啤酒。想到任何你想要的，我会停下来看报告。什么？告诉我你不会做同样的事…😆

## 人口统计数据

2021 年的报告出来得有点晚，但我不在乎，它和以前一样有趣。这清楚地证明了 JS 仍然很重要，但是所有的爱尔兰开发人员(我来自的地方)都忙着喝酒，没有时间做调查(0.4%，也就是 50 名工程师)。真的吗？！？)，这很有趣，因为你知道，我们有[硅坞站](https://en.wikipedia.org/wiki/Silicon_Docks)，是的，这是一个东西。英语仍然是主要语言，其次是西班牙语、俄语、法语和德语。有道理。令人惊讶的是，很少有人从中国回复。我不确定网站是否会被屏蔽，但我不知道为什么会被屏蔽。可悲的是，我还注意到，在 36 岁的时候，我现在是工程师群体中的一员，比 24-34 岁的群体小得多，这个群体是我的两倍多！我并不想得出无知的结论，但这确实让我想知道，软件工程师越老，他们越不关心这些统计数据，或者他们越少使用 JS，或者根本不关心？也许你可以在评论里告诉我。

没有受过高等教育的开发人员仍然是相当小的一部分，仅占 18.7%。我就是其中之一，我不确定我是否喜欢那些数字。我非常支持自学，使用 MOOCs，只有在有意义的时候才求助于高等教育。看到仍有如此多的软件工程师觉得需要有一个大学文凭来证明他们的职业和成功，我个人觉得这有点令人担忧。

更令人困惑的是，当你看到人们在哪里学习 JavaScript 时，结果只有 13.6%的人在学校或高等教育中学习。这让我相信大学和学校可能在教授*【错误】*的东西，或者依赖过时的课程。不过，我还是有一句至理名言要分享。**最终，只要你学习编程概念，你在学校学什么语言没什么关系。**在很大程度上，大多数编程语言共享主要概念，所以 JS 不是你的第一语言不应该成为你职业生涯的阻碍。

现在，就替代资源而言，作为一名媒体作者，让我感到温暖的是看到**媒体仍然是许多软件开发人员的主要资源。42.1%的份额是令人钦佩的高**，所以这里的每一个作家谁贡献了教程和文章多年来的荣誉。坚持做下去。我记得我一度认为 Medium 是比 StackOverflow 更好的资源，这是有意义的！

学习资源的另一个方面当然是提供资源的人。我承认这是一个有争议的问题，但这是我的观点，所以就这样吧。看到丹·阿布拉莫夫和肯特·C·多兹分列第一和第二位让我很困扰。我确信他们可能是好人，但我不认为他们在 JavaScript 领域做得很好。Kent 的 React 测试库是一堆乱七八糟的东西，经常会产生比解决的问题更多的问题。Dan 的 Redux 和 Create React App 是另外两个我觉得在 web 开发界做得不太好的东西。不过让我开心的是看到韦斯·博斯获得第五名。坦率地说，我认为他应该获得第一名，因为我总是发现他的资源和教学风格既有主题性，又有实用性和吸引力。说真的，[只要上他的免费 JavaScript 30 课程](https://javascript30.com)，你就会觉得自己已经淘金了！那里教的一半东西，很多 JS 开发者都不知道！高年级的也不行。

就这一点而言，工资似乎在上涨，但我认为同样值得注意的是，我们正处于通货膨胀时期，这可能意味着，这些加薪的意义远不如过去。当然，你还有一个事实，那就是远程工作或在家工作确实降低了一些成本，尽管它会增加其他成本，所以很难判断这笔钱实际上花了多少。

## 技术

虽然渐进式网络应用程序在受欢迎程度方面名列前茅，这并不奇怪，但我认为明年的结果将显示一个非常不同的现实。PWAs 并没有真正起飞。几年前，我是他们的忠实粉丝，但坦率地说，我认为他们都很难共事，支持也很糟糕，大多数人开发的都不是一个进步的网络应用。他们实际上正在做的是创建一个 web 应用程序，它有一个图标，可以离线运行某些东西，但进步的部分几乎是事后的想法。我说，首先让我们都建立可访问的 web 应用程序，然后让我们看看在 web 空间中是否还有其他真正值得破坏的东西。哦，可选链接和无效合并成为了一种最佳实践？嗯，是的，这是意料之中的。多年来，人们都去教堂为这些 JS 特性祈祷。

我很高兴看到 WASM 越来越受欢迎。我明白为什么，虽然我仍然是少数，但 web assembly 在正确使用时可以非常强大，并且与其他任何东西相比，对浏览器资源的占用非常少。我感到高兴的另一个方面是对 React 的满意度和兴趣下降，尽管似乎所有前端库和框架都在下降。**不要误会我的意思，我并不讨厌 React，但我也不喜欢它**，尤其是增加了挂钩之后。Svelte 似乎再次受到关注，尽管我会说，正如我前几年说过的，Svelte 是一个编译器，而不是一个库或框架。不管怎样，我确实喜欢它，所以也许它会越来越受欢迎。

说到后端，我就没那么开心了。**眼见 Next.js 大幅下跌，多少有些出乎意料。我一直觉得它很有前途。Express.js 也是如此，尽管我自己喜欢一些我从未听说过的好的 ol' vanilla Node.js. SvelteKit、Astro 和 Fastify，但现在我很感兴趣，因为它们在开发者满意度排行榜上排名都很高。**

测试是另一个没有好坏之分的领域。事实上，回想前几年，我并没有感到有多大的变化。笑话仍然是王，人们仍然使用柏树和故事书仍然被认为是一个测试工具，即使它不是！在某些方面，我觉得没什么变化。许多人不喜欢测试。我看到了它的价值，无论是单元测试、集成测试还是 E2E 测试，但是我不知道有多少工程师——尤其是 JavaScript 工程师——这么认为。我也是那些认为开发人员也应该编写所有测试的怪人之一，甚至包括 E2E 测试。

围绕 monorepo 工具的结果有点令人费解，但与其说是结果，不如说是今年的调查似乎包括了它。我承认 monorepos 并不太成熟，但我已经用了三年多了，不止在一家公司，而是两家公司。随着时间的推移，观察它的轨迹将会很有趣。就目前情况来看，这些数据并不十分确凿。Pnpm 在第一名的位置似乎高得令人怀疑，而 Lerna 排在第七名，感觉低得令人奇怪。 Even yarn workspaces 仅排在第五位，拜托，以 yarn 的受欢迎程度，如果这是真的，我会很惊讶。

我的一个看起来 2021 年还没有冲走的讨厌的东西是 **Moment.js** 。我明白了。处理日期并不总是最容易的，但是出于对软件开发中一切神圣事物的热爱，**为什么有 27.2%的工程师喜欢这个东西？这是腌制品。**对你的项目的包大小进行测试，我打赌 Moment.js 将是你导入的最大的代码块之一。你猜怎么着？90%的时间，它被用于你不需要 Moment.js 的简单的东西。

> 拜托，美女-拜托，试着学点普通的 JavaScript 吧。它可以做很多事情！喜欢…很多！

关于这一点，说你想说的，嘲笑你认识的任何人的生活地狱，如果成为一个巨魔是你生活中的主要功能，但是 jQuery 仍然活得很好。超过 11%的受访者仍然喜欢它，使用它，并且不会放弃它。当你考虑到网络有多庞大时，我说，11%的网络依赖于所有 JS 势利者都很酷而不能使用的东西，仍然需要尊重。

## 这一切都完了吗？

每个人都是戏剧女王。每一年，有些人都有这种奇怪的恋物癖般的希望，希望今年 JS 会完蛋。他们似乎都忘记了一个重要的事实。不管希望和观点如何，JS 仍然是所有浏览器中唯一的原生编程语言，没有一个浏览器显示出任何改变这一现实的意图。

> 不管你喜不喜欢，JavaScript 都会存在。

事情有改善吗？作为一门语言，我会说，绝对是。在很多方面。每年都有一点好转。这是一种显示出明显进化的语言。我喜欢这样。仅 Async/await 和 arrow 函数一夜之间就改变了 JS，而且在过去几年中还增加了许多新特性。一些句法糖，一些不那么。有些使语言更有能力，有些使它更难阅读，但最终，这取决于工程师使用什么对他们最有效，并保持代码易读。

话虽如此，**我也认为 JavaScript 已经不自觉地引领了一些有问题的策略**。我这么说可能听起来像只恐龙，但我认为**我们让网络变得比它应有的更加复杂。此外，用于渲染 HTML 的大量 JS 也导致了更多无法访问的网站和网络应用。我觉得这很令人担忧。我明白了。我们想建立的不仅仅是好的网站。**网络比以往任何时候都更有活力，但这不应该以可访问性为代价。****

不过，总的来说，JavaScript 领域确实比 4-5 年前稳定了一些。我不是说它停滞不前，而是说它似乎已经走出了一个*“跟不上一切”*的阶段。我个人认为这是件好事。我并不是说 JS 领域的创新需要停止，但是我确实认为我们需要更少的干扰和更多的健壮性。**总而言之，我必须同意调查的结论——**[**JavaScript 确实比 2016 年**的](https://2021.stateofjs.com/en-US/conclusion) **状态好很多，那只能是好事。让我们希望这种趋势继续下去。与此同时，继续让世界变得更美好，一次一行代码。**
