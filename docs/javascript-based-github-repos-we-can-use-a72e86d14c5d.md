许多 JavaScript 库和框架都托管在 Github 上。

在本文中，我们将看看 Github 上我们现在或以后可能会用到的顶级库、框架和完整程序。

# [vuejs / vue](https://github.com/vuejs/vue)

毫无疑问，Vue.js 是创建新的前端应用或增强现有应用的最佳前端框架之一。

它是渐进式的，所以它可以和其他 Vue 库一起包含在脚本中，这些库也可以用脚本标签添加。

此外，我们可以使用 Vue CLI 制作新的应用程序，它有许多工具可供我们创建应用程序，并按照我们想要的方式构建它们。

图书馆的生态系统和使用它来构建应用程序的用户正在有规律地增加。

# [沉香/ Motrix](https://github.com/agalwood/Motrix)

Motrix 是一个用 JavaScript 编写的下载管理器。它支持通过 HTTP、FTP、BitTorrent、Magnet 等从各种来源下载。

它可以在 Windows、macOS 或 Linux 上运行。

# [panjia Chen/vue-admin-template](https://github.com/PanJiaChen/vue-admin-template)

一个简单的 Vue.js 管理模板，我们可以添加到我们的应用程序。我们可以将它克隆到我们的系统中，并开始根据我们自己的需要进行定制。

# [zloirock / core-js](https://github.com/zloirock/core-js)

Core-js 是一个为 JavaScript 标准库中的特性提供多种填充的库，比如新的数组和字符串方法等等。

包含的方法包括像`from`和`flat`这样的数组方法。此外，该库中还包含了类似数组的对象，如集合和贴图。

它还包括自己的 polyfill for promises。

除此之外，最近几年 JavaScript 版本中包含的几乎所有东西都在这个库中实现。

然而，polyfills 比内置在浏览器中的标准库要慢，所以我们应该只在我们的应用程序面向没有这些功能的浏览器时才使用它。

# [优赞/万特](https://github.com/youzan/vant)

该存储库拥有轻量级组件，可用于 Vue.js 应用程序。

它支持 babel-plugin-import、自定义主题、国际化、TypeScript 和服务器端呈现。

根据[文档](https://youzan.github.io/vant/#/en-US/home)，有很多组件可用，包括幻灯片、按钮、单元格、图标、图像、布局容器、滑块、单选按钮、选择器等等。

# [vuejs / vuex](https://github.com/vuejs/vuex)

Vuex 是 Vue 应用的首选状态管理解决方案。它允许我们为 Vue 应用程序创建一个中央数据存储，以便我们可以在任何组件之间共享数据。

这一点很重要，因为它使得组件之间共享数据比用 props 传递数据要干净得多。

# [mozilla](https://github.com/mozilla) / [pdf.js](https://github.com/mozilla/pdf.js)

js 是一个 PDF 查看器库，我们可以将它嵌入到前端 JavaScript 应用程序中。

它正在积极开发中，所以我们可以肯定新的功能和修复将被添加。

# [ember js](https://github.com/emberjs)/[ember . js](https://github.com/emberjs/ember.js)

Ember.js 是一个 JavaScript 前端框架，不如 Angular、Vue 或 React 流行。但它仍在积极维护，并有一个活跃的生态系统，包括为它制作的库和使用它来构建应用程序的用户。

它也不难学，所以我们可能想用它来创建我们的应用程序。

# [zeit](https://github.com/zeit) / [hyper](https://github.com/zeit/hyper)

Hyper 是一个用 HTML、CSS 和 JavaScript 编写的终端程序。

因此，它也是一个跨平台的程序。我们可以在 Windows、Linux 或 macOS 上使用它。

它也有各种插件来扩展它的功能。

有扩展可以打开与当前标签目录相同的额外标签。

此外，有一个添加搜索，另一个添加更好的键盘快捷键导航。

# [zeit](https://github.com/zeit) / [next.js](https://github.com/zeit/next.js)

Next.js 是一个基于 React 的框架，它让我们可以轻松地创建静态站点和服务器端呈现的前端。

它有很多 React 没有的特性，比如数据获取、静态文件服务、路由等等。

为了使创建一个新的 Next.js 项目变得容易，有一个专门为它设计的 CLI 程序。

# [辛德索胡斯](https://github.com/sindresorhus)/[eslint-plugin-unicorn](https://github.com/sindresorhus/eslint-plugin-unicorn)

这是一个 eslint 插件，它给 ESLint 增加了更多的规则来增强它的林挺能力。

默认情况下，ESLint 有很多规则，但它们可能不适合所有环境。所以这个插件给了我们另一个选择，那就是我们想要使用的林挺规则。

# 结论

我们可以用 Next、Vue 和 Ember 构建前端。

此外，我们可以构建完整的程序，如 Hyper terminal 程序，它是用 HTML、CSS 和 JavaScript 构建的，可以在任何地方工作。

PDF.js 允许我们轻松地向页面添加 PDF viewer 小部件。
