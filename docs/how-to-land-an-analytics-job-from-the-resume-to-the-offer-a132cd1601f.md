# 谈判报价

当一家公司向你发出邀请时，他们已经对你感到兴奋了。在这一点上，他们相信你能给团队带来的价值。

Helbling 告诉我们的团队，要求更多的钱是可以的。他建议我们不要因为不问而落入“表现得像个团队成员”的陷阱。然而，当你提出要求时，一定要有理由。提前想好这些理由。

一些公司会提前给出他们的最佳报价，而不是敷衍了事，所以永远要对公司给你的报价心存感激。公司应该提供一个降低起点的理由。如果他们不能增加你的初始工资，考虑对你仍然有价值的替代方案，也许可以要求额外一周的 PTO。最后，在你谈判的时候，始终保持对角色和公司的热情。

赫尔布林的方法最让我惊讶的是他对公司如何看待第一次谈判的个人的洞察。他告诉我们，如果你要求加薪，并且做得很好，公司会因此尊重你。如果你表现出警觉和无所畏惧，他们甚至可能一开始就信任你，让你承担更多的责任。通过在谈判阶段向公司证明你知道自己的价值，你发出了这样的信号:“我是一个你可以信任的人。”

# 要求加薪

谈判是一种技巧。不要拘泥于一个特定的结果。向你的老板解释你真的喜欢为公司工作。通过提前解决问题来减轻经理的担忧。

> "不要在情感上对自己的价值不成熟。"

在提出要求之前，确保你已经做了一些调查。带着一个数字进入这个对话是很好的。解决这个问题的一个好方法是告诉你的经理，你已经看到和你职位相似的其他人获得了一定水平的报酬。解释你希望得到类似的报酬，然后解释你的价值以及为什么你应该得到相应的报酬。

从公司的角度来看，让某人接受一份工作很重要，但留住那个员工就更难了。

# 获得分析职位的技巧:

## 证书

在考虑获得哪项认证时，赫尔布林建议问问自己，“我拥有或不拥有某项认证对公司来说意味着什么？”迈克尔·赫尔布林从未获得过认证，所以他是一个活生生的例子，证明了没有证书和徽章，成为一名成功的分析师也是可能的。

证书越受欢迎，它的分量就越轻。像 GAIQ 这样的认证是如此普遍，它本身可能不足以让你找到工作。通常，证书确实是招聘人员的复选框，但它们不是一切。相比之下，Adobe Target 等不易获得的认证确实有一定意义。还有，成为人数有限的开发者社区的一员，增加了它的价值(即:谷歌开发者专家)。

如果你的公司提供免费认证，你应该接受。请记住，证书可以帮助你进入面试室，但你必须能够解释为什么你有能力。

## 迷你课程和在线学习

在赫尔布林看来，迷你课程是个人主动性的展示。这是一个很好的方式来表明你已经花时间来提高你的技能和完善你自己。本质上，它显示了努力，这是一个好迹象。然而，完成迷你课程并不一定是知识的保证。

这些在线服务的例子包括 Coursera 或 Code Camps 等平台。

## 志愿服务和公益工作

如果有意识地使用，志愿者工作是很棒的。人们志愿服务的动机各不相同，有些是内在的，有些是外在的。赫尔布林解释说，他提倡回馈社会，尤其是在你事业发展的时候。

此外，志愿服务是添加到简历中的一个有用类别。展示你的志愿者工作如何产生了某种结果，然后解释你如何能给你面试的公司提供类似的积极结果。
