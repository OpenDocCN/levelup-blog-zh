编写好的软件很难。有很多事情要考虑。但是每个人都会欣赏好的软件。

在这篇文章中，我们将看看好软件的品质。

# 软件质量的特征

好的软件有许多特征。它们包括以下内容。

## 正确性

正确性对于好的软件来说很重要。规格、设计或实现不应该有错误。

## 可用性

用户应该能够容易地学习和使用一个系统。

## 效率

一个软件使用的资源越少越好。处理器、内存和磁盘空间的使用应该最小化。

## 可靠性

能够稳定执行所需功能的系统非常重要。失败应该尽可能少。

## 完整

应该考虑安全性。我们的软件应该让攻击者访问未经授权的资源。

此外，数据验证也很重要，这样坏数据就不会保存到系统中。

## 适应性

一个无需修改就能用于不同情况的系统很好。

## 准确(性)

其输出的准确性是好的。这衡量软件是否为用户输出了正确的结果。

## 稳健性

如果一个系统在得到无效输入和有压力的环境条件后仍然在工作，那么它对我们的系统是好的。

## 可维护性

现有系统易于改变是很重要的。我们越容易做出改变，就越好。

## 轻便

一个在不同环境下运行的系统，从它最初设计的环境来看，是很好的。

## 复用性

一个软件的可重用部分越多越好。

使用可重用部件意味着我们不必从头开始重用它们。

## 可读性

易于阅读的代码易于更改代码。如果我们能更快地理解它们，那么我们就能更快地以一种更少出错的方式做出改变。

## 易测性

让我们的软件系统可测试是至关重要的。如果我们的代码易于编写单元测试，那就很好。

## 易懂

对于我们来说，从全局的角度或者在详细的代码层次上理解我们的系统的能力是非常重要的。

更容易理解的系统更连贯。

有了好的软件，我们就更难产生缺陷，做出改变也更快。

# 软件质量目标

设定维护软件质量的目标是提高质量的好方法。

这给了我们努力的方向。

# 明确的质量保证活动

质量保证总是很重要的。没有它，用户将会看到缺陷，因为没有人发现它们。

这让我们压力更大，我们的声誉也因此而恶化。

我们必须明确我们的优先事项，开发者将相应地做出回应。

# 测试策略

为了系统地测试软件，我们需要一个测试策略。否则，我们的测试将变得不一致和不可预测。

如果我们在不同的时间测试不同的东西，很容易漏掉一些东西。

# 软件工程指南

有了软件工程的指导方针，事情会变得更容易。

# 技术评论

代码评审和设计评审让每个人的生活变得更加轻松。

他们揭露缺陷，让人们检查草率的做法。

我们可以随便聊聊问题。

# 变更控制程序

不受控制的变化是不可预测的。因此，我们应该有一些程序，使它们可以预测。

例如，我们可以记录完成一个特性的步骤，编写代码，然后发布到产品中。

# 结果的衡量

我们需要一些数字来表明我们的计划是否可行。

我们上面提到的质量属性包括正确性、可用性等。也可以测量。

# 样机研究

原型让我们更容易做出决定，让我们摆弄一些可能成为现实的东西，而不仅仅是纸上谈兵或头脑中的东西。

# 检测到的缺陷百分比

我们可以检查我们系统中的缺陷率，看看它运行得如何。

然后，如果有必要，我们可以通过查看我们系统中的缺陷来改进我们的系统。

# 什么时候做质量保证

我们应该对大多数东西做质量保证。

既然设计影响到系统的每一部分，我们应该三思而行。

缺陷可以在任何地方蔓延。因此，我们应该强调质量保证的每一步。

同样需要注意的是，如果我们在代码上做得更少，那么更改代码带来的风险就会更小，产生新缺陷的风险也会更低。

# 结论

软件质量保证有很多方面。它们包括我们系统的准确性、适应性和健壮性等。

我们必须确保每一步都检查质量。
