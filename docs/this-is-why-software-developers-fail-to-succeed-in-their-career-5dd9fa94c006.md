# 这就是软件开发人员在职业生涯中失败的原因

> 原文：<https://levelup.gitconnected.com/this-is-why-software-developers-fail-to-succeed-in-their-career-5dd9fa94c006>

## 知道原因，这样你就可以避免犯同样的错误

![](img/0fa1837ddfcece211b26baf7f57a07d3.png)

来自 [Pexels](https://www.pexels.com/photo/young-troubled-woman-using-laptop-at-home-3755755/?utm_content=attributionCopyText&utm_medium=referral&utm_source=pexels) 的 Andrea Piacquadio 的照片

软件工程职业既充满挑战又令人兴奋。这也是市场上薪酬最高的职业之一。从事技术工作和赚大钱的吸引力每年吸引成千上万的人加入软件行业。

然而，在从事软件工作几年后，许多怀着发迹梦想加入软件行业的人感觉自己陷入了一份没有出路的工作。他们失去了最初的兴奋感，也找不到继续全力以赴的动力。

> 但是为什么如此多的软件开发人员没有成功，而有些人却可以实现他们为自己设定的大部分目标呢？

没有任何动力，也没有明确的未来发展道路，这些软件开发人员放弃了获得成功的梦想。但是为什么如此多的软件开发人员没有成功，而有些人却可以实现他们为自己设定的大部分目标呢？

在我从事软件行业超过 15 年的时间里，我看到了开发人员成功和失败的故事。让我们深入探究软件开发人员犯的一些错误，并最终放弃在职业生涯中获得成功的所有希望。

# 1.他们在工作中不真诚

这一点对每个软件开发人员来说应该是显而易见的。如果你没有尽全力工作，那么你就不会有想要的职业发展。但是，你仍然会发现一些开发人员没有把必要的努力投入到他们的职业生涯中。

工作不积极、不真诚的软件工程师无法实现职业目标。他们做分配给他们的最起码的任务。即使那样做，他们也要慢慢来。很多时候，整个团队都因为这些开发人员交付的低质量工作而遭受损失。

如果你继续做你被要求做的事情，你会得到你被给予的。当你把决策权交给别人的时候，那肯定会失败。你不能以这种平淡的态度向公司要求你想要的职业发展。

为了成倍增长，开发者需要对他们的工作充满热情。没有激情，你就无法在像软件工程这样具有挑战性的工作中发挥出最佳表现。你需要不断超越公司的期望，实现出色的业绩。如果不主动和真诚，就不可能实现这种契合。

# 2.他们没有导师

要在软件职业生涯中成长，你需要得到正确的指导。没有指导，你所有的努力就像在大海中划一艘没有目标的船。你可以尽你最大的努力，但你的职业之舟将无处可去。

许多软件开发人员认为他们知道什么对他们的职业最有利。他们不会向自己的同龄人、长辈或有经验的人寻求职业建议。

有时候，开发人员希望接受指导。但是他们不知道如何接近某人，请他们做他们的导师。他们犹豫地想，如果这个人拒绝成为他们的导师会怎么样。但关键是，除非你提出要求，否则你永远不会知道有人是否准备好指导你。

在这两种情况下，这些开发人员错过了利用他们可以从已经做了他们正在尝试做的事情的人那里学到的过多智慧的机会。最终，由于缺乏正确的方向，他们的职业生涯会受到影响。

# 3.他们跟不上技术的发展

软件职业如此具有挑战性的主要原因之一是技术在快速发展。与此同时，技术进步也让这一职业令人兴奋。

当软件开发人员开始他们的职业生涯时，他们对他们技术领域的新事物非常兴奋。他们会定期尝试学习一种新的语言或框架。他们的热情有助于他们尽最大努力工作。

但是随着时间的推移，许多开发人员对他们的工作越来越没有热情。当他们继续做工作场所提供的重复性任务时，他们会失去最初从事软件职业的热情。他们变得对自己日复一日的工作感到舒适，而无需付出太多努力。

一旦开发人员变得过于舒适，他们就不会自己学习任何新的东西。他们责怪公司没有给他们提供令人兴奋的工作。这些开发人员中的许多人抱怨说，他们没有时间自己学习任何新东西。他们坐等公司提供培训来提高他们的技术知识。

在一天结束时，每个人的责任是让自己跟上时代。经常为公司或时间找借口的开发人员没有机会在职业生涯中成长。

# 4.他们的态度很糟糕

在职业领域，你的行为很重要。你的态度可以成就或毁掉你的职业生涯。是的，态度还是一件大事。你有多有才华并不重要；如果没有正确的态度，你最终将无法在职业生涯中成长。

有些人觉得傲慢是坏事，如果你是一个优秀的程序员，你就可以逃脱惩罚。嗯，在某些情况下这可能是真的，但这肯定不是标准。你可以写出优秀的代码，但是如果你不能作为团队的一员工作，你就不能在职业生涯中走得很远。

然后是一些开发人员，他们在团队的每个人身上找毛病。他们总是把自己的失败归咎于别人。在团队中建立融洽的关系不是他们的强项。不管你有多有才华，如果你态度不好，团队都不会喜欢和你一起工作。

态度不好的开发人员犯的另一个错误是他们不喜欢帮助别人。他们也不是很平易近人。慢慢地，团队与这个人拉开了距离。一旦一个开发者在团队中被孤立，这个开发者在公司就没有成长的机会。

# 5.他们不寻求帮助

在任何职业中，寻求帮助都是必不可少的。没有人一开始就知道一切。学习一个职业的本质需要几年时间。学习它们最快的方法是向已经做过的人寻求帮助。

在职业生涯中未能成长的开发人员发现很难从他人那里获得帮助。当他们被困在分配给他们的任务中时，他们会犹豫是否向人们伸出援手。

有时候犹豫是因为他们自己的自负。这些开发人员有一种误解，认为他们比团队中的其他人拥有更好的知识。同时，有些人只是羞于寻求帮助。即使他们想接受帮助，他们也没有足够的勇气去请求帮助。

不管你的理由是什么，不接受别人的帮助肯定会导致失败。没有人能靠自己在软件职业生涯中成长。我们都需要来自同事、经理、上司以及组织内外许多其他人的支持。

# 6.他们不会建设性地接受反馈

反馈是软件开发人员最有力的工具之一。积极的反馈会提升你的士气，推动你做得更好。同时，建设性的反馈有助于你了解自己的不足并加以改进。

软件工程师失败的原因之一是他们不知道如何利用反馈。当他们收到对他们表现的任何改进意见时，他们会认为是针对他们个人的。事实上，他们试图在给他们提供反馈的人身上寻找错误。

因此，不知道如何处理反馈的开发人员错过了解决他们弱点的机会。你需要知道你的弱点来改进它们。因为你不容易发现自己的错误，来自同事或经理的反馈有助于你看清自己的愚蠢。

如果你注意的话，你会发现在你的日常工作中你会得到很多反馈。并非所有的反馈都是建设性的。利用每一个输入来反思你的行为。最后，如果你没有从任何反馈中发现价值，完全可以把它作为个人意见丢弃。

# 7.他们不知道什么时候离开公司

软件开发人员的职业生涯停滞不前的一个重要原因是，他们不知道何时在公司外寻找更好的机会。他们希望他们的经理能够神奇地理解他们的愿望，并关注他们的职业发展。

然而，单个开发人员的职业生涯不是他们经理的责任。开发人员需要跟上技术的步伐，并随时准备好投入市场。管理人员无疑可以帮助开发人员实现他们的目标。但所需的努力必须来自个人。

在一个团队或一个角色中工作了几年后，软件工程师开始适应他们的日常任务。做一些你重复做的事情通常需要更少的努力和脑力。一旦惯性出现，就不容易摆脱这种心态。

即使开发者知道目前的情况不适合他们的成长，他们也很难打破这种惯性。他们推迟了准备面试的计划。参加面试的想法对他们来说似乎是一项艰巨的任务。最终，他们满足于公司或经理交给他们的任何东西。

过去，[我一直为不知道什么时候离开公司而感到内疚](https://medium.com/illumination/how-my-first-job-experience-shaped-my-professional-career-71d55ac775a8\)。在我的第一份工作中，当公司作为印度历史上最大的公司骗局的一部分倒闭时，我继续尽我最大的努力。最后，我不得不在一周之内换了工作，而且工资涨幅很小。

# 最后的想法

这份清单并不是要阻止你将软件开发作为职业。相反，这些细节旨在帮助你理解，记住这些要点将促进你的职业发展。

找一份软件工作相对容易。但是得到这份工作仅仅是开始。要跟上软件行业的步伐并实现预期的增长，需要付出相当大的努力。

确保你没有犯上面列出的错误。有了正确的态度和适当的指导，我相信作为一名软件开发人员，您可以实现您为自己设定的目标。
