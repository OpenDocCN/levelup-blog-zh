# 为什么 OOP 和函数式编程的粉丝都是错的

> 原文：<https://levelup.gitconnected.com/why-both-oop-and-functional-programming-fans-are-wrong-41c5a2da8b79>

## 而且好笑的是，这和编程一点关系都没有。至少，不是你想象的那样…

![](img/707ab8ea9c0b91f2bfaa0e7e0d159a6d.png)

迈克尔·斯塔基在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上的照片

H 你见过函数式编程(FP)和面向对象编程狂热者争论哪个有意义，哪个没有意义吗？看起来既有趣又恐怖。你想去买些爆米花，但它是如此强烈，你不敢把目光移开。那致命的一击随时可能到来！忘记总统候选人辩论吧，这是有趣的东西。口吐白沫，獠牙，紧张的大脑肌肉准备展开终极辩论。绝对是作秀。此外，如果你问我，难以置信的徒劳。

当引入 React 挂钩时，函数式编程才真正引起了我的注意。突然，我开始走进关于纯功能和副作用的对话。一开始，我没有理会。然后我扬起一条眉毛，接着又扬起一条，开始更加注意了。作为一个在 OOP 开始在 PHP 流行的时候加入 web 开发的人，我记得 OOP 势利者的崛起。见鬼，当时我的一个好朋友用 OOP 重写了他的整个粉丝网站——一个相当大的项目，因为引用他的话*“过程化编程已经过时了”*。我什么也没说。我对这一切还太陌生，无法形成看法。毕竟，我还在试图用表格拼凑一个简单的网站——是的，我已经那么老了！

然而，正如每个开发人员都会遇到的那样，我很快就到达了 PHP 和 JavaScript 成为我的软件工程旅程中的下一个逻辑步骤的点。PHP5 几乎是 OOP 的同义词，因此我开始沉浸在对象的世界中。

> 面向对象编程似乎是最简单的概念之一。感觉真的毫不费力。感觉…很自然。

我花了近十年的时间来回顾那个时刻，并分析为什么它在 OOP 潮流中感觉如此自然，它是由函数式编程触发的，而我突然发现自己被 React 16 抛入了函数式编程。

深入研究 FP 的内部，我很快发现了无休止的论坛对话——也就是无休止的激烈争论——关于为什么 FP 有意义或者没有意义。不管出于什么原因，你很少会发现来自过程化编程阵营的人参与到对话中来。我想，一个人必须到了一定的(年轻)年龄，才能就编程范式展开真正的斗争。其他人只是随便写些能让它工作的代码，领了薪水，靠在一杯上好的威士忌上，听着瘦李奇在背景中即兴演奏。不一定是事实。一个假设。

**令我惊讶的是，与 OOP 相比，FP 并没有点击得那么快**，老实说，我仍然经常觉得自己是*【在黑暗中编码】*，这让我思考了很久。起初，我不明白为什么范式的改变对我的大脑有这么大的提升，为什么对其他人没有，为什么会导致两者各自的粉丝之间的紧张关系。**这可能是真正的粉丝吗？是不愿意改变吗？我的意思是，众所周知，人类不擅长应对变化，所以这并非不可能。然后它咔哒一声…**

> 不管是狂热还是无力改变，这与编程范式无关，与代码无关，而是与我们自己的编程有关——人类的大脑。

多年来已经证明，人们通常在某些活动上比其他人做得更好。有些人擅长语言，有些人擅长数学。左脑，右脑，诸如此类。我们学习某些东西的能力在整个人类范围内差异很大。因此，我们也经常把擅长某事和喜欢它联系在一起。例外并非闻所未闻，但作为一种模式，我们可以同意擅长某事通常会促使我们偏爱该活动。

现在，让我们将这些知识应用到编程范例中。我个人认为 OOP 比 FP 更容易掌握。然而，我最好的朋友 Andrew Gribben 一生都在与 OOP 作斗争。对他来说，这毫无意义。然而，我们都是软件工程师。他甚至有学位，而我没有。

> 如果只看到一种编程模式的好处，而没有看到另一种编程模式的好处，仅仅是因为，就像 Lady Gaga 一样，“你生来就是那样的”，那会怎么样？

当然，这并不意味着一个人不能学习——有效地“重新连接”自己的大脑——另一种范式，就像那些努力学习外语的人一样，仍然可以，只是需要更长的时间。突然，**看着这个理论，OOP 和 FP 阵营之间的争斗开始变得更有意义了**。双方都不明白为什么对方不能理解他们所代表的利益，因此产生了摩擦。**不是承认，也许，不是每个人都倾向于有机地理解一个或另一个，而是都试图把他们的观点和理解塞进另一个的隐喻喉咙里。**

> 也许软件开发中的多样性不应该仅仅考虑年龄、种族和性别。还有另一个变量——而且是心理上的。

这是否会让围绕编程的对话和决策变得更加复杂？也许吧。但是话说回来，**我们不能期望编写程序的人比他们编写的软件更简单。这也不意味着 OOP 和 FP 之间的对话，甚至争论需要停止，但是战斗需要停止。我们需要**将对话推进到一个有建设性和多样性意识的空间，在那里它们可以和谐共存，**因为坦率地说，任何认为另一方是错的程序员，都是在对一个已经陷入困境、技能缺口巨大的行业造成巨大伤害。**

> 选择你的战斗。OOP vs. FP 不是一场值得赢的战斗。
