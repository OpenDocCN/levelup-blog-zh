# 所以你想成为一个更好的开发者？—从初级软件工程师到高级软件工程师的旅程提示。

> 原文：<https://levelup.gitconnected.com/so-you-want-to-be-a-better-developer-1f9894a1e991>

![](img/d6490c20b3a92d8b937f94f318da2f47.png)

最近在工作中，我和我的同事被分配了一个任务，为想学习前端的初级开发人员创建一条学习路径。这是我们仍在努力的事情，但它让我想起了我的旅程，以及我是如何从一个了解相对较少的初级开发人员成为一个了解更多的高级开发人员的。

大学毕业后，我直接参加了工作。在一个经验丰富的开发团队中工作，有几件事情很快变得显而易见:

*   我在大学里学到的只是皮毛
*   如果我想成为一名优秀的开发人员，这将是一个终生学习的旅程
*   有很多事情我不知道，更重要的是我不知道我不知道

最后一点是我职业生涯早期学到的最重要的一课。首先， ***不知道的事情就说不知道*** 。这比假装好得多，这意味着你会学到更多。其次，“**你不知道你不知道的**”，这听起来很有趣，但一旦你意识到并接受它，它会非常强大。对我来说，这激发了我的好奇心。它让我提出问题，深入研究主题，不仅试图找出我知道我需要学习的东西，还找出我还没有发现我需要学习的东西。

那我做了什么？有时我感到极度不知所措，“我究竟要怎样才能和我周围的开发人员一样优秀呢？”。要学的东西太多了。剧透警告所有新开发人员，**总会有太多东西需要学习**。这是我们行业的本质。这让它变得有趣和令人兴奋，也让它变得令人恐惧和疲惫。关键是尽你最大的努力不要让它影响到你，只要专注于在你选择的任何技能上做得更好。对我来说是 JavaScript。虽然我当时的强项是 Java，因为我在大学里学过很多，但我的热情是 web 和前端。我知道如果我想走这条路，首要任务是 JavaScript，所以我已经确定了第一个重点领域，现在呢？

## 学习规则

请记住，你可以学习所有你想要的框架，但是如果你没有很好地掌握底层语言的基础，你就不能很好地处理出现的问题。你有没有听到人们抱怨一门语言，比如“啊，我讨厌 JavaScript，它没有意义。”这没有意义，因为你还不明白。如果你花时间去钻研这个问题并弄清楚它，它肯定是有意义的。当你还没有完全理解一些基本知识的时候，尝试去做一些事情是非常令人沮丧的。这就是为什么我最重要的建议，如果你没有别的收获的话，就是学习基础知识。这适用于任何语言。花时间真正掌握这门语言及其所有的细微差别。

## 不要害怕问问题

这个是关键。我知道这是多么的暗示，感觉你是房间里唯一一个没有“得到”什么的人。这种不安全感可能会阻止你问一些必要的问题来帮助你理解。但是你必须坚持到底，**问所有的问题。**如果你是团队的一员，问问你的队友。如果是自学，就去问互联网。就问这个问题。记住没有愚蠢的问题。当你可以伸出手寻求帮助时，就没有必要独自挣扎了。

## 练习。练习。练习。

你可以想读多少书就读多少书，也可以看所有的 YouTube 教程，但是最好的学习方法是通过实践。练习你的手艺。最重要的是保持一致。那是快速进步的最好方法。练习。学习。多练习。了解更多。你也不需要有什么宏大的项目去做，每天做一个形，或者在网站上找一些很酷的东西，然后试着去实现它。一些很好的灵感来源是 [codewars](https://www.codewars.com/) 上的卡式和 [reddit 每日程序员](https://www.reddit.com/r/dailyprogrammer/)上的挑战。关键是要练习。我曾经挑战自己每天解一个形，通常是在早上。你做得越多，你的技能就越快，这也很有趣。你也可以从观察其他人的解决方案中学到很多东西，以及你可以如何改进你的解决方案。

## 找一个知道的比你多的人

如果你能找到一位良师益友，他们对于培养你的技能将会是无价之宝。一个你可以问问题，讨论概念，并且可以在旅途中帮助你的人。这可能是一个同事或一个类似领域的朋友，基本上是任何你可以看到并说我想知道他们知道什么的人。

## 坚持不懈

这不是一条容易的路，这是一条漫长的学习之路，然后当新事物出现时…学习更多。这可能会让你感到不知所措，但是随着你越来越有经验，事情会变得越来越容易。坚持下去。坚持是任何开发人员最重要的品质。在问题中坚持，在骗子综合症中坚持，在错误中坚持。这只是一次大的学习经历。坚持下去，你就会胜利。

如果你已经做到了这一步，感谢你阅读我的漫谈，我希望它在某种程度上有所帮助，并祝你编码之旅好运:)
