在开始我的第一份编程工作之前，我希望知道一些事情

原文：https://levelup.gitconnected.com/a-few-things-i-wish-i-knew-before-starting-my-first-programming-job-fe62c3a85c33?source=collection_archive---------16-----------------------#2020-11-23

如果你在美国的大学里上过任何计算机科学或信息技术课程，那么你很有可能非常精通算法、数据结构和核心编程概念。虽然，这些是开发软件的一些基础，但通常还有更多。但是你还在上大学的时候没有意识到。以下是我希望在开始第一份工作前知道的几件事:

1。不要害怕提问
这一点可能是类似这篇的帖子中被提起最多的一点。但这是一个非常有效的方法。作为一个初级开发人员，我们往往想证明自己的编码能力和技能。很多时候，我们不会问问题，因为害怕看起来不够格。但是如果你为了解决一个问题而绞尽脑汁，那么向一位资深或首席开发人员寻求一些快速的指导通常会更好。人们通常会认为一个人问问题是热切和好奇的，这通常会为你赢得一些印象分。积极主动总比在花了大量时间编码后发现它是错误的并不得不重做要好。

2。 为人类编写代码
警惕编写“花哨的一行程序”或过于复杂的代码。作为初级开发人员，我们经常想通过编写复杂的代码或重用花哨的一行程序来证明我们知道如何编程。其中往往有一半是从栈溢出复制过来的，我们自己也没有完全理解。识别代码是由初级还是高级开发人员编写的最快方法是看它的可读性。80%的时间我们阅读代码，而不是编写新的代码，所以通过编写可读和可理解的代码来善待你的开发伙伴(通常是你未来的自己)。罗伯特·C·马丁的《干净的代码》是关于这个主题的一个很好的读物。我相信所有的开发人员在他们的职业生涯中都应该至少读过这本书一次。

“任何傻瓜都能写出计算机能理解的代码。优秀的程序员会写出人类能理解的代码。”—马丁·福勒

3。尽可能多地查看其他开发人员的拉式请求
经常审查拉式请求是我希望在职业生涯早期就能做的事情。阅读和浏览其他开发人员的代码是一种很好的方式，不仅可以接触到企业代码，还可以了解哪些代码有效，哪些地方可以在自己的代码和拉式请求中改进。如果你不喜欢审查更高级的开发人员的代码，在你职业生涯的早期，花些时间尽可能多地审查拉请求总是好的。

4。 经常提交新代码，保持较小的拉取请求
这与上一点密切相关。当您提交代码时，通常最好保持提交和拉取请求小而频繁。不要等到完成任务编码后才打开拉请求。如果一位高级开发人员发现您的代码有问题，并要求您更改它，而您不得不重新编写大部分代码，这将节省您重新工作的时间。如果你做错了什么或者没有遵循公司的编码标准等等，保持你的拉取请求小而频繁会在早期提醒你。除了保持较小的拉取请求，遵循良好的提交和拉取请求标准也很重要。这是Atlassian写的一篇关于拉请求的成文和不成文规则的好文章。

5。 了解你导入什么库
现在很多软件都是在一个框架中编写的，并且使用很多不同的导入库来进行各种不同的操作。每次一个库被导入到你的软件中，它要么减少样板代码，要么处理一个你的团队不必“手动”编写代码的操作。但是它也带来了安全漏洞的风险。虽然在您的个人项目中导入和探索各种库是非常好的，但是许多公司担心软件中的特殊安全风险或漏洞。即使大公司经常使用工具来扫描库中的漏洞，作为一名开发人员，您也应该理解您将什么引入到代码中。不要因为看到有人在Stack Overflow上使用某个库就导入它，而是在导入之前进行搜索并通读文档。

6。 踩刹车
这是我刚开始第一份编程工作时没有做过的事情。我会一坐就是几个小时，试图找出一个问题的解决方案，常常在一天结束时感到筋疲力尽。我后来在职业生涯中发现，经常休息不仅有助于在一天结束时留下一些精力，还经常帮助我集中思想，更快地找到解决方案。

7 .。 了解你的IDE
你的IDE是你每天都要使用的工具。花些时间学习它的快捷键和热键。这不仅会加快你的编码速度，还会通过学习如何更快地跟踪代码，让你成为一个更高效的程序员。一个技巧是，在IDE中编写代码的几天里，尽量只使用键盘，而不要使用鼠标，以便熟悉所有的快捷键。

8。 学习Git命令
大多数公司会使用某种版本控制来帮助管理和跟踪他们的代码。您的公司可能会使用GitHub进行版本控制，因为它是最流行的版本之一。花些时间学习版本控制和Git命令是如何工作的。关于这个主题有大量的教程，了解Git的基础知识会帮助你开始你的第一份工作。这一点也与第四点密切相关。这里有一个教程让你开始:Atlassian Git

9。框架由一组工具、类和函数组成，处理大量样板代码，否则开发人员必须编写这些代码来启动和运行应用程序。它让开发人员将注意力集中在与他们特定领域相关的问题上。尽管了解框架对于软件开发人员来说很重要，但是我们很容易过于专注于一个特定的框架。把你的主要学习放在基础上，因为你从基础中获得的核心知识会转移到任何框架和语言中。

10。 最后，生活中还有比编码更重要的事情
软件开发是广阔而千变万化的。在这个领域总会有新的东西需要学习和掌握。作为一名开发人员，经常很容易视野狭窄，一周大部分时间花在编码或看教程上。善待自己，远离软件开发的世界。不要在最初几年耗尽精力，因为你很可能会在IT领域工作和学习很多年。
