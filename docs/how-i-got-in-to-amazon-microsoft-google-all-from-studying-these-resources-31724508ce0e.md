# 快速背景

我现在是微软的工程师，之前分别在亚马逊工作了一年。我没有硕士学位，我毕业于 NYU 大学，不是常春藤盟校。我将很快加入谷歌，以下资源是我如何到达那里的。

是的，可购买的资源是帮助支持这个博客的附属链接。不管怎样，这些都是我用过的资源，包括可购买的和免费的。

# 编码资源

![](img/b6864510258d11ccc73614da8be18d4b.png)

奥斯卡·伊尔迪兹在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 拍摄的照片

## [破解编码面试](https://amzn.to/3wDhpVh) (CTCI)

这是让任何人开始学习编码面试的最简单的书。

如果你是一个绝对的初学者，我建议你从这里开始。这些问题有非常详细的解释，用算法和数据结构的基本知识很容易理解。

## 编程面试要素( [Python](https://amzn.to/3PYcKEq) 、 [Java](https://amzn.to/3PVKyCa) 、 [C++](https://amzn.to/3KtZ56T) )

如果你更有经验一点，这本书里的每个问题都是所有大型科技公司的面试水平。

如果你已经掌握了本书中的问题，那么你已经为一般的技术面试做好了准备。这本书不像《CTCI》那样对初学者友好，但它包括了一个学习计划，这取决于你需要为面试做多少准备。这是我个人最喜欢的书，我在大学里到处都带着。

## [neet code blind 75—YouTube](https://www.youtube.com/watch?v=KLlXCFG5TnA&list=PLot-Xpze53ldVwtstag2TL4HQhAnC8ATf&ab_channel=NeetCode)

Blind 有一个包含 75 个问题的列表[通常足以解决大多数编码面试。这是一个非常有条理和集中的列表，列出了最基本的算法来利用你的时间。](https://www.teamblind.com/post/New-Year-Gift---Curated-List-of-Top-75-LeetCode-Questions-to-Save-Your-Time-OaM1orEU)

上面的播放列表是我见过的最清晰的解释之一，如果你需要任何问题的解释，强烈推荐。

## [CSES 习题集—任务](https://cses.fi/problemset/)

这些问题很难。对于没有练习过算法并且对初学者不友好的人来说真的很难。但是如果你能够完成排序和搜索部分，你将会比一般的 LeetCode 用户更有能力，并为你的编码面试做好准备。

如果你觉得 LeetCode 中等题很舒服，并且觉得 CTCI 的题太简单，可以考虑一下这个。

# 算法学习

![](img/11f9c00b0676a051d9eea4a992f4764f.png)

照片由[андрейсизов](https://unsplash.com/@alpridephoto?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral)

## [算法简介](https://amzn.to/3RfcMJ1)(第四版)

这是任何人都可以用来学习算法的最普通和最好的教科书。它也是我的大学个人用来学习大多数编码问题的核心和基本算法的教科书。

第四版最近发布，仍然与麻省理工学院的学生相关。如果你需要结构和传统的课堂环境来学习，请点击这里学习麻省理工学院的算法课程。

## [威廉·费塞特—图论](https://www.youtube.com/watch?v=DgXR2OWQnLc&list=PLDV1Zeh2NRsDGO4--qE8yH72HFL1Km93P&ab_channel=WilliamFiset)

图论确实在面试中出现过(这也是我在彭博和谷歌遇到的一个问题)。做好准备，遵循威廉·费塞特的图论解释。

图表很全面，一步一步的解释是我见过的关于这个话题的最好的解释。

## [CSES.fi 手册](https://cses.fi/book/book.pdf)

这本手册是为精通大多数 Leetcode 算法的人准备的。这是一个免费资源，是对 CSES.fi 课程的有力补充。

## [竞争编程第 4 版。](https://amzn.to/3zwq0eN)

对于最有经验的算法爱好者来说，这本书将涵盖在任何编码面试中可能被问到的每个利基数据结构和算法。FAANG 类型的公司通常不需要这种准备，但如果你考虑对冲基金类型的公司，这种准备可能会出现。

# 系统设计

![](img/a8955ca9cb0bad961902d15e579cc251.png)

[Med Badr Chemmaoui](https://unsplash.com/@medbadrc?utm_source=medium&utm_medium=referral) 在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的照片

## 《系统设计面试》([第一卷、](https://amzn.to/3Au68rz) [第二卷、](https://amzn.to/3QYJgHO) [网络课程+社区](https://bytebytego.com?fpr=alexcancode))

在我看来，使用这些资源，你将为任何系统设计面试做好充分准备。每本书的图表都很清晰，解释也尽可能简单，以帮助你快速学习系统设计概念。

我个人推荐在线课程，因为这两本书的内容都很值得拥有，正是你接触到的在线社区的不和谐使得每年的订阅物有所值。这种不和谐包括模拟面试伙伴、薪资讨论以及与其他用户一起研究每个系统设计主题的过度关注。

## [系统设计入门](https://github.com/donnemartin/system-design-primer)

系统设计入门是关于所有系统设计的最好的免费资源。深入挖掘 Git 存储库，你将会学到你需要知道的关于系统设计的一切。所有这些都集中在一个存储库中，结构清晰，可为您提供有指导的课程。

## [Educative 的系统设计面试](https://www.educative.io/blog/complete-guide-system-design-interview?aff=xmg1)

如果你赶时间的话，这个系统设计的快速概述是很好的回顾。阅读通常需要用户 45 分钟，但是你会比一般的工程师知道更多的系统设计。

读一读。如果概念不清楚或令人困惑，那可能是你还没有准备好面试的信号。

## [进行系统设计面试](https://www.educative.io/courses/grokking-the-system-design-interview?aff=xmg1)

如果您需要更深入的东西，那么您可以查看这个资源，它为您提供了世界上最大的产品的清晰的行业标准系统设计解释。通读一遍这个和系统设计面试应该足以通过任何系统设计面试。

# 面向对象的设计

![](img/3a8a3e86e3a1c0e669c25ef2617e918c.png)

克里斯蒂娜@ wocintechchat.com 在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上的照片

## [设计模式:可重用面向对象软件的要素](https://amzn.to/3pPckp9)

不管你是否在为面向对象编程面试学习设计模式，作为这些大公司的软件工程师，你都需要了解设计模式。

这本书是当今世界上最常见的设计模式的起源，在面向对象的面试中表现出对这些模式的精通是某些大型技术公司如亚马逊的要求。

## [头先设计图案](https://amzn.to/3PZ3eRn)

上面的资源非常密集，并且用难以理解的语言编写。虽然设计模式中的原始资料很棒，但如果很难理解，就没什么帮助了。

考虑 Head First 设计模式，研究这些常见设计模式的简化解释。它可能不像原始资料那么深入，但是你对设计模式的理解将足以破解任何面向对象的面试。
