GitHub Copilot值得吗？

原文：https://levelup.gitconnected.com/is-github-copilot-worth-it-3af9d3ee1f81?source=collection_archive---------3-----------------------#2022-07-11

我对Copilot是否物有所值的初步想法

我必须承认，当我第一次听说 Github副驾驶时，我并没有那么兴奋。我认为这听起来有点噱头——像超级自动完成。

然而，当它在6月底推出时，我立即注意到它激起的兴趣——无论是在Twitter上还是在我的个人开发者网络中，所以我决定尝试一下。

好人

以下是目前为止我最喜欢Github Copilot的地方:

擅长样板文件

有时，您需要样板代码，或者与您已经编写的代码非常相似的代码。Copilot在编写整个简单的函数方面给了我很大的帮助，帮助我保持代码的整洁。

它帮助我开始

当你知道自己需要做什么，但却不太记得做这件事的最佳方式或如何开始的语法时，你知道这种感觉吗？副驾驶在这方面帮了大忙！逻辑仍然是我的，但它让我不必不止一次地搜索语法。

它很容易使用

我在Python的Visual Studio代码中使用Copilot我发现它非常容易使用，它的预览很容易查看，不会显得傲慢或令人讨厌。它不会对我写的所有东西都有建议，这比它试图填充我不需要的代码要好。

坏事

以下是我注意到的Copilot的一些不足之处:

你花更多的时间审查代码

这可能不完全是一件坏事——但你不应该盲目相信它所暗示的。有时，它建议的代码是关闭的，即使是一个微妙的名称变化。这适用于你在任何地方都能找到的代码！你仍然需要理解它在做什么。

有时候会搞混

它给我的建议不止一次变得很奇怪。或者巨大的。或者两者都有。有时质量也不在那里——它可能工作了，但是不干净。总的来说，它并不完美，但我怀疑它会随着时间的推移不断改进。

责任和安全问题

像Copilot这样的工具已经引起了很多关于责任和安全性的关注。这段代码实际上是从哪里来的？它适用什么许可证？谁拥有它产生的东西？版权呢？这些都是有效的问题和担忧，目前很难回答，但随着Copilot等工具的出现，它们将继续出现。

值得吗？

Copilot远不能取代一个开发者。然而，在我看来，它可以帮助加快发展。所以这里最大的问题是:“它值得100美元/年的价格标签吗？”

我的回答是:是的。

作为一名承包商和顾问，我认为我的时间价值超过每小时100美元。即使它只是填充样板功能，并帮助提醒我我试图调用的语法或API，我相信在一整年的过程中，它将为我节省至少一个小时的时间，在我看来这是值得的。

学生折扣

如果你是学生，我觉得副驾驶就更不用想了:对学生来说是免费的！你需要申请 Github学生开发者包，它真的包含了很多很棒的东西，不仅仅是副驾驶。

我要指出的是，我并不认为Copilot是帮助你学习编码的特别好的工具，但是我相信一旦你理解了基本原理，它可以帮助你加快速度。

密语者

不出所料，AWS也宣布了他们自己的“ML驱动的编码伴侣”:code whisper。我已经注册了预览版，并计划在有机会的时候比较这两个工具。

你试过副驾驶吗？你有什么想法？
