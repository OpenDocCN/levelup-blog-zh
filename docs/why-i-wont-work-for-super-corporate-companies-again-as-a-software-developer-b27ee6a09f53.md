# 为什么我不再以软件开发人员的身份为超级公司工作

> 原文：<https://levelup.gitconnected.com/why-i-wont-work-for-super-corporate-companies-again-as-a-software-developer-b27ee6a09f53>

## 职业创造者&职业黑仔

![](img/20d025fa75574545ae3fd3c7ce339df0.png)

达米尔·科佩扎诺夫在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的照片

在我 22 年的职业生涯中，我已经在一家公司做了 15 年的软件工程师。这就是我作为一名开发人员是如何开始的，以及有多少开发人员是如何开始的；刚从学校毕业，参加新兵训练营，或者自学成才。我欠过去工作过的公司很多；主要是稳定。

企业开发人员的工作过去是，将来也永远是为你准备的。如果你是刚起步，像大多数开发者一样；你最有可能从一个公司环境开始。这并不可耻，事实上，我非常鼓励这样做。这是你建立核心技能和理解如何评估你的项目的地方。这也是你开始对工作产生怨恨的地方；真的不是公司的问题，是文化的问题。

## 我们来分解一下

首先，这些公司。很可能已经存在了几十年。如果不是，就更久。他们不会很快去任何地方；他们很稳定……所以，如果你愿意，你可以在那里工作一辈子。让我们回顾一下在公司环境中工作的利弊。

## 优点:

*   他们给你提供丰厚的薪水，而且每年都会涨
*   健康，牙科，也许还有 HSA
*   2 周带薪休假(PTO)和少量假期；会随着时间而增加
*   也许每 5 年你会得到一支笔或一个钥匙链
*   你可以在 40 小时内停止工作

## 缺点:

*   你的工资将每年上涨 3%到 5%
*   你从工资中支付健康和牙科费用。随着受抚养人人数的增加，这一数字还会上升
*   你在一年中很快用完了你的 PTO，几年后，你要学会尽可能多地为年底储蓄，以度过一个美好的圣诞节假期
*   很可能有着装规范，所以穿牛仔裤和 t 恤的想法是不可能的，即使除了在午餐和公司会议期间没有人会看到你
*   促销可能需要更长的时间，因为有几个其他开发人员在争夺同一个领导角色
*   你不得不在一台过时的设备上工作，无论你如何乞求，你都不得不使用一台运行 16GB 内存的 Windows 7 机器，该机器被防病毒软件吸走，并被期望为本地 SQL Server 数据库和 Visual Studio 2015 高效运行
*   你领薪水，所以任何超过 40 小时的都算你的
*   你有薪水，所以如果你工作不超过 40 小时；你不是一个团队成员
*   在一些公司，你基本上只是一个数字
*   还有很多，但它会让这篇文章超级长

最终，一个公司环境可以帮助你建立职业生涯；但是，这也会让你反感，认为所有的公司都会这样……几年后想转行。

## FAANG 或 MAANG 侧

如果你运气好，或者你已经在一家公司工作了一段时间，你最终会在一家新的公司或一家文化几乎和你在电视和电影中看到的一样的公司结束。这些公司通常是完全相反的，或者至少不拘泥于一个有 100 多年历史的传统。

这些公司中的许多不再被视为创业公司，可能类似于 MAANG 公司(脸书、苹果、亚马逊、网飞、谷歌)。他们寻找可靠的开发人员，他们知道如何优化分布式系统，编写复杂和高性能的算法，并不断推动技术的极限。让我们来看看这里的一些利弊。

## 优点:

*   更高的薪水、签约奖金、季度或年度奖金、股票和股票期权
*   为整个家庭支付医疗、牙科和视力费用
*   更长的 PTO 或无限的 PTO…需要多长时间都可以。我会在另一篇文章中谈到这一点。你基本上可以每年休假 3 到 4 次，每次两周
*   付费午餐和快乐时光
*   没有着装要求
*   选择装有最新软件的顶级设备(可能是开发人员的标准配置)
*   你自然会承担更多的责任，因为你觉得这是你应得的

## 缺点:

*   你工作更长的时间，更努力地获得奖金，并证明每年 3 到 4 次为期 2 周的休假是合理的
*   你可以带着你的笔记本电脑去度假，完成一些工作
*   因为你的工作时间超过了正常的 40 个小时，所以会有更多的压力
*   你自然而然地承担了更多的责任，这导致了压力

## 最后的话

我不会列出为公司或马昂式公司工作的所有好处和坏处，因为那会是一个很长的清单。我也不会贬低这两种环境。它归结于您在软件开发旅程中的位置，您在寻找什么，以及您想去哪里。

对我来说，我已经在一个 MAANG 类型的环境中着陆；但是，我在这一行已经很长时间了，我很喜欢这种类型的环境。
