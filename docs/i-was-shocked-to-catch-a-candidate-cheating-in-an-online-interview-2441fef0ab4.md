# 我震惊地发现一名候选人在网上面试中作弊

> 原文：<https://levelup.gitconnected.com/i-was-shocked-to-catch-a-candidate-cheating-in-an-online-interview-2441fef0ab4>

## 我以前听说过这个，但是我第一次经历了

![](img/c9ce9f4e9fe7425795b338841678de28.png)

照片由 [LinkedIn 销售解决方案](https://unsplash.com/@linkedinsalesnavigator?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄

我们在招聘应用和前端团队。一位候选人表达了对后端和云团队的兴趣。虽然我们不打算招聘，但我给了他一个机会，主要是因为他毕业的学院。

根据我以前的经验，我总是从几个编程问题开始讨论，然后是 CS 概念，最后我讨论几个设计问题。这帮助我更快地淘汰候选人。

这是一次独特的经历，因为这是我第一次看到候选人作弊，并试图从一个流行的网站复制代码。

采访是这样进行的—

# 第一印象很棒

我和一位同事在 Google Meet 上开始了与候选人的讨论。我们请他分享他的屏幕，这样我们就可以看到他写的代码。

看到他使用 Linux 给我留下了深刻的印象，对于一个只有几个月经验的人来说，我们不常看到这种情况。

我们从最基本的编程问题开始，大多数人在毕业课程的第二或第三年就已经解决了这些问题。令我惊讶的是，他开始用 C++编写程序，而且编码风格非常棒。

# 我们开始注意到一个奇怪的模式

随着问题越来越严重，我们开始注意到一种模式。我们让他解决一个需要思考然后编码的编程问题。他在几分钟内就开始写代码，没有在纸上乱写任何东西。

> 我的同事打电话给我，表示他可能在作弊。我们在一个网站上找到了问题的解决方案，我们发现候选人是从那里复制代码的。

我们继续学习计算机科学概念。我们开始讨论计算机科学和操作系统中几个众所周知的问题，同样的模式出现在那里。

起初，他没有一个线索，然后盯着他的右边突然书生气的定义开始从他的嘴里出来。

# 我们开始享受它

我们继续讨论他在互联网上不容易找到的问题。只用了一个设计问题就证实了我们的推测，他在作弊。

同样，在开始的时候，他花了相当长的时间开始回答，然后是维基百科上的几个定义，然后他完全放弃了。

# 我震惊了

> 我非常尊重他毕业的学院。极少数有才华的幸运儿能去那里。我从没想到这样的候选人会这样。

也许是表现出色的压力，或者是想换工作的绝望。我不确定，但是他也不太坏。但由于这一行为，很明显，我们不想雇用他。

然后我浏览了一下 Slack 上的招聘频道。我可以看到一群其他同事与候选人有类似的经历。

# 外卖

这是我第一次亲眼见到这种行为。肯定听说过这一点，但从未想到有人从一个伟大的机构这样做。

正如我的其他同事所说，在后流感大流行时代，这在网络招聘领域已变得相当普遍。

我建议你在网上进行面试的时候要小心，并且总是寻找这样的模式。
