## 准备面试技术部分的算法。

![](img/6fca2738e614a6cb46d5eac8ec2e6281.png)

史蒂夫·哈拉马在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄的照片

通常，开发人员会花一些时间准备工作面试，以获得想要的工作机会。这个时间可以从几天到几个月不等，在此期间开发者尝试着去温习很多理论方面的东西，回忆自己过去的项目和遇到的问题，google 常见的面试问题，填补一些知识空白。

如果你必须涵盖各种复杂的话题，没有计划或算法，准备面试会很困难。

# 工作面试准备算法

那么算法是什么呢？准备面试技术部分的高级算法如下:

1.  确定你可能会被问到的话题。
2.  为每个主题找一本基础书。
3.  阅读每本书每章的**摘要**部分。

现在让我们更详细地讨论算法的每个步骤:

## **确定你可能会被问到的话题**

作为一名开发人员，您可以猜测面试官将测试您知识的主题。这个主题列表由两部分组成:你所在领域的每个开发人员都必须知道的主题，以及与你将被采访的项目相关的主题。

例如，如果你是一名. NET web 开发人员，你的主要必备主题是 C#、单元测试、设计模式、MS SQL、实体框架。但是特定于项目的主题可以是微服务架构和 NoSQL 数据库。

## 为每个主题找一本基础书

下一步是为你确定的每个主题下载至少一本基础书籍。对于我们上面确定的主题，图书列表可能如下所示:

*   C# 9.0 简括:权威参考第一版
*   设计模式:可重用面向对象软件的元素
*   实用实体框架:企业应用的数据库访问。版本
*   单元测试的艺术，第二版
*   构建微服务:设计细粒度系统
*   专业 SQL Server 内部第二版。编辑:Dmitri Korotkevitch
*   普拉莫德·j·萨达拉吉和马丁·福勒蒸馏的 NoSQL

## **阅读每本书每章的“概要”部分**

在确定了所有的主题后，你有了你的书，你应该阅读每本书的摘要部分，通常在每章的结尾。

摘要通常是对相关章节中讨论的所有内容的一页纸的总结。阅读摘要会让你回忆起你可能已经忘记的概念。在某些情况下，你可能甚至不知道你在摘要部分读到的术语。还不错。你只是发现了一个你需要弥补的缺口，例如，通过阅读整个章节。

如果你正确地确定了你的主题并找到了好书，你将会回答绝大多数你可能会被问到的理论问题。

另一个帮助你准备实际问题的技巧是:当你阅读“总结”部分时，试着详细回忆你所有的实践经验。例如，如果您阅读了散列表的摘要，请记住您为什么在代码中使用它们，是否有任何问题/替代方案？

祝你面试好运！
