> 原文：<https://levelup.gitconnected.com/youre-lucky-if-your-code-doesn-t-work-give-up-now-e1272b5a9136>

你很幸运，因为编码很难，而且你很快就发现了，这很好，所以你现在可以放弃，去找别的事情做。

## **仅出于激励目的**

取决于你现在在解决问题过程中的位置，将决定你如何理解第一句话。

如果你最近战胜了一个困扰你的问题，那可能会显得非常悲观。

如果你正在处理一个棘手的问题，你可能会想“天哪，他是对的。”

除非这是你第一次遇到编程问题，不管你是处于问题解决过程的前一阶段还是后一阶段，有一点是肯定的:**你知道你能解决这个问题。**

我写这篇文章是为了提醒你当初为什么要开始你的代码之旅。现在，花点时间来记住你的“为什么”…

## 这个故事需要存在

当我在过去的一周里感到气馁时，我决定写下这篇文章。**我希望这个故事存在**因为我多次看到“我在 3 个月内成为一名初级开发人员，并找到了一份工作！”故事在我的饲料，这些作家将详细说明他们如何努力奋斗，但它通常是过去的事情。我们从来没有“看到”斗争的发生。

我实时写这篇文章是为了让你知道**我还没有完全想通**。当我写这篇文章的时候，我的灰姑娘故事还远没有幸福快乐。

给关注我故事的人(谢谢！)，你可能注意到了，我上周没有发新的故事。这是因为我在努力应付我的考试，这让我不得不建立一个完整的网站。我没有时间去写，我担心在这么短的时间内我无法学到通过考试所需要的一切。

## 我目前的状态

在我的编码训练营中，我已经接近 Python 栈的末尾了。过去的几周非常艰难，尽管我倾向于乐观，但我也有厌恶地摇头的时候，因为我在 app.route 中漏掉了一个@ decorator，或者我遇到了一个错误，而[“Google it”](/how-to-get-un-stuck-a-letter-to-new-software-engineers-19c9990df1d3)方法没有帮我修复它，所以我的整个 Flask 应用程序无法正确运行。

但我对这个世界并不陌生，我已经有了一个工程学位。我写了一篇文章解释了拥有工程学位是否有助于学习编码，你可以在这里查看。因此，我对从周五晚上开始的周六早上 3:27 盯着一个问题、电脑屏幕或一张纸并不陌生。有时候，我觉得正是这些时刻让你成为了一个伟大的问题解决者。有时候我觉得你需要这样的时刻……*赢得你的军衔*。

**考虑一下这个**:你在 YouTube 上看到的每个人，你在 StackOverflow 或 Reddit 上看到的每个回答问题的人，每个曾经有过学习编码*的奇妙梦想*的人都有过类似的时刻。你我都没有看到他们，但感到不知所措和气馁的时刻发生了。认识到这一点很重要，因为很多时候，我们只看到成功，而不是创造成功的过程。

## 你现在真的幸运吗？

是的，如果你的代码不工作，你是幸运的。这给了你一个机会。一个揪头发的机会，一个学习桌上和口袋里的电脑如何工作的机会。因为这是我们都想知道的，对吗？我得说，到目前为止，我在编程中真正学到的东西至少有一半是我一开始不得不努力理解的结果。

当一些事情在早期对你来说很难的时候，它给了你更多关注它的机会，因为你不太明白。有了你花在这上面的额外时间，你可以把它变成一种优势。

这取决于你决定如何处理它。有些人在面临挑战时会逃跑和躲避，但一旦你决定直面挑战，不管需要多长时间都要花时间去学习，那就是你为自己建立记录的时候了，你会对自己直面挑战的能力充满信心。你向自己证明，即使你现在不明白某件事**你知道只要稍加努力就能明白。**

## 通向未来的高峰

在上大学之前的一段时间里，我怀疑自己是否能够掌握工程课程中面临的所有复杂概念。那个时候，我只是为了解决高中水平的概念而与自己建立了融洽的关系。我不知道自己是否能够应对大学带来的所有复杂性。但是一次又一次，我向自己证明了我可以在大学水平上学习最难的概念。我尝试过，失败过，但最终在最难的课上成功地学习了概念。**所以今天**，我不认为我不能学习土木或环境工程领域的概念；我知道我能做到，而且随着时间的推移，这种自信和舒适会随之而来。这只是理解的问题，这主要依赖于我多年前学到的基本原则。

我还没有达到对编码感到舒适和自信的阶段，但是根据我在其他工程学科的阅读和经验，我会达到那个阶段，**你也会。**

这实际上是我写这篇文章的主要目的。

要告诉你这个:问题就是机会。艰难的时期是伟大时代的基石。

如果你以积极的态度对待问题，我可以向你保证你会解决它们。全部都是。继续走。

保重。
