# SwiftUI 什么时候会取代 UIKit？

## 应该学 SwiftUI 还是 UIKit？

2017 年，苹果在加州 1 Infinite Loop 开设了新总部，这是一个最轻松的事件，因为苹果管理层就办公空间进行了比上次美国大选更多的政治操纵。仅仅两年过去了，有传言称，许多苹果员工非常喜欢在家工作，想象一下，他们根本不想在那个宇宙飞船大楼里工作。

但是，嘿，那只是推测——让我们来谈谈事实——在 2013 年，我决定是时候(再次)学习编码了。我不应该说真正地学习编码——我从未停止过编码，从 1984 年就开始了，但是好吧——不是在 iOS 下。我开始认真地学习，当然在网上找到了很多关于 Objective C 的教程，以及大量关于 StackOverflow 的有用信息。

但是 Objective C 和我之前接触过的 Bash、Perl、Python 和 Ruby 的世界完全不同。在 2014 年 WWDC 之前的九个月里，我进展缓慢，当时苹果公司投下了一颗名为 Swift 的重磅炸弹。在发布后的三个月内，使用 Objective C 学习 iOS 编程的教程开始枯竭。在六个版本中，你找不到任何更新过的最近流行的框架。我很恼火——我刚刚花了一年多的时间学习了一门似乎已经过时的技术。但是那个夏天我没有换。

我在 2014 年圣诞节前后换成了 Swift。我这样做是因为我在开发的一个应用程序上遇到了 CloudKit 的一个 bug。我向苹果公司报告了这件事，得到的回复是——抱歉，已经有人报告了——我们正在处理这件事，我们会忽略你。我清楚地记得坐在椅子上，想着斯威夫特。

**你认为苹果公司让他们最优秀的人在做什么——使 Swift 工作还是目标 c。当时的答案和今天一样——显然，最优秀的人在做 Swift 工作。**

回到我身上，我重新开始学习 Swift 编程，这是我喜欢看的保罗·赫加蒂教授的最好的课程之一。现在我不能确定我是哪一年在网上找到他的课程的，但我记得它不在 Objective C 中(当然)，而是在 Swift 中。时间推进到 2020 年春天，Paul 仍在教授 CS193p，他仍在教授 Swift，但他没有使用 UIKit，而是使用 SwiftUI。这又带回了一个问题…

**你怎么看苹果让他们最好的人手开发 UIKit 或 SwiftUI。答案很明显——swift ui。**

所有这些都让我回到最初的问题 SwiftUI 什么时候会取代 UIKit。这个问题的答案根本不是一个技术问题——它完全是关于人的——如果你今天寻找教程——大多数，超过一半的人不使用 UIKit，他们使用 SwiftUI。正如我已经提到的，你无法使用 Objective C 找到最新的教程，他们使用 Swift。今天的学生既没有学习 Objective C 也没有学习 UIKit。他们的重点是 Swift 和 SwiftUI。

Swift 花了 7 年时间在工作现场取代 Objective C，尽管这不是技术性的。我相信这取决于人 SwiftUI 也将遵循同样的路线，这不是一个技术问题，而是取决于人。

SwiftUI 不会取代 UIKit，直到那些学生@Stadford 进入软件公司的管理层，所以还需要几年时间。从技术上来说，行业现在就可以开始使用 SwiftUI，但他们没有，他们没有使用 SwiftUI，因为总理已经将他的职业生涯投入到 UIKit 中，他的老板和他的老板也是如此。当那个 PM 离开时，SwiftUI 将取代 UIKit 并在 2020 年被一个斯坦福班级的学生取代，或者实际上你在他之前得到提升，并让你的团队使用它。您将会理解使用该技术的优势超过了多一点代码来使其工作的不便，并为您的团队选择这条道路而感到高兴。

SwiftUI 何时会取代 ui kit——在 2029 年——当知道它如何工作的个人进入管理角色时。
