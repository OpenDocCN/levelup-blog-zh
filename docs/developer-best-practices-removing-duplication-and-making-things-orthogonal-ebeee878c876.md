# 开发人员最佳实践——消除重复，使事情正交化

要成为一名优秀的程序员，我们应该遵循一些易于采用的习惯，以保持我们的编程生涯长久。

在这篇文章中，我们将看看使我们成为更好的开发人员的最佳实践。

# 无意的重复

有时候，我们可能会犯错误，导致我们在不经意间重复一些事情。

我们可能不知道在我们的代码中已经存在某个问题的解决方案，这就是为什么我们可能会复制一些东西。

此外，我们可能有在不同表中重复的非规范化数据。

这些事情肯定会让我们的生活变得更加艰难，但我们或许可以通过消除这些重复来解决它们。

非规格化的数据可以被规格化，我们可以使用已经存在并测试过的解决方案，如果已经有了一些东西的话。

# 不耐烦复制

我们可能会复制粘贴东西，因为我们没有耐心。

这是因为我们在摆弄一些东西，我们想快速修复它们。

或者我们可能没有完全重构，在我们的代码库中留下了一些重复代码的痕迹。

一旦我们有了工作，我们应该清理我们的代码，把重复的移到一个地方，然后引用它。

否则，如果我们以后维护我们的代码，重复的代码将成为一个问题。

我们只需要在交付代码之前清理它们。

# 开发者间复制

很多人可能在做同一件事，而我们却不知道。

这可能是因为我们没有交流我们正在做的事情，或者我们可能忘记了有人可能正在做和我们一样的事情，尽管我们已经交流过了。

此外，正在实现的系统的设计可能是有缺陷的，首先，我们可能会在多个地方创建相同的东西。

在任何情况下，我们都应该确保互相更新我们正在做的事情，以确保我们不会做同样的事情。

多人做同一件事是浪费时间。

并且以后维护它也将是一件痛苦的事情，因为我们要维护同一件事情的多个实现。

让一个人来协调所有的工作是个好主意，这样可以确保这些工作不会被不同的人重复。

通过这种方式，我们可以避免多人在不知情的情况下做同一件事情。

# 使事物易于重用

从短期和长期来看，使东西易于重用会有所帮助。

重用意味着我们不必再次做同样的事情来实现本来可以重用的东西。

如果很难，那人家就不干了。

# 让事情正交

用编程术语来说，如果两个事物是正交的，那么如果我们改变系统中的一个事物，那么它不会影响系统中的任何其他事物。

这很好，因为我们不想做紧密耦合的东西。如果我们改变它们，紧密耦合的依赖关系将会破裂。

因此，我们不会想让事物彼此紧密耦合。

如果事情过于依赖彼此，那么改变它们将是一场噩梦。我们不想让事情变得不正交，让我们的生活变得更艰难。

非正交系统很难改变和控制。因此，我们应该使事情松散耦合，修复一件事情将局限于修复系统的一小部分。

为了使事情正交，我们必须确保一切都是相互独立的。

每一件事都做一件小事，这是定义明确的。只要我们不改变这个东西实现的接口，我们就不应该担心破坏任何东西。

拥有固定的接口来隐藏我们代码的实现是很重要的，这样我们就不必担心下面是什么或者引用它们。

这样，当底层实现改变时，我们就不必担心我们的东西会改变。

# 结论

我们必须消除代码中的重复和任何可能有重复的东西。

复制不一定是故意的。它也可能是无意的，因为人们可能无意中实现了同样的事情。

因此，我们应该进行沟通，以确保没有人在做同样的事情。

此外，我们应该使程序的组件独立，这样它们就彼此分离。这使得维护更容易，因为损坏东西的风险更低。
