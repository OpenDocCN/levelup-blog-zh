为我们的站点启用自定义模拟器

工具栏中出现的仿真器组是在/etc/mobile/groups/responsive节点下定义的。首先将这个节点复制到/etc/mobile/groups/archetype中。

为了让AEM在工具条中显示特定的模拟器组，我们必须使用cq:deviceGroups属性来提供该组。有两个地方必须这样做:

  1. 在你网站的根页面上(例如:/content/demo/jcr:content)
  2. 在模板的结构节点上(例如:/conf/demo/settings/wcm/templates/content-page-template/structure/jcr:content)

在这两种情况下，将cq:deviceGroups属性设置为指向/etc/mobile/groups/archetype。

警告:为根页面和模板设置cq:cq:deviceGroups属性很重要，否则模板中的断点和基于它的页面之间会有冲突的风险！

为了测试这是否可行，请转到/etc/mobile/groups/archetype/jcr:content并设置emulators属性，使其只包含/libs/wcm/mobile/components/emulators/galaxy5模拟器。您还应该更新jcr:title和jcr:description字段:

jcr:title="Archetypal Devices"jcr:description="The devices in this group represent archetypical device dimensions."emulators="[/libs/wcm/mobile/components/emulators/galaxy5]"

如果刷新页面编辑器，您会看到工具栏中只剩下Galaxy S5模拟器:

移除旋转按钮

还有最后一件事需要我们注意。因为我们已经为每个原型设备的每个方向提供了一个模拟器，所以我们不再需要旋转按钮。

我们可以使用clientlib轻松地停用它。在您的/apps项目文件夹中创建以下clientlib:

/apps/demo
└── clientlibs
 └── clientlib-emulator (cq:ClientLibraryFolder)
 ├── js (nt:folder)
 │ └── index.js (nt:file)
 └── js.txt (nt:file)

将以下属性添加到clientlib-emulator节点:

categories  --  String  --  [cq.authoring.editor.sites.page.hook]

这就是AEM知道如何在加载与编辑器相关的其他clientlib的同时加载您的client lib。
