# 你应该不惜一切代价避免在工作场所使用的 7 个短语

## 学会把自己描绘成一个高度忠诚、有能力、自信的开发人员，以获得成功的软件职业生涯

![](img/26aa024a4cca81da094c1f68866abe2f.png)

大多数程序员倾向于只关注他们软件职业的技术方面。他们很少或根本不注意他们在专业对话中使用的短语。

这些短语中的一些表明开发人员不重视他们的队友，其他的陈述削弱了他们自己的能力。不管是哪种情况，不管是无知还是缺乏软技能，频繁使用这些词语都会严重影响增长前景。

因此，在工作场所使用正确的词语或消除一些会对你的态度、诚信和承诺程度产生负面影响的词语是非常必要的。

这里列出了人们在工作场所使用的七个常用短语，如果你想成为一名有能力的、积极的、自信的软件专业人士，你应该不惜一切代价避免这些短语。

# 1.我觉得我做不到

这种说法完全表明缺乏信心和诚意。开发人员经常用它来逃避他们认为困难的责任。多做几次，你可能会被贴上懒惰或没有能力的程序员的标签。

这并不意味着你应该能够做任何事情。会有很多任务对你来说不可行。原因可能各不相同，从极其紧迫的截止日期到缺乏所需的技能。如果提供适当的工具和支持，在某些情况下你可以完成这项工作。

因此，不要直截了当地宣称你无能为力，你可以试着讨论那些让你认为自己无法胜任这份工作的挑战。

讨论之后，如果你仍然认为其他人应该接下这份工作，那也没问题。至少，现在你不会给人一种害怕承担责任的无能程序员的印象。

# 2.那不是我的问题

在工作场所，没有个人问题。当在一个项目上一起工作时，交付的任何问题都是整个团队的问题。谁对这个失误负责并不重要，但是如果项目没有按照预期交付，这将被认为是一个集体失败。

您可能不直接负责创建或解决问题。因此，你可能会觉得这不是你的问题。但是这样说就完全抛弃了它会让你看起来傲慢或者逃避责任。

所以，避免说听起来像是防御性的短语，比如— *那不是我的工作，*或者*我不在乎那件事能不能完成*。

相反，试着尽可能地帮忙。如果你不能帮忙，那么你可以礼貌地让请求者知道你的无能。您甚至可以将请求转给您认为能更好地解决问题的人。

# 3.这是个愚蠢的想法

人们试图通过拒绝他人的想法来让自己感觉更聪明，这并不罕见。他们试图淡化这些建议，认为它们很愚蠢。有时候他们甚至不让对方完成自己的想法就宣布这个想法很傻。

大声说出别人的想法是愚蠢的，这既粗鲁又不专业。这反映了不成熟。很有可能你没有理解清楚这个想法。

因此，如果你认为一个想法可能达不到要求的目标，要求进一步的澄清。让这个人知道你认为这个提议不可行的情况。找出这个人是否考虑过这些情况。

如果你觉得这个想法行不通，感谢这个人付出的努力，并在此基础上提出改进建议。这将鼓励其他成员贡献他们的建议。毕竟，软件开发的成功只能通过团队协作来实现。

# 4.你傻吗？

如果说某人的观点愚蠢是不好的，想象一下当被问到“T4，你愚蠢吗？”时，人们会是什么感觉？’。

当人们认为某人的观点没有任何意义，或者某个建议可能会适得其反时，就会产生这个问题。

不管背后的意图是什么，这种说法经常被认为是对个人的侮辱。无论你是在私下还是在集体中说，都没有关系。这种说法肯定会伤害到被针对的人，并使他们在未来不敢发表自己的意见。

如果你有说这种话的习惯，你需要明白这也反映了你的不好。隐藏不安全感的人试图冒犯他人，让自己感觉高人一等。但事实上，团队成员认为这样的人不专业且自私。

处理你认为不合上下文的想法的更好的方法是忽略它或者进一步询问它以更好地理解它。不管别人的观点多么糟糕，侮辱他们都是不值得的。

# 5.我必须做这里的一切

当开发人员工作超负荷时，他们通常会觉得这是因为其他人没有承担他们的责任。因此，他们必须努力工作来弥补差距，避免项目失败。这种感觉会对他们的队友产生怨恨。

如果你在过度工作时也有类似的反应，首先要明白的是，你不是唯一一个保持团队运转的人。如果是这样的话，那么应用程序早就失败了。

成功的项目交付从来都不是一个人的表演。只有当所有利益相关者真诚地尽自己的一份力量时，它才能成功。不这样认为意味着破坏团队其他成员的努力，这不利于你的职业态度。

也许你想表达的是你有太多的工作要做，你想得到同事的帮助来更好地平衡工作和生活。开始与你的经理讨论他们如何帮助你更好地分配任务，以专注于与你的技能相匹配的重要任务。

# 6.我不在乎

当我们在团队中工作时，每个工作问题都是团队问题。不存在个别问题。假设一个开发同事努力完成分配给他们的任务。在这种情况下，它仍然是一个团队的问题，因为不能及时交付任务会影响整个项目的交付时间表。

在任何情况下，你都不应该在工作场所用“*我不在乎，*”来拒绝别人的要求。你可能真的认为你不能满足这个要求。或者，基于你的角色和职责，你可能觉得这个话题与你无关。

然而，在职业环境中，你传达信息的方式很重要。你不想被视为只关注个人利益而不是团队成员。

如果你对讨论的话题或寻求帮助的请求不感兴趣，试着礼貌地拒绝。与其公开承认你不关心队友面临的问题，你可以提供一个你无法提供帮助的理由。这里要注意的重要一点是你表达的语气。

# 7.我不知道，但是…

这个短语的一个变体是——“*这听起来可能很愚蠢，但是……*”。在回答问题或向队友描述某事时，人们经常使用这些表达作为填充语或起始短语。我见过许多资深会员使用它们，即使他们的解释是基于事实的和正确的。

在职业环境中说话时使用这样的短语表明说话者缺乏自信。如果你觉得你要说的话很蠢，别人也会有同样的感觉。

最好对你的陈述和想法有信心，这样人们才会认真对待你。如果不清楚事实，不如问时间。找出主题的更多细节，然后回来分享你的真知灼见。

然而，如果你对自己说的话很有信心，但出于习惯使用这些表达，你需要摆脱它们。在你的空闲时间练习你的陈述。最初，你可能会觉得很难，但当这些短语不在你的词汇中时，你甚至不会意识到。

# 最后的想法

对于一个成功的软件职业生涯来说，工作场所的其他人如何看待你非常重要。为了获得最大的发展机会并取得非凡的成功，你需要在与同事、领导和客户交流时小心使用词语。

现在你知道了为了你的职业发展，你应该避免在工作场所使用的一些短语。仔细选择你的用词和短语，在你的工作场所被认为是一个积极和真诚的团队成员。

如果你仍然觉得提到的短语没有影响人们的感知，观察当别人在转换过程中使用它们时你的感觉。这将有助于你进一步了解这些条款的负面影响。

我相信，在我们的工作场所进行交流时，我们应该小心更多的短语。如果你知道上面没有列出的术语，请在评论中分享。
