做一个软件开发人员很容易，但是要成为一个更好的软件开发人员？

他们说，“编程很容易”，我也完全同意。然而，软件开发并不容易。开发和发布可以在生产环境中运行的企业级应用程序需要大量的技术和框架。

但是同样，软件开发是主观的。

如果你只是创建一个小的web应用程序，那么任何技术栈都可以。如果您正在创建一个完整的银行应用程序，它应该是可伸缩的和安全的，那么Java及其框架是首选。同样，如果你专注于机器学习或人工智能，那么Python或R是你的首选编程语言。

关键是，不管你在编码方面有多好，你总是在学习新的东西来完成你的任务。在工作中不断的学习让它变得如此有趣，因为你感觉做了一些有意义的事情。

终身学习

我一直在指导团队中的大三学生，他们中的一些人最近刚从各自的学院毕业。当我开始向他们传授我的应用知识以及业务概述时，他们不知所措。

我的低年级同事认为，他们在各自大学里学到的东西足以让他们在软件行业起步，直到他们被一系列技术、框架、工具和最佳实践轰炸。

事实上，要成为一名相关的开发人员，只需要保持对各种技术的了解，并定期更新技能。不管你的经验如何，你必须随时准备学习新技术，并在需要时实施它们。

“假装直到你成功”是最糟糕的建议

在我最初的几年里，我为在一次工作面试中谎称了解某个特定的概念或工具而感到内疚。而在一些面试中，我可以轻松地通过。然而，在一些面试中，面试官可以很容易地看出我一直在鬼混。

没错。那是我。不要那样做。

众所周知，在当今不断发展的技术世界中，没有人知道所有的事情，原因显而易见。不知道也没关系。重要的是你愿意付出多少努力去学习那项技术、编程语言，甚至一个概念。

最近，我采访了一个正在做同样事情的人。我只是告诉他我的经历，也让他感觉很舒服。他紧张得要命。我问他说他觉得什么都可以，他确实在面试中表现出色。

工作前了解你的要求

这甚至不是一个菜鸟的错误。很多有经验的人都这么做。如果你得到一个任务，你应该总是了解要求。始终充分了解要求你做的事情。否则，你会失去很多宝贵的时间。当你在做一个重要的项目时，失去的时间会困扰你。

我们所做的是写下对任务进行详细分析的实施计划，并将其发送给分配任务的人。我相信你正在遵循，或者你将遵循一个不同的过程来做同样的事情。实现计划帮助我节省了时间，并且分析确保我与请求者的需求保持一致。

危机考验你的性格，让你变得坚强

你只需要明白，并不是所有的软件开发日子都是彩虹或阳光。会有很多黑暗的日子。很多。

当然，如果你正在做一个更大的项目，对于每一个重大的失败总会有应急计划。对你和你的团队来说，事与愿违的都是小事。

有太多不可预见的问题出现在PROD环境中，尽管应用程序已经在您的开发、QA和UAT环境中针对每一个可能的场景进行了测试。您需要在4小时内解决问题。

在那个时候，最好是把注意力集中在错误上，而不是互相指责，玩指责游戏。我也看到了。😅

这是在关键时刻，它将确定你的性格，而不是在这样的时刻崩溃。关掉一切，专注于问题，和你的团队一起工作。你会想出解决办法的。

新技术总是会诱惑你

“机器人学很性感”、“数据科学目前很热”、“ Web 3.0是未来”——你听过这些说法吗？也许是这些陈述的转述版本？

当我学习工程学时，机器人技术风靡一时。每个人都在谈论它。然后是大数据和数据科学，我的意思是，两年前Twitter上充斥着“数据科学”。现在，Web 3.0和Meta已经在所有可用的社交媒体平台上吸引了我们的注意力。

不要误解我。所有这些都是伟大的技术，但是没有适当地学习任何东西就去追求它们，从长远来看肯定对你没有帮助。

但是，了解新技术总是好的。其实是鼓励的。然而，在我看来，你应该首先加强你在任何领域的基础，并在冒险进入未知领域之前变得尽可能出名。
