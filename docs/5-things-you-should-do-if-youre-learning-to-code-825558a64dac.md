# 如果你正在学习编码，你应该做的 5 件事

> 原文：<https://levelup.gitconnected.com/5-things-you-should-do-if-youre-learning-to-code-825558a64dac>

我以前是儿童和青少年的编码老师，教过数百名学生如何编码。在那些学生中，有几个成绩非常好的人。这些学生不断赢得挑战，公开发布自己的应用程序和游戏，表现远超同龄人。让这些学生与众不同的东西似乎并不归结于任何天生的能力，相反，他们做了一些不同的事情，我相信所有的学生(孩子和成人一样)都可以从中学习。

## 1.询问后续问题

这些学生不停地问“为什么”。对他们来说，仅仅学习语法而不理解为什么会这样是不够的。有时答案是相当令人失望的“这就是语言是如何被设计的”，但很多时候它会引发一场关于编码约定甚至语言如何在幕后工作的有趣讨论。

他们的好奇心也不止于语法，每当他们写代码时，他们会问计算机到底在做什么。他们经常会想出不同的方法来达到相同的结果，然后问你为什么应该或不应该使用一种方法。

然而，有一个问题我并没有经常被问到，那就是“这个为什么有用”。我相信这是因为这些学生做的另一件事不同…

## 2.有一个应用你的知识的项目

这些高成就者不需要被告知“这就是为什么 for 循环是有用的”，他们可以立即看到其中的价值，因为他们都有自己的个人项目，可以从使用循环、类或他们正在学习的任何概念中受益。

他们通常已经在一个令他们兴奋的项目中艰难地完成了一些事情，这让他们更加兴奋地去学习那些能让他们在未来做得更好的概念。

我有一个学生，他非常渴望制作一个有许多移动角色的游戏，以至于他们复制粘贴了数百个变量(“敌人 1XPos”、“敌人 1YPos”、“敌人 2XPos”)。).当我们引入类和列表的概念时，他们并不认为它们是抽象的概念，而是一个非常真实的解决方案，它解决了数百行复制粘贴的代码为每个敌人做相同事情的问题。这一新知识使他们能够通过改变代码中的一个值来创造任意数量的敌人。

## 3.实验和打破东西

我经常鼓励学生尝试改变一些值来看看它的效果，或者让他们有目的地编写会导致他们的程序崩溃的代码，这样他们就可以认识到错误，并能够在他们的代码中再次犯下错误时修复它。

一些学生太害怕故意“破坏”他们的代码，对我来说，这总是一个信号，表明他们一开始就没有很好地理解他们的代码。他们只是盲目地重新排列和复制东西，直到它能工作，然后就再也不碰它了。这些学生后来会在错误和崩溃中挣扎，不知道他们做错了什么，而之前遇到这些错误的学生却知道原因是什么。

与每次练习后花一点时间试验代码的学生相比，那些只写一次代码然后继续学习的学生在以后使用这些概念时也更加困难。

## 4.展示你的作品

每个星期，我的一些学生都会提前来上课，向我展示他们一周以来的工作成果。他们会展示他们的项目，我会向他们提供反馈和建议。他们会带着他们项目的许多新想法离开，然后会兴奋地在下周带着他们所有的改进把项目带回来。这有助于保持他们在项目中的积极性，并通过让其他人审查他们的代码学到很多东西。

即使你是自学的，你仍然可以把你的项目放在 Github 这样的网站上，并在网上分享(我可以建议[https://www.reddit.com/r/coolgithubprojects](https://www.reddit.com/r/coolgithubprojects/))。如果你的项目对其他人有吸引力，你甚至可以让其他人为你的项目做出贡献，获得反馈并学习如何与他人合作。了解诸如版本控制之类的东西也很棒，拥有一个活跃的 Github 帐户对你寻找开发人员的工作会有帮助。

## 5.继续前进！

我工作中最好的一个方面就是看着这些学生随着时间的推移发展他们的技能。这确实需要时间。但是这些学生每周都会回来，当这些学生赢得编码比赛并能够实现他们为自己制作游戏和应用程序的梦想时，看到他们的辛勤工作和努力得到回报真是太好了。

任何人都可以学习编码。这并不意味着每个人都应该，或者每个人都想，但你不需要为了学习而拥有特殊的天赋或诀窍。这不会在一夜之间发生，甚至可能不会在一年内发生，但如果你坚持下去，你会有所改善。
