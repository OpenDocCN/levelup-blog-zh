作为一名开发人员，我的第一份有报酬的工作是为我们当地的医院做网站开发，在那里我们没有 CMS，我必须用经典的 ASP 编写每个网页的代码。

那份工作可能是我最后一次做任何形式的“简单”网站创建。当然，我有一个个人网站，但那只是从我的简历中自动生成的，使用的是我几年前建立的 CI 工作流程，从那以后就再也没碰过了。

说到建网站；那是几页纸，只是说了它需要说的，没有要建模的数据、要编写的后端或要构建的前端应用程序——这已经过去了大约七年。

[作为推出我一直在工作的应用程序(JiffyCV)](https://medium.com/swlh/building-my-first-app-jiffycv-53192b24b386) 的一部分，有必要创建一个简单的网站来展示这款应用程序，并鼓励人们注册参加我们即将举行的测试。

我最近没有玩静态站点生成器，以前我玩过 Pelican 和 Jekyll，但从来没有真正跟上那个领域，所以我从来没有意识到 NodeJS 有多少不同的选项。

使用 Gatsby 的建议来自另一个开发该应用程序的人，但我很高兴他们给了我这个建议，因为它使用了 React，这是一个我非常熟悉的框架，与其他静态网站生成器不同，它比仅仅编写一堆 Markdown 文件和更改一些 mustache 模板更有活力。

# 我喜欢什么

我喜欢盖茨比的几个方面，但对我来说最大的好处是插件和为这些插件创建“启动”项目的文化。

能够在[https://www.gatsbyjs.com/plugins](https://www.gatsbyjs.com/plugins)搜索你需要解决的问题的预制解决方案，以建立你的网站，这真的很方便，虽然你可能会说像 Wordpress 这样的 CMS 已经有一段时间了，但它们从来没有选项来查看你可以下载的预制项目列表，并查看它们是如何工作的。

对于测试版注册网站，我们需要能够让人们提供他们的电子邮件地址，以便我们可以在测试版建立后向他们发送邀请，几分钟内我就可以:

*   找到我想要的插件
*   安装它并配置项目以使用它
*   创建一个利用该功能的组件
*   在本地测试
*   部署站点(虽然这不是 Gatsby 的功劳，更多的是已经建立的优秀 CI)

与我使用 CMS 和其他静态站点生成器的经验相比，我不需要运行(W/L/M)AMP 堆栈，也不需要记住在我对模板进行更改后让站点重新生成的命令。

## 它使用 React

当 JavaScript 框架开始蚕食前端开发空间时，我站在了 React 一边，因为 React 的持续改进以及相关的设计模式和范例，它让我保持了足够的警觉，让我继续投资。

因此，很自然地，能够在 Gatsby 中重用这些 React 技能使我的生活变得更加轻松，这也意味着我们为应用程序编写的任何基于 React 的库都可以轻松地放入 Gatsby 站点。

## 热重装

这不一定是一个前沿的特性，因为现在大多数工具都提供了某种形式的重载开发服务器，但是与我以前使用的工具相比，做出改变并让浏览器立即更新是非常好的。

## 利用数据源构建页面

这个很有意思。一方面，能够查询数据源并使用响应来自动构建页面是很棒的，因为这允许您快速添加大量信息，而无需将网站连接到数据库。

另一方面，它使用 GraphQL，这是我不太熟悉的东西，尽管我确实发现《权力的游戏》的 GraphQL 对于学习基础知识非常有用。

例如，我发现了一个非常棒的从吉拉获取数据的插件，它可以让我将一些关于应用程序进展的数据带到网站上，但我完全被该插件的一些更重的 GraphQL 方面所迷惑，所以我努力想出如何简化数据集，使它不会生成数千个页面。

然而，以这种方式利用数据源的好处远远超过学习新工具的坏处。

# 摘要

从本质上说，我对 Gatsby 的体验可以归结为——我已经很久没有做过小型网站开发了，但事实上，我可以使用我习惯的框架来构建网站，并利用庞大的插件库，这很棒。

在我面前有一点学习曲线，但这只会增强我的技能，看看未来事情如何发展会很有趣。
