# 学习编程的正确方法

> 原文：<https://levelup.gitconnected.com/the-right-way-to-learn-to-program-5efe742ddabf>

![](img/0a3ec3eb4a443985907f4fbc07d9b4cb.png)

当你开始学习编程时，有如此多的新的和密集的技术信息向你扑来。你是如何学会并记住这一切的？

这是我在自己的编程之旅中多次问自己的问题。为了找到答案，我一路尝试了许多学习策略。然而其中有一个很突出。我称之为先申请，后学习的方法。

如果你正在寻找一种高效的学习编程的方法，我建议你试一试。

# 在行动中学习

**先应用，后研究**就是先尝试问题。目标不是得到正确的解决方案，尽管你可以。更确切地说，是为了找到路障。然后，例如，查阅一门课程或一本书，让自己摆脱困境，更深入地探究特定的问题点。

假设你尝试一个要求你用列表理解的问题。你取得了一些进展，但是当你不得不使用条件句的时候却停滞不前了。首先尝试这个问题会让你意识到这个障碍:在列表理解中使用条件句。你已经把你的注意力集中到一件事上了，现在抓住那件事。

你会的。因为您首先尝试了这个问题，所以您将有目的地引用一个资源。你心里有一个特别的问题:我如何在列表理解中使用条件句？所以不要试图去学所有的东西，或者重新学习你已经知道的东西，你要把注意力集中在关于条件句的讲座上。

在你得到你需要的东西后，再把它应用到手头的问题上。换句话说，应用你的新知识。

如果你再次陷入困境，遵循同样的迭代过程:

*   澄清你的问题(例如，“为什么这个**不会终止循环**)).
*   找一个资源，专注于回答你的问题。
*   把你学到的东西应用到手头的问题上。

我承认，看视频教程比较轻松。然而，学习并不是越容易越好。“当学习变得更容易时，我们很容易被诱惑相信学习会更好，”这本书的作者说，“但研究显示相反的情况:当大脑必须工作时，学习会更好。”

**先申请，后学习**可能会觉得有点难受。起初，你可能看起来像是在缓慢前进。但是快进一两个星期，你会惊讶地发现你已经学到了很多东西——并且保留了下来。这是有关键原因的。

# 学习它并保留它

首先，当你第一次尝试这个问题时，你从一本书或课程中获得的信息要有意义得多。那是因为你的头脑处于警觉状态。

这就像你买了一辆新的蓝色汽车，然后“突然”你看到路上有一堆蓝色汽车。同样，当你第一次尝试时，当你专注于讲座的内容时，你遇到的问题就在你的脑海中。此外，你获得的信息会引起更多的共鸣，因为你有一个特殊的应用场合。

先申请，后学习的另一个原因是，我们更容易记住奋斗的时光。(还记得那次 10 英里跑有多难吗？还记得你为写第一篇二十多页的研究论文付出了多少努力吗？)

*解释道:“当你被要求在被展示如何解决一个问题之前努力解决这个问题时，接下来的解决方案会被更好地学习和更持久地记住……当你被展示解决方案时，灯会亮。”*

第三，你将建立我所说的“程序员的直觉”这意味着，当你用新的和不同的方式弯曲现有的知识时，你将开始发现模式并建立联系，简单地通过首先尝试问题。

# 非学校方式

**先申请，后学习**与我们大多数人在学校学习的方式背道而驰:听完一堂课，然后做作业。这种过时的方法已经被许多学习平台和训练营所支持:循环播放无数的视频。然后，(可能)最后有个练习题或者项目。

当我开始编程时，我成了这种方法的受害者，并发现它非常无效。

我们认为我们正在取得进步，因为我们循环了大量的讲座。我们认为我们在学习，因为专家让这个过程看起来很简单。所以我们*认为*我们已经“明白了”

但这往往是一种错觉。我们没有意识到这是一种错觉，直到我们面临一种迫使我们应用知识的情况，然后我们完全陷入困境。难怪我们对自己的编程进度如此沮丧:我们投入大量时间观看讲座，但我们没有保留信息，因为我们没有积极地应用它。

从头到尾看一门课程不是学习的方式，原因如下:信息不固定。*坚持*指出:“在很短的时间内，我们会失去大约 70%刚刚听到或读到的东西。”。

从一开始就让我们淹没在大量的信息中是没有好处的。是的，有很多东西要学。但是你现在不需要知道所有的事情。重新学习以前学过的概念也没有用

也就是说，当你需要的时候，专注于你需要的东西会更有效。找到你需要什么的方法是首先尝试:尝试解决问题。

# 激活它

这种方法有一个例外。假设一个问题要求您编写一个类，但是您是编程新手，从未编写过类，对类一无所知。

在这种情况下，我建议你学得刚刚好，然后马上申请。所谓“足够了”，我的意思是要有全局观念，不要迷失在细节中。然后开始迭代首先应用，研究第二种方法:继续解决问题，如果遇到困难，可以咨询资源。

你“做”得越快，学得越好。因此，从一开始就让你的学习变得积极主动，首先应用，其次学习。争取花更多的时间积极做你想做得更好的事情。毕竟厨师做饭。作家写作。程序员编程。
