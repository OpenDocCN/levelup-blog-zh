解释Swagger以及如何使用Swagger为使用SpringBoot构建的REST API创建文档。

在这篇文章中，我将解释Swagger以及如何使用它为用 SpringBoot 构建的 REST API 创建文档。

什么是霸气？

Swagger是一个开发API、与API交互、主要为API生成文档的开源工具。

为什么我们需要API的文档？

API文档是一个技术内容交付件，包含有关如何有效使用和集成API的说明。这是一本参考手册，包含了使用API所需的所有信息，包括端点、函数、类、返回类型、参数等细节。

使用您的API的第三方开发人员希望尽可能快地集成，以推进他们的软件开发。所以他们应该很快就会理解你的API的价值和用途。

开发人员在发现、学习使用和最终集成API时的总体体验被称为开发人员体验。API文档是优秀开发人员体验的关键。

你的API可以是最好的功能API，但是如果不知道怎么用，就没人会用。文档是良好开发体验的基础。

如何使用Swagger来记录API？

Swagger允许开发人员将元数据添加到API应用程序中，然后生成各种形式的文档，如JSON、YAML，以及完全可读的HTML。开发人员不需要更新文档。斯瓦格会处理的。

所以这是关于什么是Swagger以及它如何帮助API文档的。现在让我们看看如何使用Swagger为我们的Spring Boot应用程序生成API文档。

使用Swagger为SpringBoot应用程序创建API文档

添加Maven依赖项

我们将使用Swagger规范的Springfox实现。要将它添加到 Maven项目中，下面的依赖项需要包含 pom.xml 。

<dependency>
   <groupId>io.springfox</groupId>
   <artifactId>springfox-swagger2</artifactId>
   <version>2.9.2</version>
</dependency>

启用霸气

要启用Swagger，只需将其添加到主类中。通过 @EnableSwagger2 注释启用Swagger。

但不建议这样做。最好有一个单独的config类来启用和配置项目中的Swagger。所以，创建一个新的类， SwaggerConfig。

Swagger的配置主要围绕 Docket bean。docket是一个包含您设置的所有可定制属性的对象，Swagger使用它来生成文档。

定义Docket后，再用select()方法从类ApiSelectorBuilderT14】中获取一个docket builder对象。构建器对象具有定制Swagger属性的方法。然后调用build()方法来获取docket builder的准备好的对象。

因此，我已经设置了配置，以设置API公开的路径，并设置包名，这需要包括文档。

现在Swagger已经准备好为您生成文档了！为了验证这一点，您可以创建一个对http://localhost:8080/v2/API-docs的请求。

结果是一个带有大量键值对的JSON响应。它不是很容易让人读懂。所以，Swagger为此提供了 Swagger UI 。

启用Springfox的Swagger UI

您需要向pom.xml添加一个额外的依赖项来使用Swagger UI。

<dependency>
   <groupId>io.springfox</groupId>
   <artifactId>springfox-swagger-ui</artifactId>
   <version>2.9.2</version>
</dependency>

现在你可以使用http://localhost:8080/Swagger-UI . html #/获得Swagger UI。

在这里，您可以展开这些控制器，并查看它拥有的所有端点。此外，您可以使用“尝试一下”对这些端点进行API调用，并查看响应。

您可以使用之前用于生成文档的相同docket向文档添加一些应用程序元数据。您可以使用apiInfo()方法来做到这一点。

现在您可以看到您传递的元数据。

您可以使用Swagger注释向API添加更多细节。有许多注释可以用来使文档更具描述性。可以参考https://springfox.github.io/springfox/docs/current/。

因此，希望您已经了解了Swagger的基本概念，以及如何使用Swagger为您的 SpringBoot 应用程序创建文档。感谢您的阅读！
