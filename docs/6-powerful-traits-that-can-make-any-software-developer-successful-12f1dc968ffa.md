# 6 个可以让任何软件开发人员成功的强大特质

> 原文：<https://levelup.gitconnected.com/6-powerful-traits-that-can-make-any-software-developer-successful-12f1dc968ffa>

## #3.掌控职业决策

![](img/167074d5feb61d61ff51f66a3a454289.png)

信用:Canva

当大多数人想到成功的软件开发人员时，他们可能会想到像比尔·盖茨或马克·扎克伯格这样的人。但事实是，有许多不同类型的软件开发人员在他们的职业中取得了巨大的成功。

在我 16 年的软件职业生涯中，我在每个大公司、初创公司和不同规模的公司中都见过成功的软件开发人员。

所有这些软件开发人员都有一些帮助他们在职业上取得成功的共同特征。如果你也有在你的软件职业生涯中获得成功的雄心，你将需要确保你拥有这六个强大的特质。

# 1.创造性思维

一个成功的软件开发人员最重要的特征之一是创造性思维的能力。这意味着提出新的想法和解决问题的方法。这也意味着能够从不同的角度看问题，并找到解决老问题的新方法。这种特质可以帮助开发人员快速适应快速变化的软件开发世界，并在任何市场条件下建立蓬勃发展的职业生涯。

作为一名软件开发人员，你每天都要面对挑战。对我来说，这是我作为软件工程师工作中最令人兴奋的部分之一。当我说挑战时，我指的是不明确的需求、复杂的业务问题、相互冲突的优先级、紧迫的期限和困难的联锁，等等。

要想成功，你需要学习如何在不影响你作为程序员的效率的情况下克服这些挑战。为此，你必须创造性地思考，找到创新的解决方案，创造出令人惊叹的东西。

软件开发人员学习创造性思维的一个方法是阅读相关书籍，参加创造性研讨会、培训项目和研讨会。这些将帮助你发展新的技能和技术，使你在工作中更有创造性。

你还必须观察你的周围，看看你组织中的其他团队是如何解决各种问题的。扩展你创造力的一个极好的方法是和你的同龄人一起头脑风暴。这将允许你开发新的想法，并为你的创造性思维过程提供素材。

# 2.承诺和纪律

作为一名软件开发人员，你必须全身心地投入到你的任务中，并完成它们。这需要纪律和专注。从长远来看，要想取得成功，坚持自己的承诺并避免偏离方向至关重要。这样做可以在你的团队和领导之间建立必要的信任。

许多软件工程师发现很难信守承诺，因为他们总是会遇到这样或那样的问题。过度承诺是造成这种情况的主要原因之一。他们不知道如何正确估计或优先处理他们的任务。在评估时，他们没有考虑到所有可能影响交付的因素。

当开发人员致力于大量任务时，他们感觉良好。他们认为他们的领导会欣赏他们的努力工作。不幸的是，当估计值相差甚远时，就很难信守承诺了。而且，当开发人员没有使用适当的优先级技术时，他们会不断地在最后一刻抢着完成高优先级的任务。这导致错过时间表和低质量的产出。

另一方面，承诺不足也是一个问题。当开发人员持续挑选比他们能做的更少的任务时，他们会拖垮他们的团队。他们没有发挥出自己的潜力。这可能会让他们有很多时间放松，但这表明他们不是称职的软件工程师。现在，要么他们的领导不得不面对即将到来的交付的压力，要么他们的团队成员需要加班加点来弥补差距。

最终，他们的经理和团队会赶上他们懒惰的工作方式。一旦发生这种情况，他们将不会信任开发高价值的关键项目。这反过来又会减缓程序员的职业成长。如果没有任何可见的工作，他们在评估时就没有什么可展示的。

克服这个挑战的一个方法是为你自己和你的团队设定现实的目标。在你职业生涯的开始，这可能看起来很难。但是如果你注意，随着时间的推移，你会逐渐了解你的真正能力，并据此做出承诺。当你始终如一地履行你的承诺，并高质量地完成实质性的交付时，你将会在职业发展中得到应有的关注。

# 3.掌控职业决策

作为软件工程师，我们要对自己的职业生涯负责。我们可能会责怪我们的团队成员、经理或公司政策没有给我们机会，但这些都是我们给自己的借口，因为我们没有做出有助于我们成长的改变。我说的改变，可以是你工作方式的改变，可以是项目的改变，可以是团队的切换，甚至可以是跳槽到一家成长前景更好的新公司。

我看到工程师一直在抱怨他们的处境。他们觉得自己在公司受到了不公正的对待。他们把自己和不同公司的朋友进行比较。他们被他人的一般观点所左右，而不做任何事实核查。

然而，他们没有采取任何具体行动来改善他们的状况。

继续担任这个职位或者在一家公司呆很长时间都没有错。但是如果你这样做了，要认识到你是在用你自己的意志做这件事。要明白，没有人强迫软件工程师在不利于他们成长的环境中继续工作。

作为一名软件工程师，最有效的成长方式之一就是采取行动，承担责任，不管结果如何。在角色没有成长的时候不采取行动是职业杀手。同时，不承认你的职业决定会让你沮丧，转移你对责任的注意力。

# 4.知道何时以及如何提供帮助

作为一名软件开发人员，您经常需要解决问题。无论您是在调试代码还是实现新功能，找到创造性解决方案的能力对您的成功至关重要。然而，成为一名成功的开发人员不仅仅是拥有正确的技能；这也是关于知道何时以及如何帮助他人。当另一个开发人员遇到问题时，伸出援助之手可能是一个成功项目和代价高昂的延迟之间的差别。

帮助他人是一个巨大的职业助推器。你帮助别人越多，人们就越觉得你知识丰富。你帮助传播你能力的人，还有其他人。这是建立你的品牌和提高你在组织中的知名度的最有效和最有力的方法。

然而，为了从你乐于助人的天性中获得最大的益处，你需要知道何时以及如何去帮助别人。当你在需要的时候帮助别人，你是在帮助他们实现他们的承诺。这反过来帮助您的组织按时高质量地交付项目。

但是有时候，当你帮太多的时候，其他程序员就变得依赖你了。他们不是试图独立解决问题，而是一遇到困难就向你伸出援手。这建立了一种不健康的文化。这些开发人员没有从他们的错误中吸取教训，因为他们从来没有尝试找到自己的解决方案。

帮忙太多的另一个副作用是你开始看到你的工作时间增加了。你帮助别人越多，留给自己的时间就越少。为了完成工作，你不得不加班加点。你越是努力跟上帮助别人、保质保量完成工作的步伐，就越是把自己推向倦怠。

因此，要想在职业生涯中取得成功，你需要学会如何在工作中帮助他人的同时找到正确的平衡。

# 5.专注于解决方案

成为解决方案提供商对于软件开发人员的成功至关重要，这有几个原因。首先，这表明他们能够批判性地思考和识别问题。第二，这表明他们有能力与他人合作找到解决方案。第三，这表明他们愿意为自己的行为负责。

相比之下，在任何事情上都发现问题的软件工程师更倾向于指责和逃避责任。因此，从长远来看，他们不太可能成功。

如果你经常阅读我的文章，你就会知道我是多么看重软件工程师的这种特质。我们的工作是通过我们的代码为他人的问题提供解决方案。如果任何程序员不以这种方式看待他们的工作，他们需要一个能以这种观点帮助他们的导师。联系我，我们可以在你的工作中找到价值，让你长期保持动力。

回到正题，我见过好的开发者纠结于日常问题。他们看到问题并开始抱怨。他们希望他们的领导人尽快解决这些问题。当问题没有得到令他们满意的解决时，他们会感到沮丧。

然而，如果你让他们提出一些解决问题的方法，他们会一片空白。或者，他们提出的解决方案完全没有考虑到其他相关的工程师。这种态度不会产生健康的工作环境。这也无助于建立一个协作团队。

解决任何问题的最佳方法是设想几种适用于所有人的解决方案。有些问题没有解决方案是完全没问题的。但在这种情况下，向你的经理提出这个问题，并相信他们会处理好的。不要指望他们会神奇地解决问题，让你满意。给他们时间友好解决问题。

当你变得专注于解决方案而不是问题时，你的态度也会变得积极。领导者和团队成员更喜欢与能帮助解决问题的人一起工作，而不是与那些总是抄袭他们工作环境中不可行的东西的人一起工作。

因此，如果你想成为一名成功的软件开发人员，就要专注于成为解决方案的提供者，而不是问题的制造者。这是一种将在你的职业生涯中很好地为你服务的特质。

# 6.作为一名团队成员

软件开发人员需要在团队环境中有效地工作才能获得成功。这意味着与其他团队成员合作，有效沟通，并在需要时提供支持。对于任何想要专业成长的开发人员来说，成为一名团队成员是一个基本的特质。没有它，即使是最有才华的开发人员也很难获得成功。

对我来说，成为一名团队成员意味着你意识到团队中其他人的雄心和挑战。当你的同龄人成功的时候，你是快乐的。你不会因为队友的工作而受到关注。事实上，你寻找机会去欣赏别人的工作。

成为团队成员是软件工程师成功的有效途径。因此，成功的开发人员更喜欢在他们的团队成员之间分享他们的知识，这样他们都可以发挥他们最大的潜力。他们会在百忙之中抽出时间来指导他人。他们不想只为自己成功；相反，他们希望团队中的每个人都拥有它。

如果你想在你的软件生涯中获得成功，要认识到软件开发是一个团队游戏。不要犹豫，提供有价值的反馈，让你的团队变得更好。你应该为与每个工程师都有能力并为公司的成功做出贡献的团队一起工作而感到自豪。并且，为了实现它，你必须准备好付出你所需要的所有努力。

# 最后的想法

你一定想知道——编程的熟练程度如何；这难道不是成为一名成功的软件开发人员最必要的技能之一吗？

是的，它是。但是成为一名伟大的软件工程师不仅仅是技术技能。

技术专长有助于你在工作中变得更好，但不足以帮助你长期成长。相反，如何在团队中工作、承担责任以及与他人合作对成为一名成功的软件开发人员更为重要。

当然，许多其他的特质也能让一个开发者成功，但是在我看来，下面这些是最重要的。

*   创造性思维
*   承诺和纪律
*   掌控职业决策
*   知道何时以及如何提供帮助
*   专注于解决方案
*   作为一名团队成员
