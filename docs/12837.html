2022年校园VMware面试体验

原文：https://levelup.gitconnected.com/vmware-interview-experience-on-campus-2022-6a6f66e2d53b?source=collection_archive---------13-----------------------#2022-07-13

VMware于2021年8月来到我们学院进行实习。招聘过程完全是虚拟的。它从一个在线测试开始，然后我们有一个案例研究回合。之后进行了四轮面试。

在线测试

在线测试在 Hirepro 平台上进行。测试包括三个部分，持续1.5小时。

能力倾向测验(15分钟)

能力倾向问题包括数量能力倾向、逻辑、语言推理、和数据解释。

温馨提示:

  • 练习上述每个主题的能力倾向问题，因为大多数问题都基于这些主题。
  • 时间是最重要的，所以如果你在一个特定的问题上卡住了，那么跳过这个问题，进入下一个问题。
  • 研究用于解决定量能力倾向问题的不同方法，因为这些问题花费的时间最多。
  • 学几个小技巧一定能帮你快速解决这些问题。你可以在 IndiaBix 上练习能力倾向题。

技术MCQ (29分钟)

这部分由基于网络、面向对象编程、操作系统、和数据库管理系统的mcq组成。

温馨提示:

  • 确保你对以上科目都很透彻。
  • 这些问题并不容易，但如果你清楚所有的基本概念，那么这一部分应该不是问题。你可以用IndiaBix来练习这些问题。

编码(30分钟)

这部分有一个中级编码问题，我们应该在30分钟内解决。这里有一个链接到这个问题。由于时间非常有限，我使用了能够解决大多数测试案例的强力方法。

提示:

  • 总是先用蛮力方法，然后尝试优化它。
  • 蛮力可以解决很多测试用例，这给了你一个很好的机会通过编码。

基于编程语言的MCQ (16分钟)

这一节有两个部分。在这两个部分中，我们必须选择我们所选择的语言( Java 、 C 、 C++ 、 Python )。每个部分给我们8分钟。不允许两次选择同一种语言。

温馨提示:

  • 确保你能熟练掌握至少两种以上的语言。
  • 研究最常见的语言面试问题。

我是入围面试名单的12个人之一。

案例研究回合

这一轮是系统设计轮。我们得到了一个问题陈述，为此我们必须画一个类图。我们还被要求编写系统主要功能的伪代码。这不是采访。问题陈述被邮寄给我们，我们应该在3小时内提交解决方案。因此，这一轮没有淘汰。我们的解决方案在后来的采访中进行了讨论。

温馨提示:

  • 仔细阅读给出的问题陈述，了解你要解决什么问题。
  • 不要马上开始实现你的设计。先想到接近。记住，你解决问题的方式很重要。面试官会评判你解决问题的方法。
  • 对于如何设计解决方案，没有硬性规定。根据场景的不同，他们可能会要求你画一个 ER图或类图、或用例图，如果有必要的话还要附上伪代码。

面谈

技术面试1(1小时)

面试官首先让我做自我介绍。然后她翻看了我的简历，根据我在简历上提到的内容问了我各种问题。然后她在OOP上问了以下问题:

  • 什么是 OOP ？
  • OOP的主要概念有哪些？解释每个概念？
  • 如何在 Java 中实现这些概念？
  • OOP概念的一些现实世界的例子是什么？

然后，她让我给自己的Java技能打1到5分。

我是进入下一轮候选名单的9个人之一。

技术面试2(1小时)

这一轮比上一轮难度更高。

  • 我首先被要求做自我介绍，然后面试官让我打开Codepair，写一个程序来计算对角排序矩阵中负整数的个数。面试官对我最初的任何方法都不满意。我最后的方法非常接近他想要的，然后他告诉了我解决方案。
  • 他问我在第四和第六学期有哪些科目，然后他继续问关于这些科目的问题。然后我被要求解释我的案例研究解决方案。最后，他问我有什么问题要问他。

我是入围下一轮的 6人之一。

技术面试技巧:

  • 在面试中保持镇定是非常重要的。如果你被困在某个地方，面试官会帮你解决。有些时候面试官会对你的做法不满意(我就发生过这种情况)。
  • 在这些情况下，始终保持冷静并尝试想出更好的方法。如前所述，面试官会帮助你，所以大声说出你的想法并不断向面试官表达你的想法。如果你不知道某事或者你对一个主题或话题没有信心，那么说你不知道。面试官通常很有经验，他们能够识破谎言。

管理面试(30分钟)

这是一个棘手的回合。

  • 面试官先让我介绍自己，然后问了我的优势和劣势。
  • 然后他问我关于我的项目，其中一个是电子商务网站。他让我解释我已经实现的功能和我计划添加的功能。他问我如何将我的网站扩展到数百万次请求的T21，如果客户对他们在网站上的体验不满意，我会怎么做。
  • 然后，他提出了另一个场景，在这个场景中，他和我正在进行一个项目，我们是仅有的两个成员。

基于这个场景，他问了我以下问题:

  • 我如何开始开发流程？
  • 我将在项目中使用哪些技术，为什么？
  • 假设我们的应用程序应该在两周内上线，我们在应用程序的一个非常重要的组件的开发中发现了一个错误。我将怎么办?

然后，他展示了一些与团队相关的场景:

  • 我团队中的一些成员正在为项目中使用的技术而努力。作为团队成员，我会做些什么？
  • 如果一名团队成员向我寻求帮助，该怎么办？如果另一个成员告诉我不要帮助那个人怎么办？
  • 如果我参与的项目使用了我不熟悉的技术，该怎么办？
  • 然后他问了我一个很奇怪的问题，“你会对父母撒谎吗？”。在我回答之后，他解释了他为什么问这个问题以及诚实的重要性。然后他问我是否有任何问题要问他。

3人本轮入选。我是其中之一。

管理面试技巧:

  • 如果你能进入这一轮，这将是一个巨大的成就，因为你现在已经具备了在VMware这样的公司工作所必需的技能。
  • 管理轮是为了检查你是否在文化上适合这家公司。做好自己，诚实成熟的回答这些问题就好了。
  • 这一轮也观察你说话和行为的方式。
  • 记住，在大多数与团队相关的场景中，适当的沟通是必要的。
  • 给出诚实的答案很重要，因为主持这一轮的人很可能是你的经理或团队领导。

人力资源面试(15分钟)

这一轮只是一种形式。

  • 面试从我的介绍和家庭背景开始。
  • 面试官接着问了一些基本的人力资源问题，以及我是否同意调动工作。然后，她解释了CTC的分类以及其他好处，这些好处也在入职前的谈话中进行了解释。
  • 她问我是否有问题要问她，面试结束了。

结果很快就公布了，我和另外两个人一起收到了实习+安置邀请。

一般提示

这里有一些更通用的提示

  1. 数据结构和算法非常重要。如果你想通过这个或任何技术面试，你必须彻底了解所有的概念。如果你刚刚开始，解决这些问题来适应DSA。
  2. 大声思考。找到解决方案很重要，但是面试官应该能够理解你的方法。
  3. 面试官可能会扭曲你的方法来迷惑你，所以确保你对自己的方法有信心。运行一些测试案例来巩固你的方法。
  4. 总是问问题，说明你对公司有兴趣。但是问一些好的问题，不仅仅是为了问而问。
  5. 研究往年的面试经验，同样的问题很有可能会被再次问到。可以在这里翻翻往年的经验。
  6. 确保你知道你简历上写的一切。彻底了解你所有的项目和项目中使用的技术。如果这是一个小组项目，你应该能够解释你在其中的角色。
  7. 如果你没有通过管理轮，不要沮丧，因为这取决于他们是否将你视为公司文化的一部分。你已经具备了必要的技术技能。
  8. 为你在管理回合中被问到的各种情况提供适当的理由，但不要偏离主题太远。
  9. 如果你被拒绝了，不要失去希望，分析哪里出了问题，下次努力改进。记住，运气因素也会发挥作用。
