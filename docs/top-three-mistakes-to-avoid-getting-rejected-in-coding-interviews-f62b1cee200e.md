# 编码面试

编码面试很有挑战性，需要大量的准备。大型科技公司希望应聘者在 45 分钟内解决两个问题。

很多时候，考生解决问题，落实解决方案，却发现自己被拒绝了。在大多数情况下，招聘人员会告诉他们提高解决问题的技能，下次再申请。这让候选人进退两难，因为他们不明白面试中哪里出了问题。

我参加过面试。每个公司都有自己的招聘门槛和对应聘者的期望。在汇报环节，面试官讨论候选人的面试表现。

在这篇文章中，我们将讨论我们在编写面试代码时经常犯的错误，以及我们应该如何避免它们。通常，我们没有意识到这些细微的错误，所以我们在面试时必须记住这些错误。

# 1.强力解决方案

![](img/a4f535bea75a497db801c364784459ea.png)

照片由[格兰特·杜尔](https://unsplash.com/@grant_durr?utm_source=medium&utm_medium=referral)在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上拍摄

我见过很多应聘者在面试中提出暴力解决方案。蛮力或天真的解决方案是编码面试中的大忌。强力解决方案很慢&不需要很好地利用认知能力。

大多数开发人员都能想出一个强力解决问题的方法。你不能通过给出一个暴力的解决方案来区分你和其他人。

在现实世界中，你应该有效地解决问题。让我们以`Collections.sort()`方法为例。在幕后，它使用了`Quicksort`算法。使用`Selection Sort`或`Bubble Sort`，会浪费大量的 CPU 周期。此外，它还会减慢程序的速度。

最优解是快速有效的。编码面试是为了评估候选人解决问题的能力。使用正确的数据结构和最优解是良好的问题解决技能的标志。这就是为什么你需要优化解决方案，因为没有人喜欢一个缓慢的程序。

## 如何优化暴力破解？

一个人需要在面试前彻底准备好数据结构和算法。面试时，候选人应该想尽一切办法。快速浏览 Leetcode 问题的讨论部分，你会发现一个问题至少有三个解决方案。

万一遇到问题，你可以使用以下三种方法

## 👉最佳数据结构/算法

有一次我问了一道基于动态规划的候选题。他很聪明，想出了一个基于递归的方法。我让他也这样做，他提前完成了这个问题。

问题中有多个重叠的子问题。他没有认出他们。提议的解决方案不是最佳的，所以我请他优化它。看起来他对基于 DP 的问题有点生疏了。他无法看到模式并提出最佳解决方案。

许多问题都有一定的模式。诀窍是熟悉并成为识别模式的专家。例如:-使用堆可以解决寻找`top k`元素的问题。一旦识别出模式，就需要正确的数据结构和算法来解决它。

## 👉自言自语

在解决问题的同时，大声思考很重要。你需要叙述你在解决问题时想到的任何事情。

面试官在面试过程中很专注。当候选人大声说出自己的想法时，他们就能洞察候选人的思维过程。这让他们知道你是否在朝着正确的方向前进。

面试官帮助候选人，让他们知道他们是否在正确的轨道上。这样，你就不会浪费时间，一起向不同的方向前进。

## 👉获取提示

候选人的一个常见误解是，面试官是来拒绝你的。你必须在面试中摆脱这种误解。如果你通过面试并加入公司，他们可能会成为你未来的同事。

面试官会帮助你。根据我的经验，每当我遇到问题时，我都会得到很多提示。暗示是微妙的，并不总是直接的。

有一次，我被要求解决一个基于字符串的难题。我想到了两个解决方案。一个使用`trie`，另一个使用`hashmap`。因此，面试官问我基于`trie`的解决方案是否是最佳的。事实证明基于 trie 的解决方案不是最优的，所以我最终使用了`hashmap`。

所以，一个人需要全神贯注，倾听面试官给出的暗示。简单的提示可以帮助你朝着正确的方向思考，并很快找到解决方案。

## 如何避免失败的边缘案例？

你可以采用以下技巧来处理棘手的个案

1.  *For/While 循环* —确保算法完成并处理终止错误。
2.  *空检查* —在处理指针/引用时添加空检查。这在你处理与二叉树、链表、图表等相关的问题时很重要
3.  *详尽的测试用例*——试着思考非平凡的测试用例。正如上一节所讨论的，找出算法会出错的例子。
4.  *预演* —使用特定的输入并检查您的代码，就像您的计算机执行指令一样。这将避免大多数错误，并给你足够的信心写代码。

# 3.沟通

几年前，当我面试一位候选人时，我给了他两个问题让他去解决。他能够解决这两个问题，并提出了一个最佳解决方案。

但是，他全程保持沉默，直接跳到了编码上。最初，我打断了他，让他解释解决方案。他无法清楚地解释他的方法，只是在解释时触及了表面。我想他可能是一个内向的人，因此超过了他。但是，我记下了他的沟通技巧。

在候选人的汇报过程中，事实证明他在任何一轮中都无法交流。他的解释不符合标准，大家一致认为他的沟通技巧很差。

沟通是软件开发中最被低估的技能之一。在现实世界中，人们需要与多个利益相关者交谈，并推动项目向前发展。人们必须提出解决挑战性问题的想法和解决方案。因此，合作和沟通技巧与解决问题的技巧同等重要。

## 如何有效沟通？

![](img/8dd96299eee50b75532f1ee0c66b5bff.png)

Amy Hirschi 在 [Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral) 上的照片

面试时，候选人必须坐在驾驶座上。他们需要积极推动讨论。你可以使用下面的方法在面试中进行有效的交流

1.  *理解问题陈述* —澄清你是否正确理解了问题陈述。了解问题的约束条件，即输入参数的范围(正/负)、最大可用空间(内存)、输入大小等。这将表明你关注细节，不盲目地做出假设。
2.  *解释解决方案*——一旦你有了最佳解决方案，你就可以解释同样的方案。你需要确保解释简单明了。借助例子尽量简化解释。此外，谈谈您的解决方案的时间和空间复杂性，以及您正在做出的权衡。

人们需要记住，沟通在面试过程中起着积极的作用。你可以是一个很好的问题解决者，但是如果你不能交流，那会被认为是一个危险信号。

为了提高沟通技巧，你可以在镜子前练习说话。此外，和朋友进行模拟面试也是个好主意。这将帮助你克服面试恐惧，以防你已经很久没有面试了。

# 结论

如果你想在编码面试中胜出，你需要避免以下三个错误

1.  实现暴力解决方案
2.  不处理角落案件
3.  保持沉默，不能有效地沟通

这些是候选人最常犯的错误。此外，除了上述三个因素之外，候选人还需要关注其他因素。我将写另一篇文章详细介绍这些重点领域。
