# 数据科学家的工作描述

微软数据科学家的工作在不同的团队和每个人的工作方式上有很大的不同。然而，它们的最终目标都是创建可以在生产中部署的预测模型。但是在这个过程中，人们可以更多地在研究方面工作，也可以更多地在部署方面工作。例如，如果你对做研究更有热情，那么你必须深入挖掘和探索，找到你的团队中存在的问题的解决方案，因为没有其他团队已经解决了它。如果你更多的是在部署方面，你将解决在你的团队工作中发生的问题，例如类似的问题已经在不同的团队中解决了，但是需要适应你的工作。所以你会看着他们的代码，理解他们做了什么，并为你团队的用例重新实现。

微软有不同的团队，致力于语音和语言、计算机视觉、Azure 上的机器学习基础设施、云计算的数据科学咨询等等。所以你的角色和工作肯定会随着你被选入的团队而变化。

# 所需技能

微软的数据科学家职位更喜欢有几年行业经验的候选人，因为你必须在生产中部署模型，这需要你也有一些软件技能。然而，他们确实会雇用大学应届毕业生来担任数据科学家的职位，但他们确实希望有良好的编码技能和关于特定技能的特殊知识，如计算机视觉、NLP、RL 等。，取决于正在招聘的团队。话虽如此，我还是列出了他们在[职业网页](https://careers.microsoft.com/us/en)上提到的一些技能/资格。

*   候选人必须能够向领导层传达复杂的想法和概念，并交付成果。
*   候选人必须善于处理和分析来自不同来源的复杂的高维数据，以解决困难的问题。
*   计算机科学、统计学、数学、物理、工程或相关学科的学士或更高学位
*   体验基于云的架构，如 Azure

# 申请工作

像所有其他公司一样，从你认识并在该公司工作的人那里获得推荐总是好的。推荐使你的申请更容易被公司认可，与普通候选人相比，他们更喜欢面试被推荐的候选人。为了申请，你必须在微软的求职页面上申请，这基本上会要求你提供简历和附件，以展示和证明你的工作。我强烈建议增加所有的证书和奖励，使你的申请具有可识别性、透明性和真实性。

我发现一些对简历有用的建议:

*   在简历中加入你的 Github、Linkedin、Kaggle、Medium(如果你主动写的话)个人资料链接
*   提及你的出版物，因为它们对这个角色很重要
*   保持你的观点简明扼要
*   关注你的成就，而不是在简历上写满你做过的一切。

# 面试过程

一旦您通过职业页面提交了您的申请，如果您通过了他们的初步筛选流程，招聘人员将与您联系，并向您简要介绍角色期望、您将参加面试的团队以及流程简介等。在某些情况下，招聘经理会进行初次面试，在面试中他们会仔细检查候选人以前的经历，看看他们是否会对团队有价值。这一轮确保在进行多轮面试(一个漫长的过程)之前，候选人具备团队所需的技能。在这一轮之后，将进行多轮技术面试，以测试你在各个领域的技能。他们寻求的技能如下:

*   数据结构和算法
*   机器学习
*   机器学习系统设计
*   理解和解释数据
*   概率与统计

你需要清除 SDE1(软件开发工程师-1)对数据结构和算法要求的障碍。许多候选人只关注机器学习和准备简历，尤其是数据科学面试，但他们也应该为编码轮做好准备。对简单到中等水平的编码问题有信心，并对各种数据结构有基本的了解就足够了。在进行面试的时候，总是试图告诉面试官你的思维过程，并恰当地传达你的想法。这也是数据科学角色所需的一项重要技能。

# 校园安置流程

在大学，最初的筛选有点不同，因为仅仅基于简历的筛选可能不是学生的好指标，因为他们很少有或没有行业经验。每当微软来招聘数据科学家职位时，他们首先会参加基于客观的机器学习测试。这个测试非常漫长，包括各种各样的问题，如 SVM、决策树、统计学、深度学习等，所以你需要做好准备。一个人肯定应该在测试中尝试最大数量的问题。然后，被选中的候选人被邀请参加团体飞行赛。在这一轮中，他们召集一群学生，然后给所有人一个编码问题，每个人都必须用纸笔来解决它。这一轮允许考生与面试官互动，提出疑问。面试官在这里的主要任务是更多的了解应聘者的思维过程，所以一定要保证在团-飞轮的时候和他们互动。最后，入围的候选人被要求参加面试。

> 印度的大多数大学都采用这种校园安置流程。我不知道印度以外的这种情况下的过程。

# 类似的面试问题

> 这些问题并不是面试中问到的确切问题。这些是为了让你在面试中能想到一些问题

*   L1 和 L2 的正规化有何不同？
*   与普通的图像神经网络相比，卷积核有什么用处？
*   什么是假设检验？
*   在分类的情况下，如何处理偏斜的数据分布，在这种情况下应该使用什么度量？
*   什么是决策树？熵是什么？为一个示例制作一个决策树。
*   如何合并 k 排序数组？
